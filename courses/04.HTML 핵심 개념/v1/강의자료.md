### 03. 주석 ⭐

### 실습 코드

```html
<!DOCTYPE html>
<html>
  <head>
    <meta charset="utf-8">
    <title>Watch it!</title>
    <link rel="stylesheet" href="style.css">
  </head>
  <body>
    <div id="header">
      <h1 id="logo">Watch it!</h1>
    </div>
    <div id="main">
      <a href="showing.html">
        현재 상영작 보기
      </a>
    </div>
  </body>
</html>
```

### 처음 작성하는 주석

> 자 여기 간단한 HTML 파일이 있는데요. h1 태그 위에 이 안에다가 머리말이라고 쓰겠습니다. 
꺽쇠를 여시고 느낌표 그리고 대쉬 대쉬 머리말 그리고 다시
> 

```html
<!-- 머리말 -->
<h1 id="logo">Watch it!</h1>
```

> 글씨 색이 태그랑은 좀 다르죠? 이게 바로 주석입니다.
> 

### 주석 추가하기

> 하나 더 써 볼게요. 상영작 옆에 “업데이트 필요” 라고 적어준 뒤에
> 

```html
현재 상영작 보기 <!-- 업데이트 필요 -->
```

> 코드를 저장하시고 보면? 주석은 사실 웹사이트에 아무런 영향을 주지 않습니다. 브라우저가 무시해 버리기 때문이죠.
> 

### 주석을 남길 때 1

> 그럼 주석은 언제 쓰는 걸까요? 우선 첫 번째로 메모를 남길 때 사용합니다. 코드를 쓰다 보면 메모를 남기고 싶을 때가 있는데요. 코드에서 주석를 써 놓으니까 이 부분은 머리말이다, 그리고 이 부분은 나중에 수정이 필요한 부분이다, 이렇게 적을 수 있죠.
> 

### 주석을 남길 때 2

> 그리고 두 번째로 코드를 잠깐 감추고 싶을 때 사용합니다.
> 

```html
<!-- <h1 id="logo">Watch it!</h1> -->
```

> 코드 앞에 이렇게 이렇게 저장하고 확인해 보면 화면에서 사라졌습니다. 이게 코멘트의 본래 목적은 아닌데요. 많은 사람들이 코드를 잠깐만 숨길 때 코멘트를 활용하기도 하니까 이것도 팁으로 알아두시면 좋을 것 같네요.
> 

### 단축키

> 코멘트는 자주 사용하는 거기 때문에 보통 에디터에 단축키가 있습니다. VS Code에서는 원하는 줄에 커서를 옮긴 다음에 윈도우즈에서는 Ctrl+/, 맥 OS에서는 Cmd+/를 누르면 코멘트가 적용됩니다. 같은 키를 다시 누르면 코멘트가 사라집니다. 한 줄이 아니라 여러 줄을 코멘트하고 싶을 때는 여러 줄을 선택하고 단축키를 누르시면 됩니다.
> 

### 04. #Quiz HTML 시작하기 퀴즈

Q1: HTML은 무엇의 약자이며, 각각 무엇을 의미하나요?

- 정답 확인하기
    
    `Hypertext Markup Language`
    
    - Hypertext: 여러 문서가 하이퍼링크를 통해 서로 연결되어 있는 것을 의미합니다.
    - Markup Language: 내용에 구조와 의미 같은 추가적인 정보를 표시하여 문서를 만드는 언어입니다.

---

Q2: HTML 태그의 기본 구조는 어떻게 되나요? 시작 태그와 종료 태그의 차이점은 무엇인가요?

- 정답 확인하기
    
    <태그이름>으로 시작하고 </태그이름>으로 끝납니다.
    
    시작 태그와 종료 태그 사이에 내용이 들어갑니다.
    
    종료 태그는 태그 이름 앞에 슬래시(/)가 붙는다는 차이점이 있습니다.
    

---

Q3: HTML 속성(attribute)은 어떻게 작성하나요?

- 정답 확인하기
    
    시작 태그 안에 속성이름="속성값" 형태로 작성합니다.
    

---

Q4: HTML 파일의 기본 구조는 어떻게 되나요? head와 body 태그의 역할은 각각 무엇인가요?

- 정답 확인하기
    
    HTML 파일은 최상단에 <!DOCTYPE html> 선언으로 시작하며, 전체 코드는 <html> 태그로 감싸여 있습니다.
    
    그 안은 <head>와 <body>로 나뉩니다.
    
    - head: 웹 페이지 자체에 대한 정보(제목, 인코딩 방식, 외부 파일 연결 등)를 담습니다. 이 부분은 화면에 직접 보이지 않습니다.
    - body: 웹 브라우저 화면에 실제로 보이는 모든 콘텐츠(텍스트, 이미지, 비디오 등)를 담습니다.

Q5: HTML에서 주석(comment)은 어떻게 작성하며, 어떤 용도로 사용되나요?

- 정답 확인하기
    
    <!-- 주석 내용 --> 형태로 작성합니다.
    
    주석은 브라우저가 코드를 해석할 때 무시하므로 화면에 보이지 않습니다.
    
    주요 용도는 다음과 같습니다.
    
    - 메모 남기기: 코드에 대한 설명을 추가하여 다른 개발자나 미래의 자신이 코드를 쉽게 이해할 수 있도록 돕습니다.
    - 코드 임시 비활성화: 특정 코드 부분을 삭제하지 않고 잠시 실행되지 않도록 할 때 사용합니다.

> 이렇게 href의 속성 값으로는 기본적으로 사이트 주소 전체를 쓰거나 경로를 쓸 수 있다고 배웠습니다. 이번 레슨에서는 경로를 사용하는 방법에 대해 자세히 배워볼게요.

### index의 의미

### target=”_blank”

> 웹 브라우저에서 확인해 보면 클릭하면 현재 페이지에서 이동하는데요. 이게 아니라 새 탭에서 열리게 하고 싶으면 target이라는 속성을 쓰면 됩니다. 복사해서 target= "_blank", 일단 _blank라는 걸 써 볼게요. 이건 새 창, 새 탭으로 여는 겁니다.
> 

```html
<a href="https://movie.naver.com"> 네이버 영화 </a>
<br />
<a href="https://movie.naver.com" target="_blank"
  >네이버 영화 - 새 창으로 열기</a
>
```

> 확인해 보면 이번엔 이렇게 새 탭에서 열리죠? 정말 자주 쓰는 거니까 꼭 기억해 두시기 바랍니다.
> 

### target=”_self”

> 그리고 다음으로는 _self라는 게 있는데요. _self를 하면 현재 창에서 열립니다.
> 

```html
<a href="https://movie.naver.com" target="_self"> 네이버 영화 </a>
```

> 기본 값으로 설정
> 

### 창의 이름을 custom 하기

```html
</ol>
```

> 여기 p 태그 대신에 ol 태그를 감싸줄게요. 그리고 각 항목에는 List Item의 약자인 li라는 태그로 감싸주겠습니다.순위를 보면 제가 따로 숫자를 적어 준 것도 아닌데 이렇게 숫자가 적혀서 나옵니다.
> 

### 정리

> 순서 리스트는 태그 자체가 순서의 의미를 갖고 있기 때문에 여기 코드에 적혀 있는 순서대로 내용에 순서가 생깁니다. 순서가 있는 내용을 적을 때는 ol 태그 안에다가 li 태그를 쓰면 된다는 것, 꼭 기억해 주세요.
> 

### 02. 순서 없는 리스트 ⭐⭐⭐

### 04. 리스트 스타일링 ⭐

### 스타일링 변경하기

> 이렇게 순서 리스트랑 순서가 없는 리스트를 쓰면 HTML로 그 의미를 분명하게 나타낼 수 있어서 좋습니다. 근데 화면에 보이는 모양을 바꾸고 싶으면 어떻게 하면 좋을까요? 우선 type이라는 속성을 사용할 수 있습니다. 겉모양만 바꾸는 게 아니라 마크업으로 순서를 어떻게 표시할지 정하는 건데요.
> 

### type 이용하기

> 예를 들어서 type="a"라고 하면 abc로 순서가 매겨지고요. 대문자 A를 쓰면 대문자를 쓸 수 있고 소문자 i를 쓰면 소문자로 로마 숫자를 사용할 수 있고 대문자 I를 쓰면 대문자로 로마 숫자를 사용할 수 있습니다. 숫자 1을 쓰면 처음 봤던 것처럼 아라비아 숫자로 순서를 사용할 수 있습니다.
> 

```html
<ol type="a">
<ol type="A">
<ol type="i">
<ol type="I"> 
```

### css를 이용하여 스타일 적용하기

> 근데 이렇게 말고 CSS로 보여주는 모양만 바꿀 수도 있는데요.
사실 이걸 더 많이 씁니다. style 태그를 추가하고 ol 태그에다가 list-style이라는 속성을 사용해볼게요.
> 

```html
<ol style="list-style: disc">
```

> 우선 disc라고 해볼게요. 확인해 보면 순서 리스트인데 겉모양은 밑에 보이는 순서가 없는 리스트랑 똑같죠?
특이한 게 하나 있는데 decimal-leading-zero라는 게 있습니다.
> 

```html
<ol style="list-style: decimal-leading-zero">
<ol style="list-style: hangul">
```

### list-style: none

> 여기서 자주 사용하는 것을 한번 알아볼게요 
list-style을 none으로 해주겠습니다.
> 

```html
<ol style="list-style: none; margin: 0; padding: 0">
```

> 순서가 없는 리스트가 마치 그냥 본문 같이 보이는데요.
개발자 도구로 살펴보면 li 태그들 모양이 마치 div 태그로 나누어 놓은 것처럼 보입니다. HTML 코드상으로는 목록이라는 의미가 있지만 겉모습은 내가 원하는 대로 꾸미고 싶을 때 이렇게 많이 사용합니다.
> 

```html
<style>
  ol li {
    border: 1px solid red;
    display: inline-block;
    padding: 0.5rem;
    border-radius: 1rem;
    user-select: none;
  }
</style>
```

### <form> 태그

- 이용한 간단한 회원 가입 페이지 만들기
- form을 만들기 위해서는 form 태그를 사용

```html
  <body>
    <form></form>
  </body>
```

### <input> 태그

- 사용자(유저)가 입력하는 값
- self-close 태그

```html
<form><input /></form>
```

### <label> 태그

- input 태그의 설명을 적는 용도
- 라벨 클릭시에 input에 focus

```html
<form>
  <label>
    이메일
    <input />
  </label>
</form>
```

### <label> 태그에 id값 주기

- input 태그의 id값과 label 태그의 for 값을 동일하게 설정
- 라벨 클릭시에 input에 focus

```html
<form>
  <label for="signup-email">이메일</label>
  <input id="signup-email" />
</form>
```

### <input> 태그의 name 속성 사용하기

- 폼에 입력한 데이터를 전송하려면 필요함
- input에 이메일 입력후 엔터키를 누르면 url이 name을 쿼리스트링으로 가지게 됨

```html
<form>
  <label for="signup-email">이메일</label>
  <input id="signup-email" name="email" />
</form>
```

### 비밀번호 input 만들기

- inline 속성이기 때문에 div 태그 추가

```html
<form>
  <div>
    <label for="signup-email">이메일</label>
    <input id="signup-email" name="email" />
  </div>
  <div>
    <label for="signup-password">비밀번호</label>
    <input id="signup-password" name="password" />
  </div>
  <div>
    <label for="signup-password-repeat">비밀번호 확인</label>
    <input id="signup-password-repeat" name="password-repeat" />
  </div>
</form>
```

### type=”password”

- input에 입력된 값을 * 로 처리하게 됨

```html
<input id="signup-password" name="password" type="password" />

...

<input
  id="signup-password-repeat"
  name="password-repeat"
  type="password"
/>
```

### type=”email”

- 입력값이 email 인지 검증
- 모바일 환경 등에서 키보드 최적화

```html
<input id="signup-email" name="email" type="email" />
```

### 03. 버튼 ⭐⭐⭐

### <input> 태그가 하나일때와 다른 점

- 엔터키를 눌러도 더 이상 url이 변경되지 않음
    - <button> 태그를 추가하거나 <input> 태그에 `type="submit"` 을 추가해야함

### <button> 태그

- type=”submit”은 기본값

```html
  <button type="submit">회원가입</button> <!-- <input type="submit" /> 으로 해도 됨-->
</form>
```

### type=”button”

- 버튼을 눌러도 form이 전송되지 않음

```html
<button type="button">회원가입</button>
```

### type=”reset”

- form에 입력된 input 값들을 초기화 함

```html
<button type="submit">회원가입</button>
<button type="reset">취소</button>
```

### form 태그 없이 button 태그 사용하기

- 아무일도 일어나지 않음
- 자주 사용하는 태그 중 하나

```html
	</form>
	<button>제출</button>
</body>
```

### <link> 태그

- 외부에 있는 걸 불러올때 사용함
    - 구글 폰트에서 사용했음
    - 대표적으로 CSS 파일과 아이콘 파일

### CSS 파일 불러오기

- rel 속성은 relation, 관계라는 뜻으로 어떤 종류의 파일을 불러올지 적는 곳
- href는 파일의 주소를 의미함

```html
<title>코드잇 캠핑장</title>
<link rel="stylesheet" href="./style.css" />
```

### 구글 폰트에서 사용한 link

- preconnect는 불러올 파일들이 어떤 사이트에 있는지 미리 브라우저에 알려주는 역할
- href는 파일의 주소를 의미함

### <script> 태그

- 자바스크립트 코드를 작성하고 실행하도록 해줌
- 개발자도구(F12) 콘솔창에서 표시가 되는 것을 알 수 있음
- <body> 태그가 끝나는 지점에 사용해야함

```html
  <h1>파도소리 들리는 감성 캠핑!</h1>
  <script>
	  console.log("hello world");
	</script>
</body>
```

### script.js으로 분리하기

- css와 마찬가지로 외부에서 불러올 수 있음
- 자바스크립트를 통해서 웹사이트에 무한한 아이디어를 실제로 구현 할 수가 있음

```jsx
// script.js
console.log("from script.js");
```

```html
<!-- index.html -->
  <script src="./script.js" />
</body>
```
