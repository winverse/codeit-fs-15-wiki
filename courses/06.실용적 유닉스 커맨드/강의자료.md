### `Cal` Command

- 현재 월의 달력을 터미널에 출력
- 특정 월, 연도 달력 출력
    - `cal 5 2025` → 2025년 5월 달력 출력
- 특정 연도 전체 달력
    - `cal 2024` → 2024년 전체 달력 출력
- 이전/다음달 포함 3개월 출력
    - `cal -3`
- 줄리안 날짜(일수)로 보기
    - `cal -j`
- 전체 연도 달력
    - `cal -y`
- 일수 + 전체 연도 달력
    - `cal -yj`

### 08. 커맨드의 기본 형태  ⭐⭐⭐

### argument와 flag (인자와 플래그)

- `cal 5 2025`  이 커맨드에서 `5 2025` 부분이 Argument 혹은 인자라고 할 수 있습니다.
- `cal -y` 이 커맨드에서 `-y` 부분이 flag 혹은 옵션이라고 할 수 있습니다. 대시(`-`) 혹은 대시 두개 (`—-`) 와 함께 사용하는 경우가 많습니다.

### 커맨드의 기본 형태

```
// case 1
명령어 [옵션/플래그] [argument]

// case 2
명령어 [argument] [옵션/플래그]
```

- 자세한 것은 커맨드 관련해서 `-help` 혹은 `--help` 를 입력하여 자세하게 알아볼 것
    - 명령어 [argument] [옵션/플래그]

예시

- `ls -al /home/ubuntu`  (`/home/` 하고 tab키를 눌러서 나오는 걸 enter)
    - `ls` : 명령어(Command)
    - `al` : 옵션/플래그(Option/Flag)
    - `/home/user` : argument(인자/인수)
    - `ls  /home/user -al` 도 동일한 동작
- `cal -yj`
    - `cal` : 명령어(Command)
    - `yj` : 옵션/플래그(Option/Flag), 여러 옵션 조합

### **옵션과 인자의 역할**

- 옵션/플래그(Option/Flag): 명령어의 **동작 방식**을 제어하는 추가 정보입니다. (예: `cal -y`의 `-y`)
- 인자(Argument): 명령어가 **작업할 대상**을 지정하는 필수/선택 정보입니다. (예: `cal 5 2025`의 `5 2025`)

### 09. 커맨드 매뉴얼: man

### 커맨드 매뉴얼 확인하기 `man`

- 각 커맨드(명령어)는 공식 매뉴얼로 자세한 사용법을 확인할 수 있음
- 매뉴얼 보기 명령어:
    
    ```
    man [커맨드]
    ```
    
- 예시
    
    ```
    man cal
    ```
    
    - 커맨드를 아규먼트로 전달하면 해당 커맨드에 대한 공식 설명서가 출력

### man 페이지 기본 사용법

- **아래 방향키**: 한 줄씩 아래로 이동
- **위 방향키**: 한 줄씩 위로 이동
- **스페이스바/`f`키**: 한 페이지씩 아래로 이동 (Forward)
- **`b`키**: 한 페이지씩 위로 이동 (Backward)
- **`q`키**: 매뉴얼 종료(나가기)

### **매뉴얼 각 섹션 설명**

- **NAME**: 커맨드 이름과 역할을 간단히 기술
    - cal과 ncal(비슷한 기능을 가진 커맨드)의 정보가 같이 나올 수 있음
- **SYNOPSIS**: 커맨드 문법, 사용 가능한 옵션과 아규먼트 구조를 요약
    - 대괄호([])로 감싸진 부분: 선택적(옵션)
        - 예) [[month] year] 에서 `cal 3` 이라고 하면 3월이 아닌 3년임
    - 옵션에 값을 넣어야 하는 경우(예: -A number), 옵션 뒤에 아규먼트 붙임
    - month와 year처럼 여러 아규먼트가 들어가면 입력 방식에 따라 해석(연도만 입력할 수도 있음)
- **DESCRIPTION**: 커맨드와 각 옵션에 대한 상세 설명
    - 각 옵션, 예를 들어 **`j`**, **`y`**, **`A`**, **`B`** 등 실제 사용하는 방법 안내
    - A와 B 옵션은 각각 '이후/이전 n개월 달력 출력'
    - 옵션을 여러 개 조합해서 사용 가능
        - (예: `cal -j -B 3 -A 3`), 일수를 이전 3달/이후3달을 표시
- **추가 섹션**: 기타 설명, 관련 커맨드, 참고할 정보 등 표시됨

### 10. 터미널 사용 꿀팁 ⭐⭐⭐

### **이전에 실행한 커맨드 불러오기**

- 이전에 사용했던 커맨드를 다시 사용해야 하는 경우가 많음
- **위쪽 방향키(↑) 반복** : 이전에 실행한 커맨드를 한 줄씩 불러와 재사용할 수 있음
- 여러 줄 위 기록까지 쉽게 찾아서 수정·실행 가능

### 커맨드 빠른 편집

- 예를 들어서 `cal -j -B 3 -A 3` 이런 커맨드를 입력하였을때, 잘못 입력해서 화살표 하나씩 이동하는 것은 답답할때가 많음
- **Ctrl + A** : 커서를 줄 맨 앞으로 이동
- **Ctrl + E** : 커서를 줄 맨 뒤로 이동
- **Option(Alt) + ←/→** : 단어 단위로 좌우 이동(맥: Option, 윈도우: Alt)
- 긴 커맨드도 빠른 커서 이동으로 손쉽게 수정 가능

### 명령/입력 취소

- 가끔 커맨드를 입력하다가 지워야 할 때가 있음.
- **Ctrl + C**
    - 입력 중: 현재 커맨드 입력 취소, 프롬프트로 돌아감
    - 실행 중: 명령 실행 강제 종료(특히 멈추지 않는 작업 취소용)

### 터미널 화면 정리

- **clear** 명령어 입력 후 엔터 :
    
    터미널 화면을 깨끗하게, 맨 위부터 시작
    
- 이전 커맨드 히스토리는 위아래 화살표로 언제든 확인·호출 가능

### 커맨드/경로 자동 완성

- **Tab 키**
    - 커맨드 타이핑 중 Tab : 자동 완성(알려진 명령/파일명/폴더명)
        - ex) **`unc`** + Tab → **`uncompress`**(자동 완성)
    - 앞부분만 쳐도 Tab을 누르면 가능한 내용 후보 목록이 출력
    - 폴더/파일 경로 입력 시, 이름 일부 + Tab으로 경로를 빠르게 완성 가능

### 현재 디렉토리 경로 확인: pwd

- `pwd`는 Print Working Directory의 약자.
- 터미널을 새로 열면 보통 홈 디렉토리(`~`)에서 시작.
- 터미널에서 현재 작업 위치(경로)를 출력(예: `/home/user`).

### 디렉토리/파일 목록 확인: ls

- `ls`는 List의 약자. 현재 위치(디렉토리) 내용물을 리스트로 보여줌.
- 아무 인자 없이 입력하면 현재 디렉토리 내 목록 출력
- `ls [경로]`로 다른 디렉토리의 목록 확인 가능
    - 예: `ls /bin` → /bin 폴더 내 내용 확인
- 디렉토리 이동 후 `ls`로 바뀐 위치의 내용 체크 가능

### 디렉토리 이동: cd

- `cd [경로]` 명령어로 다른 디렉토리로 이동
    - 루트 디렉토리: `cd /`
        
        ![스크린샷 2025-11-05 21.57.47.png](06%20%EC%8B%A4%EC%9A%A9%EC%A0%81%20%EC%9C%A0%EB%8B%89%EC%8A%A4%20%EC%BB%A4%EB%A7%A8%EB%93%9C/%E1%84%89%E1%85%B3%E1%84%8F%E1%85%B3%E1%84%85%E1%85%B5%E1%86%AB%E1%84%89%E1%85%A3%E1%86%BA_2025-11-05_21.57.47.png)
        
    - 홈 디렉토리: `cd ~` 또는 `cd`(아무 인자 없이)
        
        ![스크린샷 2025-11-05 21.54.59.png](06%20%EC%8B%A4%EC%9A%A9%EC%A0%81%20%EC%9C%A0%EB%8B%89%EC%8A%A4%20%EC%BB%A4%EB%A7%A8%EB%93%9C/%E1%84%89%E1%85%B3%E1%84%8F%E1%85%B3%E1%84%85%E1%85%B5%E1%86%AB%E1%84%89%E1%85%A3%E1%86%BA_2025-11-05_21.54.59.png)

    - 이전 위치로 이동: `cd -`

    - 홈 디렉토리로 이동 후에 pwd를 하면 /root가 나오는 이유 (`home`이나 `user` 폴더가 아닌 이유)

### 05. ls의 중요한 옵션들 ⭐⭐⭐

### 디렉토리/파일 목록 확인: ls

- **ls**는 디렉토리(폴더)·파일 목록을 살펴볼 때 사용하는 기본 커맨드.
- **`ls`**만 입력하면 현재 위치의 파일/디렉토리 이름들을 리스트로 보여줌.
- **`ls [경로]`**로 다른 위치의 디렉토리 내용을 확인할 수도 있음.
- 디렉토리 이동 후 **`ls`**로 바뀐 위치의 내용 체크 가능.

### 숨겨진 파일까지 모두 보기: -a 옵션

- **`ls -a`**
    - all의 약자. 이름이 .으로 시작하는 숨겨진 파일/폴더까지 모두 보여줌.
    - GUI, 일반 ls에서는 안 보이던 설정파일(.bashrc, .zshrc, .config 등)까지 확인 가능.
    - 홈 디렉토리, 시스템 디렉토리 등에서 자주 사용.
    - 숨겨진 파일은 확장자가 없는 경우가 많음(.profile 등).
        - 보통 확장자가 있는 경우 어떤 파일의 형식을 의미함
        - 단순 텍스트를 담고 있는 파일은 확장자가 없어도 됨
            - `.DS_Store`: 현재 디렉토리에 대한 정보 (only mac)
            - `.zshrc` `.bashrc`: 터미널 정보
            - `.profile` `.bash_profile`: 로그인 셸 실행 시 자동으로 읽혀서, 사용자의 환경을 초기화/설정

### 상세 정보 보기: -l 옵션

- **`ls -l`**
    - long format(상세 목록). 파일마다 아래 정보들을 한 줄씩 보여줌:
        - 파일 종류(d: 디렉토리, -: 일반 파일 등)
        - 파일/디렉토리 권한, 링크 개수, 소유자/그룹, 용량(크기), 마지막 수정일, 파일 이름 등
    - 운영·관리, 파일 설정 시 자주 사용.

### 숨김 + 상세 정보 한번에: ls -al

- **`ls -al`**
    - 모든 파일/디렉토리(숨김 포함)를 상세 정보와 함께 한꺼번에 출력.
    - 디렉토리 용량(total), 각 파일의 상세 property까지 관리에 매우 유용.

        ### 권한(퍼미션) 표기 구조
        
        ```
        // 예시
        -rwxr-xr--
        ```
        
        - 첫 글자: 파일 타입(: 일반 파일, `d`: 디렉토리, `l`: 링크)
        - 이후 **9자리**가 권한 정보를 나타냄
        
        ## 9자리 구조
        
        - 3자리씩, 소유자(user), 그룹(group), 기타(others) 사용자 권한
        - 각 자리들은 다음 의미를 가짐
            1. `r` = 읽기(read)
            2. `w` = 쓰기(write)
            3. `x` = 실행(execute)
        
        | 위치 | 의미 | 예시 |
        | --- | --- | --- |
        | 1 | 파일 타입 | d/-(디렉토리/파일) |
        | 2-4 | 소유자(user) 권한 (rwx) | rwx |
        | 5-7 | 그룹(group) 권한 (rwx) | r-x |
        | 8-10 | 기타(other) 권한 (rwx) | r-- |
        
        ### 예시 해석
        
        ```
        -rwxr-xr--
        ```
        
        - `-`: 일반 파일
        - `rwx` : **소유자**는 읽기/쓰기/실행 가능
        - `r-x` : **그룹**은 읽기/실행 가능(쓰기 불가)
        - `r--` : **다른 모든 사용자**는 읽기만 가능
        
        ---
        
        ### 권한 부여 의미
        
        - **읽기(r, 4)** : 파일 내용 보기, 디렉토리는 목록 조회 허용
        - **쓰기(w, 2)** : 파일 수정/삭제/생성(디렉토리의 경우 새 파일 추가·이름 변경·삭제 등)
        - **실행(x, 1)** : 파일을 실행(프로그램/스크립트), 디렉토리 접근 가능(cd)
        
        ### 디렉토리 권한의 의미 차이
        
        - 읽기(r): 디렉토리 내 파일명 목록 조회 가능(ls)
        - 쓰기(w): 내용물 추가/삭제/이동 허용
        - 실행(x): 해당 디렉토리로 이동(cd)/접근 가능(실제 파일 열람엔 r도 필요)
        
        ---
        
        ### 8진수(숫자) 표기법
        
        - 권한은 r=4, w=2, x=1로 각 자리의 합으로 나타낼 수도 있음
            - rwxr-xr-- → **754**
            - chmod 등에서 권한 설정 시 숫자 조합 사용
        
        ---
        
        ### 권한 변경 관련 명령
        
        - chmod : 권한 설정/변경
        - chown : 소유자 변경
        - chgrp : 그룹 변경
        
        ---
        
        ### 참고
        
        - 권한 예시
            - rwxrwxrwx (777) : 모두에게 모든 권한
            - rwxr-xr-- (754) : 소유자=모두, 그룹=r,x, others=r
            - rw-r--r-- (644) : 소유자=읽기+쓰기, 그룹/others=읽기만
        - 권한이 없다고 할 때
- 두 번째 컬럼: 링크 개수 (파일/디렉토리와 연결된 다른 이름 수)
- 세 번째/네 번째 컬럼: 소유자, 그룹
- 다섯 번째 컬럼: 크기(바이트)
    - 왜 폴더는 4096바이트 크기인가요?
        - 리눅스에서 디렉토리(폴더) 자체도 하나의 파일처럼 취급되며,
            
            해당 디렉토리의 내부 구조(파일 목록, 메타정보 등)를 저장하는 최소 공간이 필요함
            
        - 이 공간은 대다수의 리눅스 파일 시스템(ext4 등)에서 블록 단위(최소 크기 보통 4096바이트 = 4KB)로 제공
        - 즉, 디렉토리가 비어 있든, 파일이 아주 많이 들어 있든, 최소 1블록(4096B)은 차지하고 있음

### 08. 디렉토리와 파일 만들기: mkdir, touch ⭐⭐⭐

### 디렉토리 생성: mkdir

- 새 디렉토리를 만들 때 사용하는 명령어
- 사용법:
    
    ```
    mkdir [디렉토리명]
    ```
    
- 예시:
    
    ```
    mkdir unix_practice               # 현재 위치에 unix_practice 디렉토리 생성
    mkdir ~/unix_practice             # 홈 디렉토리에 unix_practice 생성(절대 경로)
    mkdir reports                     # 현재 위치에 reports 디렉토리 생성
    mkdir Sep Jul Aug                 # 여러 디렉토리(Sep, Jul, Aug)를 한 번에 생성(reports안에 생성)
    ```
    

### 디렉토리 이동: cd

- 디렉토리로 이동할 때 사용하는 명령어
- 사용법:
    
    ```
    cd [디렉토리명]
    ```
    
- 예시:
    
    ```
    cd unix_practice
    cd reports
    cd ..
    ```
    

### 디렉토리/파일 목록 확인: ls

- 현재 디렉토리 또는 지정한 경로의 파일·디렉토리 목록을 확인
- 예시:
    
    ```
    ls
    ls reports
    ```
    

### 파일 생성 및 수정 시간 업데이트: touch

- 파일을 새로 만들거나, 이미 존재하는 파일의 수정/접근 시간을 변경
- 사용법:
    
    ```
    touch [파일명]
    ```
    
- 예시:
    
    ```
    touch finances.txt performance.txt      # 현재 위치에 두 파일 생성
    touch Aug/finances.txt Aug/performance.txt   # Aug 디렉토리에 각각 파일 생성
    ```
    

### 실전 활용 예시 정리

1. 홈 디렉토리에 unix_practice 디렉토리 생성
2. unix_practice 안으로 들어가서 reports 디렉토리 생성
3. reports 안에 월별 디렉토리(Jul, Aug, Sep) 한 번에 생성
4. 각 월별 폴더 안에 touch로 .txt 파일들 한번에 생성
    - touch Jul/finances.txt Jul/performance.txt

### 추가 개념

- touch는 파일이 없으면 새로 생성, 있으면 수정시간만 업데이트함
- mkdir은 여러 개의 디렉토리를 한 번에 만들 수 있어 반복 작업 효율 극대화
- cd, mkdir, touch, ls는 현업/실습/스크립트에서 가장 기본적이면서 많이 쓰이는 명령어!

CLI 환경(터미널)에서 파일 내용을 직접 작성·수정할 때 대표적으로 쓰는 텍스트 에디터

Vim은 키보드 기반으로 강력하지만 초보자에겐 다소 까다로울 수 있음

우리는 여기서 Vim의 핵심 모드와 기본 사용법만 실습!

---

## Vim 실행

- unix_practice 폴더에서

```
vim 파일명
```

- 아무 아규먼트 없이 실행: 새 빈 파일 또는 vim 자체 콘솔 실행(저장·종료 시 이름 지정)
- 파일명(경로)과 함께 실행: 해당 파일 내용 바로 편집 가능

---

## 입력·편집 실습

1. vim 실행

```
vim hello.txt
```

1. 일반 모드에서 `i` 눌러 입력 모드로 진입(화면 아래에 --INSERT-- 뜸)
2. 텍스트 입력

```
hello world
```

1. esc 눌러 일반 모드 복귀
    1. insert가 사라짐
2. 명령 모드로 이동

```
:wq
```

:wq는 파일 저장 후 종료(Write & Quit)

---

## Vim의 4가지 모드

| 모드명 | 이동 키 | 주요 역할 | 화면 표시 |
| --- | --- | --- | --- |
| 일반 모드 | (기본) | 탐색, 복사, 삭제, 명령 | 아무 표시 없음 |
| 입력 모드 | i | 텍스트 입력, 편집 | --INSERT-- |
| 비주얼 모드 | v, V | 텍스트/라인 선택 | --VISUAL--, etc. |
| 명령 모드 | : | 저장, 종료 등 명령 | (명령줄 입력) |
- 모드 전환:
    - 일반→입력: `i`, `a`, `o`
    - 입력→일반: `esc`
    - 일반→비주얼: `v`(부분)/`V`(전체라인)
    - 일반→명령: `:`(콜론)
    - 항상 esc로 일반 모드로 복귀 가능

---

## 핵심 팁, 단축키

- :q — 종료(저장 X)
- :w — 저장
- :wq — 저장하고 종료

---

**정리**

- Vim은 키보드 단축키/모드 전환이 핵심
- 일반→입력→일반→명령 모드 활용: esc, i, :, v를 조합
- 파일 편집/입력을 CLI 환경에서 편리하게 할 수 있음

### 10. vim으로 파일 작성하기

### 1. 텍스트 입력: 입력 모드 ⭐⭐

- 일반 모드에서 `i` 키
- 입력 모드로 진입(화면에 --INSERT-- 표시)
- 원하는 텍스트 입력
    
    ```markdown
    the four modes of vim are:
    - normal mode
    - insert mode
    - visual mode
    - command mode
    ```
    
- 입력 후 `esc` 키로 일반 모드 복귀

---

### 2. 텍스트 복사/잘라내기/붙여넣기 (마우스를 쓰지 않고)

### 비주얼 모드로 영역 선택

- 글자 단위: `v`
- 줄 단위: `V`
- 원하는 만큼 커서 이동하여 영역 선택

### 복사, 잘라내기, 붙여넣기 명령

- 복사: `y`
- 잘라내기(삭제): `d`
- 붙여넣기: `p`

### 전체 줄 즉시 복사/삭제

- 복사: `y`
- 잘라내기: `d`
- 마지막에 `p` 로 붙여넣기

### 비주얼 모드로 변경하지 않고 복사/붙여넣기 하기

- 복사: `yy`
- 붙여 넣기: `p`

---

### 3. 저장 및 종료: 명령 모드 ⭐⭐

- 명령 모드 진입: `:`
- 저장: `:w`
- 저장 + 종료: `:wq`
- 종료(저장 X): `:q`
- 강제 종료(수정내용 저장 안함): `:q!`
- (새 파일 저장시 `:w filename.txt` 형식으로 입력)

---

### 4. 실습 예시 흐름

1. vim 파일 실행

```
vim hello.txt
```

1. 입력 모드(i) → 텍스트 입력 → esc
2. 줄/글자 비주얼 모드(v/V) → y/d → p
3. `:wq`로 저장/종료, `:q!`로 강제 종료

### 파일 전체 내용 확인: cat

- `cat [파일명]` 형식으로 입력하면 파일의 모든 내용을 한 번에 출력할 수 있음
- cat은 `concatenate`(이어붙이다)를 의미, cat 커맨드에 여러 파일을 아규먼트로 줄 수도 있음
    - 여러 파일을 동시에 입력하면, 그 파일들의 내용이 이어져서 한 번에 표시

```markdown
cat bohemian_rhapsody.txt
cat bohemian_rhapsody.txt killer_queen.txt
```

---

### 페이지 단위로 확인: less

- `less [파일명]`을 입력하면 파일을 페이지로 나눠서 편하게 볼 수 있음
- 방향키
- 스페이스바
- 페이지 이동: `f`(앞), `b`(뒤)
- 처음: `g`,  마지막 페이지: `G`
- 종료:  `q`

```markdown
less bohemian_rhapsody.txt
```

---

### 처음 부분만 확인: head

- `head [파일명]`을 입력하면 파일의 처음 10줄만 보여줌
- `head -n [숫자] [파일명]` 으로 원하는 줄 개수 지정도 가능

```markdown
head bohemian_rhapsody.txt
head -n 5 bohemian_rhapsody.txt
```

---

### 끝 부분만 확인: tail

- tail [파일명]을 입력하면 파일의 마지막 10줄만 보여줌
- tail -n [숫자] [파일명] 으로 원하는 줄 개수 지정도 가능함

```markdown
tail bohemian_rhapsody.txt
tail -n 5 bohemian_rhapsody.txt
```

## 파일 및 디렉토리 이름 변경: mv

- 파일이나 폴더의 이름을 바꿀 때 사용
- 예시:

```markdown
// 현재 경로 reports
mv performance.txt performance2.txt
mv Sep Oct
```

---

## 디렉토리 및 파일 이동: mv

- mv에서 두 번째 인자가 이미 존재하는 “디렉토리”라면, 해당 파일이나 폴더를 그 디렉토리 안으로 이동
- 예시:

```markdown
mv Jul Aug      # Jul 디렉토리가 Aug 안으로 이동
mv finances.txt Sep   # finances.txt 파일이 Sep 디렉토리로 이동
mv performance.txt Sep    # performance.txt 파일이 Sep 디렉토리로 이동
```

- 디렉토리를 다시 꺼내오려면 상대 경로 활용:

```markdown
mv Aug/Jul .    # 현재 디렉토리(=reports)로 Jul 디렉토리를 이동
```

---

## mv 커맨드 핵심 원리

- 두 번째 인자가 **존재하는 디렉토리**면 → 그 디렉토리로 이동
- **존재하지 않는 이름**이면 → 이름 변경
- 파일/폴더 모두 mv로 이동/변경이 가능

---

## 경로 활용 팁

- 파일명/폴더명을 쓸 때 항상 “현재 위치”(`.`)와 상대 경로(예: Aug/Jul)를 잘 파악해야 정확하게 이동/이름변경 가능
- mv는 실습·프로젝트에서 파일 관리의 핵심이므로 꼭 익혀야 함

### 15. mv 커맨드 사용시 주의할 점

### mv 커맨드 사용 시 주의점: 덮어쓰기(Overwrite)

- mv로 파일/디렉토리 이름을 바꾸거나 이동할 때 목적지에 같은 이름의 파일/폴더가 이미 있으면, 기존 것이 덮어써진다는 점에 매우 유의해야 합니다.
- GUI 환경(윈도우/맥/리눅스 GUI 파일탐색기 등)에서는 덮어쓸 것인지 묻는 창이 나오지만,
    
    mv 커맨드는 물어보지 않고 곧바로 “기존 파일을 덮어쓰기” 함
    

### 예시

```markdown
// test1.txt
Test file 1

// test2.txt
Test file 2
```

```markdown
mv test1.txt test2.txt
```

- test1.txt → test2.txt로 이름 변경(동시에 test2.txt의 내용이 test1.txt로 "덮어써짐")
- 이후 test2.txt의 내용은 “Test file 1”로 바뀌고, test1.txt는 사라짐

---

### 안전하게 사용: -i (interactive) 옵션

- mv에 **i 옵션을 붙이면 덮어쓰기 전에 “진짜로 덮어쓸까요?” 라고 먼저 묻는다**
- 선택지:
    - y(Yes): 덮어쓰기 진행
    - n(No): 작업 취소

### 예시

```markdown
mv -i test1.txt test2.txt
```

- “overwrite 'test2.txt'? ”라는 메시지가 나옴
- n을 누르면 기존 파일/내용이 보존

---

### 정리

- mv는 이름 바꾸기, 위치 이동 모두 수행
- 같은 이름의 파일/폴더가 있을 경우 **기본값은 무조건 덮어쓰기(주의!)**
    - 안전하게 하려면 항상 **`mv -i`**로 방지
- 실수 방지가 최우선이라면 항상 `-i`를 습관화하세요!
- (추가) 모든 mv에 대해 mv -i가 자동으로 적용되게 별명 설정(alias)도 가능

### 실습 해보기

- **현재 디렉토리 위치**
    
    reports 디렉토리 안에서 작업 시작
    
- **디렉토리 이동**
    
    ```markdown
    cd Jul
    ```
    
- **Jul 디렉토리 내용 확인**
    - 파일: finances.txt, performance.txt
    - finances.txt → "2025 July Finance Report"
    - performance.txt → "2025 July Performance Report"

---

- **파일 복사 (같은 디렉토리 내)**
    
    ```markdown
    cp finances.txt finances2.txt
    ls
    cat finances2.txt
    ```
    
    - finances2.txt 파일 생성 및 내용 복사 확인

---

- **파일 복사 (다른 디렉토리로)**
    
    ```markdown
    cp finances.txt ../Aug
    ```
    
    - Jul 디렉토리의 finances.txt를 Aug로 복사
    - Aug 디렉토리에 동일한 이름의 파일이 있으면 **덮어씀**

---

- **덮어쓰기 방지 옵션 (-i)**
    
    ```markdown
    cp -i finances.txt ../Aug
    ```
    
    - 복사 시 목표 파일이 존재하면 덮어쓸지 확인
    - “n” 입력 시 취소
    - 실수로 기존 파일을 덮어쓰지 않도록 보호 가능

---

- **디렉토리 복사 시 옵션 사용 (-r)**
    
    ```markdown
    cd ..
    cp Jul Jul_copy     # 오류 발생
    cp -r Jul Jul_copy  # 올바른 명령
    ls
    ls Jul_copy
    ```
    
    - `Jul_copy` 디렉토리 생성 및 내부 파일 구조 동일
    - 디렉토리는 `r` 옵션으로 재귀적으로 복사해야 함

---

### 옵션 정리

| 옵션 | 의미 | 사용 예시 |
| --- | --- | --- |
| `-i` | 덮어쓰기 전 사용자에게 확인 요청 | `cp -i file1 file2` |
| `-r` | 디렉토리 및 하위 구조 재귀적으로 복사 | `cp -r dir1 dir2` |

---

### 주의사항

- 동일 이름 파일이 있으면 기본적으로 덮어씀
- 덮어쓰기 방지에는 `i` 옵션 사용
- 디렉토리 복사 시 반드시 `r` 옵션 필요

---

### 용어설명

- **재귀적(recursive)**:
    
    어떤 과정이 자기 자신을 반복적으로 호출하는 구조
    
    cp 명령어에서는 하위 디렉토리까지 포함해 전체를 복사하는 것을 의미

## 실습하기

- **현재 디렉토리 위치**
    
    unix_practice 디렉토리 안에서 작업 시작
    
    내부에 warning 디렉토리 존재
    
- **디렉토리 이동 및 내용 확인**
    
    ```markdown
    cd warning
    ls
    ```
    
    - test1.txt, test2.txt 등의 테스트 파일 존재 (없으면 생성)

---

- **단일 파일 삭제**
    
    ```markdown
    rm test1.txt
    ls
    ```
    
    - test1.txt 파일 삭제 완료
    - 여러 파일 삭제 시 아규먼트를 추가해 한 번에 삭제 가능
        
        예: `rm test1.txt test2.txt test3.txt`
        

---

- **디렉토리 삭제 (r 옵션)**
    
    ```markdown
    cd ..
    rm warning       # 오류 발생
    rm -r warning    # 올바른 명령
    ls
    ```
    
    - 디렉토리를 삭제할 때는 `r` 옵션 반드시 필요
    - 하위 내용까지 재귀적으로 삭제됨

---

- **삭제 확인 옵션 (-i)**
    
    ```markdown
    cd reports
    rm -ri Jul_copy
    ```
    
    - 각 파일 및 디렉토리 삭제 전마다 확인 요청
    - “y” 입력 시 해당 항목 삭제
    - 최종적으로 Jul_copy 디렉토리 삭제 완료
    - 주의: `rm` 명령은 파일을 휴지통으로 이동시키지 않고 **영구 삭제**

---

- **강제 삭제 옵션 (-f)**
    
    `textrm -f filename
    rm -rf path/to/dir`
    
    - 삭제 시 어떤 확인도 하지 않음
    - 읽기 전용 파일도 경고 없이 삭제
    - `r`과 함께 자주 사용 (`rf` 형태)

---

### 옵션 정리

| 옵션 | 의미 | 사용 예시 |
| --- | --- | --- |
| `-r` | 디렉토리 및 하위 구조 재귀적으로 삭제 | `rm -r folder` |
| `-i` | 각 파일 삭제 전 확인 요청 | `rm -i file1 file2` |
| `-f` | 강제 삭제, 확인 없이 실행 | `rm -f read_only.txt` |

---

### 주의사항

- rm 사용 시 삭제된 파일은 복구 불가 (휴지통 없음)
- 디렉토리 삭제 시 `r` 옵션 필수
- `rf` 조합은 매우 강력하므로 경로를 반드시 확인 후 사용
- `i`는 안전하지만 대량 삭제 시 비효율적일 수 있음

---

### 용어설명

- **재귀적(recursive)**
    
    자신 안의 하위 구조를 반복적으로 순회하는 방식
    
    디렉토리 삭제 시 내부의 모든 파일과 서브디렉토리를 포함해 처리
    
- **강제(force)**
    
    오류나 확인 절차를 무시하고 실행하는 동작
