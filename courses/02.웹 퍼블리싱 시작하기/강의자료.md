### 02. VS Code 사용법 간단하게 배워보기

VS Code는 파일과 폴더를 관리하고 코드를 편집하는 기본 개발 도구입니다. 이 절에서는 HTML/CSS 실습에 필요한 파일 관리, 검색, 저장, 테마 설정을 다룹니다.

### 파일 생성하기

왼쪽 탐색기 영역의 새 파일 아이콘을 선택하면 파일을 생성할 수 있습니다.

- HTML 실습에서는 보통 `index.html`처럼 파일 이름과 확장자를 함께 입력합니다.
- 확장자는 파일의 종류를 나타냅니다. 예를 들어 `.html`은 HTML 문서, `.css`는 CSS 파일을 의미합니다.
- 파일 이름을 입력한 뒤 Enter를 누르면 편집기에서 해당 파일이 열립니다.

### 폴더 생성하기

왼쪽 탐색기 영역의 새 폴더 아이콘을 선택하면 폴더를 생성할 수 있습니다.

- 이미지나 CSS 파일처럼 역할이 다른 파일을 구분할 때 폴더를 사용합니다.
- 예를 들어 이미지 파일은 `images`, 스타일 파일은 `css` 폴더에 모아 둘 수 있습니다.
- 폴더 이름에는 공백보다 영문 소문자, 하이픈, 언더스코어를 사용하는 것이 일반적입니다.

### 검색 기능

파일 안에서 특정 단어를 찾을 때는 검색 기능을 사용합니다.

- 현재 파일 검색: `Cmd + F` 또는 `Ctrl + F`
- 전체 프로젝트 검색: `Cmd + Shift + F` 또는 `Ctrl + Shift + F`
- 태그 이름, 클래스 이름, 파일명처럼 찾으려는 단어를 입력하면 해당 위치를 빠르게 확인할 수 있습니다.

### 바꾸기 기능

반복해서 등장하는 단어를 한 번에 수정할 때는 바꾸기 기능을 사용합니다.

- 현재 파일 바꾸기: `Cmd + Option + F` 또는 `Ctrl + H`
- 전체 프로젝트 바꾸기는 적용 범위가 넓으므로, 먼저 검색 결과를 확인한 뒤 적용합니다.
- 수업 초반에는 전체 바꾸기보다 현재 파일 안에서 하나씩 확인하며 바꾸는 방식이 적합합니다.

### 프로필

- GitHub 회원 가입과 로그인은 이후 Git/GitHub 수업에서 다시 다룹니다.
- 이 단계에서는 VS Code에서 계정 로그인이 가능하다는 점만 확인합니다.
- 설정 동기화(Settings Sync)를 사용하면 테마, 확장 프로그램, 단축키 설정을 다른 컴퓨터에서도 이어서 사용할 수 있습니다.

### 자동 저장

자동 저장을 켜면 파일을 수정한 뒤 별도로 저장하지 않아도 변경 사항이 반영됩니다.

- 메뉴에서 `File` → `Auto Save`를 선택하면 자동 저장을 켜고 끌 수 있습니다.
- HTML/CSS 실습에서는 저장 여부에 따라 브라우저 화면의 결과가 달라질 수 있으므로, 자동 저장 상태를 먼저 확인합니다.
- 자동 저장을 꺼 둔 경우에는 `Cmd + S` 또는 `Ctrl + S`로 직접 저장합니다.

### 확장자와 테마 설정

- 색 테마는 편집기 전체의 색상을 바꾸는 설정입니다.
- 아이콘 테마는 파일 종류별 아이콘을 구분하기 쉽게 바꾸는 설정입니다.
- 아이콘 테마를 설정하면 HTML, CSS, JavaScript 파일을 더 쉽게 구분할 수 있습니다.

### 06. 나의 첫 웹 사이트

### 제안 항목 트리거 단축키 안내

VS Code에서 코드 제안 목록을 직접 열어야 할 때는 다음 단축키를 사용합니다.

- Windows/Linux: `Ctrl` + `Space`
- Mac: `Cmd` + `Space`
- 일부 Mac에서 Spotlight 검색 단축키와 겹치면 `Ctrl` + `Space`를 사용합니다.
- 두 단축키가 모두 동작하지 않으면 설정의 바로 가기 키에서 `제안 항목 트리거` 또는 `editor.action.triggerSuggest`를 검색합니다.

```html
<!DOCTYPE html>
```

### <!DOCTYPE html> 이란?

`<!DOCTYPE html>`은 이 문서가 HTML5 문서라는 사실을 브라우저에 알려 주는 선언입니다. HTML 파일의 가장 위에 작성하며, 대부분의 현대 웹 페이지에서 동일하게 사용합니다.

예를 들어 네이버나 구글 같은 웹 사이트도 브라우저가 HTML 문서를 올바르게 해석할 수 있도록 문서 형식을 선언합니다.

### 태그란?

![스크린샷 2025-10-23 16.05.44.png](02%20%EC%9B%B9%20%ED%8D%BC%EB%B8%94%EB%A6%AC%EC%8B%B1%20%EC%8B%9C%EC%9E%91%ED%95%98%EA%B8%B0/%E1%84%89%E1%85%B3%E1%84%8F%E1%85%B3%E1%84%85%E1%85%B5%E1%86%AB%E1%84%89%E1%85%A3%E1%86%BA_2025-10-23_16.05.44.png)

HTML 태그는 웹 페이지의 구조와 내용을 정의하는 기본 문법 단위입니다. 일반적으로 태그는 시작 태그와 종료 태그로 내용을 감쌉니다. 시작 태그는 `<태그 이름>` 형태로 작성하고, 종료 태그는 `</태그 이름>` 형태로 작성합니다.

`html` 태그는 HTML 문서 전체를 감싸는 최상위 태그입니다. 다음 예시처럼 `html` 태그 안에 화면에 표시할 문장을 작성할 수 있습니다. HTML 문서 구조는 이후 단원에서 각 요소의 역할을 다시 정리합니다.

파일은 메뉴에서 File-Save를 선택하거나 저장 단축키를 사용해 저장합니다.

```html
<!DOCTYPE html> --> 무엇인지 소개
<html>
	안녕 HTML!
</html>
```

```html

	</head>

		안녕 HTML
	</body>
</html>
```

### <head> 태그

`<head>` 태그는 문서 자체에 대한 정보를 담는 영역입니다. 브라우저 화면에 직접 보이는 내용보다는 문서 제목, 문자 인코딩, 외부 파일 연결처럼 페이지 전체에 영향을 주는 정보를 넣습니다.

### <body> 태그

`<body>` 태그는 브라우저 화면에 실제로 보이는 내용을 담는 영역입니다. 텍스트, 이미지, 링크처럼 사용자가 보는 시각적 요소는 기본적으로 `<body>` 안에 작성합니다.

### 태그 구조 정리

- **시작 태그:** `<태그명>`
- **종료 태그:** `</태그명>`
- **내용:** 시작 태그와 종료 태그 사이에 들어가는 값
- **`<title>`:** 브라우저 탭에 표시될 문서 제목
- **`<head>`:** 문서 정보 영역
- **`<body>`:** 화면에 보이는 내용 영역

### 03. 한글이 깨져요! (<meta> 태그, UTF-8) ⭐

**인코딩이란?**

- 인코딩(Encoding)이란, 사람이 이해할 수 있는 문자, 데이터, 멀티미디어 등을 컴퓨터가 처리할 수 있는 형식(예: 0과 1로 이루어진 이진수, 또는 특정 규칙을 적용한 코드)으로 변환하는 과정이나 방법을 의미합니다.

---

**예시 설명**

- 텍스트 인코딩: '가나다'와 같은 문자를 컴퓨터가 처리할 수 있도록 숫자(코드값)로 변환
    - 예) "A"라는 문자를 ASCII 코드(65)로, "가"라는 한글을 UTF-8(234 176 128) 등으로 변환
- 파일 인코딩: 이미지를 JPG, 오디오를 MP3, 동영상을 MP4 등 특정 포맷으로 저장

---

**어디에 쓰이나?**

- **데이터 저장**: 파일, 텍스트, 이미지 등 모든 정보가 컴퓨터에 저장될 때 인코딩됨
- **데이터 전송**: 인터넷에서 문서, 파일, 데이터 송수신 시에도 인코딩 방식을 맞춰야 함
- **웹 개발**: 브라우저가 한글, 영어, 특수기호 등 다양한 문자를 깨지지 않고 잘 보여주기 위해 문자 인코딩(UTF-8, EUC-KR 등)을 사용

---

**인코딩과 암호화는 다르다!**

- **인코딩**: 누구나 규칙만 알면 원래 데이터로 복원(디코딩)할 수 있음. 표현 방식의 표준화, 호환성이 목표
- **암호화**: 보안이 목적, 임의의 키(key)가 없으면 풀 수 없음

---

**한마디로 요약**

- **인코딩**: 사람이 쓰는 정보를 컴퓨터가 처리할 수 있도록 약속된 규칙대로 바꾸는 것
- 디코딩(decoding)은 그 반대 과정(코드를 다시 문자, 데이터 등으로 해석)

**HTML 파일에 UTF-8 인코딩 적용하기**

```html
<!DOCTYPE html>
<html>
	<head>
		<title>Weekly Codeit</title>
		<meta charset="utf-8">
	</head>
	<body>
		안녕 HTML
	</body>
</html>
```

**<meta charset="utf-8">의 의미**

이 HTML 문서가 UTF-8 방식의 문자 인코딩을 사용한다는 의미입니다.

---

**좀 더 쉽게 설명하면**

- "charset"은 **character set**(문자 집합, 문자 인코딩 방식)을 뜻합니다.
- **utf-8**은 전 세계 모든 언어(한글, 영어, 일본어, 이모지 등) 문자를 폭넓게 지원하는 표준 문자 인코딩 방식입니다.
- 브라우저는 이 태그를 보고 "이 문서를 읽을 때 글자를 어떻게 해석할지"에 대한 규칙을 확인합니다.
- 이 태그가 없거나 인코딩 방식이 맞지 않으면 한글, 특수문자, 이모지 등이 깨져서 "???", "ãƒ¼" 등 이상한 글자로 보이는 문제가 발생할 수 있습니다.

---

**결론**

```html
<meta charset="utf-8">
```

덕분에, 어떤 컴퓨터/브라우저에서도 한글, 특수문자, 이모지 등이 제대로 보이게 웹페이지의 문자 인코딩 방식을 명시적으로 지정하는 역할을 합니다.

**`<h1>`** 태그

- **가장 중요한 최상위 제목**입니다.
- 한 웹페이지 또는 큰 섹션에서 **주제를 대표하는 핵심 제목**에 사용합니다.
- 기본 스타일에서는 **가장 크고 두껍게** 표시됩니다.

예시:

```html
<h1>웹사이트 구조의 이해</h1>
```

**`<h2>`** 태그

- **`<h1>`**의 **하위에 속하는 부제목(섹션 제목)**을 나타냅니다.
- 한 문서의 주요 대제목 아래에서 **세부적으로 내용을 구분**할 때 사용합니다.
- **`<h1>`**보다는 작지만, 기본적으로 글씨가 크게 나오고 눈에 잘 띕니다.

예시:

```html
<h2>HTML 태그 종류</h2>
<h2>CSS의 기초</h2>
```

### 08. 줄 바꿈 (`<br>` 태그) ⭐

HTML에서는 코드 안에 줄바꿈을 넣어도 브라우저 화면에 그대로 반영되지 않습니다. 문장 중간에서 강제로 줄을 바꾸려면 `br` 태그를 사용합니다.

`br` 태그는 별도의 내용을 감싸지 않는 빈 태그입니다. 문장 뒤에 `<br />`를 넣으면 해당 위치에서 다음 줄로 내려갑니다.

```html
  <h2>코둥이 퀴즈</h2>
  <p>
    머신 러닝에서 유저에게 유저가 좋아하거나<br /> <!-- 여기 추가 -->
    구매할만할 상품을 추천해 주는 프로그램을<br /> <!-- 여기 추가 -->
    □□ □□□ (이)라고 부른다.
  </p>
  <p>
    빈 칸에 들어갈 다섯 글자를 답장으로 보내주세요.
    선착순 200분께 코드잇 시그니쳐 굿즈를 보내드려요.
  </p>
```

### 08. #스크립트 줄 바꿈 (`<br>` 태그)

> HTML에서는 기본적으로 줄바꿈이 있더라도 화면에 반영되지 않았습니다.
> 창이 넓어지면 넓어진 만큼 글을 채워서 보여주고 창이 좁아지면 알아서 글을 넘겨서 보여주기 때문인데요.
> 내가 원하는 곳에서 줄을 바꾸고 싶으면 어떻게 해야 할까요?
> 이럴 때 바로 `<br />` 태그를 사용하는데요.
> 한번 적용해 보겠습니다.

> br은 영어로 줄을 바꾼다는 뜻인 break line의 약자입니다.
> 각 문장 끝에다가 br 태그를 넣어 줄게요.
> `br` 태그는 앞에서 살펴본 `meta` 태그처럼 내용을 감싸지 않는 빈 태그입니다. 문장 중간에 `br` 태그를 넣으면 해당 위치에서 줄바꿈이 적용됩니다. 같은 방식으로 다른 문장에도 줄바꿈을 적용할 수 있습니다.

> 확인해 보면 잘 나오는 것을 확인할 수 있습니다.

### 10. 꺾쇠 기호 넣는 법

### 꺾쇠를 그냥 쓰면 안 돼요!

HTML 문법에서는 꺾쇠 기호(`<`, `>`)를 태그 작성에 사용합니다. 따라서 페이지 내용에 꺾쇠 기호 자체를 표시하려면 별도의 표현 방법이 필요합니다. 예를 들어 `html`, `head`, `body`라는 단어를 꺾쇠와 함께 표시하는 경우를 살펴봅니다.

```html
<h2>Tips & Tricks: HTML 코드 편하게 입력하는 법</h2>
<p>
  HTML 파일을 작성하려면 DOCTYPE, <html>, <head>, <body> ... 적어야 할 것이 참
  많습니다. 이럴 때 유용하게 쓸 수 있는 기능이 있는데요. VS Code에서는
  !(느낌표)를 입력하고 Tab 키나 Enter 키를 누르면 HTML 코드를 한 번에
  입력할 수 있답니다!
</p>
```

위와 같이 작성하면 꺾쇠 기호가 문자로 표시되지 않고, 일부 단어가 사라질 수 있습니다. 웹 브라우저가 `<html>`, `<head>`, `<body>`를 일반 텍스트가 아니라 HTML 태그로 해석하기 때문입니다.

### 꺾쇠 기호를 넣는 올바른 방법

꺾쇠를 문자로 표시하려면 꺾쇠를 그대로 쓰지 않고 `&lt;`와 `&gt;`를 사용합니다. `lt`는 less than의 약자로 `<`를 의미하고, `gt`는 greater than의 약자로 `>`를 의미합니다.

```html
<h2>Tips & Tricks: HTML 코드 편하게 입력하는 법</h2>
<p>
  HTML 파일을 작성하려면 DOCTYPE, &lt;html&gt;, &lt;head&gt;, &lt;body&gt; ... 적어야 할 것이 참
  많습니다. 이럴 때 유용하게 쓸 수 있는 기능이 있는데요. VS Code에서는
  !(느낌표)를 입력하고 Tab 키나 Enter 키를 누르면 HTML 코드를 한 번에 입력할
  수 있답니다!
</p>

```

### 예시로 알아보는 Hex와 RGB

예시 색상: `#0000ff`

- **Hex 코드**
    - **앞 2자리(`00`)** : 빨간색(Red) 값 → 16진수 00 = 10진수 0
    - **중간 2자리(`00`)** : 초록색(Green) 값 → 16진수 00 = 10진수 0
    - **마지막 2자리(`ff`)** : 파란색(Blue) 값 → 16진수 ff = 10진수 255
- **RGB (0, 0, 255)의 의미**
    - R(Red) = 0 → 빨간색 없음
    - G(Green) = 0 → 초록색 없음
    - B(Blue) = 255 → 파란색 완전채도(최대값)

**즉, #0000ff는 RGB로 (0, 0, 255)이기 때문에 완전한 파란색이 됩니다.**

- 16진수 **`ff`  가 255가 되는 이유**
    - **16진수 `ff`가 10진수 `255`가 되는 이유**는 16진법의 계산 방식 때문입니다.
        - **16진법:**

            한 자리당 0~15까지 표현(**`0, 1, ... 9, a, b, c, d, e, f`**)

            - a = 10, b = 11, ... f = 15
        - **ff 계산 방법:**
            - 첫째자리(f) = 16의 1승 × 15 = 16 × 15 = 240
            - 둘째자리(f) = 16의 0승 × 15 = 1 × 15 = 15
            - 둘을 더하면 **240 + 15 = 255**

    ```html
    f × 16⁰ + f × 16¹ = (15 × 1) + (15 × 16) = 15 + 240 = 255
    ```

- 그래서 10진수로 255가 됩니다.

### 02. #스크립트 배경색

> 본격적으로 CSS를 사용해 봅시다. 우선 배경색을 넣어볼게요. 완성된 페이지를 보면 맨 위에는 보라색 배경이 있고 중간중간에 옅은 회색, 옅은 보라색 그리고 맨 아래에는 어두운 보라색이 있는데요. 이런 식으로 배경색을 넣어 볼 겁니다. 앞에서 div 태그랑 span 태그로 영역을 나눴던 거 기억나시나요? 나눠놓은 영역에다가 배경색을 넣어 볼 건데요. div 태그의 시작 태그에다가 style이라는 속성을 추가해서

> 그리고 background-color라는 건 CSS 속성이라고 부르는 문법인데요.
>
>
> 태그의 속성이랑 헷갈리지 않도록 주의하세요.
>
> CSS 속성은 이런 식으로 :으로 적으면 되는데
> background-color 속성에 purple이라는 속성 값을 넣는 코드니까 배경색을 보라색으로 하라는 의미죠.

> `purple` 대신 `#7542e0` 색상 코드를 사용하면 더 구체적인 보라색을 지정할 수 있습니다.
> 현재 단계에서는 글자색이 기본값으로 표시됩니다. 글자색은 이후 글자색 단원에서 별도로 조정합니다. 같은 방식으로 아래 영역에도 배경색을 지정할 수 있습니다.

> 마찬가지로 style 속성을 추가하고 background-color: #fbfbff 이렇게 넣어주겠습니다. 저장하고 확인해 보면 잘 안 보이실 수도 있는데 이쪽이 옅은 회색으로 들어가 있습니다.

> 마찬가지로 나머지 배경색도 지정해주겠습니다. 복사 붙여넣기를 사용할게요.
> 코둥이 퀴즈의 div에도 옅은 회색을 넣어주고 코둥이 퀴즈 안쪽에는 이렇게 #f6f1ff, 이렇게 옅은 보라색을 넣어주고요.

> 이번 레슨에서는 CSS로 배경색을 넣어 봤습니다.
>

### 06. 글꼴

글꼴은 텍스트의 모양을 결정합니다. CSS에서는 `font-family` 속성을 사용해 사용할 글꼴을 지정합니다.

브라우저는 `font-family`에 적힌 글꼴을 왼쪽부터 차례대로 확인합니다. 사용할 수 없는 글꼴이 있으면 다음 글꼴을 시도하므로, 마지막에는 `sans-serif`처럼 일반 글꼴 계열을 함께 적는 것이 일반적입니다.

### font-family 간단 설명

```html
<body style="color: #737373; font-family: ;">
```

### Sans-serif

```html
<body style="color: #737373; font-family: sans-serif;">
```

### Serif

```html
<body style="color: #737373; font-family: sans-serif;">
```

### 특정 글꼴 정하기

```html
 <body style="color: #737373; font-family: Arial">
```

### 대체 폰트 사용하기(영상하고 다를 수 있음)

```html
font-family: arial, Georgia, 'Times New Roman', Times, serif, sans-serif;
```

### 06. #스크립트 글꼴

> 페이지를 꾸미다 보니까 글자색이랑 배경색은 캐주얼하게 넣었는데 글자 모양이 너무 진지해서 안 어울립니다. 이번에는 글자 모양을 좀 더 예쁜 걸로 넣어 볼게요. body 태그에다가 이번에는 font-family라는 CSS 속성을 지정해 보겠습니다. 참고로 여기 color가 있으니까 세미콜론을 넣고 추가해 주면 됩니다.

> 이렇게 하면 페이지 모든 텍스트에 기본적으로 글꼴을 적용할 수 있는데요. 여기서 font라는 건 글자의 모양을 부르는 말인데 우리말로는 글꼴이라고도 부릅니다. 뒤에 family는 글꼴을 여러 개 지정할 수 있다는 뜻입니다. 이건 뒤에서 자세히 설명드릴게요.

> 우선 sans-serif라고 적어 봅시다. 저장하고 확인해 보면 아까랑 다르게 글자가 뭉툭해졌습니다. sans-serif는 글꼴의 종류를 부르는 말인데요. serif라는 건 글자 끝에 있는 뾰족한 부리 같이 생긴 겁니다.

> sans-serif라고 하면 serif가 없다 즉 뾰족한 부리가 없다는 뜻이죠. 한글 글꼴 중에서는 대표적으로 고딕체나 돋움체가 있습니다. sans-serif라고 해 놓으면 웹 브라우저가 내 컴퓨터에 설정되어 있는 sans-serif 기본 글꼴을 골라서 보여주는데, 저는 기본값이 sans-serif라 변화가 없습니다.

> 이번에는 sans-serif 대신에 serif라고 해 볼게요. 한글 글꼴 중에서 serif는 대표적으로 명조나 궁서체가 있는데요. 이런 뾰족한 글씨체입니다.

> 그런데 이렇게 serif, sans-serif라고 크게 크게 정하는 건 좀 별로입니다.
> 우리가 원하는 글꼴을 딱 정하고 싶은데요. 그럴 때는 글꼴 이름을 적어 주면 됩니다. 지금 제 컴퓨터에는 Arial이라는 글꼴이 깔려 있는데요. 여러분이 윈도우즈를 사용하고 계시더라도 Arial 글꼴은 아마 기본으로 깔려 있을 겁니다.

> 저장하고 확인해 보면 글꼴이 바뀐 거 보이시죠? 무슨 차이인지 모르겠다면 F12를 눌러서 확인 해보겠습니다.

> 근데 Arial 글꼴은 영문만 있는 거라서 한글은 적용이 안 되는데요. 이렇게 글꼴이 적용되지 않는 경우를 위해서 font-family를 지정해 줄 수 있습니다. `XXX`처럼 대체 글꼴을 적어 두면 Arial을 적용할 수 없는 경우에는 `XXX` 글꼴을 적용합니다. 그런데 `XXX`이 없는 컴퓨터도 있겠죠?

> 그래서 또 하나를 추가해서 이런 식으로 지정하면 XXX 이 없는 경우에는 YYYY을 보여줄 수 있고 맨 마지막에는 sans-serif를 넣어서 기본 sans-serif로 보여 달라고 정해 줄 수도 있습니다. 정리하자면 font-family라는 CSS 속성에서는 이렇게 글꼴을 여러 개 정해 놓으면 웹 브라우저가 차례대로 확인하면서 가능한 글꼴을 적용해 줍니다.

### 14. #스크립트 크기

> 우리 사이트를 작게 보면 이쁜데 윈도우를 넓게 보면 이미지가 상대적으로 작아서 볼품이 없어 보입니다. 그리고 이런 식으로 본문이 삐져 나와서 보기가 안 좋은데요. 이번 레슨에서는 이미지랑 본문 크기를 조절해 보겠습니다.

> 우선 이미지에 너비랑 높이를 지정해 볼게요. 너비는 width, 높이는 height라는 CSS 속성을 사용하면 되는데요. width: 700px; height: 375px; 이렇게 적어 보겠습니다.

> 이렇게 하면 너비는 700px, 높이는 375px이 됩니다. 저장하고 확인해 보면 크기가 더 커졌네요. 근데 이렇게 px로 하면 크기가 고정되니까 좀 아쉽습니다.

> 웹 브라우저 너비는 매번 다를 수 있거든요. 그래서 이번엔 화면에 꽉 차게 만들어 볼게요. 700px 대신에 100%라고 써 보겠습니다. 퍼센트 단위는 처음 배우는 건데요.

> 너비에 이렇게 퍼센트 단위를 쓰면 이 영역 안에서 이 퍼센트만큼 채워줍니다. 100%는 꽉 채워 주는 거죠. 저장하고 확인해 보면 너비가 넓어진 만큼 이미지도 넓어지네요. 근데 보시면 이렇게 이미지가 찌그러지죠?

> 이건 이미지 높이가 375px로 고정돼 있어서 그렇습니다. 이럴 때는 높이 값을 지워주면 이미지 같은 경우에는 비율에 맞게 알아서 보여줍니다. 저장하고 확인해 보면 비율을 유지하면서 이렇게 커지네요. 나머지 이미지들에도 이렇게 적용해 주겠습니다.

> 이번에는 이미지랑 본문을 감싸고 있는 이 div에다가 너비를 정해 줄게요. 이런 div에다가 너비를 정하고 싶은데 여기 배경색이 있는 div 태그에는 width를 추가하면 배경색이 적용되는 범위도 같이 줄어들겠죠? 그래서 배경색은 그대로 두고 이 안에 있는 거에만 너비를 적용하고 싶은데요. 그래서 안쪽에다가 div 태그를 하나 더 추가해 줄게요.
> 그리고 width: 560px을 추가합니다.

> 이렇게 하면 이 div 태그 안에 있는 건 너비가 560px이 되고 이 안에 있는 img 태그는 그 안에서 100%, 즉 560px에 꽉 차게 됩니다.

> 나머지 영역도 마찬가지로 div를 감싸고 너비를 넣겠습니다. div를 추가해 주고 너비를 넣은 다음에, div로 감싸주고요.

> 이제 영역의 너비가 560px이기 때문에 이렇게 본문이랑 이미지가 560px에 꽉 차게 됩니다. 그리고 배경색은 여전히 꽉 차게 이렇게 적용돼 있죠?

### 15. 여백 ⭐⭐⭐

여백은 요소 사이의 간격을 조절하는 핵심 CSS 개념입니다. 요소의 안쪽 여백은 `padding`, 요소의 바깥쪽 여백은 `margin`으로 지정합니다.

`padding`은 내용과 테두리 사이의 공간을 늘리고, `margin`은 요소 바깥쪽에서 다른 요소와의 간격을 만듭니다. 두 속성을 구분하면 레이아웃을 더 안정적으로 조정할 수 있습니다.

> 이번에는 뉴스레터 내용이 있는 쪽에 여백을 조절해 볼게요. 여기 위쪽이 너무 딱 붙어 있고 아래쪽도 딱 붙어 있는데요. 이 여백들을 조절해 보겠습니다. 여기부터 여기까지는 하나의 div 태그로 감싸져 있는데요.

> 이 div 태그에다가 style="padding: 40px" 이렇게 넣어 주겠습니다. 이렇게 하면 영역에 안쪽 여백으로 40px만큼 생기겠죠? 저장하고 확인해 보면 이런 식으로 여백이 들어갔습니다.

> 이렇게 하면 영역에 안쪽 여백으로 40px만큼 생기겠죠? 저장하고 확인해 보면 이런 식으로 여백이 들어갔습니다. 이 상태에서 가로 여백만 20px로 줄이고 싶으면

> 여기다가 margin: 32px을 넣으면 상자 바깥으로 간격이 늘어난 것 보이시나요? margin은 이런 식으로 바깥 여백을 조절하는 속성입니다.

> 여기서 0이라는 값은 0px이랑 똑같은 값인데 보통 짧게 0으로 씁니다. 저장하고 확인해 보면 제법 그럴싸해졌습니다.

> 확인해 보면 세로로는 바깥 여백을 추가하지 않고 가로로는 알아서 공간을 채웠습니다.

> 나머지 부분도 복사 붙여넣기로 적용해 줄게요.

### 02. HTML과 CSS를 공부하려면 책을 구매해야 하나요?

HTML과 CSS는 책 한 권을 순서대로 암기하는 방식보다, 필요한 개념을 공식 문서와 예제로 확인하며 익히는 방식이 적합합니다.

### 03. 코딩은 암기가 아니라 검색입니다

모든 태그와 CSS 속성을 암기할 필요는 없습니다. 개발자는 필요한 개념을 문서에서 확인하며 문제를 해결합니다. 다음 사이트는 HTML과 CSS를 학습할 때 자주 활용되는 자료입니다.

- **Google**
- **MDN (Mozilla Developer Network):** 웹 표준을 확인할 때 가장 기본이 되는 공식 문서입니다. 모르는 태그나 CSS 속성이 있다면 먼저 확인합니다.
- **Can I use... ([caniuse.com](https://caniuse.com/)):** 최신 CSS 속성이 크롬, 사파리, 엣지 등 주요 브라우저에서 지원되는지 확인할 수 있는 사이트입니다.

### 04. 브라우저 개발자 도구(F12)와 친해지세요.

웹 페이지에서 원하는 요소를 마우스 오른쪽 버튼으로 클릭한 뒤 **'검사'(Inspect)**를 선택하면 개발자 도구를 열 수 있습니다. 개발자 도구는 HTML 구조와 CSS 적용 상태를 확인하는 기본 도구입니다.

- **Elements 탭:** 현재 페이지의 HTML 구조와 각 요소에 적용된 CSS를 실시간으로 확인할 수 있습니다.
- **Styles 탭:** 특정 요소에 적용된 CSS를 확인하고 값을 임시로 변경할 수 있습니다. CSS가 원하는 대로 적용되지 않을 때 원인을 찾는 데 유용합니다.

### 05. 작은 토이 프로젝트로 연습하기

학습한 내용을 익히려면 직접 작은 결과물을 만들어 보는 과정이 필요합니다. 처음부터 큰 프로젝트를 만들 필요는 없습니다.

- **추천 프로젝트:**

    - 간단한 자기소개 페이지 만들기
