### 01. #스크립트 Web의 탄생 - 원시 Web

```jsx
Web의 탄생을 이해하려면 먼저 인터넷과 Web의 관계를 구분해야 합니다.

비유하자면 **인터넷은 '도시'**와 같습니다. 전 세계의 컴퓨터와 네트워크 장비를 연결하는 거대한 기반 시설입니다.
반면 **Web은 그 도시 안에 있는 '건물 하나'** 또는 **'도로 위를 달리는 자동차 한 대'**와 같습니다.
인터넷이라는 전체 안에 존재하는, 수많은 서비스 중 하나일 뿐입니다.
```

```jsx
인터넷과 Web은 역사도 다릅니다.
1960년대에는 여러 연구기관과 컴퓨터를 안정적으로 연결하는 통신 방식에 대한 연구가 활발했습니다.
당시 통신망은 일부 장비에 문제가 생기면 전체 통신이 불안정해질 수 있었고,
이를 보완하기 위해 장애에 강한 분산형 통신 구조가 연구되었습니다.

인터넷의 중요한 특징 중 하나는 **'중앙이 없는 분산형 시스템'**입니다.
수많은 통신 장치들이 서로 연결되어,
한 지점에 문제가 생겨도 다른 경로를 통해 통신을 이어갈 수 있는 구조입니다.

이 분산 구조가 오늘날 구글, 페이스북 같은 거대한 Web 서비스를 지탱하는 기반이 되었습니다.

인터넷은 1960년대부터 약 30년 동안 천천히 전 세계로 확산되었습니다.
이 기간 동안 인터넷은 기업, 연구소, 대학, 군대 등 거대한 기관에서 사용되던 통신 시스템이었기
때문에 대중에게는 잘 알려지지 않았습니다.

Web은 1990년에 등장하여 인터넷이라는 기반 시설의 활용 범위를 대중적인 정보 공유 서비스로 확장했습니다.
```

```jsx
Web은 의외의 장소, 스위스 제네바에 위치한 **유럽 입자 물리 연구소**, CERN에서 태어났습니다.
이곳은 거대한 입자 가속기를 이용해 우주의 근원을 탐구하는 연구 기관입니다.
전 세계에서 모인 만 명 이상의 과학자와 엔지니어가 방대한 연구 데이터를 만들어 내는 곳이었습니다.
1980년대 말, **유럽 입자 물리 연구소**는 정보의 바다에서 허우적대는 '작은 지구'와 같았습니다.
이처럼 정보가 빠르게 늘어나는 환경에서는, 문서를 쉽게 연결하고 공유할 수 있는 새로운 시스템이 필요했습니다.
```

```jsx
1980년, **팀 버너스리**라는 비정규직 프로그래머가 **유럽 입자 물리 연구소**에 합류합니다.

그리고 1990년 말, 몇 달 사이에 Web의 핵심 요소들이 만들어졌습니다.

- 10월: 세계 최초의 '웹페이지 편집기' 개발
- 11월: 세계 최초의 'Web 브라우저' (이름도 'World Wide Web'이었습니다) 개발
- 12월 말: 세계 최초의 'Web서버'가 가동되고, `info.cern.ch` 라는 인류 최초의 Web 주소가 부여됩니다.

`info.cern.ch`는 인류 최초의 웹사이트 주소입니다.
이 사이트에는 Web이 무엇인지, 어떻게 사용할 수 있는지에 대한 가장 기본적인 정보가 담겨 있었습니다.

오늘날의 웹사이트와 비교하면 매우 단순하지만, 정보 문서를 링크로 연결한다는 Web의 본질은 이미 갖추고 있었습니다.
이러한 초기 형태를 원시 Web이라고 부를 수 있습니다.
```

### 01. #퀴즈 Web의 탄생 - 원시 Web (7분)

**문제 1**

인터넷과 Web의 관계를 흔히 도시에 비유합니다. '인터넷'이 도시 전체라면, 'Web'은 무엇에 해당한다고 볼 수 있을까요?

1. 또 다른 도시
2. 도시를 연결하는 도로
3. 도시 안의 건물 하나
4. 도시의 시장

- 정답 및 해설

    정답: 3. 도시 안의 건물 하나

    해설: 인터넷이라는 거대한 인프라(도시) 안에 Web은 수많은 서비스(건물) 중 하나라고 설명하고 있습니다.


**문제 2**

초기 인터넷이 일부 장비에 문제가 생겨도 통신을 이어가기 위해 활용한 핵심적인 구조는 무엇이었나요?

1. 중앙집중형 시스템
2. 분산형 시스템
3. 단일 서버 시스템
4. 암호화 통신 시스템

- 정답 및 해설

    정답: 2. 분산형 시스템

    해설: 분산형 시스템은 하나의 중앙 장치에 모든 통신이 의존하지 않는 구조입니다. 일부 장비나 경로에 문제가 생겨도 다른 경로를 통해 통신을 이어갈 수 있다는 점이 중요합니다.


**문제 3**

Web의 아버지, 팀 버너스리가 1990년 말에 Web을 탄생시킬 당시 소속되어 있던 연구소의 이름은 무엇일까요?

1. NASA (미국 항공우주국)
2. Bell Labs (벨 연구소)
3. CERN (유럽 입자 물리 연구소)
4. IBM Watson 연구소

- 정답 및 해설

    정답: 3. CERN (유럽 입자 물리 연구소)

    해설: Web은 스위스 제네바에 위치한 CERN에서, 수많은 연구자들의 정보 공유 문제를 해결하기 위한 목적으로 탄생했습니다.


**문제 4**

1990년 말, 팀 버너스리가 가동한 인류 최초의 웹사이트 주소는 무엇이었을까요?

1. cern.com
2. worldwideweb.org
3. timbl.com
4. info.cern.ch

- 정답 및 해설

    정답: 4. info.cern.ch

    해설: info.cern.ch는 팀 버너스리가 운영한 최초의 Web 서버와 연결된 주소입니다. 초기 Web의 목적과 사용 방법을 설명한 첫 웹사이트로 볼 수 있습니다.


**문제 5** · 단답형

스크립트에서 팀 버너스리가 1990년에 완성한, Web의 본질만을 담고 있는 순수한 형태의 Web을 무엇이라고 부르나요?

- 정답 및 해설

    정답: 원시 Web

    해설: 원시 Web은 복잡한 디자인이나 동적 기능보다, 문서와 문서를 링크로 연결해 정보를 공유한다는 Web의 핵심 기능이 두드러진 초기 형태를 의미합니다.


**문제 6** · 심화

info.cern.ch가 Web의 역사에서 중요한 이유를 한 문장으로 설명해 보세요.

- 정답 및 해설

    예시 답안: info.cern.ch는 최초의 Web 서버와 연결된 주소로, Web의 목적과 사용 방법을 설명한 초기 웹사이트였기 때문에 중요합니다.

    해설: 이 주소는 Web이 단순한 개념이 아니라 실제로 접속 가능한 정보 공유 시스템으로 구현되었음을 보여주는 대표적인 사례입니다.

### 01. #퀴즈 Web에서 정보 주고받기 (6분)

**문제 1**

정보를 요청하는 컴퓨터를 무엇이라고 부르나요?

1. 서버 (Server)
2. 클라이언트 (Client)
3. 인터넷 (Internet)
4. Web (Web)

- 정답 및 해설

    정답: 2. 클라이언트 (Client)

    해설: 클라이언트는 서버에게 정보를 요청하는 역할을 합니다.


**문제 2**

다음 중 서버(Server)에 대한 설명으로 가장 올바른 것은 무엇인가요?

1. 유튜브 영상을 시청하는 나의 스마트폰
2. 정보를 요청해서 제공받는 컴퓨터
3. 서비스를 사용하는 일반 사용자의 컴퓨터
4. 요청을 받아 정보를 제공해주는 컴퓨터

- 정답 및 해설

    정답: 4. 요청을 받아 정보를 제공해주는 컴퓨터

    해설: 서버는 클라이언트의 요청에 따라 정보를 제공(Serve)하는, 즉 응답하는 역할을 합니다. 1, 2, 3번은 모두 클라이언트에 대한 설명입니다.


**문제 3** · 단답형

우리가 앞으로 배울 내용에서, Web 브라우저에 보이는 화면을 만드는 '프론트엔드'는 서버와 클라이언트 중 어떤 역할을 하게 될까요?

- 정답 및 해설

    정답: 클라이언트

    해설: '프론트엔드'는 사용자가 브라우저에서 직접 보고 상호작용하는 화면을 의미합니다. 이 화면은 사용자에게 필요한 데이터(예: 게시글 목록)를 보여주기 위해 서버에게 정보를 요청하는 역할을 합니다. 이처럼 정보를 요청하는 쪽이 '클라이언트'이므로, 프론트엔드는 클라이언트의 역할을 담당하게 됩니다.


**문제 4** · 응용

온라인 쇼핑몰에서 옷을 주문하려고 합니다. 이 상황에서 '서버'의 역할을 하는 것은 무엇일까요?

1. 내 컴퓨터의 Web 브라우저 (크롬, 사파리 등)
2. 주문 요청을 처리하는 쇼핑몰의 컴퓨터 시스템
3. 내 컴퓨터의 운영체제 (윈도우, macOS 등)
4. 주문한 옷을 배송하는 택배 회사

- 정답 및 해설

    정답: 2. 주문 요청을 처리하는 쇼핑몰의 컴퓨터 시스템

    해설: 내 Web 브라우저(클라이언트)가 '주문'이라는 요청을 보내면, 쇼핑몰의 컴퓨터 시스템(서버)이 이 요청을 처리하고 주문 완료 결과를 응답합니다.


**문제 5** · 응용

Gmail을 사용해서 친구에게 이메일을 보냈습니다. 이때 내 컴퓨터는 ( A ) 역할을 한 것이고, Google의 컴퓨터는 ( B ) 역할을 한 것입니다. A와 B에 들어갈 말로 올바른 것은?

1. A: 서버, B: 클라이언트
2. A: 클라이언트, B: 서버
3. A: Web, B: 인터넷
4. A: 인터넷, B: Web

- 정답 및 해설

    정답: 2. A: 클라이언트, B: 서버

    해설: 내 컴퓨터(클라이언트)가 '이메일을 보내달라'는 요청을 한 것이고, Google의 컴퓨터(서버)가 그 요청을 처리한 뒤 결과를 응답한 것입니다.


**문제 6** · 응용

사용자가 브라우저에서 게시글 목록 페이지를 열 때 일어나는 흐름으로 가장 올바른 것은 무엇인가요?

1. 브라우저가 서버에 요청한다 → 서버가 응답한다 → 브라우저가 화면을 표시한다.
2. 서버가 먼저 브라우저에 요청한다 → 브라우저가 HTML을 만든다 → 서버가 화면을 표시한다.
3. DNS가 게시글을 만든다 → 브라우저가 데이터베이스를 저장한다 → 서버가 URL을 보여준다.

### 개발자 도구 여는 방법

- 단축키 (Windows): Ctrl + Shift + I 또는 F12

### 초보자를 위한 4가지 핵심 패널

개발자 도구에는 여러 탭(패널)이 있습니다. 처음에는 아래 네 가지 패널의 역할만 구분해도 충분합니다.

### 1. Elements 패널: HTML 구조와 CSS 스타일 확인하기

Elements 패널은 웹페이지의 현재 HTML 구조와 CSS 스타일을 보여주는 곳입니다.

![](01%20%ED%94%84%EB%A1%9C%EA%B7%B8%EB%9E%98%EB%B0%8D%20%EC%8B%9C%EC%9E%91%ED%95%98%EA%B8%B0/%E1%84%89%E1%85%B3%E1%84%8F%E1%85%B3%E1%84%85%E1%85%B5%E1%86%AB%E1%84%89%E1%85%A3%E1%86%BA_2026-06-02_18.03.10.png)

주요 기능:

- HTML 구조 확인: 삼각형(▶)을 클릭해 태그를 열어보며 페이지가 어떻게 구성되었는지 파악할 수 있습니다.
- CSS 스타일 확인 및 수정: 특정 HTML 요소를 클릭하면 오른쪽 Styles 탭에 해당 요소에 적용된 CSS 속성이 나타납니다. 체크박스를 끄거나 값을 바꾸면 페이지가 실시간으로 변합니다.
