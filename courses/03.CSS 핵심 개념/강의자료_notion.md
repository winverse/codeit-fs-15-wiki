#### CSS 규칙 복습

앞에서는 CSS 규칙, 선택자, 선언을 배웠습니다. 다음 예시는 제주 여행 정보를 보여 주는 간단한 페이지입니다. `body` 태그 안에는 `img`, `h1`, `p`, `h3` 같은 여러 태그가 들어 있습니다.


#### CSS 규칙과 선택자, 그리고 선언

`head` 태그 안에는 `style` 태그가 있습니다.
태그 이름과 중괄호로 이루어진 부분을 CSS 규칙이라고 합니다. 태그 이름이 적힌 부분은 CSS 선택자이고, 중괄호 안에 있는 CSS 속성, 콜론, 속성값, 세미콜론의 조합은 선언입니다.
선택자에는 태그 이름뿐 아니라 여러 가지 값을 사용할 수 있습니다. 이 절에서는 아이디(id)를 선택자로 사용하는 방법을 다룹니다.


#### 태그 선택자와 ID 선택자

`h3` 선택자는 문서 안의 모든 `h3` 태그를 선택합니다. 따라서 같은 색상이 모든 `h3` 태그에 적용됩니다. 이 페이지에서 특정 태그 하나에만 다른 스타일을 적용하려면 `id` 속성과 ID 선택자를 사용할 수 있습니다.
먼저 한라산 국립공원 제목의 `h3` 태그에 `id="hallasan"`을 추가합니다. 그런 다음 `style` 태그 안에서 `h3` 대신 `#hallasan` 선택자로 CSS 규칙을 작성합니다.


```html
<h3 id="hallasan">한라산 국립공원</h3>
```

```css
body {
	/* ... */
}

#hallasan { /* h3을 hallasan으로 변경 */
  color: #f56513;
}
```

`#`은 ID 이름을 나타내는 기호입니다. `#hallasan`은 `hallasan`이라는 ID를 가진 요소를 선택한다는 뜻입니다. 이 규칙을 적용하면 한라산 국립공원 제목에만 다른 색상이 적용됩니다.


#### 이미지 스타일 분리하기

인라인 스타일로 작성되어 있던 `img` 태그의 스타일도 CSS 규칙으로 옮길 수 있습니다.


```html
<!-- 변경 전 -->
<img style="width: 100%" src="jeju.png" />

<!-- 변경 후 -->
<img src="jeju.png" />
```

```css
p {
	/* ... */
}

img {
  width: 100%;
  margin-bottom: 56px;
}
```

#### ID 선택자로 특정 이미지 선택하기

`img` 선택자를 사용하면 문서 안의 모든 이미지에 같은 스타일이 적용됩니다.
지도 이미지 하나에만 스타일을 적용하려면 해당 `img` 태그에 `id="map"`을 추가하고, CSS에서는 `img` 선택자 대신 `#map` 선택자를 사용합니다. 이렇게 작성하면 지도 이미지만 화면 너비에 맞게 표시됩니다.


```html
<img id="map" src="jeju.png" />
```

```css

#map { /* img 태그에서 #map으로 변경 */
  width: 100%;
  margin-bottom: 56px;
}
```

#### ID를 사용할 때 주의할 점

ID를 만들 때는 한 가지 주의해야 합니다.
**아이디 이름은 한 문서 안에서 중복해서 쓰면 안 됩니다.**
예를 들어 성산 일출봉에 `hallasan`이라는 아이디를 똑같이 쓰면 안 됩니다. 아이디를 쓸 때는 각 요소마다 고유한 이름을 지정해야 합니다.


#### ID를 중복해서 쓰면 안 되는 이유

ID가 중복되면 다음과 같은 문제가 발생합니다.

1. **선택자의 모호성**
    - `document.getElementById()`를 사용할 때 동일한 ID가 여러 개 있으면 보통 첫 번째 요소만 선택됩니다.
    - JavaScript로 특정 요소를 조작하고 싶을 때 의도한 요소가 아닌 다른 요소가 선택될 수 있습니다.
2. **HTML 명세 위반**
    - HTML 표준에 따르면 ID는 문서 내에서 유일해야 합니다.
    - `<a>` 태그의 `href`에 ID를 사용할 수 있는데, ID를 중복해서 쓰면 의도한 위치로 이동하지 않을 수 있습니다.

### 04. 클래스 ⭐⭐⭐

#### 하나만 사용할 수 있는 ID

ID는 문서 안에서 하나의 요소에만 사용하는 고유한 이름입니다. 따라서 ID 선택자는 특정 요소 하나를 구분할 때 적합합니다.
여러 요소에 같은 스타일을 적용하려면 ID 대신 클래스(class)를 사용합니다. 이 절에서는 우도, 한라산 국립공원 같은 제목의 크기와 굵기를 클래스 선택자로 바꾸는 방법을 다룹니다.


클래스를 사용하면 여러 태그를 하나의 그룹으로 묶어 같은 스타일을 적용할 수 있습니다.
먼저 제목 태그의 `class` 속성에 `place`라는 이름을 지정합니다.


```html
<h3 class="place">우도</h3>
```

#### 여러 요소에 클래스 적용

같은 스타일을 적용할 다른 제목 태그에도 동일한 클래스를 추가합니다.


```html
<h3 class="place">우도</h3>
<h3 id="hallasan" class="place">한라산 국립공원</h3>
<h3 class="place">성산 일출봉</h3>
<h3 class="place">군산 오름</h3>
```

#### 클래스 선택자 작성

CSS 코드에서는 ID 선택자 대신 `.place` 선택자를 사용합니다.
`font-size: 16px;`와 `font-weight: 400;`을 지정하면 해당 클래스를 가진 요소의 글자 크기와 굵기가 바뀝니다.
ID 선택자에 `#`을 쓰는 것처럼, 클래스 선택자에는 `.`을 사용합니다. `.place`는 `place`라는 클래스를 가진 요소를 선택한다는 뜻입니다.


```css
#hallasan {
	/* ... */
}

.place {
  font-size: 16px;
  font-weight: 400;
}
```

이 규칙은 `place` 클래스를 가진 제목에만 적용됩니다. 같은 문서 안의 다른 문단에는 적용되지 않습니다.
클래스 선택자를 사용하면 여러 태그를 한 번에 선택해서 스타일을 지정할 수 있습니다.


#### 클래스의 단일 요소 사용

클래스는 여러 요소에 사용할 수 있지만, 반드시 여러 번 사용해야 하는 것은 아닙니다.
예를 들어 현재 모든 `p` 태그에 배경색이 들어가 있다면, 첫 번째 문단에만 `description` 클래스를 지정해 배경색을 적용하고 나머지 `p` 태그에는 글자색만 적용할 수 있습니다.


```css
.place {
	/* ... */
}

.description { /* p태그를 .description 으로 변경해줌 */
  background-color: #f8f8f8;
  color: #555555;
  padding: 16px 24px;
  margin: 24px 0 56px;
}

p { /* 추가 */
  color: #919191;
}
```

그리고 배경색을 적용할 `p` 태그에 클래스 이름을 추가합니다.


```html
<p class="description">
  일정은 짧고 갈 곳은 많고. 먹고 싶은 것도 보고 싶은 것도 많은데 ... 여행
  인플루언서들이 강력 추천하는 여행지들을 모았습니다. 한눈에 보이는 제주
  여행지 위시리스트 4곳!
</p>
```

이렇게 작성하면 맨 위에 있는 `p` 태그에만 배경색이 적용되고, 나머지 `p` 태그에는 글자색만 적용됩니다.
참고로 클래스 이름은 숫자로 시작하면 안 됩니다.


#### 정리

요소 하나만 구분해서 스타일링해야 할 때는 ID를 사용할 수 있습니다.
여러 요소를 같은 그룹으로 묶어 스타일링해야 할 때는 클래스를 사용합니다.
실무에서는 재사용성과 유지보수를 고려해 클래스 선택자를 더 자주 사용합니다. ID는 문서 안에서 유일해야 하는 대상을 구분할 때 제한적으로 사용합니다.


### 05. CSS 파일 ⭐⭐

#### 외부 CSS 파일을 사용하는 이유

앞에서는 CSS 코드를 모두 `style` 태그 안에 작성했습니다. HTML과 CSS가 한 파일 안에 섞이면 문서가 길어질수록 구조를 파악하기 어려워집니다. 그래서 보통 CSS 코드는 별도의 파일에 저장합니다. 이 절에서는 CSS 파일을 만들고 HTML 문서에 연결하는 방법을 다룹니다.


#### style.css 파일

`style.css`라는 이름으로 CSS 파일을 만듭니다. `style.css`는 스타일을 담는 파일에 자주 사용하는 일반적인 파일명입니다. 파일을 만든 뒤 기존 `style` 태그 안에 있던 CSS 코드를 `style.css`로 옮깁니다.


#### link 태그로 CSS 연결

HTML 문서에서는 기존 `style` 태그를 제거하고 `link` 태그로 CSS 파일을 연결합니다. `link` 태그는 현재 문서 바깥에 있는 파일을 불러올 때 사용합니다.
`rel` 속성에는 `stylesheet`를, `href` 속성에는 불러올 CSS 파일의 경로인 `style.css`를 작성합니다.


```html
<link rel="stylesheet" href="style.css" />
```

#### link 태그 속성 값

`rel`은 relationship의 줄임말로, 현재 문서와 연결할 파일의 관계를 나타냅니다. `rel="stylesheet"`는 연결할 파일이 스타일시트, 즉 CSS 파일이라는 뜻입니다.
`href`는 파일의 주소나 경로를 지정하는 속성입니다. 여기서는 HTML 파일과 `style.css`가 같은 폴더에 있으므로 파일 이름만 작성해도 됩니다.


#### 외부 CSS 파일 적용 결과

CSS 파일을 올바르게 연결하면 `style` 태그를 제거해도 같은 스타일이 적용됩니다. 일반적으로 HTML은 구조를, CSS는 스타일을 담당하도록 파일을 분리해서 작성합니다.

### 07. 개발자 도구로 선택자 확인하기

#### 개발자 도구의 역할

개발자 도구를 사용하면 선택자가 어떤 요소에 적용되는지 확인할 수 있습니다. 브라우저에서 `F12` 키를 누르면 개발자 도구를 열 수 있습니다.


#### 요소 선택 도구

왼쪽 위의 요소 선택 도구를 사용하면 화면에서 특정 요소를 선택할 수 있습니다.
`Styles` 탭에는 선택한 요소에 적용된 CSS 규칙이 표시됩니다. 직접 작성한 CSS뿐 아니라 브라우저가 기본으로 제공하는 `user agent stylesheet`도 함께 볼 수 있습니다.


#### 선택자 적용 범위 확인하기

`h3` 선택자 위에 마우스를 올리면 이 선택자가 선택하는 요소들이 화면에 표시됩니다.
마찬가지로 `.place` 선택자 위에 마우스를 올리면 `place` 클래스를 가진 요소만 표시됩니다.
개발자 도구를 사용하면 CSS 규칙이 적용되는 범위를 빠르게 확인할 수 있습니다. 선택자가 예상한 요소를 선택하는지 확인할 때 유용합니다.

### 11. #퀴즈 기본 개념 정리

**문제 1.**

여러 개의 `<h3>` 태그에 동일한 스타일(예: 글자색, 굵기)을 한 번에 적용하려고 합니다. 이때 가장 적합한 CSS 선택자 유형은 무엇이며, CSS 코드에서 해당 선택자는 어떤 기호로 시작해야 할까요?

- 정답 및 해설

    정답

    - 선택자 유형: 클래스 (class)
    - 시작 기호: `.` (마침표)


**문제 2.**

HTML 문서의 `<head>` 태그 안에서 `style.css`라는 외부 CSS 파일을 연결하려고 합니다. 아래 코드의 `[ A ]`와 `[ B ]`에 들어갈 올바른 속성(attribute)은 무엇일까요?

- 정답 및 해설

    ```html
    <link [ A ]="stylesheet" [ B ]="style.css" />
    ```

    정답: `[ A ]`는 `rel`, `[ B ]`는 `href`입니다.

    해설: `rel="stylesheet"`는 연결할 파일이 스타일시트라는 뜻이고, `href="style.css"`는 불러올 CSS 파일의 경로를 지정합니다.



**문제 3.**

CSS에서 색상을 표현하는 `rgba()` 단위는 투명도를 조절할 수 있습니다. 순수한 파란색(`rgb(0, 0, 255)`)을 50%의 투명도로 표현하려면 `rgba()` 값을 어떻게 작성해야 할까요?

- 정답 및 해설
    - `rgba(0, 0, 255, 0.5)`


**문제 4.**

아래와 같은 CSS 코드가 있고 `.child` 요소가 `.parent` 요소 안에 있을 때, `child` 클래스를 가진 요소의 최종 `font-size`는 몇 px이 될까요? (단, `rem`의 기준이 되는 `<html>` 태그의 `font-size`는 16px이라고 가정합니다.)

```css
.parent {
  font-size: 20px;
}

.child {
  font-size: 1.5em;
}
```

- 정답 및 해설

    30px (`em`은 부모 요소의 `font-size`를 기준으로 하므로, 20px * 1.5 = 30px가 됩니다.)



**문제 5.**

CSS에서 특정 요소 **단 하나만**을 선택하여 고유한 스타일을 적용하고 싶을 때 사용하는 선택자는 `id`와 `class` 중 무엇일까요? 그리고 이 선택자는 CSS에서 어떤 기호로 시작하나요?

- 정답 및 해설
    - 선택자: 아이디 (id)
    - 시작 기호: `#` (샵)

### 03. 배경 이미지: background-image ⭐

#### 실습 코드 다운로드

[2-03.배경 이미지.zip](03%20CSS%20%ED%95%B5%EC%8B%AC%20%EA%B0%9C%EB%85%90/2-03.%E1%84%87%E1%85%A2%E1%84%80%E1%85%A7%E1%86%BC_%E1%84%8B%E1%85%B5%E1%84%86%E1%85%B5%E1%84%8C%E1%85%B5.zip)

#### 예제 코드 구조

이 절에서는 피자 주문 페이지를 수정하면서 자주 사용하는 CSS 속성을 살펴봅니다. 예제에는 `index.html`, `pizza.png`, `style.css` 파일이 있습니다. `index.html`에는 제목, 가격, 본문, 그리고 "주문하기" 링크가 들어 있습니다. 이 페이지의 카드 영역에 배경 이미지를 추가합니다.


#### card 배경 이미지

`pizza.png` 파일을 `card` 클래스를 가진 `div` 태그의 배경 이미지로 사용합니다.
배경색을 지정할 때는 `background-color`를 사용했습니다. 배경 이미지를 지정할 때는 `background-image`를 사용하고, 이미지 경로는 `url("pizza.png")`처럼 작성합니다.


```css
.card {
  background-image: url("pizza.png"); /* 수정 */
  color: #ffffff;
  line-height: 1.7;
  margin: 40px auto;
  padding: 40px;
  width: 480px;
}
```

#### url() 값

`url()`은 파일의 주소나 경로를 괄호 안에 적는 CSS 함수입니다.
따옴표는 생략할 수 있는 경우도 있지만, 파일 경로에 특수 문자나 공백이 있을 때 문제가 생기지 않도록 따옴표로 감싸는 것이 좋습니다.


#### 배경 이미지의 기본 반복

이렇게 작성하면 브라우저는 현재 페이지의 경로를 기준으로 `pizza.png` 파일을 찾습니다. 배경 이미지는 기본적으로 요소의 영역 안에서 반복되어 표시됩니다.


#### 배경 이미지 배치 설정

배경 이미지가 반복되지 않고 카드 전체를 자연스럽게 채우도록 `background-repeat`, `background-position`, `background-size`를 함께 지정합니다.


```css
.card {
  background-image: url("pizza.png");
  background-repeat: no-repeat;  /* 수정 */
  background-position: center;  /* 수정 */
  background-size: cover;  /* 수정 */
  color: #ffffff;
  line-height: 1.7;
  margin: 40px auto;
  padding: 40px;
  width: 480px;
}
```

`background-repeat: no-repeat;`은 배경 이미지를 반복하지 않도록 지정합니다. `background-position: center;`는 이미지를 가운데 기준으로 배치합니다. `background-size: cover;`는 이미지 비율을 유지하면서 카드 영역을 꽉 채우도록 크기를 조절합니다.


#### 개발자 도구로 속성 확인하기

개발자 도구에서 `card`에 해당하는 `div` 태그를 선택하면 `Styles` 탭에서 적용된 CSS 규칙을 확인할 수 있습니다. 각 속성의 체크 박스를 끄거나 켜면서 `background-repeat`, `background-position`, `background-size`가 화면에 어떤 영향을 주는지 비교할 수 있습니다.


#### background-repeat 값

`background-repeat`에는 대표적으로 `no-repeat`, `repeat`, `repeat-x`, `repeat-y` 값을 사용할 수 있습니다. `repeat`은 기본값으로 이미지를 가로와 세로 방향으로 반복합니다. `repeat-x`는 가로 방향으로만, `repeat-y`는 세로 방향으로만 반복합니다. 배경 이미지를 한 번만 보여 주고 싶을 때는 `no-repeat`을 사용합니다.


#### background-position 값

`background-position`은 배경 이미지의 위치 기준을 지정합니다. `center`를 사용하면 이미지가 가운데 기준으로 배치됩니다. 이 외에도 `top`, `right`, `bottom`, `left` 같은 값을 사용할 수 있습니다.


#### background-size 값

`background-size`는 배경 이미지의 크기를 지정합니다. `cover`를 사용하면 이미지가 요소의 영역을 채우도록 확대되거나 축소됩니다.


`cover`는 이미지 비율을 유지하면서 영역을 꽉 채우는 값입니다. `contain`은 이미지가 잘리지 않는 범위에서 최대 크기로 배치하는 값입니다.

가로 크기와 세로 크기를 직접 지정할 수도 있습니다. 예를 들어 `100px 50px`처럼 작성하면 배경 이미지의 가로 크기와 세로 크기를 각각 지정합니다.


```css
background-size: 100px 50px;
```

배경 이미지를 카드 전체에 채울 때는 `cover`를 자주 사용합니다. 다만 이미지 위에 글자가 올라가면 글자가 잘 보이지 않을 수 있으므로, 뒤에서 그라디언트를 겹쳐 가독성을 높이는 방법을 다룹니다.


#### cover와 contain의 차이

#### cover

- **이미지가 잘릴 수 있음**: 요소의 전체 영역을 완전히 덮도록 이미지의 가로/세로 비율을 유지하면서 확대/축소됩니다.
- **빈 공간 없음**: 이미지의 일부가 요소 바깥으로 벗어날 수는 있지만, 요소 내부에 빈 공간은 절대 생기지 않습니다.
- **용도**: 주로 요소의 배경을 이미지로 가득 채우고 싶을 때 사용합니다. (예: 전체 화면 배경)

#### contain

- **이미지가 잘리지 않음**: 이미지 전체가 요소 안에 모두 보이도록 이미지의 가로/세로 비율을 유지하면서 확대/축소됩니다.
- **빈 공간 생길 수 있음**: 이미지의 비율과 요소의 비율이 다를 경우, 요소 내부에 빈 공간(레터박스처럼)이 생길 수 있습니다.
- **용도**: 이미지의 전체 모습을 손상 없이 보여주는 것이 중요할 때 사용합니다. (예: 로고, 갤러리 썸네일)

### 04. 그라디언트: linear-gradient

#### 그라디언트의 의미

그라디언트(gradient)는 한 색상에서 다른 색상으로 자연스럽게 이어지는 표현입니다. 예를 들어 검정색에서 남색으로 점차 변하는 배경을 만들 수 있습니다. 이 절에서는 CSS에서 선형 그라디언트를 만드는 `linear-gradient()`를 사용합니다.


![스크린샷 2025-10-23 17.21.38.png](03%20CSS%20%ED%95%B5%EC%8B%AC%20%EA%B0%9C%EB%85%90/%E1%84%89%E1%85%B3%E1%84%8F%E1%85%B3%E1%84%85%E1%85%B5%E1%86%AB%E1%84%89%E1%85%A3%E1%86%BA_2025-10-23_17.21.38.png)

#### linear-gradient() 사용하기

그라디언트는 `background-image` 속성에 지정할 수 있습니다.


그라디언트는 CSS에서 배경 이미지의 한 종류처럼 사용할 수 있습니다. 이미지 경로 대신 `linear-gradient()`를 값으로 작성합니다.


```css
.card {
  background-image: linear-gradient(rgba(0, 0, 0, 1), rgba(0, 0, 0, 0)); /* 변경 */
  background-repeat: no-repeat;
  /* ... */
}
```

첫 번째 색은 불투명한 검정색, 두 번째 색은 투명한 검정색입니다. 이렇게 작성하면 아래로 갈수록 검정색에서 투명에 가까워지는 그라디언트가 만들어집니다.


#### 그라디언트 방향

그라디언트의 방향을 바꾸려면 `linear-gradient()`의 첫 번째 값으로 각도를 작성합니다. `90deg`를 사용하면 색상 변화가 가로 방향으로 만들어집니다.


```css
background-image: linear-gradient(90deg, rgba(0, 0, 0, 1), rgba(0, 0, 0, 0));
```

#### 색상 지점 지정하기

그라디언트의 색상이 바뀌는 지점을 조절하려면 색상 뒤에 `40%`처럼 위치 값을 함께 작성합니다.


```css
  background-image: linear-gradient(
    90deg,
    rgba(0, 0, 0, 1) 40%,
    rgba(0, 0, 0, 0)
  );
```

#### 그라디언트 생성 도구

그라디언트는 다양한 형태로 만들 수 있지만, 직접 작성해야 하는 값이 많아질 수 있습니다. 이럴 때는 gradient generator 같은 생성 도구를 사용하면 필요한 CSS 코드를 빠르게 만들 수 있습니다.


검색 결과나 사이트 UI는 달라질 수 있지만, 일반적으로 마우스로 색상과 방향을 조절한 뒤 CSS 코드를 복사해서 사용할 수 있습니다.


#### 그라디언트와 이미지 겹치기

배경 이미지 위에 반투명한 그라디언트를 겹치면 글자의 가독성을 높일 수 있습니다. `background-image`에 여러 값을 작성할 때는 쉼표로 구분합니다. 먼저 작성한 배경이 위에 쌓이므로, 그라디언트를 먼저 쓰고 그 뒤에 이미지 URL을 작성합니다.


```css
.card {
  background-image: linear-gradient(
      90deg,
      rgba(0, 0, 0, 1) 30%,
      rgba(0, 0, 0, 0)
    ),
    url("pizza.png");
    /* ... */
  }
```

이미지만 배경으로 넣으면 글자가 잘 보이지 않을 수 있습니다. 이미지 위에 반투명한 그라디언트를 겹치면 배경 이미지를 유지하면서도 글자를 더 잘 읽을 수 있습니다.

### 06. 그림자: box-shadow ⭐

카드 전체에 그림자를 넣을 때는 `box-shadow` 속성을 사용합니다. 처음 두 값은 그림자의 가로 위치와 세로 위치를 나타내고, 마지막 값은 그림자의 색상을 나타냅니다. 다음 예시는 그림자를 오른쪽으로 `10px`, 아래쪽으로 `15px` 이동시키고 색상은 `rgba(0, 0, 0, 0.4)`로 지정합니다.


```css
.card {
  /* ... */
  box-shadow: 10px 15px rgba(0, 0, 0, 0.4); /* 추가 */
  /* ... */
}
```

이렇게 작성하면 그림자가 요소 기준으로 오른쪽 `10px`, 아래쪽 `15px` 위치에 생깁니다.


#### 흐림 효과

그림자에 흐림 효과를 추가하려면 세 번째 길이 값을 작성합니다. 이 값을 blur radius라고 하며, 값이 클수록 그림자 경계가 더 부드럽게 퍼집니다. 다음 예시는 흐림 값을 `20px`로 지정합니다.


```css
.card {
  /* ... */
  box-shadow: 10px 15px 20px rgba(0, 0, 0, 0.4); /* 변경 */
  /* ... */
}
```

#### 퍼지는 정도

네 번째 길이 값은 그림자가 퍼지는 정도를 지정합니다. 이 값을 spread radius라고 합니다. 다음 예시는 그림자를 모든 방향으로 `5px`만큼 더 크게 퍼지도록 지정합니다.


```css
.card {
  /* ... */
  box-shadow: 10px 15px 20px 5px rgba(0, 0, 0, 0.4); /* 변경 */
  /* ... */
}
```

`box-shadow`는 여러 값을 함께 사용하므로 처음에는 복잡해 보일 수 있습니다.


#### box-shadow 값의 의미

값의 순서를 단순히 외우기보다 각 값이 어떤 역할을 하는지 이해하는 것이 중요합니다. 그림자의 위치, 흐림 정도, 퍼지는 정도, 색상을 차례로 지정한다고 보면 됩니다. 개발자 도구에서 값을 조절하면 각 값이 화면에 미치는 영향을 확인할 수 있습니다.


![스크린샷 2025-10-23 18.32.12.png](03%20CSS%20%ED%95%B5%EC%8B%AC%20%EA%B0%9C%EB%85%90/%E1%84%89%E1%85%B3%E1%84%8F%E1%85%B3%E1%84%85%E1%85%B5%E1%86%AB%E1%84%89%E1%85%A3%E1%86%BA_2025-10-23_18.32.12.png)


#### 최종 그림자 값

디자인에 맞게 그림자 값을 다음과 같이 조정할 수 있습니다.


```css
box-shadow: 0px 4px 15px rgba(0, 0, 0, 0.4);
```

디자인 도구에서 그림자에 해당하는 CSS 코드를 제공하는 경우도 많습니다. 이런 코드는 그대로 가져와 사용할 수 있지만, 각 값이 어떤 의미인지 읽을 수 있어야 합니다.


값이 헷갈릴 때는 MDN 같은 문서를 찾아보거나 box shadow generator 같은 도구로 값을 조절하며 확인할 수 있습니다.


### 07. 불투명도: opacity ⭐

#### opacity의 역할

`opacity`는 요소 전체의 불투명도를 조절하는 속성입니다. 품절된 상품의 버튼처럼 요소를 흐리게 표시하고 싶을 때 사용할 수 있습니다.
먼저 "주문하기" 텍스트를 "품절"로 바꾸고, `class="order"`가 지정된 링크에 스타일을 적용합니다.


```html
<a class="order" href="https://codeit.kr">
  품절
</a>
```

#### opacity 속성 적용

CSS 파일에서는 `order` 클래스에 `opacity: 0.5;`를 추가합니다.


```css
.order {
  background-color: #f13030;
  color: #ffffff;
  font-weight: 800;
  padding: 6px 16px;
  text-decoration: none;
  opacity: 0.5; /* 추가 */
}
```

`opacity` 값은 `0`부터 `1`까지 사용할 수 있습니다. 값이 `1`에 가까울수록 불투명하고, `0`에 가까울수록 투명하게 보입니다.


#### 개발자 도구에서 opacity 값 조정

개발자 도구에서 "품절" 요소를 선택하면 `opacity` 속성이 적용된 것을 확인할 수 있습니다.
값을 `0.1`부터 `1`까지 바꿔 보면 불투명도가 어떻게 달라지는지 확인할 수 있습니다.


#### opacity 사용 시 주의점

`opacity`를 사용할 때 주의할 점은 요소 전체가 반투명해진다는 것입니다. 배경색뿐 아니라 글자, 테두리, 자식 요소까지 함께 흐려집니다. 배경색만 반투명하게 만들고 싶다면 앞에서 배운 `rgba()` 색상 값을 사용합니다.
요소 전체를 반투명하게 만들 때는 `opacity`를 사용합니다.

### 10. #퀴즈 자주 쓰는 CSS 속성

**문제 1.**

`background-size` 속성의 값인 `cover`와 `contain`의 차이점을 설명하세요. 어떤 상황에서 각각 사용하면 좋을까요?

- 정답 및 해설
    - **cover**: 이미지의 비율을 유지하면서 요소를 완전히 덮습니다. 요소에 빈 공간이 생기지 않지만, 이미지의 일부가 잘릴 수 있습니다. (예: 페이지 전체 배경)
    - **contain**: 이미지의 비율을 유지하면서 요소 안에 이미지 전체가 보이도록 크기를 조절합니다. 이미지가 잘리지 않지만, 요소와 이미지의 비율이 다를 경우 빈 공간이 생길 수 있습니다. (예: 로고, 갤러리 이미지)



**문제 2.**

CSS를 사용하여 이미지 위에 반투명한 검정색 그라데이션을 겹치려고 합니다. 다음 중 올바른 코드는 무엇일까요?

1. `background-image: url("image.png"), linear-gradient(rgba(0,0,0,0.5), rgba(0,0,0,0));`
2. `background-image: linear-gradient(rgba(0,0,0,0.5), rgba(0,0,0,0)), url("image.png");`
3. `background: url("image.png");`와 `opacity: 0.5;`를 함께 사용한다.
4. 위 코드 모두 동일하게 동작한다.

- 정답 및 해설
    - **정답: 2번**
    - `background-image`에 여러 값을 사용하면 먼저 선언된 것이 위로 쌓입니다. 따라서 그라데이션을 먼저 쓰고 이미지 URL을 써야 이미지 위에 그라데이션이 겹쳐 보입니다.


**문제 3.**

`box-shadow: 0px 10px 20px 5px rgba(0, 0, 0, 0.2);` 코드에서 `10px`, `20px`, `5px`는 각각 무엇을 의미하나요?

- 정답 및 해설
    - `10px`: 그림자의 **세로 위치(vertical offset)**. 양수 값이면 아래쪽으로 그림자가 생깁니다.
    - `20px`: **흐림 효과(blur radius)**. 값이 클수록 그림자 경계가 부드럽고 넓게 퍼집니다.
    - `5px`**: 퍼지는 정도(spread radius).** 양수 값이면 그림자가 모든 방향으로 지정된 크기만큼 커집니다.
    - (참고: 첫 번째 값 `0px`는 가로 위치(horizontal offset)입니다.)


**문제 4.**

"주문하기" 버튼의 배경색만 반투명하게 만들고 싶을 때와, 버튼 전체(텍스트 포함)를 반투명하게 만들고 싶을 때 사용해야 하는 CSS 속성은 각각 무엇인가요?

- 정답 및 해설
    - **배경색만 반투명하게**: `background-color` 속성에 `rgba()` 색상 값을 사용합니다. (예: `background-color: rgba(0, 0, 0, 0.5);`)
    - **요소 전체를 반투명하게**: `opacity` 속성을 사용합니다. (예: `opacity: 0.5;`)

#### 박스 모델의 의미

웹 페이지의 모든 요소는 박스 모델을 따릅니다. 각 요소는 화면에서 눈에 보이지 않는 사각형 영역을 가진다고 볼 수 있습니다. 개발자 도구에서 요소 위에 마우스를 올리면 각 요소가 차지하는 박스 영역을 확인할 수 있습니다.


#### Computed 탭에서 확인하기

개발자 도구의 `Computed` 탭에서는 선택한 요소의 박스 모델을 볼 수 있습니다. 예제에서는 `terms` 클래스를 가진 `div` 태그를 살펴봅니다. 박스 모델은 네 가지 영역으로 구성됩니다.


#### 박스 구성 요소

### 05. border와 border-radius ⭐⭐

이번에는 테두리를 만드는 `border` 속성을 배웁니다. `terms` 클래스를 가진 `div` 태그에 `border: 1px solid #dedede;`를 추가합니다.


```css
.terms {
  background-color: #ffffff;
  box-shadow: 0px 4px 12px rgba(0, 0, 0, 0.05);
  margin: 32px auto;
  padding: 32px 19px 40px;
  width: 400px;
  border: 1px solid #dedede; /* 추가 */
}
```

`1px`은 테두리의 굵기입니다. 값이 커질수록 테두리가 더 굵게 표시됩니다. 가운데의 `solid`는 테두리의 선 스타일을 나타냅니다.


#### border 선 스타일

`dotted`를 사용하면 점선 테두리가 생기고, `dashed`를 사용하면 짧은 선으로 이어진 테두리가 생깁니다. 기본적인 실선 테두리는 `solid`로 지정합니다.


#### border 색상

마지막에 있는 색상 값은 테두리의 색상을 나타냅니다. `red`, `blue` 같은 색상 이름이나 HEX 색상 코드를 사용할 수 있습니다.


#### border-radius

`border-radius`는 요소의 모서리를 둥글게 만드는 속성입니다. `terms` 클래스에 `border-radius: 8px;`을 추가하면 박스 모서리가 둥글게 표시됩니다.


```css
.terms {
	/* ... */
  border: 1px solid #dedede;
  border-radius: 8px;
}
```

`border-radius`는 `border`가 없어도 사용할 수 있습니다. 예를 들어 `terms-content`라는 `div` 태그에 테두리를 지정하지 않고 `border-radius`만 적용해도 배경 영역의 모서리가 둥글게 표시됩니다.

### 07. box-sizing ⭐⭐

박스 모델에서 요소의 실제 크기는 어떻게 계산될까요? `terms` 클래스의 `width`를 `400px`로 지정했더라도 개발자 도구에서는 전체 너비가 `440px`로 보일 수 있습니다. 이 차이는 `content`, `padding`, `border`가 크기 계산에 포함되는 방식 때문에 발생합니다.


![스크린샷 2025-10-26 15.00.15.png](03%20CSS%20%ED%95%B5%EC%8B%AC%20%EA%B0%9C%EB%85%90/%E1%84%89%E1%85%B3%E1%84%8F%E1%85%B3%E1%84%85%E1%85%B5%E1%86%AB%E1%84%89%E1%85%A3%E1%86%BA_2025-10-26_15.00.15.png)


#### Computed 탭의 크기 계산

기본적으로 `width`와 `height`는 `content` 영역의 크기를 지정합니다. `Computed` 탭에서 확인하면 `content`의 너비가 `400px`로 표시됩니다.
전체 너비 `440px`은 `content` 너비에 좌우 `padding`과 좌우 `border`를 더한 값입니다. 예제에서는 한쪽의 `border`가 `1px`, `padding`이 `19px`이므로 한쪽에 `20px`, 양쪽에 `40px`이 더해집니다. 따라서 전체 너비는 `400px + 40px = 440px`이 됩니다.


#### 전체 너비를 400px로 맞추는 문제

요소의 전체 너비를 정확히 `400px`로 만들려면 `padding`과 `border` 값을 제외한 `content` 너비를 직접 계산해야 합니다.
하지만 `padding`이나 `border` 값을 바꿀 때마다 다시 계산해야 하므로 관리가 번거로워집니다.


#### box-sizing으로 크기 계산 방식 바꾸기

`width`와 `height`에 `padding`과 `border`까지 포함해서 계산하고 싶을 때는 `box-sizing` 속성을 사용합니다.


#### border-box 값

box-sizing: border-box;


```css
* {
  box-sizing: border-box;
}
```

`box-sizing: border-box;`를 적용하면 `width`와 `height`에 `content`, `padding`, `border`가 함께 포함됩니다. 따라서 전체 너비가 지정한 값인 `400px`에 맞춰지고, `content` 영역은 남는 공간에 맞게 자동으로 계산됩니다.


#### 전체 선택자 *

`box-sizing: border-box;`는 크기를 직관적으로 계산할 수 있어서 모든 요소에 적용하는 경우가 많습니다. 모든 요소에 같은 스타일을 적용하려면 전체 선택자 `*`를 사용할 수 있습니다. 여기서 `*`는 문서 안의 모든 요소를 뜻합니다.

### 09. overflow

약관처럼 내용이 긴 영역은 제한된 높이 안에서 스크롤할 수 있게 만드는 경우가 많습니다. 예제에서는 연보라색 박스의 높이를 지정한 뒤, 넘치는 내용을 처리하는 방법을 살펴봅니다.


#### height 값

`terms-content` 클래스에 `height: 324px;`을 추가합니다.


```css
.terms-content {
  background: #f4f3f8;
  border-radius: 8px;
  font-size: 12px;
  padding: 24px;
  height: 324px; /* 추가 */
}
```

#### overflow

높이를 지정하면 박스는 작아지지만 내용이 박스 밖으로 넘칠 수 있습니다. 이렇게 요소의 내용이 영역 밖으로 넘치는 상태를 오버플로우(overflow)라고 합니다. 넘친 내용은 감추거나 스크롤할 수 있게 만들 수 있습니다.


#### overflow: hidden

`height` 아래에 `overflow: hidden;`을 추가하면 요소 밖으로 넘친 내용이 보이지 않습니다.


#### overflow: scroll

`overflow: scroll;`을 사용하면 넘치는 내용을 스크롤해서 볼 수 있습니다. 스크롤은 세로 방향뿐 아니라 가로 방향으로도 만들 수 있습니다.


```css
  overflow-y: scroll;
```

세로 방향 스크롤만 지정하려면 `overflow-y: scroll;`을 사용할 수 있습니다.


#### overflow-x: scroll

가로 방향 스크롤만 지정하려면 `overflow-x: scroll;`을 사용할 수 있습니다.


```css
  overflow-x: scroll;
```

`overflow-x: scroll;`을 지정하면 가로 방향의 넘침을 스크롤로 처리합니다. 브라우저는 상황에 따라 다른 축의 `overflow` 값을 자동으로 조정할 수 있습니다. 기본 예제로 돌아가려면 `overflow: scroll;`을 다시 지정합니다.


```css
overflow: scroll;
```

#### white-space

개발자 도구에서 `white-space: nowrap;`을 추가하면 줄 바꿈 방식도 확인할 수 있습니다. `white-space`는 요소 안의 공백 문자와 줄 바꿈을 처리하는 방식을 정하는 속성입니다. `nowrap`은 줄 바꿈을 하지 않겠다는 뜻입니다.


#### 가로 스크롤

`white-space: nowrap;`으로 줄 바꿈을 막으면 내용이 가로 방향으로 넘칠 수 있습니다. 이때 가로 스크롤을 사용하면 넘친 내용을 확인할 수 있습니다. 마우스를 사용하는 환경에서는 `Shift` 키를 누른 상태로 휠을 돌려 가로 스크롤을 할 수 있습니다.

### 11. 마진 상쇄 ⭐⭐

#### 부모 요소와 자식 요소

HTML 파일을 보면 다음과 같은 구조가 있습니다.


```html
<div class="terms">
  <h1 class="terms-title">이용 약관</h1>
  <p class="terms-description">...</p>
  <div class="terms-content">...</div>
</div>
```

여기서 `<div class="terms">` 태그는 그 안에 있는 `<h1>`, `<p>`, 또 다른 `<div>` 태그를 감싸고 있습니다.


이처럼 바깥에서 다른 요소를 감싸는 요소를 부모 요소라고 하고, 그 안에 포함된 요소를 자식 요소라고 부릅니다.

HTML에서도 같은 부모를 가진 자식 요소들은 서로 형제 관계라고 부릅니다.
예제의 `<h1 class="terms-title">`, `<p class="terms-description">`, `<div class="terms-content">`는 모두 같은 부모 요소 안에 나란히 들어 있습니다.

이 세 태그는 모두 `<div class="terms">`라는 동일한 부모를 가지므로 서로 형제 요소입니다.


#### 형제 요소의 마진 상쇄

부모, 자식, 형제 관계를 이해하면 마진 상쇄를 설명할 수 있습니다.
어떤 요소에 세로 방향의 `margin`이 지정되어 있을 때, 브라우저는 두 요소 사이의 간격을 단순히 두 마진의 합으로 계산하지 않을 수 있습니다.


![스크린샷 2025-10-26 17.37.15.png](03%20CSS%20%ED%95%B5%EC%8B%AC%20%EA%B0%9C%EB%85%90/%E1%84%89%E1%85%B3%E1%84%8F%E1%85%B3%E1%84%85%E1%85%B5%E1%86%AB%E1%84%89%E1%85%A3%E1%86%BA_2025-10-26_17.37.15.png)

`Computed` 창에서 확인하면 `h1` 태그와 `p` 태그에 각각 `8px`의 마진이 들어가 있을 수 있습니다. 이 경우 두 요소 사이의 간격이 `16px`이 될 것처럼 보일 수 있습니다.

하지만 실제 간격은 `16px`이 아니라 `8px`로 표시될 수 있습니다.
`h1` 태그와 `p` 태그 사이의 세로 마진은 서로 더해지는 것이 아니라 겹쳐지고, 그중 더 큰 값이 적용됩니다.
이 현상을 마진 상쇄, 영어로는 margin collapsing이라고 합니다.


#### 부모 자식 관계에서 마진 상쇄

부모 요소와 자식 요소 사이에서도 세로 마진이 상쇄될 수 있습니다. 예를 들어 `terms` 요소의 `border`와 `padding`을 제거하면, 부모 요소의 마진과 그 안의 `terms-title` 요소의 마진이 겹쳐 보일 수 있습니다.
개발자 도구에서 `terms`와 `terms-title`에 서로 다른 배경색을 지정하면 각 요소의 영역을 더 쉽게 확인할 수 있습니다. 부모 요소와 자식 요소의 `margin-top` 값을 바꿔 보면, 두 마진이 더해지는 것이 아니라 더 큰 값으로 겹쳐 적용되는 상황을 확인할 수 있습니다.


#### 마진 상쇄 해결 방법

부모 요소에 `border`나 `padding`이 있으면 부모와 자식 사이에 경계가 생기므로 마진 상쇄가 일어나지 않습니다.
개발자 도구에서 `border`와 `padding`을 켜고 끄면서 `terms-title`의 `margin-top` 영역이 부모 요소 안쪽에 머무는지, 바깥쪽으로 겹쳐 나가는지 비교할 수 있습니다.


#### 마진 상쇄 정리

이웃한 요소끼리는 세로 마진이 상쇄될 수 있습니다. 부모 요소와 자식 요소 사이에서도 조건에 따라 세로 마진이 상쇄될 수 있습니다.
다만 부모 요소에 `border`나 `padding`이 있어 경계가 생기면 부모와 자식 사이의 마진 상쇄는 일어나지 않습니다.
세로 간격이 예상과 다르게 보일 때는 마진 상쇄가 일어났는지 확인해야 합니다. 가로 마진에는 마진 상쇄가 일어나지 않습니다.


### 12. #퀴즈 마진 상쇄 퀴즈 (10분)

```plain text

```

문제를 풀 때는 태그 구조와 스타일을 읽은 뒤, 각 요소의 박스 모델을 간단히 그려 보면 좋습니다.
마진 상쇄의 핵심 특징은 다음과 같습니다.
1. 마진 상쇄는 세로 방향에만 생깁니다.
2. 겹치는 마진 중 더 큰 값으로 적용됩니다.
3. 부모와 자식 관계에서는 `padding`이나 `border` 같은 경계가 있으면 마진 상쇄가 일어나지 않습니다.

### 03. 인라인 블록 ⭐⭐⭐

#### <a> 태그와 padding

본문 아래쪽에는 "자세히 읽기" 링크가 있습니다. 이 링크의 세로 `padding` 값을 크게 지정하면 인라인 요소의 특징을 확인할 수 있습니다.


![스크린샷 2025-10-27 16.34.25.png](03%20CSS%20%ED%95%B5%EC%8B%AC%20%EA%B0%9C%EB%85%90/%E1%84%89%E1%85%B3%E1%84%8F%E1%85%B3%E1%84%85%E1%85%B5%E1%86%AB%E1%84%89%E1%85%A3%E1%86%BA_2025-10-27_16.34.25.png)


#### 블록 vs 인라인

화면을 보면 주변 텍스트와 링크 영역이 어색하게 겹쳐 보일 수 있습니다. 앞에서 배운 배치 방식을 기준으로 원인을 살펴봅니다.
블록 요소는 위에서 아래로 쌓이고, 인라인 요소는 왼쪽에서 오른쪽으로 한 줄 안에 배치됩니다.


`a` 태그는 기본적으로 인라인 요소입니다.
일반적인 인라인 요소는 `width`와 `height`를 지정해도 레이아웃에서 차지하는 크기가 블록 요소처럼 바뀌지 않습니다. `img`, `button`, `input`, `textarea`처럼 예외적인 요소도 있지만, 기본적인 인라인 요소는 글자 흐름 안에서 배치된다고 이해하면 됩니다.


#### 인라인 요소의 주변 흐름

링크 옆에 본문을 이어서 배치하면 인라인 요소가 주변 텍스트와 같은 줄 흐름 안에서 배치된다는 점을 더 분명히 볼 수 있습니다.


```html
<a class="read-more" href="#">자세히 읽기</a>
2020년 7월 9일, 테슬라의 일론 머스크 회장이 "올해 안에 5단계 자율주행 기본
기능을 갖게 될 것"이라 밝혔다. 김필수(대림대 교수) 전기차협회장은 "5단계
자율주행은 전문가들도 꿈으로 남겨 놓고, 4단계 기술에 집중하는 상황"이라며
"5단계를 운운하는 건 허풍이 분명하지만, 다른 완성차업체보다 자율주행
기술이 우위에 있는 것 또한 사실"이라고 말했다.
```

이 결과에서 인라인 요소의 좌우 `padding`은 주변 텍스트의 위치에 영향을 줍니다. 하지만 세로 방향 `padding`은 시각적으로 영역을 키울 수 있어도, 주변 요소를 블록 요소처럼 밀어내지는 않습니다.


![스크린샷 2025-10-27 16.54.07.png](03%20CSS%20%ED%95%B5%EC%8B%AC%20%EA%B0%9C%EB%85%90/%E1%84%89%E1%85%B3%E1%84%8F%E1%85%B3%E1%84%85%E1%85%B5%E1%86%AB%E1%84%89%E1%85%A3%E1%86%BA_2025-10-27_16.54.07.png)


#### 인라인 요소의 크기 제한

인라인 요소에 `width: 300px; height: 300px;`을 추가해도 일반적인 블록 요소처럼 크기가 바뀌지 않습니다.


#### inline-block

한 줄 안에 배치하면서도 크기와 여백을 지정하고 싶을 때는 `display: inline-block;`을 사용합니다. `inline-block`은 인라인처럼 같은 줄에 배치되지만, 블록 요소처럼 `width`, `height`, 세로 여백을 가질 수 있습니다.


```css
.read-more {
  background: #528d9f;
  border-radius: 8px;
  display: inline-block;
  /* ... */
}
```

이렇게 지정하면 요소가 인라인 흐름 안에 배치되면서도 크기를 가지므로, 세로 방향으로도 주변 요소와의 간격이 자연스럽게 계산됩니다.
`display` 속성을 바꾸면 인라인 요소를 인라인 블록으로 바꾸거나, 블록 요소를 인라인처럼 배치하는 등 요소의 배치 방식을 조절할 수 있습니다.


#### display none 값

`display`에는 `none` 값도 있습니다. `display: none;`은 요소를 화면에서 보이지 않게 만들고, 요소가 차지하던 공간도 제거합니다.
`block`, `inline`, `inline-block`은 CSS 레이아웃을 이해할 때 특히 중요한 값입니다.


#### 인라인 블록 정리

CSS를 작성할 때 주변 요소가 자리를 비켜 주지 않거나 `width`, `height`가 적용되지 않는다면 해당 요소의 `display` 값이 `inline`인지 확인해야 합니다. 같은 줄에 배치하면서 크기도 지정해야 한다면 `inline-block`을 사용할 수 있습니다.

### 06. #퀴즈 Display 퀴즈

**문제 1.**

`display: block`과 `display: inline`의 가장 큰 차이점은 무엇인가요?

- 정답 및 해설

    정답

    - block은 한 줄 전체를 차지하는 박스 요소
    - inline은 글자처럼 한 줄에 이어 나란히 배치되는 요소

    해설

    - `block` 요소는 새로운 줄에서 시작하며 사용할 수 있는 전체 너비를 차지합니다.
    - `inline` 요소는 새로운 줄에서 시작하지 않고, 필요한 만큼의 너비만 차지합니다.


**문제 2.**

기본적으로 display: inline인 `<a>` 태그에 padding-top: 20px, height: 100px을 주면 주변 요소와 어떤 현상이 발생하나요?

- 정답 및 해설

    정답

    - 가로(좌우) `padding`은 적용되지만, 세로(상하) `padding`과 `height`는 레이아웃에 충분히 반영되지 않아 다른 요소들과 겹쳐 보일 수 있습니다.

    해설

    - `inline` 요소는 글자처럼 취급되며 `width`, `height`가 적용되지 않습니다.
    - `padding`의 좌우 값은 주변 인라인 요소에 영향을 주지만, 상하 값은 시각적으로만 확장되고 레이아웃에는 제한적으로 반영됩니다.


**문제 3.**

한 줄에 나란히 배치되면서도 width, height 값을 지정하고 싶을 때 사용하는 display 속성 값은?

- 정답 및 해설

    정답

    - inline-block

    해설

    - `inline-block`은 같은 줄에 나란히 배치하면서 `block`처럼 크기와 여백을 지정할 수 있습니다.


**문제 4.**

CSS로 요소를 화면에서 완전히 숨기고 공간조차 없애고 싶을 때 사용하는 display 값은?

- 정답 및 해설

    정답

    - display: none

    해설

    - `display: none`은 요소를 렌더링 트리에서 제외하므로 화면에 보이지 않고, 차지하던 공간도 사라집니다.


**문제 5.**

width 값을 줬는데 크기가 변하지 않는 경우 그 원인과 해결법은?

- 정답 및 해설

    정답

    - 원인: `display` 값이 `inline`일 가능성이 높습니다.
    - 해결법: `display` 값을 `block` 또는 `inline-block`으로 변경합니다.

    해설

    - `inline` 요소는 내용만큼의 너비만 가지며 `width`와 `height`가 적용되지 않습니다. 크기를 지정해야 한다면 `block` 또는 `inline-block`으로 바꿔야 합니다.

이런 표현은 모두 선택자입니다. 여러 선택자를 조합해서 필요한 요소를 정확히 선택할 수 있으면 CSS를 더 효율적으로 작성할 수 있습니다.


### 02. 선택자 목록

CSS를 작성하다 보면 여러 선택자에 같은 선언을 적용해야 할 때가 있습니다.
예를 들어 다음 코드에서는 `.book-description`과 `.book-info`에 같은 글꼴 관련 속성이 반복해서 작성되어 있습니다.


```css
/* style.css */
.book-description {
  font-size: 12px;
  font-weight: 400;
  line-height: 17px;
}

.book-info {
  background-color: #000000;
  border-radius: 12px;
  color: #ffffff;
  font-size: 12px;
  font-weight: 400;
  line-height: 17px;
  margin: 24px 0;
  padding: 12px 32px;
  text-align: center;
}
```

두 요소에 같은 스타일을 적용해야 한다면 선택자를 따로 작성하지 않고 **선택자 목록**을 사용할 수 있습니다.
선택자 목록은 쉼표(`,`)로 여러 선택자를 이어서 작성합니다.


```css
.book-description,
.book-info {
  font-size: 12px;
  font-weight: 400;
  line-height: 17px;
}

.book-info {
  background-color: #000000;
  border-radius: 12px;
  color: #ffffff;
  margin: 24px 0;
  padding: 12px 32px;
  text-align: center;
}
```

#### 선택자 목록의 의미

쉼표로 선택자를 나열한 것을 선택자 목록, 영어로는 selector list라고 합니다. 쉼표 뒤에 바로 다음 선택자를 이어서 쓸 수도 있고, 가독성을 위해 줄을 바꿔 작성할 수도 있습니다.


#### 정리

개발자 도구에서 확인하면 `.book-description`과 `.book-info`에 같은 글꼴 관련 속성이 적용된 것을 볼 수 있습니다.

### 04. 선택자 붙여 쓰기 ⭐⭐

지금까지는 선택자를 주로 하나씩 사용했습니다. 이 절에서는 선택자를 여러 개 붙여서 더 구체적인 요소를 선택하는 방법을 다룹니다.


먼저 HTML에서 한 요소에 여러 클래스를 사용하는 방법을 살펴봅니다. 예를 들어 `book-info` 클래스가 있는 요소에 `delay` 클래스를 추가하려면 `class="delay book-info"`처럼 클래스 이름을 공백으로 구분해서 작성합니다.


```html
<p class="delay book-info">
  지금 택배로 주문하면 <span class="date">11월 14일</span>에 받아 볼 수
  있어요
</p>
```

#### 클래스 선택자 붙여 쓰기

CSS 코드에서 `.delay` 클래스를 만들고 배경색을 지정할 수 있습니다.
여기서 `.delay` 클래스만 가진 요소를 선택할 수도 있지만, `.delay`와 `.book-info`를 모두 가진 요소만 선택할 수도 있습니다. 이때 클래스 선택자를 공백 없이 붙여 씁니다.


```css
.delay.book-info { /* .book-info 추가 */
  background-color: #c76546;
  white-space: nowrap; /* 한줄이 넘어가면 추가 */
}
```

#### 순서를 바꾸어도 상관 없음

`.delay.book-info`는 `delay` 클래스와 `book-info` 클래스를 모두 가진 요소를 선택합니다. 클래스 선택자를 붙여 쓸 때 클래스 이름의 순서는 영향을 주지 않습니다. 따라서 `.book-info.delay`라고 써도 같은 요소를 선택합니다.


#### 태그 선택자와 함께 붙여 쓰기

클래스 선택자는 태그 선택자와 함께 붙여 쓸 수도 있습니다. `p` 태그이면서 `delay` 클래스와 `book-info` 클래스를 모두 가진 요소를 선택하려면 맨 앞에 `p`를 붙입니다.


```css
p.delay.book-info { /* p 추가 */
  background-color: #c76546;
  white-space: nowrap;
}
```

이 선택자는 `p` 태그이면서 `delay` 클래스와 `book-info` 클래스를 모두 가진 요소를 선택합니다. ID 선택자와 함께 사용할 때도 같은 방식으로 붙여 씁니다.


#### ID 선택자와 함께 붙여 쓰기

예를 들어 `p` 태그에 `id="shipping"`을 추가했다면, `#shipping.delay.book-info`처럼 ID 선택자와 클래스 선택자를 붙여 쓸 수 있습니다.


```css
#shipping.delay.book-info {
	background-color: #c76546;
  white-space: nowrap;
}
```

#### 정리

선택자를 공백 없이 붙여 쓰면 조건을 모두 만족하는 요소를 선택합니다. 클래스 선택자를 여러 개 붙이면 해당 클래스를 모두 가진 요소를 선택합니다. 맨 앞에 태그 선택자를 붙이면 특정 태그이면서 해당 클래스를 가진 요소를 선택합니다. ID 선택자와 클래스 선택자를 함께 붙이면 특정 ID와 클래스를 모두 가진 요소를 선택합니다.


선택자를 조합할 수 있다고 해서 항상 복잡하게 작성하는 것이 좋은 것은 아닙니다. 기본적으로는 의미가 분명한 클래스 하나를 사용하는 편이 읽기 쉽습니다. 다만 같은 클래스를 여러 곳에서 사용하면서 특정 경우만 구분해야 할 때는 `.delay.book-info`처럼 두 클래스를 함께 붙여 쓰는 방식을 사용할 수 있습니다.

### 06. 자식, 자손 선택하기 ⭐⭐

#### class 속성

도서 목록에서 책 제목에 해당하는 `h2` 태그의 마진을 바꾸려면 각 제목에 `title` 클래스를 추가할 수 있습니다.


```html
<h2 class="title">
  <a href="https://codeit.kr">Pachinko (Paperback)</a>
</h2>

...

<h2 class="title">Shoko's Smile (Paperback)</h2>

...

<h2 class="title">The White Book (Paperback)</h2>
```

#### h1 태그와 겹치는 클래스

하지만 같은 문서의 `h1` 태그에도 `title` 클래스가 이미 있을 수 있습니다. 복잡한 웹사이트에서는 같은 클래스 이름이 여러 위치에서 쓰이면서 의도하지 않은 요소에 스타일이 적용되는 일이 생길 수 있습니다.


#### 부모 클래스와 조합하기

이럴 때는 부모 요소의 클래스 이름과 자식 요소의 클래스 이름을 조합해서 더 구체적인 선택자를 만들 수 있습니다.


#### 자식 선택하기

예제에서 부모 요소는 `book-container` 클래스를 가지고 있고, 그 안의 제목 요소는 `title` 클래스를 가지고 있습니다.


```css
.book-container {
	/* ... */
}

.book-container > .title {
  color: #12172a;
  font-size: 24px;
  font-weight: 700;
  line-height: 35px;
  margin: 24px 0 8px;
}
```

여기 있는 `>`는 자식 결합자입니다.
`.book-container > .title`은 `book-container` 클래스를 가진 요소의 직접적인 자식이면서 `title` 클래스를 가진 요소를 선택합니다. 이 선택자를 사용하면 같은 `title` 클래스라도 특정 부모 안에 있는 제목에만 스타일을 적용할 수 있습니다.


![스크린샷 2025-10-28 09.05.22.png](03%20CSS%20%ED%95%B5%EC%8B%AC%20%EA%B0%9C%EB%85%90/%E1%84%89%E1%85%B3%E1%84%8F%E1%85%B3%E1%84%85%E1%85%B5%E1%86%AB%E1%84%89%E1%85%A3%E1%86%BA_2025-10-28_09.05.22.png)

이런 식으로 자식 결합자를 쓰면 똑같은 클래스 이름이라도 다른 스타일을 적용할 수 있습니다.


#### 자손 결합자

직접적인 자식이 아니라 더 깊이 들어 있는 하위 요소를 선택해야 할 때도 있습니다. 예를 들어 `book` 클래스 안에 있는 `date` 클래스를 선택할 수 있습니다.


![스크린샷 2025-10-28 09.08.06.png](03%20CSS%20%ED%95%B5%EC%8B%AC%20%EA%B0%9C%EB%85%90/%E1%84%89%E1%85%B3%E1%84%8F%E1%85%B3%E1%84%85%E1%85%B5%E1%86%AB%E1%84%89%E1%85%A3%E1%86%BA_2025-10-28_09.08.06.png)


```css
.book .date { /* .book 추가 */
  font-weight: 700;
}
```

`.book .date`처럼 공백으로 구분해서 적으면 `book` 클래스를 가진 요소의 자손 중 `date` 클래스를 가진 요소를 선택합니다.


#### 자식 결합자와 자손 결합자 비교

HTML 구조는 작업 중에 바뀔 수 있으므로, 직접적인 자식 관계가 꼭 필요하지 않다면 자손 결합자를 사용하는 편이 더 유연할 수 있습니다. 앞에서 쓴 선택자도 `>` 대신 공백을 사용해 자손 결합자로 바꿀 수 있습니다.


```css
.book-container .title { /* > 제거 */
  /* ... */
}
```

#### 정리

자식 요소를 선택할 때는 `>`를 사용하고, 자손 요소를 선택할 때는 공백을 사용합니다. 다만 결합자를 많이 사용하면 선택자가 복잡해질 수 있습니다. 예제처럼 특정 제목만 선택해야 한다면 클래스 이름을 `book-title`처럼 더 구체적으로 바꾸는 방법도 있습니다.


```css
/* 변경 전 */
.book-container .title {
  /* ... */
}

/* 변경 후 */
.book-title {
  /* ... */
}
```

명확한 클래스 이름을 사용하면 어떤 요소가 선택되는지 이해하기 쉽고, 나중에 코드를 수정하기도 편합니다.

#### 링크 HTML

가상 클래스 선택자는 요소의 특정 상태를 선택할 때 사용합니다.
웹사이트에서 링크에 마우스를 올리면 밑줄이 생기거나 글자색이 바뀌는 경우가 있습니다. 이런 상태 기반 스타일을 만들기 위해 먼저 책 제목을 링크로 감쌉니다.


```html
<h2 class="title">
  <a href="https://codeit.kr">Pachinko (Paperback)</a>
</h2>
```

#### 링크 기본 스타일

그런 다음 링크의 기본 스타일을 다음과 같이 지정합니다.


```css
a {
  color: #12172a;
  text-decoration: none;
}
```

`color`로 링크의 글자색을 지정하고, `text-decoration: none;`으로 기본 밑줄을 없앱니다.


#### hover 상태의 밑줄

마우스를 올렸을 때 밑줄이 생기게 하려면 가상 클래스를 사용합니다.
가상 클래스는 클래스 선택자의 마침표(`.`) 대신 콜론(`:`)으로 시작합니다. 이름은 미리 정해져 있습니다.


![스크린샷 2025-10-28 09.57.00.png](03%20CSS%20%ED%95%B5%EC%8B%AC%20%EA%B0%9C%EB%85%90/%E1%84%89%E1%85%B3%E1%84%8F%E1%85%B3%E1%84%85%E1%85%B5%E1%86%AB%E1%84%89%E1%85%A3%E1%86%BA_2025-10-28_09.57.00.png)

VS Code의 제안 목록을 보면 여러 값을 확인할 수 있습니다. 콜론 하나(`:`)로 시작하는 것은 가상 클래스이고, 콜론 두 개(`::`)로 시작하는 것은 가상 요소입니다. 여기서는 `hover`를 사용합니다.



```css
a:hover {
  text-decoration: underline;
}
```

#### hover 가상 클래스

`a:hover`는 `a` 태그에 마우스를 올린 상태를 선택합니다. 이 선택자에 `text-decoration: underline;`을 지정하면 링크에 마우스를 올렸을 때 밑줄이 생깁니다.


#### 개발자 도구의 Force state

개발자 도구에서 링크 요소를 선택하고 `Force state`를 사용하면 특정 가상 클래스 상태를 강제로 적용해 볼 수 있습니다.
자주 사용하는 가상 클래스에는 `:active`, `:hover`, `:focus`, `:visited`가 있습니다. `:active`는 요소를 클릭하는 순간, `:hover`는 마우스를 올린 상태, `:focus`는 입력 요소 등에 포커스가 간 상태, `:visited`는 이미 방문한 링크를 나타냅니다.


```css
a:active { color: green; }
a:visited { color: purple; }
```

```html
<input name="input" class="input" placeholder="이름을 입력해주세요" />
```

```css
input:focus { outline: none; border: 2px solid blue; }
```

#### 정리

이 절에서는 가상 클래스 중 `:hover`를 사용해 링크에 마우스를 올렸을 때의 스타일을 지정했습니다. 가상 클래스는 요소의 상태에 따라 다른 스타일을 적용할 때 사용합니다.

## 06. 스타일 계산하기 (우선순위 정하기)

### 01. 캐스케이드 ⭐⭐

#### 적용된 CSS 규칙

개발자 도구에서 `delay book-info` 클래스를 가진 `p` 태그를 선택하면 이 요소에 적용된 CSS 규칙을 확인할 수 있습니다.


![스크린샷 2025-10-28 11.24.08.png](03%20CSS%20%ED%95%B5%EC%8B%AC%20%EA%B0%9C%EB%85%90/%E1%84%89%E1%85%B3%E1%84%8F%E1%85%B3%E1%84%85%E1%85%B5%E1%86%AB%E1%84%89%E1%85%A3%E1%86%BA_2025-10-28_11.24.08.png)

오른쪽에 `style.css`라고 파일 이름이 적힌 규칙은 직접 작성한 CSS 파일에서 온 규칙입니다. 그 밖에 직접 작성하지 않은 규칙도 보일 수 있는데, `user agent stylesheet`는 웹 브라우저가 기본으로 적용하는 CSS 규칙입니다.


#### 캐스케이드의 의미

브라우저는 여러 CSS 규칙을 합쳐 최종 스타일을 계산하고 화면에 표시합니다. 그런데 여러 규칙에서 같은 속성을 지정하면 어떤 값을 최종으로 사용할지 결정해야 합니다.


![스크린샷 2025-10-28 11.27.33.png](03%20CSS%20%ED%95%B5%EC%8B%AC%20%EA%B0%9C%EB%85%90/%E1%84%89%E1%85%B3%E1%84%8F%E1%85%B3%E1%84%85%E1%85%B5%E1%86%AB%E1%84%89%E1%85%A3%E1%86%BA_2025-10-28_11.27.33.png)

이처럼 여러 CSS 규칙을 어떤 기준으로 합칠지 정하는 방식이 캐스케이드(Cascade)입니다.


#### cascade라는 이름

CSS는 Cascading Style Sheets의 약자입니다. 여기서 cascade는 계단식으로 이어지는 폭포를 뜻합니다. CSS에서도 여러 규칙이 일정한 기준에 따라 계단식으로 합쳐진다는 의미로 이 단어를 사용합니다.


#### CSS에서의 캐스케이드

CSS에서 캐스케이드는 여러 규칙을 우선순위에 따라 합쳐 최종 스타일을 결정하는 과정입니다. 개발자 도구에서는 어떤 규칙이 적용되고, 어떤 규칙이 다른 규칙에 의해 덮어써졌는지 확인할 수 있습니다. 예를 들어 `.delay.book-info`의 `background-color`가 `.book-info`의 `background-color`보다 우선 적용될 수 있습니다.


#### element.style의 의미

개발자 도구에서 `element.style`에 값을 추가하면 가장 위쪽에 인라인 스타일처럼 표시됩니다. `element.style`은 HTML 태그의 `style` 속성에 직접 작성한 스타일과 같은 성격을 가집니다.


#### 정리

캐스케이드는 여러 CSS 규칙이 적용될 때 최종 값을 정하는 방식입니다. 다음 절에서는 이 우선순위가 어떤 기준으로 정해지는지 살펴봅니다.


### 02. 캐스케이드 순서 ⭐⭐⭐

#### 캐스케이드 순서의 기준

CSS 규칙은 캐스케이드 기준에 따라 합쳐져 최종 스타일로 적용됩니다. 여러 기준이 있지만, 이 절에서는 초반 학습에서 꼭 알아야 할 세 가지 기준을 살펴봅니다.


#### 스타일시트의 종류

첫 번째 기준은 스타일시트의 종류입니다. 브라우저가 기본으로 제공하는 CSS를 `user agent stylesheet`라고 합니다. 사용자가 작성한 `style.css`는 일반적으로 `user agent stylesheet`보다 우선순위가 높습니다.
따라서 직접 작성한 CSS로 브라우저 기본 스타일을 덮어쓸 수 있습니다.


#### 인라인 스타일

HTML 태그의 `style` 속성에 직접 작성한 스타일을 인라인 스타일이라고 합니다. 인라인 스타일은 일반적인 외부 CSS 파일의 규칙보다 우선순위가 높습니다.


#### 코드상의 순서

두 번째 기준은 코드상의 순서입니다. 같은 우선순위를 가진 규칙이 같은 속성을 지정한다면, 나중에 작성한 규칙이 먼저 작성한 규칙을 덮어씁니다. 예를 들어 같은 `.description` 선택자를 다시 작성하고 `color: blue;`를 지정하면 뒤에 있는 규칙이 최종적으로 적용됩니다.


```css
.description {
  color: #818181;
  font-size: 16px;
  font-weight: 400;
  line-height: 23px;
  margin: 0;
}

.description {
  color: blue;
}
```

#### 선택자 명시도

세 번째 기준은 선택자의 명시도입니다. 영어로는 specificity라고 합니다. 선택자마다 계산되는 점수가 있고, 명시도가 높은 선택자가 더 높은 우선순위를 가집니다. 예를 들어 `.delay.book-info`는 `.book-info`보다 더 구체적인 선택자입니다.

#### specificity 점수

명시도는 보통 ID 개수, 클래스/속성/가상 클래스 개수, 태그 이름 개수를 기준으로 계산합니다. 예를 들어 `.delay.book-info`는 클래스가 두 개이므로 `.book-info`보다 명시도가 높습니다.
개발자 도구의 `Styles` 탭에서는 어떤 규칙이 더 우선 적용되는지 확인할 수 있습니다. 같은 속성이 여러 규칙에 있을 때, 명시도가 높은 규칙이 최종 값으로 적용됩니다.


#### 정리

이 절에서는 캐스케이드 순서를 결정하는 주요 기준을 살펴봤습니다.


CSS 코드가 예상과 다르게 동작할 때는 캐스케이드 순서를 확인해야 합니다. 개발자 도구에서 요소를 선택하고 문제가 되는 CSS 속성을 검색하면 어떤 규칙이 값을 덮어쓰고 있는지 확인할 수 있습니다.

### 04. 상속

#### inherited from

캐스케이드된 CSS 규칙을 살펴보면 `inherited from`이라고 표시된 규칙을 볼 수 있습니다. 이는 조상 요소에서 물려받은 CSS 속성을 뜻합니다. 흐릿하게 표시된 속성은 해당 요소에 상속되지 않는 속성입니다.


![스크린샷 2025-10-28 13.07.50.png](03%20CSS%20%ED%95%B5%EC%8B%AC%20%EA%B0%9C%EB%85%90/%E1%84%89%E1%85%B3%E1%84%8F%E1%85%B3%E1%84%85%E1%85%B5%E1%86%AB%E1%84%89%E1%85%A3%E1%86%BA_2025-10-28_13.07.50.png)

`font-family` 같은 속성은 자식 요소로 상속되는 대표적인 속성입니다. 상속이라는 이름은 처음 배우지만, 앞에서 `body`에 글꼴을 지정했을 때 이미 사용한 개념입니다.

우리 코드에서는 `body`에 글꼴을 지정했습니다.
자식 요소는 이 글꼴을 상속받아 사용합니다.


#### 정리 및 개발자 도구의 중요성

정리하자면 일부 CSS 속성은 조상 태그에서 자식 태그로 상속됩니다. 개발자 도구에서 `inherited from`으로 표시되는 규칙은 조상 요소에서 물려받은 규칙입니다.
상속 여부와 최종 적용 값을 확인하려면 개발자 도구를 꼼꼼히 살펴보는 것이 중요합니다. 이 흐름을 이해하면 CSS가 예상과 다르게 적용될 때 원인을 찾는 데 도움이 됩니다.
