### 배웠던 내용 복습하기

> 앞에서 CSS 규칙, 선택자, 선언에 대해서 배웠습니다. 여기 코드를 보면서 잠깐 복습해 볼게요. 제주 여행 정보가 있는 간단한 페이지인데요. 보시면 body 태그 안에 img 태그, h1, p, h3 이런 것들이 있습니다.
> 

### CSS 규칙과 선택자, 그리고 선언

> 그리고 head 태그 안에는 style 태그가 있는데요. 
여기 이렇게 태그 이름이랑 중괄호가 있는 부분을 CSS 규칙이라고 했고 여기 태그 이름이 적혀 있는 부분이 CSS 선택자, 중괄호 안에 CSS 속성이랑 콜론, 속성 값, 세미콜론이 있는 부분이 선언이라고 했습니다. 
선택자에는 태그 이름 말고도 다양한 것들을 쓸 수 있는데요. 이번에는 아이디(id)라는 걸 배워 볼게요.
> 

### h3 선택자

> 여기 보시면 h3라는 선택자는 모든 h3 태그를 선택하는 선택자입니다. 그래서 보면 **오렌지 색깔**이 모든 h3 태그에 적용되어 있죠? 만약에 이 페이지에서 어떤 태그 딱 하나만 선택해서 적용하고 싶다면 id라는 선택자를 쓸 수 있습니다. 여기 한라산 국립공원 부분에만 색깔을 다르게 해 볼게요.
우선 h3 태그에다가 id라는 속성을 추가하고 hallasan이라고 할게요. 그리고 style 태그 안에다가 h3 대신에 #hallasan이라는 선택자로 규칙을 만들어 줍니다.
> 

```html
<h3 id="hallasan">한라산 국립공원</h3>
```

```css
body {
	...
}

#hallasan { /* h3을 hallasan으로 변경 */
  color: #f56513;
}
```

> 여기 #이라는 건 아이디 이름을 표시하는 건데 hallasan이라는 아이디를 선택하라는 의미입니다. 저장하고 새로고침해보면 이렇게 한라산 국립공원 제목만 색깔이 바뀌었네요.
> 

### img 태그 규칙 추가하기

> 여기 밑에 있는 img 태그에 있는 속성도 규칙으로 만들어 주겠습니다.
> 

```html
// 변경 전
<img style="width: 100%" src="jeju.png" /> 
 
// 변경 후
<img src="jeju.png" />
```

```css
p {
	...
}

img {
  width: 100%;
  margin-bottom: 56px;
}
```

### 모든 img 태그에 스타일이 적용된 것을 확인

> 확인해 보면 여기 이렇게 이미지가 두 개나 있어서 모든 이미지에 적용됩니다. 
지도를 보여 주는 img 태그는 제목 바로 밑에 있는 이거인 거 같은데요.
여기에다가 map이라는 아이디를 만들고요. 그냥 img 대신에 #map이라고 아이디 선택자를 쓸게요. 그러면, 그럼 지도 이미지만 이렇게 화면에 꽉 찹니다.
> 

```html
<img id="map" src="jeju.png" />
```

```css

#map { /* img 태그에서 #map으로 변경 */
  width: 100%;
  margin-bottom: 56px;
}
```

### Id를 사용할때 주의할 점

> 참고로 아이디를 만들 때 한 가지 주의할 점이 있는데요.
**바로 아이디 이름은 중복해서 쓰면 안된다는 점이에요.**
예를 들어서 이렇게 성산 일출봉에다가 hallasan이라는 아이디를 똑같이 쓰면 안 됩니다. 이럴 때는 seongsan 뭐 이런 식으로 아이디를 쓸 때는 각 요소마다 고유한 이름을 지어 줘야 하니까 이것도 함께 기억해주세요.
> 

### 어떤 점 때문에 중복해서 쓰면 안되나요?

> ID가 중복되면 발생하는 문제점들에 대해 설명해드리겠습니다:
> 
> 1. **선택자의 모호성**
>     - `document.getElementById()`를 사용할 때 동일한 ID가 여러 개 있으면 항상 첫 번째 요소만 선택됩니다
>     - JavaScript로 특정 요소를 조작하고 싶을 때 의도한 요소가 아닌 다른 요소가 선택될 수 있습니다
> 2. **HTML 명세 위반**
>     - HTML 표준에 따르면 ID는 문서 내에서 유일해야 합니다
>     - `<a>` 태그 `href` 에 아이디를 사용할 수 있는데, 중복으로 쓰면 정상 작동하지 않습니다.

### 04. 클래스 ⭐⭐⭐

### 하나만 사용할 수 있는 Id

> 아이디를 쓸 때는 요소 하나당 하나의 이름을 지어줬기 때문에 id 선택자로는 요소 하나 밖에 선택할 수 없었습니다.
만약에 두 개 세 개 이렇게 여러 개 선택하고 싶다면 id가 아니라 클래스라는 걸 써야 하는데요. 
이번에는 클래스라는 걸 배워봅시다. 여기 우도, 한라산 국립공원 같은 제목의 크기랑 굵기를 바꿔 볼게요.
> 

> 아이디를 쓰면 태그 하나만 딱 정해서 스타일을 지정할 수 있는데 클래스를 쓰면 여기 네 개만 딱 정해서 적용할 수 있습니다. 
일단 class라는 속성에다가 place라는 이름을 적어 줄게요.
> 

```css
<h3 class="place">우도</h3>
```

### 다른 곳에 적용하기

> 그리고 다른 태그들에도 붙여 넣겠습니다.
> 

```css
<h3 class="place">우도</h3>
<h3 id="hallasan" class="place">한라산 국립공원</h3>
<h3 class="place">성산 일출봉</h3>
<h3 class="place">군산 오름</h3>
```

### CSS 선택자 작성하기

> CSS 코드에서는 id 선택자 대신에 .place라고 적겠습니다.
font-size: 16px; font-weight: 400; 
id 선택자를 쓸 때는 #을 썼던 것처럼 여기 마침표(.)는 place라는 게 클래스 이름이라는 뜻 입니다.
> 

```css
#hallasan {
	...
}

.place {
  font-size: 16px;
  font-weight: 400;
}
```

> 제목들에만 글자 크기랑 굵기가 잘 적용됐네요. 밑에 소개나 전설에는 적용이 안 됐죠?
이렇게 클래스 선택자를 쓰면 태그 여러 개를 선택해서 스타일을 지정할 수 있습니다.
> 

### 단일 태그에 클래스 사용하기

> 그런데, 꼭 여러 개일 필요는 없는데요.
예를 들어서 지금 p 태그들에 배경색이 들어가 있는데 맨 위에 있는 거에만 배경색을 넣고 나머지는 글자색만 바꿔 볼게요. 일단 선택자를 .description으로 바꾸고 나머지 
p 태그들은 글자색만 넣어 줍니다.
> 

```css
.place {
	...
}

.description { /* p태그를 .description 으로 변경해줌 */
  background-color: #f8f8f8;
  color: #555555;
  padding: 16px 24px;
  margin: 24px 0 56px;
}

p { /* 추가 */
  color: #919191;
}
```

> 그리고 p 태그에 클래스 이름을 추가해 줘야겠죠?
> 

```html
<p class="description">
  일정은 짧고 갈 곳은 많고. 먹고 싶은 것도 보고 싶은 것도 많은데 ... 여행
  인플루언서들이 강력 추천하는 여행지들을 모았습니다. 한눈에 보이는 제주
  여행지 위시리스트 4곳!
</p>
```

> 맨 위에 있는 p 태그만 배경색이 적용되어 있고 나머지 p 태그들은 글자색만 바뀌어 있습니다.
참고로 클래스 이름이 숫자로 시작하면 안됩니다.
> 

### 정리

> 정리하자면 하나만 스타일링하고 싶을 때는 아이디를 쓰고 
여러 개를 한번에 스타일링하고 싶을 때는 클래스를 쓰면 되는데요.
혹시 뭘 써야 할지 헷갈리신다면 그럴 때는 클래스 선택자를 쓰는 걸 추천드립니다. 
많이 쓰다 보면 어떤 상황에서 어떤 걸 쓰는지 자연스럽게 배울 수 있으니까 처음에는 너무 걱정하지 않으셔도 됩니다.
> 

### 05. CSS 파일 ⭐⭐⭐

### 이번 강의 목적

> 앞에서 CSS 코드는 전부 style 태그 안에다가 썼는데요. 이렇게 위아래로 왔다 갔다 하면서 코드를 보는 게 너무 힘들기 때문에 보통 CSS 파일을 따로 저장합니다. 이번 레슨에서는 CSS 파일을 만들고 사용하는 법에 대해 배워볼게요. VS Code에서 파일을 만들 건데요.
> 

### style.css 파일 만들기

> style.css라는 이름으로 파일을 만듭니다. 여기 style이라는 이름은 딱히 이유가 있는 건 아니고 그냥 무난한 이름으로 쓴 거니까 여러분이 원하는 이름을 쓰시면 됩니다. 그리고 여기다가 style 태그 안에 있는 내용을 가져옵니다.
> 

### link 태그 사용하기

> 그리고 이제 style 태그는 지우고 link라고 쓸게요. 앞에서 구글 폰트를 쓰는 법을 배울 때 봤던 태그인데요. link 태그는 현재 문서 바깥에 있는 파일을 불러오는 태그입니다. 구글 폰트에서는 구글 서버에 있는 CSS 파일을 불러왔던 건데요.
rel이라는 속성에는 stylesheet라고 적어주고 href라는 속성에는 style.css라고 파일 경로를 써줍니다.
> 

```html
<link rel="stylesheet" href="style.css" />
```

### link 태그 속성 값 보기

> 하나씩 살펴보면 rel이라는 건 영어로 Relationship의 약자인데 불러올 파일과의 관계를 나타냅니다. stylesheet라고 적었으니까 우리가 불러올 파일이 스타일 시트, 즉 CSS 파일이라는 뜻이죠.
그리고 href는 a 태그에서도 봤던 건데 파일의 주소, 경로라고 생각하면 됩니다. 여기서는 style.css라고 파일 이름만 써줬는데요. 현재 문서의 경로를 기준으로 파일을 불러오는데 같은 폴더에 있으니까 이렇게 이름만 써도 되는 겁니다.
> 

### 확인하기

### 07. 개발자 도구로 선택자 확인하기

### 개발자 도구 알아보기

> 이번에는 개발자 도구라는 걸 사용해서 선택자를 확인하는 법에 대해 알아볼게요. 우선 개발자 도구를 열어 볼 건데요. 앞에서 짧게 배워 봤었는데 F12버튼을 누르시면 개발자 도구가 열립니다.
> 

### 선택 도구

> 쪽 위에 있는 요소 선택 도구를 사용해서 제목을 하나 선택해 볼게요.
여기 Styles라는 탭을 보시면 CSS 규칙이 보이는데요. 여기 내가 작성한 코드도 있고 밑에는 작성한 적 없는 코드도 나옵니다. 예를 들어서 여기 user agent stylesheet라는 건 웹 브라우저에서 제공하는 기본 스타일인데 이렇게 Styles 탭에서는 이 요소에 적용된 모든 규칙들이 보입니다.
> 

### h3 태그, place 클래스 확인하기

> 자 이번에는 여기 h3 위에다가 마우스를 올려 볼게요.
그럼 이 선택자가 선택한 요소들이 화면에 보입니다. 이번에는 place라는 클래스 선택자를 살펴볼게요. 마찬가지로 마우스를 올려보면 이번에는 클래스 이름이 있는 것만 선택돼서 보이네요. 앞으로 수업에서 계속 개발자 도구를 쓸 건데요.
개발자 도구를 사용하면 CSS 규칙이 잘 적용되고 있는지 확인할 수 있고 특히 선택자가 어디 어디 적용되는지 한눈에 확인할 수 있으니까 잘 알아 두시면 좋을 거 같습니다.

### card에 배경추가하기

> 저는 미리 pizza.png라는 파일을 다운받았는데요. 
이걸 여기 card라는 div 태그에다가 배경 이미지로 넣을 겁니다. 
이제 card 부분에 코드를 추가할 건데요.
(css 파일로 옮겨서)
앞에서 배경색을 넣을 때는 background-color라는 걸 썼던 거 기억나시나요? 이미지를 넣을 때는 color 대신에 image를 써주면 됩니다. 그리고 url('pizza.png') 이렇게 넣어 줄게요.
> 

```css
.card {
  background-image: url("pizza.png"); /* 수정 */
  color: #ffffff;
  line-height: 1.7;
  margin: 40px auto;
  padding: 40px;
  width: 480px;
}
```

### url에 대해서

> 여기 url이라는 건 처음 배우는 건데 어렵지 않습니다. 
파일 주소나 경로를 여기 괄호 안에다가 쓰는 건데요. 
참고로 따옴표는 보통 없어도 괜찮지만 특수한 문자들(쉼표 등)이 있을 때는 반드시 따옴표가 필요한데요. 
그래서 저는 따옴표로 항상 주소를 감싸주는 것을 추천드립니다.
> 

### 결과 확인하기

> 이렇게 쓰면 pizza.png라는 파일을 현재 페이지의 경로를 기준으로 찾아줄 겁니다. 그럼 이런 식으로 이미지가 반복되는데요. 배경 이미지는 기본적으로 주방 타일처럼 반복해서 깔립니다.
> 

### 배경 채우기

> 타일은 조금 촌스러운 느낌이 있으니까 배경 이미지를 꽉 차게 만들어 보겠습니다. 코드부터 일단 쭉 써볼게요.
> 

```css
.card {
  background-image: url("pizza.png");
  background-repeat: no-repeat;  /* 수정 */
  background-position: center;  /* 수정 */
  background-size: cover;  /* 수정 */
  color: #ffffff;
  line-height: 1.7;
  margin: 40px auto;
  padding: 40px;
  width: 480px;
}
```

> 여기 background-repeat은 배경을 반복할지 정하는 건데 no-repeat, 반복 안 한다는 거고요. background-position은 이미지를 어떤 위치를 기준으로 배치할지 정하는 건데 중앙 정렬을 해줬고요. background-size는 이미지 크기를 정하는 건데 cover, 꽉 차게 했습니다. 확인해 보면 이번에는 꽉 차게 보이네요.
> 

### 개발자 도구에서 확인하기

> 개발자 도구로 보면서 코드를 한번 더 정리해볼게요. 자 card에 해당하는 div 태그를 선택하고요, 여기 Styles을 보시면 CSS 코드가 보이는데요. 마우스를 올려보면 속성을 껐다 켰다할 수 있습니다. 일단 background-position이라는 속성이랑 size 속성은 끄고 background-repeat만 보면 이런 식으로 이미지가 반복이 안 되죠?
> 

### background-repeat 옵션 알아보기

> no-repeat 쪽을 더블 클릭해서 지운 다음에 다른 값들을 간단히 살펴보자면 대표적으로 no-repeat, repeat, repeat-x, repeat-y 같은 게 있는데 repeat은 기본 값인데 전체를 반복하는 거고요. repeat-x는 가로만 반복하는 거구요, repeat-y는 세로만 반복하는 겁니다. 다시 no-repeat으로 바꿔 놓을게요. 보통 no-repeat을 제일 많이 쓰게 되실 겁니다.
> 

### background-position 알아보기

> 이번에는 background-position을 볼게요. 보시면 center로 하니까 배경 이미지가 가운데로 정렬됐죠? center 말고도 bottom, center, left, right, top 이런 식으로 이미지 위치 기준을 바꿔 줄 수 있습니다.
> 

### background-size 알아보기

> 마지막으로 background-size를 보면 cover라고 했을 때 이미지가 꽉 차는 거 보이시나요?
> 
> 
> cover는 이미지 비율을 유지하면서 영역에 꽉 채우는 겁니다. cover 말고 
> contain이라는 값도 있는데 이건 이미지가 잘리지 않는 선에서 최대 크기로 배치해 줍니다.
> 
> 가로 크기, 세로 크기를 직접 지정해 줄 수도 있는데 
> 예를 들어서 가로 100px, 세로 50px, 모양이 좀 웃기지만 이렇게도 정할 수 있습니다. 
> 다시 cover로 바꿔 놓을게요.
> 

```css
background-size: 100px 50px;
```

> 이거도 아마 대부분의 경우에는 cover 값을 쓰실 겁니다. 여기 글자랑 이미지가 잘 구분이 안 가서 아쉬운데 이건 뒤에서 좀 더 고쳐 보도록 하겠습니다.
> 

### cover와 contain의 차이

### `cover`

- **이미지가 잘릴 수 있음**: 요소의 전체 영역을 완전히 덮도록 이미지의 가로/세로 비율을 유지하면서 확대/축소됩니다.
- **빈 공간 없음**: 이미지의 일부가 요소 바깥으로 벗어날 수는 있지만, 요소 내부에 빈 공간은 절대 생기지 않습니다.
- **용도**: 주로 요소의 배경을 이미지로 가득 채우고 싶을 때 사용합니다. (예: 전체 화면 배경)

### `contain`

- **이미지가 잘리지 않음**: 이미지 전체가 요소 안에 모두 보이도록 이미지의 가RO/세로 비율을 유지하면서 확대/축소됩니다.
- **빈 공간 생길 수 있음**: 이미지의 비율과 요소의 비율이 다를 경우, 요소 내부에 빈 공간(레터박스처럼)이 생길 수 있습니다.
- **용도**: 이미지의 전체 모습을 손상 없이 보여주는 것이 중요할 때 사용합니다. (예: 로고, 갤러리 썸네일)

### 04. 그라디언트: linear-gradient ⭐

### 그라데이션 소개하기

> 매킨토시, 아이팟, 아이폰 수많은 혁신을 만들어낸 애플의 스티브 잡스 스티브 잡스는 항상 검은 상의에 청바지를 입고 발표 키노트에도 검정색이랑 물 빠진 청바지 색을 섞은 배경으로 유명하죠. 여기 배경을 보시면 검정색에서 점점 남색으로 바뀌는 배경이 있는데 이런 걸 그라데이션(Gradation)이라고 부릅니다. 이번 레슨에서는 CSS에서 그라데이션 넣는 법을 배워볼게요.
>

### 그라데이션 추가하기

> 그라디언트는 background-image라는 속성으로 넣을 수 있는데요.
> 
> 
> 그라디언트도 일종의 배경 이미지처럼 쓰는 거라고 이해하시면 됩니다. 아까 넣었던 이미지 대신에 linear-gradient()를 넣어주고요.
> 

```css
.card {
  background-image: linear-gradient(rgba(0, 0, 0, 1), rgba(0, 0, 0, 0)); /* 변경 */
  background-repeat: no-repeat;
  ...
}
```

> 시작 색깔은 검정색에서 마지막 색깔은 투명한 검정색으로 할게요. 그럼 이렇게 아래로 갈수록 검정에서 투명에 가까워지는 그라디언트가 만들어집니다.
> 

### 각도 변경하기

> 만약에 방향을 바꾸어 주고 싶다면 맨 앞에다가 각도를 적어주면 됩니다. 90deg, 이렇게 추가하고 확인해 보면 오른쪽 각도로 만들어지네요.
> 

```css
background-image: linear-gradient(90deg, rgba(0, 0, 0, 1), rgba(0, 0, 0, 0));
```

### 중간에 색상 변경하기

> 그라디언트의 단계마다 비율을 다르게 하고 싶다면 중간에 이렇게 40% 이런 식으로 적어주면 되는데요.
> 

```css
  background-image: linear-gradient(
    90deg,
    rgba(0, 0, 0, 1) 40%,
    rgba(0, 0, 0, 0)
  );
```

### 구글 gradient generator

> 그라디언트는 정말 다양하게 만들 수 있는데 문법이 복잡하기도 하고 이런 걸 여러분이 직접 작성하시는 것보다는 도구를 이용하는 걸 추천드릴게요.
> 
> 
> 구글에서 gradient generator라고 검색해볼게요. 예를 들어서 이런 사이트에 들어가보면 마우스 클릭 만으로 다양한 그라디언트 코드를 만들 수 있습니다.
> 

### 이미지에 그라데이션 넣기

> 마지막으로 꿀팁을 하나 알려드릴게요. 정말 유용한 기능인데요.
배경 위에다가 반투명한 그라디언트를 겹쳐보겠습니다. 그라디언트 코드 맨 끝에 ,로 끝낸 다음에 다음 줄에다가 이미지를 넣는데요. 이 순서가 중요하니까 기억해주세요. 이미지 위에 반투명한 그라디언트를 겹쳐놓는다고 생각하면 됩니다.
> 

```css
.card {
  background-image: linear-gradient(
      90deg,
      rgba(0, 0, 0, 1) 30%,
      rgba(0, 0, 0, 0)
    ),
    url("pizza.png");
    ...
  }
```

> 그냥 이미지만 넣으면 글씨가 잘 안 보였는데 이러니까 훨씬 낫죠? 이런 식으로 이미지랑 그라디언트를 같이 넣는 건 자주 쓰니까 팁으로 알아둡시다.

### 07. 불투명도: opacity ⭐⭐

### 이번 시간 배울 내용 소개

> 페퍼로니 피자를 팔다 보니까 너무 인기가 많아서 페퍼로니 햄이 다 떨어져 버렸습니다. 그래서 품절되었다고 홈페이지를 고치려고 하는데요. 보통 이럴 때는 주문 버튼을 품절이라는 텍스트로 바꾸고 버튼을 반투명하게 만들죠? 이번 레슨에서는 품절 버튼을 만들면서 요소 전체를 반투명하게 만드는 법을 배워봅시다.

> 

```html

</a>
```

### opacity 추가하기

> CSS 파일에서는 order 클래스에다가 opacity: 0.5;를 추가할게요.
> 

```css
.order {
  background-color: #f13030;
  color: #ffffff;
  font-weight: 800;
  padding: 6px 16px;
  text-decoration: none;
  opacity: 0.5; /* 추가 */
}
```

> 앞에서 색상을 배울 때 rgb, rgba를 배우면서 알파(Alpha)라는 값 배웠던 거 기억나시나요? 알파 값처럼 opacity도 값이 1에 가까울수록 불투명해지고 값이 0에 가까워질수록 투명해지기 때문에 불투명도라고 부릅니다.

### 개발자 도구에서 변경하기

그러면 스타일에 opacity 가 들어가 있는 것을 볼 수 있는데요.
여기에 0.1부터 값을 1로 변경해보며 어떻게 변화하는지 알아보도록 할게요
> 

### 주의점 및 정리

> opacity를 사용할 때 주의할 점은 태그 전체가 반투명해진다는 겁니다. 배경색만 반투명하게 하고 싶다면 앞에서 배웠던 rgba를 활용하면 됩니다. 이건 뒤에서 따로 한번 더 정리해 드릴게요.
요소 전체를 반투명하게 만들고 싶으면 opacity를 사용한다는 것 정말 자주 쓰니까 꼭 알아둡시다.

    - `background-image`에 여러 값을 사용하면 먼저 선언된 것이 위로 쌓입니다. 따라서 그라데이션을 먼저 쓰고 이미지 URL을 써야 이미지 위에 그라데이션이 겹쳐 보입니다.

## 문제 3

`box-shadow: 0px 10px 20px 5px rgba(0, 0, 0, 0.2);` 코드에서 `10px`, `20px`, `5px`는 각각 무엇을 의미하나요?

- 정답 확인하기
    - `10px`: 그림자의 **세로 위치(vertical offset)**. 양수 값이면 아래쪽으로 그림자가 생깁니다.
    - `20px`: **흐림 효과(blur radius)**. 값이 클수록 그림자 경계가 부드럽고 넓게 퍼집니다.
    - `5px`**: 퍼지는 정도(spread radius).** 양수 값이면 그림자가 모든 방향으로 지정된 크기만큼 커집니다.
    - (참고: 첫 번째 값 `0px`는 가로 위치(horizontal offset)입니다.)

"주문하기" 버튼의 배경색만 반투명하게 만들고 싶을 때와, 버튼 전체(텍스트 포함)를 반투명하게 만들고 싶을 때 사용해야 하는 CSS 속성은 각각 무엇인가요?

### 07. box-sizing ⭐⭐

### 개발자 도구 Computed로 확인하기

> 사실 너비나 높이 같은 것들은 기본적으로 content의 크기를 정하는 건데요. 아까처럼 Computed로 가서 확인해 볼게요. 보시면 content의 너비가 400으로 되어 있죠?

400 + 40px 은 440px이 되는 것이죠.

### 진짜로 400px로 만들려면

> 너비를 400으로 하고 싶으면 위에서 했던걸 역으로 계산하면 되잖아요.
그런데 어때요? 너무 귀찮지 않나요? 
또 padding 값이나 border 값을 바꿀 때마다 다시 계산해줘야 되기도 하구요.

### 자동으로 계산하려면

> 가로 길이랑 세로 길이를 설정했을 때 border, padding 계산 되어서 그냥 딱 맞춰서 나왔으면 좋겠는데요.

### border-box 사용하기

> 


### * (Asterisk)의 의미

> 이 box-sizing: border-box;를 쓰는 게 너무 편하다 보니까 요즘은 일반적으로 모든 요소에다가 이걸 추가하는 추세입니다. 모든 요소에다가 같은 스타일을 추가하고 싶으면 이런 식으로 별(*)을 써주면 되는데요. 나중에 뒤에서 자세히 배울 건데 여기서 *은 모든 요소라는 뜻입니다. 여러분도 앞으로는 웬만하면 이걸 쓰고 개발을 시작하세요.

### overflow

> 이렇게 넣겠습니다. 그러면 이렇게 박스는 작아졌는데 내용은 넘쳐버립니다. 이런 거를 오버플로우(overflow)라고 부르는데요. 이럴 때는 넘친 내용을 감추거나 스크롤할 수 있게 해주면 됩니다.
> 

### overflow: hidden

> height 밑에다가 overflow: hidden; 이렇게 넣으면 삐져나온 게 안 보이죠?
> 

### overflow: scroll

> 이번에는 overflow: scroll;이라고 넣어줄게요. 그럼 이렇게 넘치는 내용을 스크롤해서 볼 수 있습니다. 스크롤은 꼭 세로만 되는건 아니고 가로로도 할 수 있는데요.
> 

```css
  overflow-y: scroll;
```

> 이렇게 해도 마찬가지로 작동하는 것을 확인 할 수 있습니다.
> 

### overflow-x: scroll

> 이번엔  ****overflow-x: scroll;을 해볼건데,
> 

```css
  overflow-x: scroll;
```

> 마찬가지로 잘 작동하는 것을 확인할 수 있습니다.
왜 가로축인데(x)인데 잘 되는 것 일까요?
그건 브라우저에서 자동으로 조정하기 때문입니다.
overflow-x: scroll이 되었을때, overflow-y: auto로 자동으로 설정하기 때문이죠.
다시 overflow: scroll로 설정해주겠습니다.
> 

```css
overflow: scorll;
```

> 이때 바깥에서 이 태그들을 감싸고 있는 `<div class="terms">`를 **부모 요소**라고 하고, 그 안에 포함된 `<h1>`, `<p>` 등을 **자식 요소**라고 부릅니다.
> 
> 마치 'terms'라는 이름의 폴더 안에 세 개의 다른 파일/폴더가 들어있는 것과 똑같은 구조입니다.
> 
> 다음은 '형제 관계'입니다. 하나의 폴더 안에 여러 파일이 나란히 들어있을 수 있죠? 이것과 마찬가지로, HTML에서도 **같은 부모를 가진 자식 요소들**을 서로 '형제 관계'에 있다고 말합니다.
> 여기에는 `<h1 class="terms-title">` 태그, `<p class="terms-description">` 태그, 그리고 `<div class="terms-content">` 태그가 나란히 들어있습니다.
> 
> 이 세 태그는 모두 `<div class="terms">`라는 동일한 부모를 가지고 있으므로, 서로 **'형제 관계'**에 있는 것입니다.
> 

### 형제 관계 태그 마진 상쇄

> 이렇게 알아두고서 한번 마진 상쇄에 대해서 알아보도록 할게요
어떤 태그에 스타일로 세로 마진값이 들어갔을 때 브라우저는 단순히 우리가 정한 값을 더해서 간격을 만들지 않습니다. 예를 들어서 현재 개발자 도구로 보면
> 
> 
> ![스크린샷 2025-10-26 17.37.15.png](03%20CSS%20%ED%95%B5%EC%8B%AC%20%EA%B0%9C%EB%85%90/%E1%84%89%E1%85%B3%E1%84%8F%E1%85%B3%E1%84%85%E1%85%B5%E1%86%AB%E1%84%89%E1%85%A3%E1%86%BA_2025-10-26_17.37.15.png)
> 
> computed창에서 확인해보면 h1 태그에도 마진이 8px, p태그에도 마진이 8px이 적혀 있기 때문에 이 사이가 16px이 되는 것이 아닌가 할 수 있는데요.
> 
> 그러나 그 둘 사이 값을 확인해보면 16px이 아닌 8px가 들어가 있는 것을 볼 수 있습니다.
> 여기 h1 태그랑 p 태그 사이의 간격은 h1 태그 마진이랑 p 태그의 마진을 더 한 게 아니라 둘을 겹쳐서 더 큰 마진을 적용하는데요.
> 이런 걸 마진 상쇄, 영어로는 margin collapsing이라고 부릅니다. 쉽게 말해서 세로 마진은 서로 겹친다라고 이해하시면 편한데요.
> 

### 부모 자식 관계에서 마진 상쇄

> 부모랑 자식 태그도 겹치는데요. 여기 terms라는 div 태그에서 border랑 padding을 없애볼게요. 그리고 나서 확인해 보면 여기 부모 div 태그의 마진이랑 안에 있는 terms-title 태그의 마진이랑 겹친 거 보이시나요? 부모 자식 간에도 세로 마진이 겹칠 수 있습니다.
잘 이해가 안되시면 한번 개발자 도구를 이용해서 확인할 건데, 일단 배경색을 주어서 terms와 terms-title의 영역을 확인해보겠습니다. terms에 red, title에 gold를 추가하고 그런 다음에
terms에 margin-top: 0px을 추가한 뒤에 terms-title에 margin-top: 32px을 넣어보겠습니다. 자 그러면 이렇게 공간이 생긴 것을 확인 할 수 있습니다. 그런 다음에 terms 에 margin-top: 32px를 넣어주면 아무 변화가 없는 것을 확인 할 수 있습니다. 
이렇게 되는 이유가 바로 마진 상쇄 때문이지요; 마진 상쇄는 더 큰 마진을 적용한다고 했는데요.
terms-title에 margin-top:42px로 변경하겠습니다. 한번 화살표 키를 이용해서 변경해보면 계속해서 1px씩 변화가 있는 것을 알 수 있습니다.
> 

### 마진 상쇄 해결 방법

> 다시 새로 고침해서 border랑 padding이 있을 때를 살펴보면 이번에는 padding에 막혀서 세로 마진이 안 겹치는 거 보이시죠?
잘 모르겠으면 역시 개발자 도구를 열어서 한번 보면 terms에 background-color: gold;를 주고 영역을 정확히 파악한 뒤에 padding과 border를 제거(체크해제)하고 terms-title의 margin-top영역이 gold 색상 바깥쪽으로 나가 있는 것을 확인할 수 있습니다. 
그러면 우선 terms의 border를 다시 추가하고 terms-title을 보면 margin-top 영역이 gold 색상 안쪽에 있는 것을 확인할 수 있습니다. padding도 마찬가지에요 한번 border를 체크해제하고 padding을 체크하고 확인해보면 padding 영역 그리고 terms-title의 margin-top 영역이 따로 잘 들어가 있는 것을 확인할 수 있습니다.
> 

### 정리하기

> 다시 정리하자면 이웃하는 요소들끼리는 마진 상쇄가 일어나고 부모 자식 사이에도 마진 상쇄가 일어날 '수' 있습니다. 여기서 일어날 '수' 있다고 한 것은 부모 요소에 border나 padding이 있어서 어떤 경계가 생기는 경우에는 마진이 상쇄되지 않습니다. 그리고 형제 관계에서 마진 상쇄도 생기는데 이때 이걸 막으려면 중간에 다른 태그를 추가하거나 둘 중 하나의 태그에만 마진을 주어야 합니다. 
이제 앞으로 여러분들이 CSS를 해보면서 세로 마진이 내가 원하는 대로 안 나온다 이럴 때는 마진이 상쇄된 게 아닌지 확인해 보시면 됩니다. 참고로 가로 마진에는 마진 상쇄가 일어나지 않습니다.

> 

### inline-block

> 배치는 인라인처럼 하면서 크기도 정하고 싶을 때는 display 속성으로 인라인 블록(inline-block)이라는 걸 쓰면 되는데요. 인라인 블록은 배치는 인라인처럼 되지만 블록처럼 크기가 있는 녀석이라고 보시면 됩니다. display: inline-block; 이렇게 추가하고 확인해보면
> 

```css
.read-more {
  background: #528d9f;
  border-radius: 8px;
  display: inline-block;
  ...
}
```

> 이런 식으로 인라인으로 배치되고 크기도 갖고 있기 때문에 세로로 자리를 밀어내서 배치됩니다.
여기 테스트하려고 추가했던 내용은 다시 지워줄게요. 이렇게 display 속성을 바꿔주면 인라인을 인라인 블록으로 바꿀 수도 있고 블록을 인라인으로 바꾸거나 자유롭게 바꿔줄 수 있습니다.
> 

### display: none;

> 참고로 display에 none이라는 값도 있는데요. 이건 CSS로 HTML 태그를 감출 때 사용하는 값입니다. display 값 중에서 block, inline, inline-block 이 세 가지는 정말 중요한데요. 이것들의 공통점과 차이점을 뒤에서 표로 정리해드릴 거니까 나중에 꼭 읽어보세요.
> 

### 정리하기

> 자 다시 정리할게요. 여러분이 CSS를 작성하다가 다른 요소들이 자리를 안 비켜준다 너비나 높이가 적용되지 않는다 그럴 때는 그 요소가 인라인인지 확인해보고 inline-block으로 바꿔서 써보세요.

문제 1

`display: block`과 `display: inline`의 가장 큰 차이점은 무엇인가요?

- 정답 확인하기
    
    정답
    
    - block은 한 줄 전체를 차지하는 박스 요소
    - inline은 글자처럼 한 줄에 이어 나란히 배치되는 요소
    
    해설
    
    - block 요소는 새로운 줄에서 시작하며 사용할 수 있는 전체 너비를 차지함.
    - inline 요소는 새로운 줄에서 시작하지 않고, 필요한 만큼의 너비만 차지함.

---

문제 2

기본적으로 display: inline인 `<a>` 태그에 padding-top: 20px, height: 100px을 주면 주변 요소와 어떤 현상이 발생하나요?

- 정답 확인하기
    
    정답
    
    - 가로(좌우) padding은 적용되지만, 세로(상하) padding과 height는 적용되지 않아 다른 요소들과 겹칠 수 있다.
    
    해설
    
    - inline 요소는 글자처럼 취급되며 width, height가 적용되지 않는다.
    - padding의 좌우 값은 주변 인라인 요소에 영향을 주지만 상하 값은 시각적으로만 확장되고 레이아웃에는 영향 없음.

### 04. 선택자 붙여 쓰기 ⭐⭐⭐

> 지금까지는 선택자를 쓸 때 주로 하나만 썼는데요. 이번 레슨에서는 선택자를 여러 개 붙여서 쓰는 방법을 배워 보겠습니다.
> 

> 일단 HTML에서 클래스를 여러 개 쓰는 법부터 살펴볼게요. 예를 들어서 여기 book-info라는 클래스만 쓴 데다가 delay라는 클래스를 추가로 쓰고 싶으면 이런 식으로 delay book-info, 이렇게 쓰면 됩니다.
> 

```html
<p class="delay book-info">
  지금 택배로 주문하면 <span class="date">11월 14일</span>에 받아 볼 수
  있어요
</p>
```

### 클래스를 이용한 선택자 붙여쓰기

> CSS 코드에서는 delay 클래스를 만들고 간단하게 배경색을 지정해 줄게요.
개발자 도구로 확인해 보면 delay랑 book-info가 둘 다 적용됩니다. 근데 여기서 delay 클래스만 갖고 있는 태그를 선택할 수도 있지만 delay랑 book-info를 둘 다 갖고 있는 요소를 선택할 수도 있는데요. 이런 식으로 클래스 선택자를 붙여서 쓰면 됩니다.
> 

```css
.delay.book-info { /* .book-info 추가 */
  background-color: #c76546;
  white-space: nowrap; /* 한줄이 넘어가면 추가 */
}
```

### 순서를 바꾸어도 상관 없음

> 이렇게 하면 delay 클래스, book-info 클래스 둘 다 갖고 있는 요소를 선택합니다. 여기 순서는 상관없으니까 .book-info.delay 이런 식으로 써도 됩니다. 근데 헷갈릴 수 있으니까 HTML 코드랑 순서를 맞춰서 쓸게요.
> 

### 태그를 이용한 선택자 붙여쓰기

> 이거 말고도 클래스 선택자랑 태그 선택자를 같이 쓰거나 아이디 선택자랑 태그 선택자를 같이 쓸 수도 있는데요. .delay.book-info를 보면 p 태그인데 만약에 p 태그이면서 delay 클래스랑 book-info 클래스 둘 다 갖고 있는 걸 선택하고 싶으면 맨 앞에다가 p 태그를 붙여 주면 됩니다.
> 

```css
p.delay.book-info { /* p 추가 */
  background-color: #c76546;
  white-space: nowrap;
}
```

> 이 선택자는 p 태그이면서 delay 클래스랑 book-info 클래스 둘 다 갖고 있는 요소를 선택하는 겁니다. 아이디 선택자랑 쓸 때도 마찬가지입니다.
> 

### 아이디를 이용한 선택자 붙여쓰기

> 예를 들어서 p 태그에다가 id="shipping" 이렇게 추가해 볼게요. 그리고 이렇게 선택자를 붙이면 p 태그이면서 shipping이라는 아이디를 갖고 있는 요소를 선택합니다. 마지막으로 아이디 선택자랑 클래스 선택자를 붙여서 쓰는 걸 보여 드릴게요. 여기서 .delay.book-info
> 

```css
#shipping.delay.book-info {
	background-color: #c76546;
  white-space: nowrap;
}
```

### 정리

> 정리해볼게요, 이번 레슨에서는 선택자를 붙여서 쓰는 법을 살펴봤는데요. 우선 클래스를 여러 개 붙여서 쓰면 이 클래스를 모두 갖고 있는 걸 선택하는 거고 맨 앞에다가 태그 선택자를 붙이면 이 태그에 해당하고 이런 클래스를 갖고 있다, 이 태그에 해당하고 이 아이디에 해당한다, 이런 식으로 선택할 수도 있죠. 마지막으로 아이디랑 클래스를 같이 쓰면 이 아이디에 해당하고 이 클래스에 해당하는 요소를 선택할 수 있었습니다. 여러 가지 방법이 있어서 어떤 걸 쓸지 고민되실 텐데요.
> 

> 이런 조합 방법을 알아두는 건 좋지만 조합을 많이 쓴다고 무조건 좋은 건 아닙니다. 저는 되도록이면 클래스 한 개 짜리를 위주로 사용하시는 것을 추천드리고요. 이렇게 book-info 클래스를 여러 군데서 쓰면서 특별한 book-info를 만들고 싶다, 이런 경우에는 클래스 이전에 봤던 .delay.book-info 클래스 2개를 붙여서 쓰는 걸 추천드릴게요.
>

> 

```css
a {
  color: #12172a;
  text-decoration: none;
}
```

> 글자 색을 넣고 text-decoration: none;으로 해주겠습니다. 링크에 있는 기본 글자색을 검정색으로 바꾸고 text-decoration을 none으로 해서 밑줄을 없애줍니다.
확인해 보면 밑줄 없는 검정색 링크가 잘 나오네요.
> 

### hover시 밑줄

> 자 이제 마우스를 올렸을 때 밑줄이 생기게 해볼 건데요.
가상 클래스라는 걸 쓸 겁니다. 가상 클래스는 마침표 대신에 콜론으로 시작하고 이름이 미리 정해져 있습니다. a를 입력하고 콜론 그리고 VScode 추천 단축키(제안 항목 트리거)를 눌러보시면
> 
> 
> ![스크린샷 2025-10-28 09.57.00.png](03%20CSS%20%ED%95%B5%EC%8B%AC%20%EA%B0%9C%EB%85%90/%E1%84%89%E1%85%B3%E1%84%8F%E1%85%B3%E1%84%85%E1%85%B5%E1%86%AB%E1%84%89%E1%85%A3%E1%86%BA_2025-10-28_09.57.00.png)
> 
> 여러가지 나오는 것을 알 수 있는데요. (: 하나는 가상 클래스, :: 두개는 가상 요소) 
> 이 중에서 hover라는 것을 사용하겠습니다.

```css
a:hover {
  text-decoration: underline;
}
```

### hover 설명하기

> a:hover 이렇게 쓰고 text-decoration: underline;을 줄게요. 여기 hover라는 게 가상 클래스의 이름인데 마우스를 올렸을 때를 가리키는 게 :hover라고 생각하시면 됩니다. 이러면 a 태그에 마우스를 올렸을 때 이 가상 클래스가 선택돼서 밑줄이 생길 겁니다.
> 

### 개발자 도구에서 보기

> 개발자 도구로 좀 더 살펴볼게요. a 태그를 선택해서 마우스를 오른쪽 클릭해서 Force state를 보시면 쓸 수 있는 가상 클래스들이 있는데요. active, hover, focus, visited 이 네 개를 자주 쓰는데 이 중에서도 hover를 앞으로 제일 많이 쓰실 겁니다. hover는 마우스를 올린 상태였죠?
다른 것도 잠깐 설명을 드리면 active는 요소(버튼, 링크 등)를 클릭하는 "순간"에 적용됨, focus는 폼 입력 등 요소에 포커스가 갔을 때(클릭, Tab 등) 적용됨
****:visited는 이미 방문한 링크에 적용 됩니다.
> 

```css

a:active { color: green; }
a:visited { color: purple; }

<input name="input" class="input" placeholder="이름을 입력해주세요" /> 

input:focus { outline: none; border: 2px solid blue; }
```

### 캐스캐이드란?

> 이 규칙들을 합쳐서 이렇게 하나로 만들고 이걸 화면에 보여주는 건데요. 근데 규칙들 중에는 여기 있는 background-color처럼 겹치는 속성도 있습니다.
> 
> 
> ![스크린샷 2025-10-28 11.27.33.png](03%20CSS%20%ED%95%B5%EC%8B%AC%20%EA%B0%9C%EB%85%90/%E1%84%89%E1%85%B3%E1%84%8F%E1%85%B3%E1%84%85%E1%85%B5%E1%86%AB%E1%84%89%E1%85%A3%E1%86%BA_2025-10-28_11.27.33.png)
> 
> 이럴 때는 어떤 값을 쓰는 게 좋을까요? 이런 상황에서 CSS 규칙을 어떻게 합칠지 정해주는 방식이 바로 캐스케이드(Cascade)입니다.
> 

### 구글 검색하기

> CSS는 Cascading StyleSheet의 약자인데요. 좀 생소한 단어라서 구글에서 cascade라는 걸 한번 검색해볼게요. 캐스케이드라는 건 이런 계단 같은 게 있는 작은 폭포입니다. 이런 자연물도 있고 이런 분수를 통해서 만들어지는 인공적인 캐스케이드도 있습니다.
> 

### 캐스캐이드의 의미

> 이런 것처럼 CSS에서는 규칙을 순서에 따라서 계단식으로 합치는 걸 캐스케이드라고 하죠. 다시 우리가 만든 페이지로 돌아가서 보면 여기 개발자 도구에서 보여주는 CSS 규칙 순서는 캐스케이드되는 순서입니다. 위에 있는 규칙이 아래에 있는 규칙을 덮어쓴 건데요. 보시면 delay book-info에 있는 background-color가 book-info에 있는 background-color를 덮어썼죠?
> 

### element.style 이용하여 배경 색상 변경하기

> 만약에 제가 여기 맨 위에다가 background-color를 #abcabc 이렇게 주면 그 밑에 있는 것들을 덮어씁니다. 참고로 여기 element.style이라고 되어 있는 건 HTML 태그에다가 직접 스타일 속성으로 넣는 거랑 똑같습니다.
> 

### 정리

> 정리하자면 캐스케이드라는 건 여러 CSS 규칙이 적용될 때 순서에 따라서 합치는거라고 할 수 있는데요. 그럼 이때 합쳐지는 순서는 어떻게 정해지는 걸까요? 이건 뒤에서 알아보도록 하겠습니다.
> 

### 02. 캐스케이드 순서 ⭐⭐⭐

### 캐스케이드 순서 의문

> CSS 규칙들은 순서에 따라서 합쳐서 적용한다고 했습니다. 이런 걸 캐스케이드라고 했는데요. 그럼 캐스케이드 순서는 어떻게 정해지는 걸까요? 여러 가지 기준이 있지만 이번 레슨에서는 가장 핵심적인 기준 세 가지에 대해서 알아보겠습니다.
> 

### user agent stylesheet

> 우선 첫 번째로 스타일 시트의 종류입니다. 크롬 개발자 도구에서 보면 브라우저에 기본 CSS가 있었죠? 이런 거를 user agent stylesheet라고 하는데요. 우리가 만든 style.css보다 우선 순위가 낮습니다. 
즉, 우리가 css를 작성하여, user agent stylesheet에 있는 값을 덮어 씌울 수가 있습니다.
마치 이전에 background-color에서봤듯이요.
> 

### 캐스케이드 우선순위1, 인라인 스타일

> 추가로 HTML에다가 직접 스타일을 작성하는 걸 인라인 스타일이라고 하는데요. 인라인 스타일은 캐스케이드에서 가장 우선 순위가 높습니다. 예를 들어서 개발자 도구에서 컬러 값을 #defdef 이렇게 하면 맨 위에 가장 높은 우선순위로 들어갑니다
> 

### 캐스케이드 우선순위2, 코드상의 순서

> 두 번째로는 코드상의 순서입니다. 똑같은 선택자를 쓰는 경우에는 나중에 쓸수록 우선 순위가 높습니다. 예를 들어서 제가 description이라는 선택자 밑에다가 선택자를 똑같이 쓰고 color: blue 이렇게 넣어 볼게요.
> 

```css
.description {
  color: #818181;
  font-size: 16px;
  font-weight: 400;
  line-height: 23px;
  margin: 0;
}

.description { 
  color: blue;
}
```

>

> 선택자는 book-info보다 우선순위가 높죠? 이건 선택자의 명시도가 높아서 그런데요. 
> VS Code에서 선택자에 마우스를 올려보시면 이런 명시도가 나옵니다.

### selector specificity

> book-info 이렇게 적어 보면 이렇게 점수를 계산해서 보여주는데요. 여기서 앞에 있는 숫자가 클수록 명시도 점수가 높습니다. 일단 첫 번째 숫자는 전부 0으로 똑같고 두 번째 숫자를 보면 .delay.book-info가 2라서 점수가 높습니다. 그 다음으로는 book-info가 점수가 1이고요. 
명시도 점수는 명시도는 아이디 개수, 클래스 개수, 요소 개수를 순서대로 숫자를 나열해서 점수를 매깁니다.
개발자 도구 style에서 보면 위쪽에 있는 CSS 규칙일수록 이 점수가 더 높다고 보시면 됩니다.
자 정리하자면 선택자에는 명시도라는 게 있고 선택자가 복잡할수록 명시도 점수가 높아서 우선적으로 캐스케이드한다라고 이해하시면 됩니다.
