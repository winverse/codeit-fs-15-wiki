# Day 13. 커링과 부분 적용

### 오늘의 목표

커링과 부분 적용은 함수를 한 번에 모두 호출하지 않고, 일부 인자를 먼저 고정해 재사용하는
방식입니다. 오늘은 클로저를 이용한 커링, `bind`를 이용한 부분 적용, predicate 조합을
복습하고, TypeScript의 generic 함수 타입으로 실무 필터 코드를 안전하게 표현하는 방법을
확인합니다.

### 레퍼런스 연결

- 스코프와 함수형 패턴 관련 내용 복습
- 핵심 복습 포인트: 커링, 부분 적용, 고차 함수, 재사용 가능한 함수 조합
- 다시 볼 질문: 함수형 패턴은 어디까지 쓰면 읽기 쉬운 코드가 될까?

### 문제 1. 기본

다음 코드의 실행 결과를 순서대로 작성하세요.

```jsx
function add(a) {
  return function (b) {
    return a + b;
  };
}

const add10 = add(10);

console.log(add10(3));
console.log(add(2)(5));
```

- 정답과 해설
    
    ### 정답
    
    ```
    13
    7
    ```
    
    ### 해설
    
    `add`는 숫자 `a`를 받은 뒤, 다시 숫자 `b`를 받는 함수를 반환합니다. 반환된 내부 함수는
    클로저를 통해 바깥 함수의 `a`를 기억합니다.
    
    `const add10 = add(10)`은 `a`가 `10`으로 고정된 함수를 만듭니다. 이후 `add10(3)`을 호출하면
    기억된 `10`과 새로 받은 `3`을 더해 `13`이 됩니다.
    
    `add(2)(5)`는 같은 동작을 한 줄로 표현한 것입니다. 먼저 `add(2)`가 `a = 2`를 기억하는
    함수를 반환하고, 그 반환된 함수에 `5`를 넘겨 `7`을 만듭니다.
    
    이처럼 여러 인자를 받는 함수를 인자 하나씩 받는 함수들의 연속으로 바꾸는 방식을 커링이라고
    부릅니다. 핵심은 “앞에서 받은 값을 나중 호출까지 기억한다”는 클로저 동작입니다.
    

### 문제 2. 중급

다음 코드의 실행 결과를 순서대로 작성하고, `bind`를 이용한 부분 적용이 어떻게 동작하는지
설명하세요.

```jsx
function multiply(a, b, c) {
  return a * b * c;
}

const double = multiply.bind(null, 2);

console.log(double(3, 4));

const doubleAndTriple = multiply.bind(null, 2, 3);

console.log(doubleAndTriple(4));
console.log(multiply.length);
console.log(double.length);
```

- 정답과 해설
    
    ### 정답
    
    ```
    24
    24
    3
    2
    ```
    
    ### 해설
    
    `multiply`는 세 인자 `a`, `b`, `c`를 곱합니다.
    
    `multiply.bind(null, 2)`는 `this`를 `null`로 고정하고 첫 번째 인자 `a`를 `2`로 미리
    고정한 새 함수를 만듭니다. 이 예제의 `multiply`는 `this`를 사용하지 않으므로 첫 번째 인자인
    `null`은 결과에 영향을 주지 않습니다.
    
    `double(3, 4)`는 실제로 `multiply(2, 3, 4)`처럼 실행되어 `24`를 반환합니다.
    
    `multiply.bind(null, 2, 3)`은 `a = 2`, `b = 3`을 미리 고정합니다. 따라서
    `doubleAndTriple(4)`는 `multiply(2, 3, 4)`가 되어 역시 `24`입니다.
    
    `function.length`는 함수가 기대하는 매개변수 개수를 나타냅니다. `multiply.length`는
    선언된 매개변수가 세 개이므로 `3`입니다.
    
    `double.length`는 이미 하나의 인자가 bind로 고정되었기 때문에 남은 기대 인자 수가 `2`로
    계산됩니다. 이 값은 런타임 메타데이터일 뿐 타입 안정성을 보장하는 것은 아니지만, 부분 적용된
    함수의 형태를 이해하는 데 도움이 됩니다.
    

### 문제 3. 중급

다음 코드의 실행 결과를 순서대로 작성하고, predicate를 커링하면 어떤 장점이 있는지
설명하세요.

```jsx
const users = [
  { name: "Ada", role: "admin" },
  { name: "Lin", role: "user" },
];

const hasRole = (role) => (user) => user.role === role;

console.log(users.filter(hasRole("admin")).map((user) => user.name).join(","));
console.log(users.filter(hasRole("guest")).length);
```

- 정답과 해설
    
    ### 정답
    
    ```
    Ada
    0
    ```
    
    ### 해설
    
    `hasRole`은 역할 이름 `role`을 먼저 받고, 사용자를 검사하는 함수를 반환합니다.
    
    `hasRole("admin")`은 `user.role === "admin"`을 검사하는 predicate 함수가 됩니다. 이 함수를
    `filter`에 넘기면 `admin` 역할을 가진 사용자만 남습니다. 예제에서는 `Ada`만 해당하므로
    첫 번째 출력은 `"Ada"`입니다.
    
    `hasRole("guest")`는 `guest` 역할을 찾는 predicate를 만듭니다. 배열에는 해당 사용자가 없으므로
    필터 결과의 길이는 `0`입니다.
    
    이 패턴의 장점은 조건을 재사용하기 쉽다는 점입니다. `hasRole("admin")`, `hasRole("user")`,
    `hasRole("guest")`처럼 같은 형태의 조건 함수를 쉽게 만들 수 있습니다.
    
    다만 커링을 남용하면 호출 구조가 지나치게 간접적이 되어 읽기 어려울 수 있습니다. 반복되는
    조건 생성이 실제로 있고, 이름을 붙였을 때 의미가 분명해지는 경우에 쓰는 편이 좋습니다.
    

### 문제 4. 심화

다음 코드는 상품 목록에서 비싼 상품만 필터링합니다. 실행 결과를 예측하고, 이 코드의
장단점을 JS 관점과 TS 관점에서 설명한 뒤 더 안전한 TypeScript 코드로 개선하세요.

```jsx
const products = [
  { name: "Book", price: 12000 },
  { name: "Pen", price: 1500 },
  { name: "Bag", price: 45000 },
];

const greaterThan = (min) => (value) => value > min;
const getPrice = (product) => product.price;
const isExpensive = (product) => greaterThan(10000)(getPrice(product));

console.log(products.filter(isExpensive).map((product) => product.name).join(","));
```

- 정답과 해설
    
    ### 정답
    
    ```
    Book,Bag
    ```
    
    안전하게 개선한 TypeScript 예시는 다음과 같습니다.
    
    ```tsx
    type Product = {
      name: string;
      price: number;
    };
    
    type Predicate<T> = (value: T) => boolean;
    
    const greaterThan = (min: number): Predicate<number> => {
      return (value) => value > min;
    };
    
    const by = <T, U>(selector: (value: T) => U, predicate: Predicate<U>): Predicate<T> => {
      return (value) => predicate(selector(value));
    };
    
    const products: Product[] = [
      { name: "Book", price: 12000 },
      { name: "Pen", price: 1500 },
      { name: "Bag", price: 45000 },
    ];
    
    const isExpensive = by((product: Product) => product.price, greaterThan(10000));
    
    console.log(products.filter(isExpensive).map((product) => product.name).join(","));
    ```
    
    ### 해설
    
    ### JS 관점
    
    `greaterThan(10000)`은 `value > 10000`을 검사하는 함수를 반환합니다. 이것은 `min` 값을
    먼저 고정한 커링 형태의 predicate입니다.
    
    `getPrice(product)`는 상품에서 가격만 꺼냅니다. `isExpensive(product)`는 상품 가격을 꺼낸
    뒤 `10000`보다 큰지 검사합니다.
    
    상품 세 개 중 `Book`의 가격은 `12000`, `Pen`은 `1500`, `Bag`은 `45000`입니다. `10000`보다
    큰 상품은 `Book`과 `Bag`이므로 출력은 `"Book,Bag"`입니다.
    
    이 코드는 작은 predicate를 조합한다는 장점이 있지만, 매번 `greaterThan(10000)(getPrice(product))`
    처럼 읽어야 하므로 코드가 복잡해질 수 있습니다. 조합이 반복되고 이름을 붙일 가치가 있을 때
    사용하는 편이 좋습니다.
    
    ### TS 관점
    
    TypeScript에서는 이런 함수 조합을 타입으로 표현할 수 있습니다. `Predicate<T>`는 어떤 값
    `T`를 받아 Boolean을 반환하는 함수입니다.
    
    `greaterThan`은 `number`를 검사하는 predicate를 반환하므로 반환 타입은
    `Predicate<number>`입니다.
    
    `by` 함수는 selector와 predicate를 연결합니다. selector가 `T`를 받아 `U`를 반환하고,
    predicate가 `U`를 검사한다면, 최종적으로는 `T`를 검사하는 predicate를 만들 수 있습니다.
    예제에서는 `Product -> number` selector와 `number` predicate를 연결해
    `Predicate<Product>`를 만듭니다.
    
    이 generic 타입 덕분에 가격이 숫자가 아닌 속성을 잘못 연결하거나, predicate가 기대하는
    타입과 selector의 반환 타입이 맞지 않는 경우를 컴파일 단계에서 잡을 수 있습니다.
    
    ### 실무 관점
    
    커링과 부분 적용은 “조건 생성”이 반복될 때 유용합니다. 예를 들어 가격 기준, 역할 기준,
    상태 기준, 날짜 기준 같은 필터를 여러 화면에서 재사용할 수 있습니다.
    
    하지만 모든 함수를 커링 형태로 바꾸면 오히려 코드가 어려워집니다. 실무에서는 다음 기준으로
    판단하는 편이 좋습니다.
    
    1. 고정할 인자가 실제로 반복되는가
    2. 만들어진 함수에 좋은 이름을 붙일 수 있는가
    3. 타입 추론이 오히려 복잡해지지 않는가
    4. 일반 함수 호출보다 의도가 더 잘 드러나는가
    
    함수형 패턴은 문법을 과시하기 위한 도구가 아니라, 반복되는 조건과 변환을 안전하게 재사용하기
    위한 도구로 쓰는 것이 좋습니다.
