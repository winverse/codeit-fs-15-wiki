# Day 15. 콜백과 고차 함수

### 오늘의 목표

배열 메서드는 JavaScript에서 가장 자주 만나는 고차 함수 사용 사례입니다. 오늘은
`map`, `filter`의 콜백 실행 방식, 부수효과가 있는 콜백, 콜백이 받는 인자 개수 때문에
생기는 실수를 복습하고, TypeScript에서 predicate와 타입 가드를 어떻게 표현할지
정리합니다.

### 레퍼런스 연결

- 함수 챕터 복습
- 핵심 복습 포인트: 콜백, 고차 함수, 배열 메서드, 순수 함수와 부수효과
- 다시 볼 질문: 콜백이 많아질 때 코드 이해 비용을 어떻게 줄일 수 있을까?

### 문제 1. 기본

다음 코드의 실행 결과를 순서대로 작성하세요.

```jsx
const numbers = [1, 2, 3];
const result = numbers.map((number) => number * 2);

console.log(result.join(","));
console.log(numbers.join(","));
```

- 정답과 해설
    
    ### 정답
    
    ```
    2,4,6
    1,2,3
    ```
    
    ### 해설
    
    `map`은 배열의 각 요소를 콜백 반환값으로 변환해 새 배열을 만듭니다. `number * 2`를
    반환하므로 `[1, 2, 3]`은 `[2, 4, 6]`으로 변환됩니다.
    
    `result.join(",")`는 새 배열을 쉼표로 연결해 `"2,4,6"`을 출력합니다.
    
    중요한 점은 `map`이 원본 배열을 직접 바꾸지 않는다는 것입니다. `numbers`는 그대로
    `[1, 2, 3]`입니다. 그래서 두 번째 출력은 `"1,2,3"`입니다.
    
    다만 `map`이 항상 안전하다는 뜻은 아닙니다. 콜백 안에서 객체 내부를 변경하면 원본 객체가
    바뀔 수 있습니다. `map` 자체가 새 배열을 만든다는 사실과 배열 안의 참조 값까지 복사된다는
    사실은 구분해야 합니다.
    

### 문제 2. 중급

다음 코드의 실행 결과를 순서대로 작성하고, 콜백 안의 부수효과가 왜 주의해야 할 대상인지
설명하세요.

```jsx
const logs = [];

const filtered = [1, 2, 3, 4].filter((value) => {
  logs.push(value);
  return value % 2 === 0;
});

console.log(filtered.join(","));
console.log(logs.join(","));
```

- 정답과 해설
    
    ### 정답
    
    ```
    2,4
    1,2,3,4
    ```
    
    ### 해설
    
    `filter`는 각 요소에 대해 콜백을 호출하고, 콜백 반환값이 Truthy인 요소만 새 배열에
    남깁니다.
    
    예제의 콜백은 먼저 `logs.push(value)`를 실행합니다. 이 코드는 외부 배열 `logs`를 변경하는
    부수효과입니다. 그 다음 `value % 2 === 0`을 반환합니다.
    
    `[1, 2, 3, 4]` 중 짝수는 `2`와 `4`이므로 `filtered.join(",")`는 `"2,4"`입니다.
    
    하지만 콜백은 모든 요소에 대해 실행됩니다. `1`과 `3`은 결과 배열에는 남지 않지만, 콜백은
    호출되었기 때문에 `logs`에는 `1`, `2`, `3`, `4`가 모두 들어갑니다.
    
    부수효과가 있는 콜백은 디버깅이나 로깅에는 유용할 수 있지만, 데이터 변환 로직에서는 주의해야
    합니다. `filter`, `map`, `reduce`의 콜백이 외부 상태를 바꾸면 코드의 결과를 예측하기
    어려워지고 테스트도 복잡해집니다.
    

### 문제 3. 중급

다음 코드의 실행 결과를 순서대로 작성하고, `map(parseInt)`가 기대와 다르게 동작하는 이유를
설명하세요.

```jsx
const words = ["1", "2", "3"];

console.log(words.map(Number).join(","));
console.log(words.map(parseInt).join(","));
```

- 정답과 해설
    
    ### 정답
    
    ```
    1,2,3
    1,NaN,NaN
    ```
    
    ### 해설
    
    `words.map(Number)`는 각 문자열을 `Number` 함수에 넘깁니다. `Number("1")`, `Number("2")`,
    `Number("3")`은 각각 `1`, `2`, `3`입니다.
    
    `words.map(parseInt)`는 자주 나오는 함정입니다. `map`의 콜백은 요소 값만 넘기는 것이
    아니라 `(value, index, array)` 세 인자를 넘깁니다.
    
    `parseInt`의 두 번째 매개변수는 진법(radix)입니다. 따라서 실제 호출은 다음과 비슷합니다.
    
    ```jsx
    parseInt("1", 0);
    parseInt("2", 1);
    parseInt("3", 2);
    ```
    
    `parseInt("1", 0)`은 보통 10진수처럼 해석되어 `1`이 됩니다. `parseInt("2", 1)`은 유효하지
    않은 진법이므로 `NaN`입니다. `parseInt("3", 2)`는 2진수에서 숫자 `3`이 유효하지 않으므로
    `NaN`입니다.
    
    해결 방법은 인자를 명확히 감싸는 것입니다.
    
    ```jsx
    words.map((word) => parseInt(word, 10));
    ```
    
    콜백으로 함수를 그대로 넘길 때는 그 함수가 몇 개의 인자를 받고, 배열 메서드가 몇 개의 인자를
    전달하는지 확인해야 합니다.
    

### 문제 4. 심화

다음 코드는 활성 사용자 이름만 정리하려고 작성되었습니다. 실행 결과를 예측하고, 이 코드의
장단점을 JS 관점과 TS 관점에서 설명한 뒤 더 안전한 TypeScript 코드로 개선하세요.

```jsx
function getActiveNames(users) {
  return users
    .filter((user) => user.active)
    .map((user) => user.name.trim())
    .filter(Boolean);
}

const users = [
  { name: " Ada ", active: true },
  { name: " ", active: true },
  { name: "Lin", active: false },
  { name: "Ken", active: true },
];

console.log(getActiveNames(users).join(","));
```

- 정답과 해설
    
    ### 정답
    
    ```
    Ada,Ken
    ```
    
    안전하게 개선한 TypeScript 예시는 다음과 같습니다.
    
    ```tsx
    type User = {
      name: string;
      active: boolean;
    };
    
    function isNonEmptyString(value: string): value is string {
      return value.length > 0;
    }
    
    function getActiveNames(users: readonly User[]): string[] {
      return users
        .filter((user) => user.active)
        .map((user) => user.name.trim())
        .filter(isNonEmptyString);
    }
    
    const users: User[] = [
      { name: " Ada ", active: true },
      { name: " ", active: true },
      { name: "Lin", active: false },
      { name: "Ken", active: true },
    ];
    
    console.log(getActiveNames(users).join(","));
    ```
    
    ### 해설
    
    ### JS 관점
    
    첫 번째 `filter`는 `active`가 Truthy인 사용자만 남깁니다. `Ada`, 공백 이름 사용자,
    `Ken`이 남고, `Lin`은 제외됩니다.
    
    `map((user) => user.name.trim())`은 이름 앞뒤 공백을 제거합니다. `" Ada "`는 `"Ada"`가 되고,
    `" "`는 빈 문자열 `""`이 되며, `"Ken"`은 그대로 `"Ken"`입니다.
    
    마지막 `.filter(Boolean)`은 각 문자열을 Boolean 문맥으로 평가합니다. 빈 문자열 `""`은
    Falsy이므로 제거되고, `"Ada"`와 `"Ken"`만 남습니다. 그래서 출력은 `"Ada,Ken"`입니다.
    
    이 코드는 간결하지만, `.filter(Boolean)`이 무엇을 제거하는지 코드만 보고 명확하지 않을 수
    있습니다. 문자열 배열에서는 빈 문자열 제거로 볼 수 있지만, 숫자 배열에서는 `0`까지 제거될 수
    있습니다.
    
    ### TS 관점
    
    TypeScript에서 `users: readonly User[]`로 받으면 함수가 입력 배열 자체를 바꾸지 않는다는
    의도를 드러낼 수 있습니다.
    
    `isNonEmptyString(value: string): value is string`은 타입 가드 형태입니다. 이 예제에서는
    입력과 출력 타입이 모두 `string`이라 타입 좁히기 효과가 크지는 않지만, `.filter(Boolean)`보다
    의도가 더 분명합니다. “빈 문자열이 아닌 문자열만 남긴다”는 정책을 이름으로 표현하기 때문입니다.
    
    실제 프로젝트에서는 `Array<string | null>`에서 `null`을 제거하거나, `Array<User | undefined>`에서
    사용자만 남기는 식으로 타입 가드의 효과가 더 커집니다.
    
    중요한 점은 `Boolean`을 무조건 predicate로 쓰지 않는 것입니다. `Boolean`은 Falsy 값을 모두
    제거합니다. 도메인에서 `0`, `false`, `""` 중 무엇을 제거해야 하는지에 따라 명시적인 predicate를
    작성해야 합니다.
    
    ### 실무 관점
    
    배열 메서드 체인은 읽기 쉬울 때 강력합니다. 하지만 각 단계의 역할이 불분명하면 디버깅이
    어려워집니다.
    
    좋은 흐름은 다음과 같습니다.
    
    1. `filter`, `map`, `reduce` 콜백은 가능하면 순수하게 유지한다.
    2. `parseInt`처럼 여러 인자를 받는 함수를 콜백으로 바로 넘길 때는 주의한다.
    3. `.filter(Boolean)`은 제거 기준이 도메인에 맞는지 확인한다.
    4. 재사용되는 조건은 이름 있는 predicate 또는 타입 가드로 분리한다.
    
    고차 함수는 짧게 쓰는 기술이 아니라, 데이터 흐름을 작은 단계로 나누어 읽기 쉽게 만드는 도구로
    사용해야 합니다.
