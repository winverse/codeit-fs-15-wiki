### 01. 자바스크립트 소개

### 웹의 발전과 자바스크립트

오늘날 우리는 구글, 페이스북, 넷플릭스 등 수많은 웹 서비스를 통해 정보를 얻고, 소통하며, 여가를 즐깁니다. 이 모든 서비스의 기반에는 **웹 기술**이 있습니다.

최초의 웹사이트는 단순한 정보를 공유하는 목적의 텍스트 위주였지만, 기술이 발전하면서 이미지, 영상을 넘어 게임, 채팅, 은행 업무까지 가능한 **인터랙티브(interactive)한 공간**으로 진화했습니다.

이러한 웹의 발전에 가장 크게 기여한 프로그래밍 언어가 바로 **자바스크립트(JavaScript)**입니다. 자바스크립트는 웹 페이지를 동적으로 만들고, 사용자와 상호작용하는 핵심적인 역할을 담당합니다.

### 자바스크립트의 확장

자바스크립트는 이제 웹 브라우저를 넘어 훨씬 더 넓은 분야에서 사용되고 있습니다.

- **모바일 앱** 개발
- PC용 데스크톱 프로그램 개발
- VR/AR과 같은 3D 콘텐츠
- 블록체인 기술

활용 범위가 넓어지면서, 자바스크립트 개발자에 대한 수요는 전 세계적으로 매우 높아졌습니다.

### 어떻게 공부해야 할까?

프로그래밍 언어를 처음 배울 때 많은 사람들이 하는 실수는 **문법**에만 집중하는 것입니다. 하지만 이런 방식은 응용력을 키우기 어렵고, 새로운 기술이 등장했을 때 적응하기 힘듭니다.

하지만 설명을 듣고, 기초 문법을 공부해도 모든 것이 명확하게 이해되지 않을 수 있습니다. 왜 그럴까요? 직접 코드를 써보지 않았기 때문입니다. 프로그래밍 지식은 눈으로만 익히는 것이 아니라, 직접 코드를 작성하고 실행하는 '실습'을 통해 비로소 자신의 것이 됩니다.

중요한 것은 언어의 문법이 아닌 **프로그래밍의 기본 원리**를 이해하는 것입니다.

- **왜** 이 개념이 필요한가?
- **어떤 문제**를 해결하는가?
- **어떻게** 동작하는가?

이러한 원리를 중심으로 학습하면, 특정 언어에 종속되지 않는 탄탄한 기본기를 갖출 수 있습니다.

### 이 강의의 목표

이 강의는 두 가지 목표를 가지고 있습니다.

1. **프로그래밍의 기본 개념과 원리 이해**: 단순히 따라 하는 것을 넘어, 컴퓨터처럼 생각하는 방법을 배웁니다.
2. **자바스크립트 활용 능력 습득**: 배운 개념을 바탕으로 실제 코드를 작성하고 문제를 해결하는 능력을 기릅니다.

이 과정을 통해 여러분은 앞으로 웹 개발뿐만 아니라 더 넓은 IT 분야로 나아가기 위한 튼튼한 발판을 마련하게 될 것입니다.

## 2. 프로그래밍 기본 개념

### 01. 세미콜론

### 문장의 마침표, 세미콜론

일반적으로 글을 쓸 때 마침표로 문장의 끝을 나타내는 것처럼, 많은 프로그래밍 언어에서는 **세미콜론(;)**을 사용하여 코드의 각 문장이 끝났음을 명시적으로 표시합니다.

자바스크립트에서는 특정 상황에 세미콜론을 생략할 수 있습니다.

---

### 세미콜론을 생략할 수 있는 경우

한 줄에 하나의 문장만 작성할 경우, 자바스크립트 엔진이 코드를 실행할 때 자동으로 세미콜론을 추가해주기 때문에 생략해도 문제가 없습니다.

```jsx
console.log('첫 번째 문장')
console.log('두 번째 문장')
```

---

## 세미콜론이 반드시 필요한 경우

하지만 한 줄에 여러 문장을 이어서 작성할 경우에는 각 문장을 구분하기 위해 세미콜론이 반드시 필요합니다. 세미콜론이 없으면 문법 오류(SyntaxError)가 발생합니다.

**오류가 발생하는 코드:**

```jsx
console.log('첫 번째 문장') console.log('두 번째 문장') *// SyntaxError 발생*
```

**올바른 코드:**

```jsx
console.log('첫 번째 문장'); console.log('두 번째 문장');
```

## 세미콜론, 써야 할까? 말아야 할까?

개발자들 사이에서는 세미콜론 사용에 대해 두 가지 의견이 있습니다.

1. **생략하자**: 꼭 필요한 경우에만 쓰고, 나머지는 생략하여 코드를 간결하게 유지하자는 의견.
2. **항상 쓰자**: 문장의 끝을 명확하게 표시하여 예상치 못한 오류를 방지하자는 의견.

두 의견 모두 타당하지만, 프로그래밍을 처음 배우는 입장에서는 문장의 단위를 명확하게 인지하고 실수를 줄이기 위해 **의식적으로 세미콜론을 작성하는 습관**을 들이는 것이 좋습니다.

따라서 앞으로 진행될 강의에서는 코드 작성 시 문장의 끝에 세미콜론을 붙이는 것을 원칙으로 합니다.

### 01. #Quiz 세미콜론 (5분)

### 문제 1

아래 코드의 실행 결과를 예상해보고, 세미콜론(`;`)의 역할에 대해 설명해보세요.

```bash
console.log('Hello'); console.log('World');
```

- 정답 및 해설
    
    실행 결과:
    
    ```bash
    Hello
    World
    ```
    
    해설:
    
    세미콜론(`;`)은 문장(statement)의 끝을 명확하게 표시하는 역할을 합니다. 위 코드처럼 한 줄에 여러 개의 문장을 작성해야 할 경우, 각 문장을 구분하기 위해 세미콜론이 **반드시** 필요합니다.
    
    자바스크립트는 세미콜론이 없어도 자동으로 문장의 끝을 추측하여 세미콜론을 삽입해주는 기능(ASI)이 있지만, 위와 같은 경우에는 개발자가 직접 세미콜론을 붙여 문장의 구분을 명확히 해주어야 합니다.

## 문제 2

아래 코드는 오류를 발생시킵니다. 세미콜론(`;`)을 추가하여 코드를 올바르게 수정하고, 오류가 발생하는 이유를 설명해보세요. (let과 forEach는 나중에 배웁니다.)

**수정 전 코드:**

```bash
let message = 'Hello'
['W', 'o', 'r', 'l', 'd'].forEach(char => console.log(char))
```

- 정답 및 해설
    
    수정 후 코드:
    
    ```bash
    let message = 'Hello';
    ['W', 'o', 'r', 'l', 'd'].forEach(char => console.log(char));
    ```
    
    해설:
    
    오류의 원인은 자바스크립트 엔진이 첫 번째 줄의 `'Hello'`와 두 번째 줄의 `['W', ...]`를 하나의 문장으로 잘못 해석하기 때문입니다.
    
    세미콜론이 없으면, 엔진은 `message` 변수에 할당된 `'Hello'` 문자열의 특정 속성(property)에 접근하려는 것으로 오해하여 `Uncaught TypeError: Cannot read properties of undefined (reading 'forEach')`와 같은 오류를 발생시킵니다.
    
    첫 번째 문장의 끝에 세미콜론을 추가함으로써, `'Hello'`에서 문장이 끝났음을 명확히 알려주어 다음 줄의 코드가 독립적인 새 문장으로 해석되게 할 수 있습니다.
    
    이처럼 세미콜론은 코드의 의도치 않은 해석을 방지하는 중요한 역할을 합니다.

### 02. 코멘트

### 컴퓨터가 읽지 않는 코드, 코멘트

프로그래밍을 하다 보면 특정 코드를 컴퓨터가 읽지 않고 그냥 넘어가게 만들어야 할 때가 있습니다. 코드에 특별한 규칙에 따라 표시를 해두면, 컴퓨터(정확히는 자바스크립트 엔진)는 그 부분을 실행하지 않고 건너뜁니다.

이러한 표시를 **코멘트(Comment)** 또는 **주석**이라고 부릅니다.

코멘트는 주로 다음과 같은 목적으로 사용됩니다.

- **메모 남기기**: 코드에 대한 설명을 작성하여 나중에 다시 보거나 다른 개발자가 이해하기 쉽게 만듭니다.
- **코드 비활성화**: 기존 코드를 삭제하지 않고 임시로 실행되지 않게 만들고 싶을 때 사용합니다.

---

### 코멘트 사용법

자바스크립트에서는 두 가지 종류의 코멘트를 사용할 수 있습니다.

### 1. 한 줄 코멘트 (Single-line Comment)

한 줄 전체를 코멘트로 처리하거나, 코드의 오른쪽 부분만 코멘트로 만들고 싶을 때 사용합니다. **슬래시 두 개(`//`)**로 시작하며, 그 뒤부터 해당 줄의 끝까지 모든 내용이 코멘트로 처리됩니다.

```jsx
*// 이 줄은 전체가 코멘트입니다.*

console.log('Hello, World!'); *// 코드 오른쪽의 이 부분만 코멘트입니다.

// console.log('이 코드는 실행되지 않습니다.');*
```

---

## 2. 여러 줄 코멘트 (Multi-line Comment)

여러 줄에 걸쳐 긴 설명을 작성하고 싶을 때 사용합니다. **`/*`** 로 시작해서 **`*/`** 로 끝나며, 그 사이에 있는 모든 내용이 코멘트로 처리됩니다.

```jsx
*/*
  이것은
  여러 줄
  코멘트입니다.
*/*
console.log('코멘트 밖의 코드는 정상적으로 실행됩니다.');

*/*
  console.log('이 안의 코드는');
  console.log('모두 실행되지 않습니다.');
*/*
```

---

## VS Code 코멘트 단축키

Visual Studio Code에서는 단축키를 사용하여 코멘트를 쉽게 추가하거나 제거할 수 있습니다.

- **한 줄 코멘트**: `Cmd` + `/` (macOS) 또는 `Ctrl` + `/` (Windows/Linux)
    
    원하는 코드 라인에 커서를 두거나 여러 라인을 선택한 후 단축키를 누르면, 각 라인이 한 줄 코멘트로 변환됩니다. 한 번 더 누르면 코멘트가 해제됩니다.
    
- **여러 줄 코멘트**: `Option` + `Shift` + `A` (macOS) 또는 `Alt` + `Shift` + `A` (Windows/Linux)
    
    코드를 선택한 후 단축키를 누르면, 선택된 전체 블록이 여러 줄 코멘트로 감싸집니다.

## 코멘트 작성 Tip!

코멘트는 다른 개발자나 미래의 나를 위한 중요한 소통 수단입니다. 하지만 너무 길거나 불필요한 코멘트는 오히려 코드를 읽기 어렵게 만들 수 있습니다.

**필요한 내용만, 간결하고 명확하게** 작성하는 습관을 들이는 것이 중요합니다.

### 02. #Quiz 코멘트 (5분)

### 문제 1

아래 코드에서 잘못 사용된 코멘트(주석)를 찾아 올바르게 수정하고, 두 가지 코멘트 방식(`//` 와 `/* */`)의 차이점을 설명해보세요.

```jsx
*// 변수 a에 숫자 5를 할당합니다.*
let a = 5;

*/*
  변수 b에 숫자 10을 할당하고,
  a와 b를 더한 값을 출력하려고 합니다.
  console.log(a + b);
*/*
let b = 10;
console.log(a + b);
```

- 정답 및 해설
    
    수정 후 코드:
    
    ```jsx
    *// 변수 a에 숫자 5를 할당합니다.*
    let a = 5;
    
    */*
      변수 b에 숫자 10을 할당하고,
      a와 b를 더한 값을 출력하려고 합니다.
    */*
    let b = 10;
    console.log(a + b); *// 이 코드는 실행되어야 하므로 주석 처리되면 안 됩니다.*
    ```
    
    해설
    
    - 한 줄 주석 (`//`): // 이후 줄의 끝까지 모두 주석 처리합니다. 주로 코드 한 줄 혹은 특정 부분을 잠시 설명하거나 비활성화할 때 사용합니다.
    - 여러 줄 주석 (`/* */`): /*와 */ 사이에 있는 모든 내용을 주석 처리합니다. 여러 줄에 걸쳐 설명하거나 코드 블록을 비활성화할 때 유용합니다.
    
    위 예제처럼 실행되어야 할 코드를 주석 안에 넣으면, 실제로 실행되지 않으니 반드시 주석 밖으로 빼주어야 합니다.

### 문제 2

아래 코드에서 result의 최종 값은 무엇일까요? 코멘트가 코드 실행에 미치는 영향을 생각하며 답을 골라보세요.

```jsx
let result = 10;

*// result = result + 10;*
result = result * 2;
*/*
result = result - 5;
result = result / 2;
*/*

console.log(result);
```

1. 10
2. 20
3. 15
4. 7.5

---

- 정답 및 해설
    
    정답: 2번 (20)
    
    해설
    
    주석 처리된 코드는 자바스크립트 엔진이 무시합니다.
    
    1. result에는 10이 들어갑니다.
    2. result = result * 2;만 실행되니, result 값은 20이 됩니다.
    3. 그 외 연산(result = result + 10 등)은 모두 주석 처리되어 코드에 영향을 주지 않습니다.
        
        그래서 최종적으로 20이 출력됩니다.
        

### 03. 자료형 개요 ⭐⭐

### 프로그래밍은 계산이다

'컴퓨터는 복잡한 계산기'라는 말을 들어본 적 있나요? 실제로 프로그래밍은 컴퓨터를 통해 다양한 종류의 값을 다루고, 계산하고, 처리하는 과정이라고 할 수 있습니다.

일반 계산기는 숫자만 다루지만, 프로그래밍에서는 숫자 외에도 훨씬 다양한 종류의 값을 사용합니다. 값의 종류를 **자료형(Data Type)**이라고 부릅니다.

프로그래밍의 가장 기초가 되는 기본 자료형 몇 가지를 알아봅시다.

---

### 1. 숫자 (Number)

이름 그대로 숫자를 나타내는 자료형입니다. 우리가 일상에서 쓰는 모든 종류의 숫자를 포함합니다.

- **정수(Integer)**: `0`, `100`, `55` 와 같이 소수점이 없는 숫자
- **소수(Floating Point)**: `3.14`, `0.5`, `1.234` 와 같이 소수점이 있는 숫자

숫자 자료형은 덧셈(`+`), 뺄셈(`-`), 곱셈(`*`), 나눗셈(`/`) 같은 모든 종류의 산술 연산이 가능합니다.

```jsx
console.log(10 + 5);   *// 15*
console.log(3.14 * 2); *// 6.28*
```

---

### 2. 문자열 (String)

'Hello, world!', 'JavaScript' 와 같이 텍스트를 나타내는 자료형입니다. 따옴표로 텍스트를 감싸서 만듭니다.

- **작은따옴표(`'`)** 또는 **큰따옴표(`"`)** 를 사용할 수 있습니다.
- **주의!** 시작하는 따옴표와 끝나는 따옴표는 반드시 같아야 합니다. (`'Hello"` 와 같이 사용하면 오류 발생)

```jsx
console.log('안녕하세요!');
console.log("Let's learn JavaScript!");
```

문자열은 더하기(`+`) 연산을 통해 서로 이어붙일 수 있습니다.

```jsx
console.log('Hello, ' + 'world!'); *// "Hello, world!"*
```

> **잠깐! 숫자와 문자열의 덧셈**
> 
> 
> `10 + 5`는 숫자 15를 만들지만, `'10' + '5'`는 문자열 "105"를 만듭니다. `typeof` 연산자로 값의 자료형을 확인해보면 차이를 명확히 알 수 있습니다. 이 내용은 실습 코드 파일에서 직접 확인해보세요!
> 

---

### 3. 불린 (Boolean)

'참'과 '거짓', 단 두 가지 상태만을 나타내는 자료형입니다.

- `true`: 참, 맞음, 켜짐 등을 의미
- `false`: 거짓, 틀림, 꺼짐 등을 의미

불린은 값을 직접 쓰기보다는, 주로 두 값을 비교한 결과로 사용됩니다.

```jsx
console.log(8 > 6);  *// 8은 6보다 크므로, 결과는 true*
console.log(10 === 5); *// 10과 5는 같지 않으므로, 결과는 false*
```

이러한 비교 결과는 나중에 특정 조건에 따라 코드를 다르게 동작하게 만들 때 아주 중요하게 사용됩니다.

---

지금까지 배운 숫자, 문자열, 불린은 자바스크립트뿐만 아니라 대부분의 프로그래밍 언어가 공통으로 가지고 있는 가장 기본적인 자료형입니다.

### 04. 추상화 개요 ⭐⭐

### 원활한 소통의 열쇠, 추상화

우리는 혼자가 아닌, 다른 사람들과 '소통'하며 살아갑니다. 프로그래밍도 마찬가지입니다. 코드를 통해 미래의 나, 동료, 컴퓨터, 그리고 프로그램을 사용하는 사용자와 소통하는 과정이라고 할 수 있습니다.

이때 원활한 소통을 위해 꼭 필요한 개념이 바로 **추상화(Abstraction)**입니다.

---

**"복잡한 것들은 숨기고, 목적에 맞게 꼭 필요한 핵심만 드러내어 단순하게 만드는 것"**

우리는 일상에서 이미 수많은 추상화의 결과물을 접하고 있습니다.

- **지도**: 현실 세계의 모든 건물, 나무, 사람을 보여주지 않습니다. '길을 찾는' 목적에 맞게 도로, 지하철역, 주요 건물 등 꼭 필요한 정보만 뽑아 단순하게 시각화합니다.
- **자판기:** 사용자가 ****복잡한 원리를 이해할 필요없이, 이용자는 동전을 넣고, 음료 선택, 받기만 이해하면 자판기를 이용할 수 있습니다.

## 프로그래밍과 추상화

프로그래밍은 '추상화의 연속'이라고 해도 과언이 아닙니다. 사실 우리는 이미 추상화의 산물을 사용해왔습니다.

바로 `console.log()` 입니다.

```jsx
console.log('Hello, Abstraction!');
```

우리는 `console.log`가 어떤 복잡한 과정을 거쳐 컴퓨터 화면에 글자를 보여주는지 전혀 알지 못합니다. 하지만 '콘솔에 괄호 안의 값을 보여준다'는 핵심 기능만 알고 있으면, 그 내부 동작을 몰라도 얼마든지 원하는 값을 출력하며 잘 사용할 수 있습니다.

이처럼, 프로그래밍에서의 추상화는 **복잡한 내부 구현은 감추고, 사용하기 쉬운 기능(인터페이스)만 제공**하여 우리가 더 중요한 문제에 집중할 수 있도록 돕습니다.

---

앞으로 변수, 함수, 객체 등 프로그래밍의 중요한 개념들을 배우게 될 것입니다. 이 모든 것들이 사실은 '추상화'라는 개념을 바탕으로 만들어진 도구들입니다.

지금 당장 추상화라는 단어가 어렵게 느껴져도 괜찮습니다. 앞으로의 학습을 통해 자연스럽게 **'복잡한 것을 단순하게 만들어 문제를 해결하는'** 추상적 사고방식을 익히게 될 것입니다.

**목표를 명확히 하고, 불필요한 것은 숨기고, 간결하게 핵심만 드러내는 것.** 이 과정만 기억해주세요!

### 05. 변수 ⭐⭐⭐

### 값을 저장하는 상자, 변수

프로그램을 만들다 보면 같은 값을 여러 번 사용해야 할 때가 많습니다. 예를 들어, 편의점에서 음료수 판매 금액을 계산하는 프로그램을 만든다고 생각해봅시다.

```jsx
console.log(1500 * 3);
```

첫 번째 주문을 계산하는 코드입니다. 잘 작동하는 것을 확인했으니, 나머지 주문들도 계산해봅시다.

```jsx
console.log(1500 * 3);
console.log(2000 * 2 + 2500);
console.log(1400 + 2000);
console.log(1500 * 2 + 2000 + 2500 * 2);
console.log(2500 * 4);
```

위 코드는 정상 작동하지만 몇 가지 문제가 있습니다. 자세히 보면 세 번째 줄에서 콜라 가격을 `1500`이 아닌 `1400`으로 잘못 입력했습니다!

**문제 1: 반복적인 입력과 오타의 위험**

콜라 가격 `1500`원을 여러 번 입력하다 보면 실수로 `1500` 대신 `1400`이나 `1600`을 입력할 수 있습니다. 이런 실수는 찾아내기 어렵고, 코드가 길어질수록 더욱 위험해집니다.

**문제 2: 가격 변경 시 불편함**

만약 콜라 가격이 1500원에서 1700원으로 인상된다면? 코드 전체에서 `1500`을 찾아 일일이 수정해야 합니다. 50줄, 500줄의 코드라면 문제가 심각해집니다.

**문제 3: 코드의 의미 파악 어려움**

다른 사람이 이 코드를 보면 `1500`, `2000`, `2500`이 각각 무엇을 의미하는지 바로 알기 어렵습니다. 각 숫자마다 코멘트를 작성해야 할 것입니다.

이런 문제를 해결하는 것이 바로 **변수(Variable)**입니다.

---

### 변수란?

**변수는 값을 저장하기 위해 이름을 붙인 상자**라고 생각하면 됩니다. 상자에 이름표를 붙이고 그 안에 값을 넣어두면, 필요할 때마다 이름만 불러서 그 값을 사용할 수 있습니다.

---

### 변수 선언하기

자바스크립트에서 변수를 사용하려면 먼저 **변수를 선언**해야 합니다. 변수 선언은 `let` 키워드를 사용합니다.

```jsx
let colaPrice;
```

- `let`: 변수를 선언하는 키워드
- `colaPrice`: 변수의 이름 (원하는 이름으로 지을 수 있습니다)

이제 `colaPrice`라는 이름의 빈 상자가 만들어졌습니다!

---

### 변수에 값 할당하기

변수에 값을 넣는 것을 **할당(assignment)**이라고 합니다. **등호(`=`)**를 사용합니다.

```jsx
let colaPrice;
colaPrice = 1500;

console.log(colaPrice); *// 1500*
```

여기서 `=`는 수학에서처럼 "같다"는 의미가 아닙니다. **할당 연산자**로, **오른쪽 값을 왼쪽 변수에 저장한다**는 의미입니다.

---

### 선언과 동시에 할당하기

변수를 선언하면서 동시에 값을 할당할 수도 있습니다.

```jsx
let colaPrice = 1500;
let juicePrice = 2000;
let coffeePrice = 2500;
```

이 방식이 더 간결하고 많이 사용됩니다.

---

### 변수를 사용한 코드 개선

이제 처음의 코드를 변수를 사용해 개선해봅시다.

**변수를 사용하기 전:**

```jsx
console.log(1500 * 3);
console.log(2000 * 2 + 2500);
console.log(1500 + 2000);
console.log(1500 * 2 + 2000 + 2500 * 2);
console.log(2500 * 4);
```

**변수를 사용한 후:**

```jsx
let colaPrice = 1500;
let juicePrice = 2000;
let coffeePrice = 2500;

console.log(colaPrice * 3);
console.log(juicePrice * 2 + coffeePrice);
console.log(colaPrice + juicePrice);
console.log(colaPrice * 2 + juicePrice + coffeePrice * 2);
console.log(coffeePrice * 4);
```

---

### 변수 사용의 장점

### 1. 오타 방지

변수 이름을 잘못 입력하면 오류가 발생하여 즉시 알 수 있습니다.

```jsx
console.log(colaPrice); *// 정상 출력*
console.log(colaPirce); *// ReferenceError: colaPirce is not defined*
```

### 2. 유지보수 편리

가격이 변경되어도 변수의 값만 한 번 수정하면 모든 코드에 자동으로 반영됩니다.

```jsx
let colaPrice = 1700; *// 여기만 수정하면 됨!*
```

### 3. 코드 가독성 향상

숫자만 봐서는 알기 어려운 의미를 변수 이름을 통해 명확히 전달할 수 있습니다.

```jsx
*// 1500 * 3 보다*
colaPrice * 3; *// 훨씬 이해하기 쉽습니다!*
```

---

### 변수 이름 짓기

좋은 변수 이름은 코드의 가독성을 크게 높입니다. 다음 규칙을 따르는 것이 좋습니다.

- **의미가 명확해야 함**: `x`, `a` 보다는 `score`, `userName`처럼 의미를 담은 이름
- **카멜 케이스(camelCase) 사용**: 여러 단어를 연결할 때 첫 단어는 소문자, 이후 단어의 첫 글자는 대문자 (`mathScore`, `totalAmount`)
- **영문으로 작성**: 자바스크립트 관례상 영어로 작성합니다.

---

## 정리

- **변수**는 값을 저장하기 위해 이름을 붙인 상자
- `let` 키워드로 변수를 선언
- `=` 할당 연산자로 값을 저장
- 변수를 사용하면 코드의 안정성, 유지보수성, 가독성이 모두 향상됨

### 08. 함수 ⭐⭐⭐

### 명령들을 저장하는 상자, 함수

지난 시간에 배운 **변수**가 '값'을 저장하는 상자였다면, **함수**는 '명령들의 묶음'을 저장하는 상자입니다.

예를 들어, 우리가 아침에 일어나서 하루를 시작하는 과정을 생각해봅시다. '아침 루틴'은 다음과 같은 여러 동작으로 이루어져 있습니다.

1. 잠에서 깨어난다.
2. 세수를 한다.
3. 옷을 입는다.
4. 아침 식사를 한다.

이런 여러 명령을 하나로 묶어두고 필요할 때마다 쉽게 사용할 수 있게 해주는 것이 바로 **함수**입니다.

---

### 함수 선언하기

자바스크립트에서 함수를 사용하려면 먼저 **함수를 선언**해야 합니다. 함수 선언은 `function` 키워드를 사용합니다.

```jsx
function 함수이름() {
  *// 실행할 명령들*
}
```

- `function`: 함수를 선언하는 키워드
- `함수이름`: 원하는 이름으로 지정
- `()`: 소괄호 (지금은 비워두세요)
- `{}`: 중괄호. 이 안에 실행할 코드(명령)를 작성합니다.

그럼, 우리의 '아침 루틴'을 함수로 만들어 볼까요?

```jsx
function morningRoutine() {
  console.log('잠에서 깨어납니다.');
  console.log('세수를 합니다.');
  console.log('옷을 입습니다.');
  console.log('아침 식사를 합니다.');
}
```

이제 `morningRoutine`이라는 이름의 함수가 만들어졌습니다. 이 함수는 4개의 `console.log` 명령을 담고 있습니다.

---

### 함수 호출하기

함수를 선언하는 것만으로는 아무 일도 일어나지 않습니다. 함수는 '이런 명령들을 실행할 준비가 되어있다'고 정의만 해둔 상태입니다.

함수를 실제로 실행하려면 **함수를 호출**해야 합니다. 함수 호출은 **함수 이름 뒤에 소괄호 `()`**를 붙여서 합니다.

```jsx
morningRoutine();
```

이렇게 코드를 작성하고 실행하면, `morningRoutine` 함수 안에 있던 4개의 명령이 순서대로 실행되는 것을 볼 수 있습니다.

```jsx
잠에서 깨어납니다.
세수를 합니다.
옷을 입습니다.
아침 식사를 합니다.
```

---

### 함수를 사용하는 이유

### 1. 코드의 재사용

만약 '아침 루틴'을 주말에도 반복해야 한다면 어떨까요? 함수가 없다면 4줄의 코드를 2번, 총 8줄을 작성해야 합니다.

하지만 함수를 사용하면 간단하게 해결할 수 있습니다.

```jsx
morningRoutine();
morningRoutine();
```

함수를 한 번만 잘 만들어두면, 필요할 때마다 몇 번이든 다시 불러서 사용할 수 있습니다. 이것이 **코드의 재사용성**입니다.

### 2. 코드의 가독성 및 유지보수

함수에는 의미를 담은 이름을 붙일 수 있습니다. `morningRoutine()`이라는 이름만 봐도 '아, 아침 루틴을 실행하는 코드구나'라고 쉽게 짐작할 수 있습니다.

또한, 만약 루틴의 내용이 변경된다면 어떨까요? 예를 들어 "세수를 합니다."를 "샤워를 합니다."로 바꾸고 싶다면, 함수 내부의 코드 한 줄만 수정하면 됩니다.

```jsx
function morningRoutine() {
  console.log('잠에서 깨어납니다.');
  console.log('샤워를 합니다.'); *// 여기만 수정!*
  console.log('옷을 입습니다.');
  console.log('아침 식사를 합니다.');
}
```

함수를 호출하는 모든 곳에 이 변경사항이 자동으로 반영되므로, 코드를 관리하기가 훨씬 쉬워집니다.

---

### 정리

- **함수**: 여러 개의 명령(코드)을 묶어놓은 실행 단위
- **함수 선언**: `function` 키워드를 사용하여 함수를 만듦
- **함수 호출**: `함수이름()` 형태로 함수를 실행함
- **함수의 장점**: 코드의 재사용성을 높이고, 가독성과 유지보수를 편리하게 함

함수는 프로그래밍의 핵심 개념 중 하나입니다.

복잡한 문제들을 기능 단위로 나누어 함수로 만들어 해결하는 것은 매우 중요한 프로그래밍 기술입니다.

"Hello, [이름]!" 과 "Welcome to Codeit!" 이라는 두 개의 환영 메시지를 출력하는 `printWelcomeMessage` 라는 이름의 함수를 선언하고, 이 함수를 두 번 호출해보세요.

    해설

    함수를 두 번 호출하면 메시지가 두 번 출력됩니다. 함수 내용을 수정하면 모든 호출에 자동으로 반영되어 코드 관리가 편해집니다.

### 10. 파라미터 ⭐⭐⭐

### 함수에 값을 전달하는 통로, 파라미터

이전 시간에 배운 함수는 호출할 때마다 항상 똑같은 결과를 만들어냈습니다.

```jsx
function morningRoutine() {
  console.log('잠에서 깨어납니다.');
  console.log('세수를 합니다.');
  console.log('옷을 입습니다.');
  console.log('아침 식사를 합니다.');
}

morningRoutine(); *// 언제나 같은 결과 출력*
```

하지만 만약 함수가 외부로부터 값을 전달받을 수 있다면 어떨까요? 마치 자판기에 동전을 넣으면 음료수가 나오는 것처럼, 함수에 값을 '입력'해서 상황에 따라 다른 '결과'를 만들 수 있습니다.

이렇게 함수에 값을 전달하는 통로 역할을 하는 것이 바로 **파라미터(Parameter)**입니다.

---

### 파라미터 선언과 사용

파라미터는 함수를 선언할 때, 함수 이름 뒤의 소괄호 `()` 안에 변수처럼 선언합니다.

```jsx
function 함수이름(파라미터) {
  *// 파라미터를 변수처럼 사용*
}
```

음료를 만드는 함수 `makeDrink`를 예로 들어봅시다. 이 함수는 어떤 음료를 만들지 외부로부터 전달받아야 합니다.

```jsx
function makeDrink(drinkName) {
  console.log(drinkName + '를 만듭니다.');
}
```

- `drinkName`: `makeDrink` 함수의 파라미터입니다. 함수 안에서 변수처럼 사용할 수 있습니다.

---

### 아규먼트(Argument) 전달하기

함수를 호출할 때, 파라미터에 전달할 실제 값을 소괄호 `()` 안에 넣어줍니다. 이 값을 **아규먼트(Argument, 전달인자)**라고 합니다.

```jsx
makeDrink('아메리카노');
```

실행 과정:

1. `makeDrink` 함수를 `'아메리카노'`라는 아규먼트와 함께 호출합니다.
2. 아규먼트 `'아메리카노'`가 `makeDrink` 함수의 파라미터 `drinkName`에 값으로 전달(할당)됩니다.
3. 함수 내부에서 `drinkName`은 `'아메리카노'`라는 값을 가진 변수처럼 동작합니다.
4. `console.log('아메리카노' + '를 만듭니다.');` 코드가 실행됩니다.

실행 결과:

```jsx
아메리카노를 만듭니다.
```

아규먼트를 다르게 전달하면 함수의 결과도 달라집니다.

```jsx
makeDrink('아메리카노');
makeDrink('녹차라떼');
makeDrink('자몽에이드');
```

실행 결과:

```jsx
아메리카노를 만듭니다.
녹차라떼를 만듭니다.
자몽에이드를 만듭니다.
```

---

### 다양한 자료형을 파라미터로 활용하기

파라미터는 문자열뿐만 아니라 숫자, 불린 등 모든 자료형의 값을 전달받을 수 있습니다.

이번에는 수량을 파라미터로 받아, 총 주문 수량을 알려주는 함수를 만들어 봅시다.

```jsx
function printOrderCount(count) {
  console.log('총 ' + count + '잔을 주문하셨습니다.');
}

printOrderCount(3); *// 숫자 3을 아규먼트로 전달*
printOrderCount(5); *// 숫자 5를 아규먼트로 전달*
```

실행 결과:

```jsx
총 3잔을 주문하셨습니다.
총 5잔을 주문하셨습니다.
```

이처럼 파라미터를 활용하면, 고정된 결과만 내놓던 함수를 훨씬 더 유연하고 재사용성 높은 코드로 만들 수 있습니다.

---

## 정리

- **파라미터(Parameter)**:함수의 내부에서 사용할 변수, 즉 데이터가 담길 **"공간"**을 말합니다. (함수의 '입력' 부분)
- **아규먼트(Argument)**: 그 공간에 실제로 넣어주는 **"값"**을 의미합니다.
- 파라미터를 사용하면 함수가 호출될 때마다 다른 값을 받아, 상황에 따라 다른 결과를 만들어내는 동적인 함수를 작성할 수 있습니다.

### 12. 여러 개의 파라미터 ⭐⭐

### 함수에 더 많은 정보 전달하기

지난 시간에는 함수에 값을 전달하는 파라미터에 대해 배웠습니다. 그런데 만약 함수에 한 개가 아닌 여러 개의 정보를 전달하고 싶다면 어떻게 해야 할까요?

예를 들어, 두 수의 덧셈을 계산하는 함수를 만든다면, 두 개의 숫자를 모두 전달받아야 합니다.

---

### 여러 개의 파라미터 선언하기

여러 개의 파라미터를 사용하려면, 함수를 선언할 때 소괄호 `()` 안에 쉼표(`,`)로 파라미터들을 구분해주면 됩니다.

```jsx
function 함수이름(파라미터1, 파라미터2, 파라미터3) {
  *// ...*
}
```

두 개의 숫자를 파라미터로 받아 그 합을 출력하는 `printSum` 함수를 만들어 봅시다.

```jsx
function printSum(num1, num2) {
  console.log(num1 + num2);
}
```

- `num1`과 `num2`라는 두 개의 파라미터를 선언했습니다.

---

### 여러 개의 아규먼트 전달하기

함수를 호출할 때도 마찬가지로 쉼표(`,`)를 사용하여 파라미터의 개수와 순서에 맞게 아규먼트를 전달합니다.

```jsx
printSum(10, 5);
```

실행 과정:

1. `printSum` 함수를 `10`과 `5`라는 두 개의 아규먼트와 함께 호출합니다.
2. 첫 번째 아규먼트 `10`은 첫 번째 파라미터 `num1`에 전달됩니다.
3. 두 번째 아규먼트 `5`는 두 번째 파라미터 `num2`에 전달됩니다.
4. 함수 내부에서 `console.log(10 + 5);` 코드가 실행됩니다.

실행 결과:

```jsx
15
```

아규먼트의 순서를 바꾸면 결과도 달라지므로, **파라미터와 아규먼트의 순서를 일치시키는 것**은 매우 중요합니다.

---

### 여러 파라미터 활용 예제

이번에는 택배 배송 정보를 출력하는 함수를 만들어 봅시다. 이 함수는 '받는 사람'과 '상품 이름'이라는 두 가지 정보가 필요합니다.

```jsx
function printShippingInfo(recipient, item) {
  console.log(recipient + '님에게 ' + item + '을(를) 배송합니다.');
}

printShippingInfo('김코딩', '노트북');
printShippingInfo('박해커', '키보드');
```

실행 결과:

```jsx
김코딩님에게 노트북을(를) 배송합니다.
박해커님에게 키보드를(를) 배송합니다.
```

`recipient`와 `item`이라는 의미 있는 파라미터 이름을 사용함으로써, 코드를 읽는 사람이 각 파라미터의 역할을 쉽게 이해할 수 있습니다.

### 정리

- 함수를 선언할 때 쉼표(`,`)를 사용하여 여러 개의 파라미터를 정의할 수 있습니다.
- 함수를 호출할 때도 파라미터의 **순서와 개수**에 맞게 아규먼트를 쉼표로 구분하여 전달해야 합니다.
- 파라미터의 이름을 의미 있게 지으면 코드의 가독성이 크게 향상됩니다.

### 14. return문

### 함수의 결과값을 바깥으로! `return`

지금까지 우리가 만든 함수는 `console.log`를 이용해 단순히 값을 '출력'만 했습니다. 함수가 계산한 결과를 함수 바깥에서 변수에 저장하거나 다른 계산에 활용할 수는 없었죠.

수학에서의 함수처럼, 프로그래밍의 함수도 입력(파라미터)을 받아 처리한 후, 그 결과를 출력(output)할 수 있습니다. 이 '출력' 역할을 하는 것이 바로 `return` 문입니다.

`return`은 함수 안에서 만들어진 값을 함수 바깥 세상으로 '반환(되돌려주기)'합니다.

---

### `return` 사용하기

`return` 키워드 뒤에 함수가 반환할 값을 명시해주면 됩니다.

```jsx
function getTen() {
  return 10;
}
```

`getTen` 함수는 호출되면 숫자 `10`을 반환합니다.

이렇게 `return`이 있는 함수를 호출하면, 함수를 호출한 그 자리가 함수의 **반환값(return value)**으로 대체됩니다.

```jsx
getTen(); *// 이 코드는 숫자 10과 같습니다.*
```

`console.log`로 확인해볼까요?

```jsx
console.log(getTen()); *// console.log(10); 과 동일하게 동작*
```

**실행 결과:**

```jsx
10
```

---

### `return`과 파라미터 함께 사용하기

`return`은 파라미터와 함께 사용될 때 더욱 강력해집니다. 달러를 원화로 환전해주는 함수를 만들어 봅시다. (1달러 = 1350원이라고 가정)

```jsx
function krwToUsd(krw) {
  let usd = krw / 1350;
  return usd;
}
```

**실행 과정:**

1. `krwToUsd(27000)`를 호출하면, 파라미터 `krw`에 `27000`이 전달됩니다.
2. 함수 내부에서 `27000 / 1350`이 계산되고, 그 결과인 `20`이 `usd` 변수에 저장됩니다.
3. `return usd;` 코드를 통해 `usd` 변수의 값, 즉 `20`이 함수 바깥으로 반환됩니다.
4. `krwToUsd(27000)`를 호출했던 자리는 반환값인 `20`으로 대체됩니다.

---

### 반환값을 변수에 저장하기

함수가 값을 반환한다는 것은, 그 값을 변수에 저장할 수도 있다는 의미입니다.

```jsx
let myMoneyInUsd = krwToUsd(27000);
console.log('내 돈은 ' + myMoneyInUsd + '달러입니다.');
```

**실행 결과:**

```jsx
내 돈은 20달러입니다.
```

`krwToUsd(27000)`의 반환값인 `20`이 `myMoneyInUsd`라는 변수에 성공적으로 저장되었습니다. 이제 이 변수를 이용해 다른 계산을 하거나, 필요한 곳에 자유롭게 사용할 수 있습니다.

이처럼 `return`을 사용하면 함수의 실행 결과를 값으로 만들어, 코드의 다른 부분에서 재사용할 수 있게 되므로 훨씬 더 유연하고 확장성 있는 프로그래밍이 가능해집니다.

---

### 정리

- **`return`**: 함수의 실행 결과를 함수 바깥으로 반환합니다.
- **반환값(Return Value)**: `return` 키워드 뒤에 오는 값.
- 함수 호출문은 그 함수의 반환값으로 대체될 수 있습니다.
- 함수의 반환값은 변수에 저장하거나 다른 연산에 즉시 활용할 수 있습니다.

## 문제 2

아래 코드의 실행 결과를 예상해보세요. `return` 키워드의 역할과 함수의 실행 흐름을 생각하며 답을 골라보세요.

```jsx
function getGrade(score) {
  if (score >= 90) {
    return 'A';
  }
  
  if (score >= 80) {
    return 'B';
  }
  
  if (score >= 70) {
    return 'C';
  }
  
  return 'F';
}

let myGrade = getGrade(85);
console.log(myGrade);
```

1. A
2. B
3. C
4. F

---

- 정답 및 해설
    
    정답: 2번 (B)
    
    해설:
    
    `return` 문은 함수의 결과값을 반환하는 동시에 그 즉시 함수의 실행을 종료시킨다
    
    `getGrade(85)`가 호출되고, 파라미터 score는 85가 된다
    
    첫 번째 `if (score >= 90)` 문은 false이므로 넘어간다
    
    두 번째 `if (score >= 80)` 문은 true이므로, 코드 `return 'B'`가 실행된다
    
    함수는 'B'를 반환하고 그 자리에서 즉시 실행을 멈춘다
    
    아래에 있는 `if (score >= 70)` 문과 `return 'F'`는 테스트조차 되지 않는다
    
    반환된 값 'B'가 myGrade 변수에 저장되고, 콘솔에 B가 출력된다
