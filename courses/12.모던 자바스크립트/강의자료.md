### 01. 모던 자바스크립트란?

## 1. ECMAScript란?

JavaScript를 공부하다 보면 **ECMAScript(에크마스크립트)**라는 용어를 자주 보게 됩니다.

ECMAScript는 **JavaScript의 표준 사양**을 의미합니다.

JavaScript도 다른 언어처럼 계속 발전하며 업데이트되고 있으며, 새로운 기능과 문법이 ECMAScript 규격에 추가됩니다.

---

## 2. ECMAScript의 관리

### 2-1. 표준화 기구

JavaScript 표준은 **Ecma International**이라는 국제 표준화 기구가 관리합니다.

이 기구는 JavaScript의:

- 규칙
- 동작 방식
- 새로운 기능

이 모든 내용을 **ECMA-262 문서**에 정의합니다. 이 문서가 바로 ECMAScript입니다.

### 2-2. 버전 관리

ECMAScript는 1997년에 첫 버전이 등장했습니다.

**초기 버전 (1997~2014)**

- ES1 (1997)
- ES2 (1998)
- ES3 (1999)
- ES4 (폐기)
- ES5 (2009)

**현대 버전 (2015~현재)**

- ES6 = ES2015 (2015)
- ES2016
- ES2017
- ES2018
- … 매년 1회 업데이트

2015년 이후부터는 **연도 기반 버전명(ES2015)**을 공식 명칭으로 사용합니다.

---

## 3. ES6(ES2015)의 혁명

### 3-1. 왜 중요한가?

ES6는 JavaScript 역사에서 가장 중요한 업데이트입니다.

- 기존 문제점 개선
- 현대적 프로그래밍 기능 추가
- 실무 코드 품질 향상

### 3-2. ES6+ 용어

ES6 이후의 모든 버전을 **ES6+**라고 부릅니다.

즉:

- ES6 = 기준점
- ES6+ = 그 이후 최신 기능을 포함한 모던 자바스크립트

### 3-3. ES6에서 추가된 주요 기능

**변수/스코프**

- `let`, `const`
- 블록 스코프

**함수**

- 화살표 함수
- 기본 파라미터
- Rest 파라미터

**객체/배열**

- 구조 분해 할당
- Spread 문법
- 템플릿 문자열

**그 외**

- 클래스
- 모듈(import/export)

### 01. 데이터 타입의 특징과 종류

### 1. JavaScript의 유연한 데이터 타입

JavaScript는 다른 언어에 비해 **데이터 타입이 매우 유연**합니다.

유연하다는 것은 **상황에 따라 타입이 자동으로 변할 수 있다**는 의미입니다.

### 다른 언어와 비교

**Python**

```python
2 + "3"   # TypeError
```

**Java**

```java
int num = 10;
num = "hello"; // 컴파일 에러
```

**JavaScript**

```jsx
let x = 10;
x = "hello";
x = true;
x = [1, 2, 3];
```

JavaScript는 타입 선언이 없고, 연산 중에도 타입이 자동으로 변환됩니다.

---

### 2. 자동 타입 변환

### 덧셈 연산

```jsx
2 + "3"; // "23"
```

숫자 2가 문자열 "2"로 변환되어 문자열 결합이 일어납니다.

### 곱셈, 뺄셈, 나눗셈

```jsx
"10" - 5;  // 5
"20" / 4;  // 5
"5" * "3"; // 15
"hello" * 3; // NaN
```

변환 가능한 경우에는 숫자로 변환됩니다.

### 변수 재할당

```jsx
let v = 100;
v = "문자열";
v = true;
v = { name: "홍길동" };
```

언어 자체가 타입을 강제하지 않기 때문에 가능한 일입니다.


### 문제 1

다음 코드의 실행 결과를 예측하세요.

```jsx
console.log(5 + "10");
console.log(5 - "10");
console.log(5 * "2");
console.log("20" / 4);
```

- 정답 및 해설
    
    정답:
    
    ```jsx
    console.log(5 + "10"); // "510"
    console.log(5 - "10"); // -5
    console.log(5 * "2"); // 10
    console.log("20" / 4); // 5
    
    ```
    
    해설:
    
    `5 + "10"` = `"510"` (문자열)
    
    - 덧셈 연산에서 한쪽이 문자열이면 다른 쪽도 문자열로 변환
    - 숫자 `5`가 문자열 `"5"`로 변환되어 문자열 연결
    - `"5" + "10"` = `"510"`
    
    `5 - "10"` = `-5` (숫자)
    
    - 뺄셈 연산에서는 문자열이 숫자로 변환
    - `"10"` → 10
    - `5 - 10` = `5`
    
    `5 * "2"` = `10` (숫자)
    
    - 곱셈에서는 문자열이 숫자로 변환
    - `"2"` → 2
    - `5 * 2` = `10`
    
    `"20" / 4` = `5` (숫자)
    
    - 나눗셈은 숫자 우선
    - `"20"` → 20
    - `20 / 4` = 5

### 문제 3

다음 코드를 실행했을 때, `arr1`과 `arr2`의 값은 각각 무엇인가요? 그리고 그 이유를 설명하세요.

```jsx
const arr1 = [1, 2, 3];
const arr2 = arr1;

arr1.push(4);

console.log(arr1);
console.log(arr2);

```

- 정답 및 해설
    
    정답:
    
    ```jsx
    console.log(arr1); // [1, 2, 3, 4]
    console.log(arr2); // [1, 2, 3, 4]
    
    ```
    
    해설:
    
    배열(Array)은 **참조형(Reference Type)**입니다.
    
    `arr1`과 `arr2`는 같은 메모리 주소를 가리킵니다.
    
    ```jsx
    const arr1 = [1, 2, 3]; // 주소 예: 0x001
    const arr2 = arr1;      // 같은 주소 0x001
    arr1.push(4);           // 해당 주소의 배열에 값 추가
    
    ```
    
    둘 다 동일한 배열을 바라보기 때문에 `[1, 2, 3, 4]`가 출력됩니다.
    
    독립적인 복사본을 만들고 싶다면:
    
    ```jsx
    const arr2 = arr1.slice();
    const arr2 = [...arr1];
    
    ```

### 05. 불린인 듯 불린 아닌 불린같은 값

### 1. if문에 문자열이?

### 1-1. 예상 밖의 코드

다음 코드를 보세요:

```jsx
if ("codeit") {
  console.log("I love JavaScript!");
} else {
  console.log("...");
}

```

질문: if문의 조건 부분에 문자열 `"codeit"`이 들어있습니다. 무엇이 출력될까요?

### 1-2. 실행 결과

```jsx
// 실행 결과
I love JavaScript!
```

"I love JavaScript!"가 출력됩니다.

이유: 문자열 `"codeit"`이 **true처럼 평가**되었기 때문입니다.

---

### 2. 불린 타입으로의 자동 형 변환

### 2-1. JavaScript의 특징

JavaScript에서는 if, for, while문의 조건처럼 **불린이 필요한 맥락**에서 불린이 아닌 값도 **불린으로 평가**됩니다.

즉, 불린이 아닌 타입이 불린으로 **자동 형 변환**됩니다.

### 2-2. 불린 값이 필요한 상황

**1. if문**

```jsx
if ("hello") {
  console.log("실행됨!");
}
```

**2. while문**

```jsx
let count = 5;
while (count) {
  console.log(count);
  count--;
}
```

**3. 논리 연산자**

논리 연산자도 값을 Truthy/Falsy로 평가합니다:

```jsx
// && 연산자: 왼쪽이 Truthy면 오른쪽 값 반환
const name1 = "김철수" && "유효한 이름";
console.log(name1); // "유효한 이름"

const name2 = "" && "유효한 이름";
console.log(name2); // "" (빈 문자열은 Falsy)

// || 연산자: 왼쪽이 Falsy면 오른쪽 값 반환
const value1 = null || "기본값";
console.log(value1); // "기본값"

const value2 = "있음" || "기본값";
console.log(value2); // "있음"
```

**4. 삼항 연산자**

```jsx
const message = "hello" ? "있음" : "없음"; // "hello"가 Truthy로 평가 됨
console.log(message) // 있음
```

### 3. Falsy 값: False처럼 평가되는 값들

### 3-1. Falsy 값 목록

JavaScript에서 **false로 평가되는 값은 딱 8가지**입니다:

```jsx
1. false
2. null
3. undefined
4. NaN
5. 0
6. -0
7. 0n
8. ""

```

### 3-2. Falsy 값 예제

```jsx
if (false) {}
if (null) {}
if (undefined) {}
if (NaN) {}
if (0) {}
if (-0) {}
if (0n) {}
if ("") {}
```

모두 false로 평가됩니다.

### 3-3. 실전 활용

```jsx
function greet(name) {
  if (!name) {
    console.log("이름을 입력해주세요!");
    return;
  }
  console.log(`안녕하세요, ${name}님!`);
}
```

### 4. Truthy 값: True처럼 평가되는 값들

### 4-1. Truthy 값이란?

Falsy 값을 제외한 **모든 값은 true처럼 평가**됩니다.

예시:

```jsx
"hello"
"0"
"false"
" "
1
-1
Infinity
{}
[]
function() {}
true
new Date()
Symbol()

```

### 4-2. 빈 배열과 빈 객체

```jsx
if ([]) {
  console.log("실행됨!");
}

if ({}) {
  console.log("실행됨!");
}
```

둘 다 Truthy입니다.

### 4-3. 올바른 배열 체크

```jsx
const items = [];

if (items.length > 0) {
  console.log("배열에 항목이 있습니다");
} else {
  console.log("배열이 비어있습니다");
}
```

### 4-4. 문자열 "0"과 "false"

```jsx
if ("0") {
  console.log("실행됨!");
}

if ("false") {
  console.log("실행됨!");
}
```

내용이 있는 문자열은 모두 Truthy입니다.


### 8. 정리

### 핵심 개념

1. Falsy 값: false, null, undefined, NaN, 0, -0, 0n, ""
2. Truthy 값: Falsy를 제외한 모든 값
3. 빈 배열 / 빈 객체: Truthy
4. Boolean(): 명시적 불린 변환

### 주의사항

- 문자열 `"0"`, `"false"`는 Truthy 값이다.
- 빈 배열, 빈 객체도 Truthy이므로 조건문에서 주의해야 한다.
- `||` 기본값 패턴은 숫자 0을 오해할 수 있음. → `??` (null 병합 연산자)로 처리하면 됨!
    
    ```jsx
    function printNumber(num) {
        const target = num || "없음"
        console.log(`받은 숫자는 ${target}.`)
    }
    
    printNumber(5); // 받은 숫자는 5.
    printNumber(0); // 받은 숫자는 없음.
    ```
    

### 용어설명

- **Falsy**: false처럼 평가되는 값
- **Truthy**: true처럼 평가되는 값
- **형 변환(Coercion)**: JavaScript가 자동으로 타입을 바꿔 평가하는 과정

### 07. AND와 OR의 연산 방식 ⭐⭐

### 1. 논리 연산자의 특별한 동작

지난 시간에 Truthy와 Falsy 값에 대해 배웠습니다. 이번 시간에는 논리 연산자 `&&`(AND)와 `||`(OR)가 실제로 어떻게 동작하는지 살펴보겠습니다.

다음 코드를 실행하면 어떤 결과가 나올까요?

```jsx
console.log("banana" && "apple");
```

`"banana"`와 `"apple"` 모두 Truthy 값이므로 `true && true`가 되어 `true`가 출력될 것 같지만, 실제 실행해보면 `"apple"`이 출력됩니다. 둘다 `true`이면 **오른쪽에 있는 것이 반환** 됩니다.

이유: 자바스크립트의 논리 연산자는 **항상 true 또는 false를 반환하는 것이 아니라, 두 값 중 하나를 그대로 반환**하기 때문입니다. 

---

### 2. AND 연산자(`&&`)의 동작 원리

### 2-1. 불린 값으로 확인하기

```jsx
console.log(true && true); // true
console.log(true && false); // false
console.log(false && true); // false
console.log(false && false); // false
```

### 2-2. 실제 동작 방식

자바스크립트의 AND 연산자는 아래처럼 동작합니다:

1. **왼쪽 값 평가**
2. **왼쪽이 Falsy면 → 왼쪽 값 반환하고 종료** (오른쪽은 평가 안 함)
3. **왼쪽이 Truthy면 → 오른쪽 값 평가해서 반환**

예시 분석:

- `true && true` → 왼쪽 Truthy → 오른쪽 평가 → `true` 반환
- `true && false` → 왼쪽 Truthy → 오른쪽 평가 → `false` 반환
- `false && true` → 왼쪽 Falsy → `false` 반환 (오른쪽 평가 안 함)
- `false && false` → 왼쪽 Falsy → `false` 반환 (오른쪽 평가 안 함)

따라서 `"banana" && "apple"`은 `"banana"`가 Truthy이므로 `"apple"`이 반환됩니다.

---

### 3. OR 연산자(`||`)의 동작 원리

### 3-1. 불린 값으로 확인하기

```jsx
console.log(true || true); // true
console.log(true || false); // true
console.log(false || true); // true
console.log(false || false); // false
```

### 3-2. 실제 동작 방식

OR 연산자는 다음 순서로 평가합니다:

1. **왼쪽 값 평가**
2. **왼쪽이 Truthy면 → 왼쪽 값 반환하고 종료** (오른쪽은 평가 안 함)
3. **왼쪽이 Falsy면 → 오른쪽 값 평가해서 반환**

예시 분석:

- `true || true` → 왼쪽 Truthy → `true` 반환 (오른쪽 평가 안 함)
- `true || false` → 왼쪽 Truthy → `true` 반환 (오른쪽 평가 안 함)
- `false || true` → 왼쪽 Falsy → 오른쪽 평가 → `true` 반환
- `false || false` → 왼쪽 Falsy → 오른쪽 평가 → `false` 반환

---

### 4. 다양한 값으로 연습하기

```jsx
console.log(null && undefined); // null
console.log(0 || true); // true
console.log("0" && NaN); // NaN
console.log({} || 123); // {}
```

### 분석

1. **`null && undefined`**
    - `null`은 Falsy
    - AND는 왼쪽이 Falsy면 왼쪽 값 반환
    - 결과: `null`
2. **`0 || true`**
    - `0`은 Falsy
    - OR는 왼쪽이 Falsy면 오른쪽 값 반환
    - 결과: `true`
3. **`"0" && NaN`**
    - `"0"`은 Truthy
    - AND는 왼쪽 Truthy → 오른쪽 반환
    - 결과: `NaN`
4. **`{} || 123`**
    - `{}`는 Truthy
    - OR는 왼쪽 Truthy → 왼쪽 값 반환
    - 결과: `{}`

---

### 5. 실전 활용: 기본값 설정

```jsx
function greet(name) {
  const userName = name || "손님";
  console.log(`안녕하세요, ${userName}님!`);
}

greet("철수"); // 안녕하세요, 철수님!
greet(); // 안녕하세요, 손님님!

```

함수 호출 시 인자를 전달하지 않으면 `name`은 `undefined`(Falsy)입니다.

그래서 `name || "손님"`은 오른쪽 `"손님"`을 반환하게 됩니다.

### 6. 정리

올바른 정리 표는 다음과 같습니다:

| 연산자 | 왼쪽이 Truthy일 때 | 왼쪽이 Falsy일 때 |
| --- | --- | --- |
| `&&` | 오른쪽 값 반환 | 왼쪽 값 반환 |
| `||` | 왼쪽 값 반환 | 오른쪽 값 반환 |

논리 연산자는 단순히 true/false를 반환하는 것이 아니라, **실제 값을 선택해서 반환**한다는 점을 기억하세요.

- 객체(`{}`), 배열(`[]`)은 항상 Truthy이다.
- `||`를 기본값 설정에 사용할 때 숫자 0은 Falsy라서 오작동할 수 있다.

### 문제 2

다음 함수의 문제점을 찾고 수정하세요:

```jsx
function createUser(name, age) {
  const userName = name && "익명";
  const userAge = age && 0;

  console.log(`이름: ${userName}, 나이: ${userAge}`);
}

createUser("김철수", 25);
```

- 정답 및 해설

    ```jsx
    }
    ```

    ```jsx
    name || "익명";
    age || 0;
    
    ```

    - AND는 왼쪽 Falsy → 왼쪽 반환
    - 결과: `0`
    
    **2. `b && 100`**
    
    - `b`는 `"0"` → Truthy
    - AND는 왼쪽 Truthy → 오른쪽 반환
    - 결과: `100`
    
    **3. `a || 100`**
    
    - `a`는 `0` → Falsy
    - OR는 왼쪽 Falsy → 오른쪽 반환
    - 결과: `100`
    
    **4. `b || 100`**
    
    - `b`는 `"0"` → Truthy
    - OR는 왼쪽 Truthy → 왼쪽 반환
    - 결과: `"0"`
    
    핵심:
    
    ```jsx
    0     // Falsy
    "0"   // Truthy
    ```
    
    이 차이 때문에 사용자 입력 처리 시 주의해야 함.
    
    **예시**
    
    ```jsx
    function setVolume(volume) {
      const vol = volume || 50; // 0 입력 → 50이 돼버림 (문제)
    }
    ```
    
    → Null 병합 연산자 사용
    
    ```jsx
    const vol = volume ?? 50; // null/undefined일 때만 기본값
    ```
    
    0은 유지되고 기본값이 적용되지 않음.

### 11. 변수와 스코프 ⭐⭐⭐

### 1. 변수 선언 키워드의 변화

변수는 값에 이름을 부여하는 프로그래밍의 기본 추상화 수단입니다. JavaScript에서는 ES2015 이후 변수 선언 방식에 큰 변화가 있었습니다.

### 1-1. ES2015 이전: `var`

```jsx
var title = "JavaScript";
```

### 1-2. ES2015 이후: `let`과 `const`

```jsx
let title = "JavaScript"; // 재할당 가능
const language = "JavaScript"; // 재할당 불가능
```

ES2015 이전에는 `var`만 존재했지만 여러 문제점이 있었기 때문에, 현대 JavaScript에서는 **`let`과 `const`를 사용하는 것이 권장됩니다.**

---

### 2. `var`의 문제점과 `let`/`const`의 해결책

### 2-1. 호이스팅 문제

**`var`의 문제:**

```jsx
console.log(title); // undefined
var title = "JavaScript";
```

`var`로 선언한 변수는 선언 이전에도 접근 가능하며, 값은 `undefined`입니다. 이것을 **호이스팅(Hoisting)**이라고 합니다.

```jsx
console.log(price); // undefined
var price = 5000;
console.log(price); // 5000

```

**`let`/`const`의 해결:**

```jsx
console.log(title); // ReferenceError
let title = "JavaScript";
```

`let`과 `const`는 선언 전에 접근하면 **명확한 에러**를 발생시킵니다.

---

### 2-2. 중복 선언 문제

**`var`의 문제:**

```jsx
var title = "JavaScript";
var title = "Python"; // 덮어쓰기 발생
```

중복 선언이 가능하여 실수를 유발합니다.

**`let`/`const`의 해결:**

```jsx
let title = "JavaScript";
let title = "Python"; // SyntaxError
```

중복 선언이 허용되지 않아 실수를 방지합니다.

---

### **2-3. 스코프 문제 (가장 중요!)**

`var`는 **오직 함수의 중괄호만 스코프로 인정**합니다. 다른 중괄호(if, for, while 등)는 무시합니다.

**함수 안에서 선언하면 지역 변수:**

```jsx
var x = 10; // 전역 변수

function test() {
  var y = 20; // 지역 변수 (함수 중괄호 안에서만 사용 가능)
  console.log(x); // 10 (전역 변수 접근 가능)
  console.log(y); // 20
}

test();
console.log(x); // 10
console.log(y); // ReferenceError: y is not defined
```

여기서 `y`는 함수 안에서 선언되었으므로 **함수 밖에서는 접근 불가**합니다. 이건 정상입니다.

**문제: if, for, while의 중괄호는 무시됨**

```jsx
var x = 10;

if (true) {
  var z = 30; // if문 안에서 선언
  console.log(z); // 30
}

console.log(z); // 30 (if문 밖에서도 접근 가능!)
```

⚠️ **이게 문제입니다!**

- `z`는 `if`문 안에서 선언했습니다
- 하지만 `var`는 **함수 중괄호만 스코프로 인정**합니다
- `if`문의 중괄호는 **스코프를 만들지 못합니다**
- 결과적으로 `z`는 **전역 변수**가 되어버립니다!

**for문도 마찬가지:**

```jsx
for (var i = 0; i < 3; i++) {
  console.log(i); // 0, 1, 2
}

console.log(i); // 3 (for문 밖에서도 접근 가능!, i++가 실행되어 3이 되어 for문 종료)
```

`i`는 `for`문 안에서만 사용하려고 했지만, `var`는 함수 중괄호가 아니면 스코프를 만들지 못하므로 전역 변수가 됩니다.

**while문도 마찬가지:**

```jsx
var count = 0;

while (count < 3) {
  var message = "반복 중";// while문 안에서 선언
  console.log(message);
  count++;
}

console.log(message);// "반복 중" (while문 밖에서도 접근 가능!)
```

**var가 스코프로 인정하는 것:**

```jsx
// ✅ 함수 중괄호만 스코프로 인정
function myFunction() {
  var a = 1;// 함수 안에서만 사용 가능
}

// ❌ 이런 중괄호들은 스코프를 만들지 못함
if (true) {
  var b = 2;// 전역 변수가 됨
}

for (var i = 0; i < 3; i++) {
  var c = 3;// 전역 변수가 됨
}

while (true) {
  var d = 4;// 전역 변수가 됨break;
}

{
  var e = 5;// 단순 블록도 무시, 전역 변수가 됨
}

```

**`let`/`const`의 해결: 블록 스코프**

`let`과 `const`는 **모든 중괄호(`{}`) 단위로** 스코프가 구분됩니다:

```jsx
let x = 10; // 전역 변수

function test() {
  let y = 20; // 함수의 지역 변수
  console.log(x); // 10
  console.log(y);// 20
}

if (true) {
  let z = 30; // if문의 지역 변수
  console.log(z); // 30
}

console.log(x); // 10 
console.log(z); // ReferenceError: z is not defined
```

심지어 단순한 코드 블록도 스코프를 만듭니다:

```jsx
{
  let message = "Hello";
  console.log(message); // Hello
}

console.log(message); // ReferenceError: message is not defined
```

---

### 3. 스코프 용어 정리

### 3-1. 함수 스코프 (Function Scope)

- `var`가 사용하는 스코프
- 함수 단위로만 유효 범위가 생김
- if, for 등의 블록은 스코프를 만들지 못함

### 3-2. 블록 스코프 (Block Scope)

- `let`, `const`가 사용하는 스코프
- 모든 `{}` 단위로 유효 범위 결정
- 함수, 조건문, 반복문, 단순 블록 모두 포함

---

### 4. 비교 표

| 특징 | `var` | `let` | `const` |
| --- | --- | --- | --- |
| 재할당 | 가능 | 가능 | **불가능** |
| 재선언 | 가능 | **불가능** | **불가능** |
| 호이스팅 | 가능(초기값: undefined) | **접근 시 에러** | **접근 시 에러** |
| 스코프 | 함수 스코프 | **블록 스코프** | **블록 스코프** |

---

### 5. 어떤 키워드를 사용해야 할까?

### 5-1. 기본 원칙

1. **기본은 `const`**
2. **변경이 필요하면 `let`**
3. **`var`는 사용하지 않기**

### 5-2. 예시

```jsx
const PI = 3.14159;
const userName = "김철수";

let count = 0;
count++;

let status = "pending";
status = "completed";

```

---

### 주의사항

- `var`는 호이스팅과 스코프 문제로 인해 모던 JS에서 비권장된다.
- 선언 전 접근이 필요한 경우는 존재하지 않으며, 에러가 발생하는 것이 오히려 안전하다.

### 용어설명

- **호이스팅(Hoisting)**: 선언이 코드 최상단으로 끌어올려진 것처럼 동작하는 현상
- **함수 스코프(Function Scope)**: 함수 단위로 스코프가 구분
- **블록 스코프(Block Scope)**: `{}` 단위로 스코프가 구분

### 문제 1

다음 코드의 출력 결과를 예측하고, 그 이유를 설명하세요:

```jsx
console.log(name);
var name = "철수";

console.log(age);
let age = 25;
```

- 정답 및 해설
    
    정답:
    
    ```jsx
    console.log(name); // undefined
    var name = "철수";
    
    console.log(age); // ReferenceError: Cannot access 'age' before initialization
    let age = 25;
    
    ```
    
    해설:
    
    **1. `var name`의 경우:**
    
    - `var`는 호이스팅되어 선언 전에 접근 가능
    - 선언만 끌어올려지고 할당은 그대로 유지됨
    - 따라서 `undefined` 출력
    
    **2. `let age`의 경우:**
    
    - `let`은 선언 전에 접근하면 ReferenceError 발생
    - 호이스팅은 되지만 "Temporal Dead Zone"(TDZ)에 있음
    - 초기화 이전에는 접근 불가능
    
    **TDZ(Temporal Dead Zone)란?**
    
    TDZ는 "일시적 사각지대"라는 뜻으로, 변수가 선언되었지만 아직 초기화되지 않아 접근할 수 없는 구간을 말합니다.
    
    ```jsx
    // TDZ 시작
    console.log(name); // ReferenceError! (TDZ 구간)
    // TDZ 구간
    let name = "철수"; // TDZ 끝 (초기화 완료)
    console.log(name); // "철수" (접근 가능)
    
    ```
    
    - `var`: 호이스팅되면서 `undefined`로 자동 초기화 → TDZ 없음
    - `let`/`const`: 호이스팅되지만 초기화 안 됨 → TDZ 존재
    
    **호이스팅이 실제로 동작하는 방식:**
    
    ```jsx
    // var의 경우 (실제 동작 방식)
    var name; // 선언만 끌어올려짐
    console.log(name); // undefined
    name = "철수"; // 할당은 원래 위치에서
    
    // let의 경우
    // 선언은 되지만 TDZ 상태
    console.log(age); // 에러! 초기화 전 접근 불가
    let age = 25;
    
    ```
    
    **핵심:**
    
    - `var`: 호이스팅되어 undefined로 초기화
    - `let`/`const`: 호이스팅되지만 초기화 전 접근 불가 (TDZ)

### 문제 2

선행지식!

> setTimeout이란?
> 
> 
> `setTimeout`은 특정 시간(밀리초) 후에 함수를 실행하는 내장 함수입니다.
> 
> ```jsx
> setTimeout(() => {
>   console.log("1초 후 실행");
> }, 1000);
> 
> setTimeout(() => {
>   console.log("2초 후 실행");
> }, 2000);
> 
> setTimeout(() => {
>   console.log("3초 후 실행");
> }, 3000);
> ```
> 
> 중요한 점은 **즉시 실행되지 않고 나중에 실행**된다는 것입니다.
> 
> [링크](https://www.programiz.com/javascript/online-compiler/) 여기서 테스트 해보세요!
> 

다음 코드에서 예상과 다른 결과가 나옵니다. 왜 그럴까요?

```jsx
const numbers = [1, 2, 3, 4, 5];

for (var i = 0; i < 3; i++) {
  setTimeout(() => {
    console.log(numbers[i]);
  }, 100);
}
```

이 코드를 실행하면 `1 2 3`이 출력되길 기대하지만, 실제로는 예상과 다른 결과가 나옵니다. 왜 그럴까요?

- 정답 및 해설
    
    정답:
    
    코드를 실행하면 `4 4 4`가 출력됩니다.
    
    해설:
    
    **실행 순서 이해하기:**
    
    ```jsx
    // 1. for문이 매우 빠르게 실행됨
    // i=0: setTimeout 예약 (100ms 후 실행)
    // i=1: setTimeout 예약 (100ms 후 실행)
    // i=2: setTimeout 예약 (100ms 후 실행)
    // i=3: 조건 불만족, 반복 종료
    
    // 2. for문이 끝난 시점에 i는 3
    
    // 3. 100ms 후 setTimeout들이 실행됨
    // console.log(numbers[3]) → 4
    // console.log(numbers[3]) → 4
    // console.log(numbers[3]) → 4
    
    ```
    
    **이유:**
    
    1. `var`는 함수 스코프를 가지므로 `i`는 반복문 밖에서도 접근 가능
    2. `setTimeout`의 콜백 함수들은 100ms 후에 실행됨
    3. 그 사이에 반복문은 이미 완료되어 `i`는 3이 됨
    4. 세 콜백 함수 모두 같은 `i`(3)를 참조
    5. `numbers[3]`은 `4`이므로 `4 4 4`가 출력됨
    
    **수정된 코드 (let 사용):**
    
    ```jsx
    const numbers = [1, 2, 3, 4, 5];
    
    for (let i = 0; i < 3; i++) {
      setTimeout(() => {
        console.log(numbers[i]);
      }, 100);
    }
    // 출력: 1 2 3
    
    ```
    
    **`let`을 사용하면 해결되는 이유:**
    
    ```jsx
    // var: 반복문 전체에서 하나의 i 공유
    for (var i = 0; i < 3; i++) {
      // 세 번 반복하지만 모두 같은 i 변수 사용
    }
    console.log(i); // 3 (접근 가능)
    
    // let: 각 반복마다 새로운 i 생성
    for (let i = 0; i < 3; i++) {
      // 각 반복마다 독립적인 i가 생성됨
      // 첫 번째 반복: i=0 (이 i는 첫 번째 setTimeout에 고정)
      // 두 번째 반복: i=1 (새로운 i, 두 번째 setTimeout에 고정)
      // 세 번째 반복: i=2 (새로운 i, 세 번째 setTimeout에 고정)
    }
    console.log(i); // ReferenceError (접근 불가)
    
    ```
    
    - `let`은 블록 스코프를 가짐
    - 반복문의 각 반복마다 새로운 `i`가 생성됨
    - 각 콜백 함수는 자신만의 `i`를 참조
    
    **핵심 차이:**
    
    | 키워드 | 반복문에서의 동작 | setTimeout 실행 시 |
    | --- | --- | --- |
    | `var` | 하나의 `i` 공유 | 모두 같은 `i`(3) 참조 |
    | `let` | 각 반복마다 새로운 `i` 생성 | 각자의 `i` 참조 |

### 문제 3

다음 코드의 에러를 모두 찾고 수정하세요:

```jsx
function processOrder() {
  if (true) {
    var productName = "맥북";
    let quantity = 2;
  }

  console.log(productName);
  console.log(quantity);

  const total = 100;
  total = 200;

  const userInfo = {
    name: "김철수",
    age: 30,
  };
  userInfo = { name: "이영희", age: 25 };
  userInfo.age = 31;
}

processOrder();

```

- 정답 및 해설
    
    정답:
    
    다음과 같은 세 가지 에러가 발생합니다.
    
    1. `console.log(quantity);` → ReferenceError
    2. `total = 200;` → TypeError
    3. `userInfo = { name: "이영희", age: 25 };` → TypeError
    
    해설:
    
    **1. `quantity` 접근 에러:**
    
    ```jsx
    if (true) {
      var productName = "맥북"; // 함수 스코프
      let quantity = 2; // 블록 스코프
    }
    
    console.log(productName); // OK (var는 함수 스코프)
    console.log(quantity); // 에러! (let은 블록 스코프)
    
    ```
    
    `let`은 블록 스코프이므로 if문 밖에서 접근할 수 없습니다.
    
    **2. `const` 재할당 에러:**
    
    ```jsx
    const total = 100;
    total = 200; // 에러! const는 재할당 불가
    
    ```
    
    `const`로 선언한 변수는 재할당할 수 없습니다.
    
    **3. `const` 객체 재할당 에러:**
    
    ```jsx
    const userInfo = { name: "김철수", age: 30 };
    userInfo = { name: "이영희", age: 25 }; // 에러! 객체 자체는 재할당 불가
    userInfo.age = 31; // OK! 객체의 프로퍼티는 변경 가능
    
    ```
    
    `const`는 변수 자체의 재할당을 막지만, 객체의 내부 프로퍼티는 변경 가능합니다.
    
    **수정된 코드:**
    
    ```jsx
    function processOrder() {
      let productName; // 함수 스코프 전역으로 선언
      let quantity;
    
      if (true) {
        productName = "맥북";
        quantity = 2;
      }
    
      console.log(productName); // 맥북
      console.log(quantity); // 2
    
      let total = 100; // const → let 변경
      total = 200; // 재할당 가능
    
      let userInfo = {
        // const → let 변경 (객체 재할당이 필요한 경우)
        name: "김철수",
        age: 30,
      };
      userInfo = { name: "이영희", age: 25 }; // 재할당 가능
      userInfo.age = 31; // 프로퍼티 변경 가능
    }
    
    processOrder();
    
    ```
    
    **더 나은 방법 (const 활용):**
    
    ```jsx
    function processOrder() {
      let productName;
      let quantity;
    
      if (true) {
        productName = "맥북";
        quantity = 2;
      }
    
      console.log(productName);
      console.log(quantity);
    
      let total = 100;
      total = 200;
    
      // 객체 재할당이 아니라 프로퍼티 변경만 필요한 경우
      const userInfo = {
        name: "김철수",
        age: 30,
      };
      // userInfo = {...};  // 이렇게 하지 말고

    processOrder();
    
    ```
    
    **핵심 정리:**
    
    | 키워드 | 재할당 | 재선언 | 스코프 |
    | --- | --- | --- | --- |
    | `const` | 불가능 | 불가능 | 블록 |
    | `let` | 가능 | 불가능 | 블록 |
    | `var` | 가능 | 가능 | 함수 |
    
    **const 객체의 특별한 점:**
    
    ```jsx
    const obj = { a: 1 };
    obj = { b: 2 }; // 에러: 객체 자체 재할당 불가
    obj.a = 2; // 프로퍼티 변경 가능
    obj.c = 3; // 새 프로퍼티 추가 가능
    ```

## 3. 함수 다루기

### 01. 함수를 만드는 방법 ⭐⭐⭐

### 함수를 만드는 방법

### 1. 함수 선언 (Function Declaration)

우리가 이미 잘 알고 있는 함수 만드는 방법입니다:

```jsx
function greet() {
  console.log("안녕하세요!");
}

greet(); // 안녕하세요!
```

### 파라미터와 return문 추가

```jsx
function add(a, b) {
  return a + b;
}

const result = add(3, 5);
console.log(result); // 8
```

이렇게 `function` 키워드로 시작해서 함수를 만드는 방식을 **함수 선언(Function Declaration)**이라고 합니다.

---

### 2. 함수 표현식 (Function Expression)

함수를 만드는 또 다른 방법이 있습니다. **함수 선언을 변수에 할당하는 방식**입니다:

```jsx
const greet = function () {
  console.log("안녕하세요!");
};

greet(); // 안녕하세요!

```

함수 선언을 마치 값처럼 사용하고 있습니다. 이런 방식을 **함수 표현식(Function Expression)**이라고 합니다.

### 파라미터와 return문 추가

```jsx
const add = function (a, b) {
  return a + b;
};

const result = add(3, 5);
console.log(result); // 8
```

### **함수 표현식의 진짜 의미**

함수 표현식은 "변수에 할당하는 것"이 핵심이 아닙니다. **함수 선언을 값처럼 사용하는 모든 경우**를 말합니다.

예를 들어, 이벤트 리스너에 함수를 전달하는 경우도 함수 표현식입니다:

```jsx
const button = document.querySelector("#myButton");

button.addEventListener("click", function () {
  console.log("버튼 클릭!");
});
```

함수를 `addEventListener`의 파라미터로 **값처럼** 전달하고 있죠. 이것도 함수 표현식입니다.

---

### 3. 함수 선언 vs 함수 표현식

### 3-1. 호이스팅 차이

**함수 선언: 선언 전에 호출 가능**

```jsx
greet(); // 안녕하세요! (에러 없음)

function greet() {
  console.log("안녕하세요!");
}
```

함수 선언은 호이스팅되어, 선언 전에도 호출할 수 있습니다.

**함수 표현식: 선언 전에 호출 불가**

```jsx
greet(); // ReferenceError: Cannot access 'greet' before initialization

const greet = function () {
  console.log("안녕하세요!");
};
```

함수 표현식은 변수에 할당되므로, 변수의 호이스팅 규칙(`let`/`const`와 같은 TDZ)의 영향을 받습니다. 그래서 선언(초기화) 전에 사용할 수 없습니다.

---

### 3-2. 스코프 차이

**함수 선언: 선언된 위치의 스코프를 따름**

```jsx
function test() {
  function inner() {
    console.log("내부 함수");
  }
  inner(); // 내부 함수
}

test();
inner(); // ReferenceError (함수 밖에서 호출 불가)
```

함수 안에 선언된 함수는 **그 함수를 둘러싼 스코프(여기서는 `test` 함수 본문)** 안에서만 사용할 수 있습니다.

블록 안에서의 함수 선언은 환경과 모드(엄격 모드 여부)에 따라 동작이 달라질 수 있어, 일반적으로 다음처럼 쓰는 패턴은 피하는 것이 좋습니다:

```jsx
if (true) {
  function greet() {
    console.log("Hello");
  }
  greet(); // 대부분의 환경에서 가능
}

greet(); // 환경/모드에 따라 동작이 달라질 수 있어 권장되지 않음

```

실무에서는 **블록 안에서는 함수 선언 대신 함수 표현식**을 사용하는 것이 더 안전합니다.

**함수 표현식: 변수가 가진 스코프를 그대로 따름**

```jsx
if (true) {
  const greet = function () {
    console.log("Hello");
  };
  greet(); // Hello
}

greet(); // ReferenceError (블록 밖에서 호출 불가)

```

`const`로 선언했으므로 **블록 스코프**를 가지며, 블록 밖에서는 접근할 수 없습니다.

---

### 4. 비교 표

| 특징 | 함수 선언 | 함수 표현식 |
| --- | --- | --- |
| 문법 | `function name() {}` | `const name = function () {}` |
| 호이스팅 | 선언 전 호출 가능 | 선언 전 호출 불가 |
| 스코프 | 선언된 위치의 스코프를 따름 | 변수가 가진 스코프를 그대로 따름 |
| 이름 | 필수 | 선택 (익명 함수 가능) |

---

### 5. 어떤 방식을 사용해야 할까?

### 5-1. 함수 선언을 선호하는 이유

- 선언부가 명확하게 눈에 들어옴
- 파일 어디에서든 자유롭게 호출 가능
- 개념 설명용 예제에서 가독성이 좋음

### 5-2. 함수 표현식을 선호하는 이유

- 선언 이후에만 호출 가능해서 **실행 흐름을 따라가며 읽기 쉬움**
- 변수 스코프를 그대로 활용 가능
- `const`로 선언하면 **실수로 재할당되는 것 방지**

### 5-3. 결론

둘 중 어느 방식을 선택하든 큰 문제는 없지만, **한 프로젝트 안에서는 일관성 있게 사용하는 것**이 중요합니다.

일반적인 방향은 다음과 같습니다.

- **개념 설명, 교재, 기본 예제**: 함수 선언 (의미가 더 명확)
- **실무 코드, 모듈/컴포넌트 정의**: 함수 표현식 (`const fn = () => {}` 포함)

이 강의에서는 **함수 선언을 주로 사용**하여 설명하겠습니다. 함수의 선언부가 더 명확하게 구분되기 때문입니다.

---

### 6. 정리

- **함수 선언(Function Declaration)**
    - `function` 키워드로 시작하는 전통적인 방식
    - 호이스팅되어 선언 전에도 호출 가능
- **함수 표현식(Function Expression)**
    - 함수를 값처럼 변수에 담거나, 인수로 전달하는 모든 방식
    - 변수의 스코프, 호이스팅 규칙을 그대로 따름
- 두 방식은 **호이스팅과 스코프**에서 차이가 있으며,
    
    코드 스타일과 목적에 따라 선택해서 사용한다.

### 05. 값으로서 함수

### 값으로서 함수

JavaScript에서 함수는 매우 특별한 위치를 차지합니다. 다른 언어와 달리, JavaScript의 함수는 **값(value)**으로 취급됩니다.

### 함수의 데이터 타입

함수의 타입을 확인해볼까요?

```jsx
function greet() {
  console.log("안녕하세요!");
}

console.log(typeof greet); // function
```

`typeof` 연산자로 확인하면 `function`이 출력됩니다. 하지만 함수의 실제 데이터 타입은 **객체(Object)**입니다. 

```jsx
console.log(greet.name); // "greet"
console.log(greet.length); // 0 (파라미터 개수)
console.log(greet.constructor); // [Function: Function]
```

함수는 `name`, `length`, `constructor` 같은 프로퍼티를 가진 객체입니다. 그래서 함수는 값처럼 다룰 수 있습니다.

### 함수는 어디에나 할당될 수 있다

### 1. 변수에 할당

가장 기본적인 방식입니다:

```jsx
const greet = function () {
  console.log("Hello!");
};

greet(); // Hello!
```

### 2. 객체의 메소드로 할당

```jsx
const person = {
  name: "철수",
  sayHi: function () {
    console.log(`${this.name}입니다!`);
  },
};

person.sayHi(); // 철수입니다!
```

### 3. 배열의 요소로 할당

```jsx
const operations = [
  function (a, b) {
    return a + b;
  },
  function (a, b) {
    return a - b;
  },
  function (a, b) {
    return a * b;
  },
];

console.log(operations[0](5, 3)); // 8 (덧셈)
console.log(operations[1](5, 3)); // 2 (뺄셈)
console.log(operations[2](5, 3)); // 15 (곱셈)
```

흔한 방법은 아니지만, 필요에 따라 배열에도 함수를 담을 수 있습니다!

### 콜백 함수 (Callback Function)

함수를 다른 함수의 **파라미터로 전달**할 수 있습니다. 이렇게 전달된 함수를 **콜백 함수**라고 합니다.

```jsx
function checkAnswer(answer, onCorrect, onWrong) {
  const correctAnswer = 7;

  if (answer === correctAnswer) {
    onCorrect();
  } else {
    onWrong();
  }
}

function handleCorrect() {
  console.log("정답입니다!");
}

function handleWrong() {
  console.log("오답입니다!");
}

checkAnswer(7, handleCorrect, handleWrong); // 정답입니다!
checkAnswer(3, handleCorrect, handleWrong); // 오답입니다!

```

조건에 따라 다른 동작을 실행하도록 만들 수 있습니다.

### 콜백 함수의 활용 예시

```jsx
// 배열 메소드에서 콜백 함수 사용
const numbers = [1, 2, 3, 4, 5];

numbers.forEach(function (num) {
  console.log(num * 2);
});

// 2
// 4
// 6
// 8
// 10

```

`forEach`의 파라미터로 전달된 익명 함수가 콜백 함수입니다.

### 고차 함수 (Higher-Order Function)

**함수를 리턴하는 함수**를 **고차 함수**라고 합니다.

```jsx
function createGreeting(name) {
  return function () {
    console.log(`${name}님, 반갑습니다!`);
  };
}

const greetJohn = createGreeting("존");
greetJohn(); // 존님, 반갑습니다!

const greetSarah = createGreeting("사라");
greetSarah(); // 사라님, 반갑습니다!

```

`createGreeting` 함수는 함수를 리턴합니다. 이렇게 리턴된 함수를 변수에 담아 나중에 호출할 수 있습니다.

### 고차 함수 바로 실행하기

특별한 경우에는 **이중 괄호**를 사용해서 리턴된 함수를 바로 실행할 수 있습니다:

```jsx
function createMultiplier(multiplier) {
  return function (number) {
    return number * multiplier;
  };
}

// 이중 괄호로 바로 실행
const result = createMultiplier(3)(5);
console.log(result); // 15

// 위 코드는 아래와 같습니다:
// const multiply3 = createMultiplier(3);
// const result = multiply3(5);

```

### 일급 함수 (First-Class Function)

JavaScript 함수의 특징을 정리하면:

1. 변수나 데이터 구조에 할당될 수 있다
2. 다른 함수의 파라미터로 전달될 수 있다
3. 다른 함수의 리턴값이 될 수 있다

프로그래밍 언어에서 이런 특징을 가진 함수를 **일급 함수(First-Class Function)**라고 합니다.

> JavaScript는 일급 함수를 지원하는 프로그래밍 언어입니다.
> 

### 정리

- JavaScript의 함수는 **객체 타입의 값**입니다
- 함수는 **값이 들어갈 수 있는 곳이라면 어디든** 할당 가능합니다 (변수, 객체 프로퍼티, 배열 요소 등)
- **콜백 함수**: 다른 함수의 파라미터로 전달되는 함수
- **고차 함수**: 함수를 리턴하는 함수
- JavaScript는 **일급 함수**를 가진 언어입니다

이러한 특징 덕분에 JavaScript는 함수형 프로그래밍이 가능하며, React와 같은 모던 프레임워크에서 핵심적인 역할을 합니다!

- 함수 이름: `createGreeter`
- 파라미터: `greeting` (인사말 종류, 예: "안녕하세요", "Hello")

### 07. Parameter ⭐⭐⭐

이미 parameter의 기본 개념은 잘 알고 있지만, 용어와 특징들을 정리하고 복습해보겠습니다.

### Parameter와 Argument

### Parameter (파라미터)

```jsx
function greeting(name) {
  // name이 parameter
  console.log(`Hi! My name is ${name}!`);
}
```

**파라미터**: 함수 선언 시 소괄호 안에 작성하는 변수

### Argument (아규먼트)

```jsx
greeting("JavaScript"); // "JavaScript"가 argument
greeting("Codeit"); // "Codeit"가 argument
```

**아규먼트**: 함수 호출 시 파라미터로 전달하는 값

### 구분하기

- **Parameter**: 함수 **선언부**의 변수
- **Argument**: 함수 **호출 시** 전달하는 값

```jsx
function introduce(name, age) {
  // name, age는 parameter
  console.log(`${name}, ${age}살`);
}

introduce("철수", 25); // "철수", 25는 argument
```

### Parameter의 기본값

### Argument를 전달하지 않으면?

```jsx
function greeting(name) {
  console.log(`Hi! My name is ${name}!`);
}

greeting(); // Hi! My name is undefined!
```

Argument를 전달하지 않으면 parameter는 `undefined` 값을 가집니다.

### 기본값 설정하기

```jsx
function greeting(name = "Codeit") {
  console.log(`Hi! My name is ${name}!`);
}

greeting(); // Hi! My name is Codeit!
greeting("JavaScript"); // Hi! My name is JavaScript!
```

할당 연산자(`=`)로 기본값을 설정하면, argument가 없을 때 기본값이 사용됩니다.

### 기본값은 undefined일 때만 동작

```jsx
function greeting(name = "Codeit") {
  console.log(`Hi! My name is ${name}!`);
}

greeting(); // Hi! My name is Codeit! (기본값 사용)
greeting(undefined); // Hi! My name is Codeit! (기본값 사용)
greeting(null); // Hi! My name is null! (기본값 사용 안됨)
greeting(""); // Hi! My name is ! (기본값 사용 안됨)
```

기본값은 argument가 **생략되거나 `undefined`일 때만** 동작합니다. 예를 들어, `null`을 전달하면 기본값이 아닌 `null`이 그대로 parameter에 할당됩니다.

### 여러 Parameter의 기본값

### Argument는 순서대로 전달됨

```jsx
function greeting(name = "Codeit", interest) {
  console.log(`Hi! My name is ${name}!`);
  console.log(`I like ${interest}!`);
}

greeting("JavaScript");
// Hi! My name is JavaScript!
// I like undefined!
```

첫 번째 argument가 `name`에 전달되고, 두 번째는 생략되어 `interest`는 `undefined`입니다.

### 기본값이 있는 Parameter는 오른쪽에!

```jsx
function greeting(interest, name = "Codeit") {
  // 순서 바꿈
  console.log(`Hi! My name is ${name}!`);
  console.log(`I like ${interest}!`);
}

greeting("JavaScript");
// Hi! My name is Codeit!
// I like JavaScript!
```

**권장 사항**: 기본값이 있는 parameter는 **오른쪽**에 배치하세요!

### 중간 Parameter의 기본값 사용하기

```jsx
function greeting(
  name = "Codeit",
  interest = "JavaScript"
) {
  console.log(`Hi! my name is ${name}!`);
  console.log(`I like ${interest}!`);
}

greeting(undefined, "Python");
// Hi! my name is Codeit!
// I like Python!

```

`undefined`를 명시적으로 전달하면 기본값이 사용됩니다!

### Parameter 기본값에 표현식 사용

### 앞의 Parameter 활용

```jsx
function defaultTest(x, y = x + 3) {
  console.log(`x: ${x}`);
  console.log(`y: ${y}`);
}

defaultTest(2);
// x: 2
// y: 5

defaultTest(2, 10);
// x: 2
// y: 10

```

앞쪽에 정의된 parameter를 기본값에 사용할 수 있습니다!

### 함수 호출도 가능

```jsx
function getDefault() {
  return 100;
}

function calculate(x, y = getDefault()) {
  console.log(x + y);
}

calculate(50); // 150
calculate(50, 25); // 75

```

### 정리

**Parameter vs Argument:**

- Parameter: 함수 선언 시 소괄호 안의 변수
- Argument: 함수 호출 시 전달하는 값

**기본값 설정:**

- 문법: `function func(param = defaultValue) {}`
- Argument가 없거나 `undefined`일 때 사용됨
- 기본값에는 앞의 parameter나 함수 호출도 가능

**베스트 프랙티스:**

- 기본값이 있는 parameter는 오른쪽에 배치
- 중간 parameter의 기본값을 사용하려면 `undefined` 전달

ES2015 이전에는 기본값 문법이 없었으므로, 구형 환경에서는 사용할 수 없습니다!

### 07. #Quiz Parameter

### 문제 1

다음 코드의 실행 결과를 예측하세요.

```jsx
function createMessage(name = "Guest", greeting = `Hello, ${name}!`) {
  return greeting;
}

console.log(createMessage());
console.log(createMessage("김코딩"));
console.log(createMessage("이개발", "Welcome!"));

```

- 정답 및 해설
    
    정답:
    
    ```
    Hello, Guest!
    Hello, 김코딩!
    Welcome!
    
    ```
    
    해설:
    
    `createMessage()`는 `name`과 `greeting` 모두 기본값을 사용해서 `Hello, Guest!`를 반환한다.
    
    `createMessage("김코딩")`은 `name`이 `"김코딩"`으로 전달되고, 이 값을 이용해 기본값 표현식으로 `greeting`이 `Hello, 김코딩!`이 되어 반환된다.
    
    `createMessage("이개발", "Welcome!")`은 두 Argument가 모두 전달되어 기본값이 사용되지 않고, `greeting`에 전달된 `"Welcome!"`이 그대로 반환된다.
    
    기본값 표현식에서 이전 Parameter를 참조할 수 있으며, 기본값은 Argument가 전달되지 않았거나 `undefined`일 때만 평가된다.

### 문제 2

다음 함수의 문제점을 찾고, 개선된 코드를 작성하세요.

```jsx
function calculateTotal(price = 10000, quantity, discount = 0) {
  return price * quantity * (1 - discount);
}

// 사용 예시
const total = calculateTotal(5); // quantity에 5를 전달하고 싶음
```

- 정답 및 해설
    
    정답:
    
    ```jsx
    function calculateTotal(quantity, price = 10000, discount = 0) {
      return price * quantity * (1 - discount);
    }
    
    // 사용 예시
    const total1 = calculateTotal(5); // 50000 (5 * 10000 * 1)
    const total2 = calculateTotal(3, 20000); // 60000 (3 * 20000 * 1)
    const total3 = calculateTotal(2, 15000, 0.1); // 27000 (2 * 15000 * 0.9)
    
    ```
    
    해설:
    
    원래 코드에서는 기본값이 없는 `quantity`가 중간에 위치해 있다.
    
    `calculateTotal(5)`를 호출하면 `5`는 첫 번째 Parameter인 `price`에 전달되고, `quantity`는 `undefined`가 되어 곱셈 과정에서 `NaN`이 된다.
    
    **개선된 코드에서는 필수 Parameter인 `quantity`를 가장 앞에 두고, 기본값이 있는 `price`와 `discount`를 뒤에 배치했다.**
    
    이렇게 하면 `calculateTotal(5)`처럼 필수 값만 전달해서 호출할 수도 있고, 필요하다면 `price`, `discount`를 선택적으로 덮어쓸 수 있다.
    
    핵심은 기본값이 있는 Parameter를 뒤쪽에 배치해야 함수 사용이 직관적이고 안전해진다는 점이다.

### 문제 3

다음 코드의 실행 결과를 예측하고, 그 이유를 설명하세요.

```jsx
function test(a, b = a * 2, c = b + 1) {
  console.log(a, b, c);
}

test(5);
test(5, undefined);
test(5, null);
test(5, 10);

```

- 정답 및 해설
    
    정답:
    
    ```
    5 10 11
    5 10 11
    5 null NaN
    5 10 11
    
    ```
    
    해설:
    
    `test(5)`에서는 `a = 5`, `b`는 기본값인 `a * 2`가 적용되어 `10`, `c`는 기본값 `b + 1`이 적용되어 `11`이 되어 `5 10 11`이 출력된다.
    
    `test(5, undefined)`에서는 `b`에 `undefined`가 전달되지만, Parameter 기본값 규칙에 따라 `undefined`가 들어오면 기본값이 적용되어 `b = 10`이 된다. 이어서 `c = b + 1`이 되어 `11`이므로 `5 10 11`이 출력된다.
    
    `test(5, null)`에서는 `b`에 `null`이 전달되며, `null`은 기본값을 트리거하지 않는다. 따라서 `b = null` 그대로 사용되고, `c = b + 1`은 `null + 1`이 되어 `NaN`이 되므로 `5 null NaN`이 출력된다.
    
    `test(5, 10)`에서는 `b`에 `10`이 전달되고, `c`는 기본값 표현식 `b + 1`에 의해 `11`이 되어 `5 10 11`이 출력된다.
    
    정리하면, `undefined`일 때만 기본값이 적용되고, `null`은 실제 값으로 취급된다는 것, 그리고 기본값 표현식에서 앞에 온 Parameter를 참조할 수 있다는 점이 핵심이다.

### 09. Arguments ⭐⭐⭐

### Arguments

함수 내부에서 사용할 수 있는 특별한 객체인 `arguments`를 살펴보겠습니다.

### 문제 상황

```jsx
function printArguments(a, b, c) {
  console.log(a);
  console.log(b);
  console.log(c);
}

printArguments("Young", "Mark", "Koby"); // 정상 동작
printArguments("Captain"); // b, c는 undefined
printArguments("S", "J", "J", "N"); // 4번째는 무시됨
```

Parameter 개수와 Argument 개수가 다르면 문제가 발생합니다.

### arguments 객체

### 기본 사용법

```jsx
function printArguments() {
  console.log(arguments);
}

printArguments("Young", "Mark", "Koby");
// [Arguments] { '0': 'Young', '1': 'Mark', '2': 'Koby' }

printArguments("Captain");
// [Arguments] { '0': 'Captain' }

printArguments("S", "J", "J", "N");
// [Arguments] { '0': 'S', '1': 'J', '2': 'J', '3': 'N' }

```

`arguments` 객체는 함수에 전달된 **모든 argument**를 담고 있습니다!

### 유사 배열 (Array-like)

```jsx
function printArguments() {
  console.log(arguments.length); // length 프로퍼티 있음
  console.log(arguments[0]); // 인덱싱 가능
}

printArguments("A", "B", "C");
// 3
// A
```

배열처럼 생겼지만 **유사 배열**입니다:

- `length` 프로퍼티 있음
- 인덱싱 가능
- 배열 메소드 사용 불가

### for...of 문 사용 가능

```jsx
function printArguments() {
  for (const arg of arguments) {
    console.log(arg);
  }
}

printArguments("Young", "Mark", "Koby");
// Young
// Mark
// Koby
```

`for...of` 문으로 반복 작업이 가능합니다!

### 유연한 함수 만들기

```jsx
function printArguments() {
  for (const arg of arguments) {
    console.log(arg);
  }
  console.log("---------------");
}

printArguments("Young", "Mark", "Koby");
printArguments("Captain");
printArguments("Jayden", "Scott");
printArguments("Suri", "Jack", "Joy", "Noel");
```

Argument 개수에 관계없이 모두 출력할 수 있습니다!

### 주의사항

### arguments는 예약어

```jsx
// ❌ 이렇게 사용하면 안 됨!
function test(arguments) {
  // Parameter 이름으로 사용 금지
  // ...
}

function test() {
  let arguments; // 변수 이름으로 사용 금지
  function arguments() {} // 함수 이름으로 사용 금지
}
```

`arguments`는 함수 내부에서 자동으로 생성되는 특별한 객체이므로, 다른 용도로 사용하면 안 됩니다!

### 한계점

### 1. 배열 메소드 사용 불가

```jsx
function test() {
  console.log(arguments.push("New")); // ❌ TypeError
}
```

유사 배열이므로 `push`, `pop`, `splice` 등의 배열 메소드를 사용할 수 없습니다.

### **2. Arrow Function에서는 사용 불가**

```jsx
// 일반 함수: arguments 사용 가능
function normalFunc() {
  console.log(arguments);
}

normalFunc("A", "B", "C"); // [Arguments] { '0': 'A', '1': 'B', '2': 'C' }

// Arrow Function: arguments 사용 불가
const arrowFunc = () => {
  console.log(arguments); // ❌ ReferenceError: arguments is not defined
};

arrowFunc("A", "B", "C");
```

Arrow Function에서는 `arguments` 객체가 제공되지 않습니다. Arrow Function을 사용할 때는 **Rest Parameter**(`...args`)를 사용해야 합니다!

### 정리

**arguments 객체:**

- 함수에 전달된 **모든 argument**를 담고 있음
- **유사 배열**로 배열 메소드 사용 불가
- `length`, 인덱싱, `for...of` 사용 가능
- Argument 개수가 유동적인 함수를 만들 수 있음

**한계:**

- 배열 메소드 사용 불가
- 항상 전체 argument만 다룸
- 일부만 묶어서 다루기 어려움
- **Rest Parameter**가 이런 한계를 해결해줌!

### 문제 2

다음 함수의 문제점을 찾고, 왜 에러가 발생하는지 설명하세요.

```jsx
function sumAll() {
  let total = 0;
  for (let i = 0; i < arguments.length; i++) {
    total += arguments[i];
  }
  return arguments.reverse(); // 에러 발생!
}

console.log(sumAll(1, 2, 3, 4, 5));
```

- 정답 및 해설
    
    정답:
    
    ```
    TypeError: arguments.reverse is not a function
    ```
    
    해설:
    
    `arguments`는 배열처럼 보이지만 실제 배열이 아닌 **유사 배열 객체**이기 때문에 `reverse`, `push`, `map` 같은 배열 메소드를 사용할 수 없다.
    
    문제를 해결하려면 배열 메소드를 사용하지 않거나, `arguments`를 실제 배열로 변환해야 한다.
    
    예시 1 — 합계만 반환:
    
    ```jsx
    function sumAll() {
      let total = 0;
      for (let i = 0; i < arguments.length; i++) {
        total += arguments[i];
      }
      return total;
    }
    ```
    
    예시 2 — 배열 메소드 사용을 위해 변환:
    
    ```jsx
    function sumAll() {
      const arr = Array.from(arguments);
      return arr.reduce((sum, num) => sum + num, 0);
    }
    ```
    
    핵심은 `arguments`는 유사 배열이며 배열 메소드를 직접 사용할 수 없다는 점이다.

### 문제 3

다음 코드의 실행 결과를 예측하고, 그 이유를 설명하세요.

```jsx
function outer() {
  function inner() {
    console.log(arguments[0]);
  }

  inner("Inner");
  console.log(arguments[0]);

outer("Outer");
```

    ```
    Outer
    ```

### 11. Rest Parameter ⭐⭐⭐

### Rest Parameter

`arguments` 객체의 한계를 극복한 **Rest Parameter**를 살펴보겠습니다.

### arguments 객체의 한계 복습

```jsx
function printArguments() {
  for (const arg of arguments) {
    console.log(arg);
  }
}

printArguments("Young", "Mark", "Koby");
```

**한계점:**

1. 유사 배열이라 배열 메소드 사용 불가
2. 항상 전체 argument만 다룸

### Rest Parameter

### 기본 문법

```jsx
function printArguments(...args) {
  for (const arg of args) {
    console.log(arg);
  }
}

printArguments("Young", "Mark", "Koby");
```

Parameter 앞에 `...`(마침표 3개)를 붙이면 **Rest Parameter**가 됩니다!

### arguments vs Rest Parameter

| 특징 | arguments | Rest Parameter |
| --- | --- | --- |
| 타입 | 유사 배열 | **진짜 배열** |
| 배열 메소드 | ❌ 불가 | ✅ 가능 |
| 부분 선택 | ❌ 어려움 | ✅ 쉬움 |

### Rest Parameter의 장점

### 1. 배열 메소드 사용 가능

```jsx
function printArguments(...args) {
  console.log(args.splice(0, 2));
}

printArguments("Young", "Mark", "Koby");
// ["Young", "Mark"]
```

진짜 배열이므로 `splice`, `map`, `filter` 등 모든 배열 메소드를 사용할 수 있습니다!

**arguments로 시도하면?**

```jsx
function printArguments() {
  console.log(arguments.splice(0, 2)); // ❌ TypeError!
}
```

### 2. 일반 Parameter와 함께 사용

```jsx
function printRank(first, second, ...others) {
  console.log("코드잇 레이스 최종 결과");
  console.log(`우승: ${first}`);
  console.log(`준우승: ${second}`);
  for (const other of others) {
    console.log(`참가자: ${other}`);
  }
}

printRank("Phil", "Won", "Emma", "Min", "Luke");
// 코드잇 레이스 최종 결과
// 우승: Phil
// 준우승: Won
// 참가자: Emma
// 참가자: Min
// 참가자: Luke
```

**동작 원리:**

1. 첫 번째 argument → `first`
2. 두 번째 argument → `second`
3. **나머지** argument → `others` 배열

이름처럼 **나머지(rest)**를 묶어줍니다!

### 3. 명확한 구분

```jsx
function introduce(name, age, ...hobbies) {
  console.log(`이름: ${name}`);
  console.log(`나이: ${age}`);
  console.log(`취미: ${hobbies.join(", ")}`);
}

introduce("철수", 25, "독서", "운동", "코딩");
// 이름: 철수
// 나이: 25
// 취미: 독서, 운동, 코딩
```

필수 정보는 일반 parameter로, 유동적인 정보는 Rest Parameter로 받을 수 있습니다!

### 4. Destructuring과 함께 사용

```jsx
function processData(name, ...scores) {
  const [first, second, ...rest] = scores;
  console.log(`${name}의 최고 점수: ${first}`);
  console.log(`${name}의 두번째 점수: ${second}`);
  console.log(`나머지 점수들:`, rest);
}

processData("철수", 95, 88, 92, 85, 90);
// 철수의 최고 점수: 95
// 철수의 두번째 점수: 88
// 나머지 점수들: [92, 85, 90]
```

Rest Parameter는 **진짜 배열**이므로 destructuring(구조 분해)을 바로 적용할 수 있습니다!

```jsx
function calculateStats(...numbers) {
  const [min, ...middle] = numbers.sort((a, b) => a - b);
  const max = middle.pop();

  console.log(`최솟값: ${min}`);
  console.log(`최댓값: ${max}`);
  console.log(`중간값들:`, middle);
}

calculateStats(45, 23, 78, 12, 67);
// 최솟값: 12
// 최댓값: 78
// 중간값들: [23, 45, 67]
```

배열이기 때문에 destructuring, 메소드 체이닝 등 다양한 배열 기능을 자유롭게 활용할 수 있습니다!

### 주의사항

### 1. 반드시 마지막에 위치

```jsx
// ✅ 올바름
function test(a, b, ...rest) {}

// ❌ 에러
function test(...rest, a, b) {}

// ❌ 에러
function test(a, ...rest, b) {}
```

Rest Parameter는 **나머지**를 모으는 역할이므로 반드시 **마지막**에 와야 합니다!

### 2. 하나만 사용 가능

```jsx
// ❌ 에러
function test(...rest1, ...rest2) {}
```

### Spread vs Rest: 문맥에 따라 다른 역할

같은 `...` 기호를 사용하지만 **사용 위치**에 따라 역할이 다릅니다:

### Rest Parameter: 함수 매개변수에서만

```jsx
// ✅ Rest Parameter: 함수 parameter 위치
function sum(...numbers) {
  // 여러 값을 배열로 묶기
  return numbers.reduce((a, b) => a + b);
}

sum(1, 2, 3, 4); // 10
```

**Rest는 함수 parameter에서만** 사용됩니다. 여러 argument를 **하나의 배열로 묶습니다**.

### Spread 구문: 나머지 모든 상황

```jsx
// ✅ Spread: 배열/객체를 펼치기
const arr = [1, 2, 3];
console.log(...arr); // 1 2 3 (함수 호출 시)

const newArr = [...arr, 4, 5]; // [1, 2, 3, 4, 5] (배열 리터럴)

const obj = { name: "철수", age: 25 };
const newObj = { ...obj, city: "서울" }; // 객체 리터럴

// ✅ Spread: 배열 destructuring
const [first, ...rest] = [1, 2, 3, 4];
console.log(first); // 1
console.log(rest); // [2, 3, 4]
```

**Spread는 함수 parameter 외의 모든 곳**에서 사용됩니다:

- 함수 호출 시: `func(...arr)`
- 배열 리터럴: `[...arr]`
- 객체 리터럴: `{...obj}`
- Destructuring: `const [a, ...rest] = arr`

### 정리: 문맥으로 구분

```jsx
// Rest Parameter (함수 parameter에서)
function test(...args) {} // Rest

// Spread 구문 (나머지 모든 곳)
test(...array); // Spread (함수 호출)
const copy = [...array]; // Spread (배열 리터럴)
const [x, ...y] = array; // Spread (destructuring)
```

- **Rest**: 묶기 (여러 개 → 하나) - **함수 parameter에서만**
- **Spread**: 펼치기 (하나 → 여러 개) - **나머지 모든 상황**

### arguments vs Rest Parameter: 언제 사용할까?

### Rest Parameter 권장

```jsx
// ✅ 권장: Rest Parameter
const sum = (...numbers) => {
  return numbers.reduce((a, b) => a + b);
};
```

**장점:**

- 진짜 배열
- 명확한 의도
- Arrow Function에서도 사용 가능

### arguments는?

```jsx
// Arrow Function에서는 arguments가 아예 제공되지 않음
const test = () => {
  console.log(arguments); // ❌ ReferenceError: arguments is not defined
};

test(1, 2, 3);
```

Arrow Function에서는 `arguments` 객체가 **아예 제공되지 않습니다**. 일반 함수와 달리 Arrow Function은 자체적인 `arguments`를 생성하지 않으므로, 접근하려고 하면 `ReferenceError`가 발생합니다.

ES6 이후로는 **Rest Parameter가 표준**입니다. 대부분의 스타일 가이드에서 Rest Parameter를 권장합니다!

### 정리

**Rest Parameter:**

- 문법: `function func(...rest) {}`
- **진짜 배열**: 모든 배열 메소드 사용 가능
- 일반 parameter와 함께 사용 가능
- **반드시 마지막에 위치**
- 나머지 argument를 배열로 묶어줌

**베스트 프랙티스:**

- arguments 대신 Rest Parameter 사용
- 명확한 parameter 이름 지정
- 일반 parameter와 Rest Parameter 조합으로 유연한 함수 작성

### 문제 2

다음 함수 중 **문법 에러**가 발생하는 것을 모두 찾고, 그 이유를 설명하세요.

```jsx
// A
function funcA(...rest, a, b) {
  console.log(rest, a, b);
}

// B
function funcB(a, ...rest, b) {
  console.log(a, rest, b);
}

// C
function funcC(a, b, ...rest) {
  console.log(a, b, rest);
}

// D
function funcD(...rest1, ...rest2) {
  console.log(rest1, rest2);
}

// E
function funcE(a, b, ...rest, c) {
  console.log(a, b, rest, c);
}
```

- 정답 및 해설
    
    정답:
    
    에러 발생: **A, B, D, E**
    
    정상 동작: **C**
    
    해설:
    
    Rest Parameter에는 두 가지 중요한 규칙이 있다.
    
    1. **항상 마지막에 위치해야 한다.**
    2. **하나만 사용할 수 있다.**
    
    각 함수별로 보면:
    
    - **A**: `function funcA(...rest, a, b)`
        
        Rest Parameter가 가장 앞에 있고 뒤에 일반 parameter가 이어진다.
        
        “나머지”를 모으는 Rest가 마지막이 아니므로 ❌
        
    - **B**: `function funcB(a, ...rest, b)`
        
        Rest Parameter가 중간에 위치하고 뒤에 `b`가 있다.
        
        `rest`가 어디까지를 포함해야 하는지 모호해지므로 ❌
        
    - **C**: `function funcC(a, b, ...rest)`
        
        일반 parameter 뒤에 Rest Parameter가 마지막으로 온다. 규칙을 잘 지키고 있으므로 ✅
        
    - **D**: `function funcD(...rest1, ...rest2)`
        
        Rest Parameter를 두 개 사용했다. Rest는 하나만 사용할 수 있으므로 ❌
        
    - **E**: `function funcE(a, b, ...rest, c)`
        
        Rest 뒤에 `c`가 와서 마지막이 아니다. Rest는 반드시 마지막에 위치해야 하므로 ❌
        
    
    따라서 올바른 패턴은 다음과 같은 형태들이다.
    
    ```jsx
    function correct1(...rest) {}
    function correct2(a, ...rest) {}
    function correct3(a, b, c, ...rest) {}
    ```

### 문제 3

다음 코드를 실행하면 어떤 결과가 나올까요? 그리고 `arguments`와 Rest Parameter의 차이를 설명하세요.

```jsx
function sumWithArguments() {
  let total = 0;
  for (let i = 0; i < arguments.length; i++) {
    total += arguments[i];
  }
  return total;
}

function sumWithRest(...numbers) {
  return numbers.reduce((sum, num) => sum + num, 0);
}

console.log(sumWithArguments(1, 2, 3, 4, 5));
console.log(sumWithRest(1, 2, 3, 4, 5));

// Arrow Function으로 바꾸면?
const sumArrow = (...numbers) => {
  return numbers.reduce((sum, num) => sum + num, 0);
};

console.log(sumArrow(1, 2, 3, 4, 5));

const sumArrowArguments = () => {
  return Array.from(arguments).reduce((sum, num) => sum + num, 0);
};

console.log(sumArrowArguments(1, 2, 3, 4, 5)); // 에러?
```

- 정답 및 해설
    
    정답:
    
    ```
    15
    15
    15
    ReferenceError: arguments is not defined
    ```
    
    해설:
    
    앞의 세 줄은 모두 `1 + 2 + 3 + 4 + 5 = 15`를 잘 계산해서 `15`를 출력한다.
    
    - `sumWithArguments(1, 2, 3, 4, 5)`는 일반 함수에서 `arguments`를 사용해 합을 구한다.
    - `sumWithRest(1, 2, 3, 4, 5)`는 Rest Parameter `...numbers`를 배열로 받아 `reduce`로 합을 구한다.
    - `sumArrow(1, 2, 3, 4, 5)`도 Rest Parameter를 사용하므로 동일하게 동작한다.
    
    마지막 `sumArrowArguments(1, 2, 3, 4, 5)`에서 에러가 발생하는 이유는 **Arrow Function에는 자체적인 `arguments` 객체가 없기 때문**이다.
    
    Arrow Function 안에서 `arguments`를 참조하면 `ReferenceError: arguments is not defined`가 발생한다.
    
    정리하면 다음과 같다.
    
    **arguments vs Rest Parameter**
    
    | 특징 | arguments | Rest Parameter |
    | --- | --- | --- |
    | 타입 | 유사 배열 (Array-like) | 진짜 배열 (Array) |
    | 배열 메소드 | 직접 사용 불가 | 자유롭게 사용 가능 |
    | Arrow Function 지원 | ❌ 없음 | ✅ 있음 |
    | 일반 parameter와 조합 | ❌ 안 됨 | ✅ 됨 |
    | 도입 시기 | ES5 이전부터 | ES6(ES2015)부터 |
    
    실무와 현대 JavaScript 스타일 가이드에서는 대부분 `arguments` 대신 **Rest Parameter** 사용을 권장한다.
    
    ```jsx
    // 권장되는 패턴
    const sum = (...numbers) =>
      numbers.reduce((sum, num) => sum + num, 0);
    ```

### 13. Arrow Function ⭐⭐⭐

ES2015에서 등장한 **Arrow Function(화살표 함수)**는 함수를 더 간결하게 작성할 수 있는 새로운 문법입니다.

### 기본 문법

### 1. 기존 함수 표현식

```jsx
const double = function (number) {
  return number * 2;
};

console.log(double(5)); // 10
```

### 2. Arrow Function으로 변환

```jsx
const double = (number) => {
  return number * 2;
};

console.log(double(5)); // 10
```

변환 방법:

1. `function` 키워드 제거
2. 파라미터 뒤에 `=>` (화살표) 추가

### Arrow Function 단축 문법

Arrow Function은 상황에 따라 더 간결하게 작성할 수 있습니다.

### 1. 파라미터가 하나일 때: 소괄호 생략 가능

```jsx
// 소괄호 O
const double = (number) => {
  return number * 2;
};

// 소괄호 X (파라미터가 1개일 때만 가능)
const double = (number) => {
  return number * 2;
};
```

**주의:** 파라미터가 0개 또는 2개 이상이면 소괄호 필수!

```jsx
// 파라미터 0개
const greet = () => {
  console.log("Hello!");
};

// 파라미터 2개 이상
const add = (a, b) => {
  return a + b;
};
```

### 2. 함수 본문이 return문만 있을 때: 중괄호와 return 생략 가능

```jsx
// 일반 형태
const double = (number) => {
  return number * 2;
};

// 축약 형태
const double = (number) => number * 2;

console.log(double(7)); // 14
```

훨씬 간결해졌습니다!

### 3. 최종 축약 형태

```jsx
// 기존 함수 표현식
const add = function (a, b) {
  return a + b;
};

// Arrow Function (일반)
const add = (a, b) => {
  return a + b;
};

// Arrow Function (축약)
const add = (a, b) => a + b;

console.log(add(3, 5)); // 8
```

**비교:**

- 기존: `function(a, b) { return a + b; }`
- 축약: `(a, b) => a + b`

### 축약 시 주의사항

### return 외에 다른 코드가 있으면 축약 불가

```jsx
// ❌ 축약 불가 (변수 선언이 있음)
const add = (a, b) => {
  const result = a + b;
  return result;
};

// ✅ 축약 가능 (return만 있음)
const add = (a, b) => a + b;

```

### 객체를 리턴할 때는 소괄호로 감싸기

```jsx
// ❌ 에러 발생
const getPerson = () => { name: "철수", age: 25 };

// ✅ 올바른 방법: 소괄호로 감싸기
const getPerson = () => ({ name: "철수", age: 25 });

console.log(getPerson());  // { name: "철수", age: 25 }

```

**이유:** 중괄호 `{}`를 객체가 아닌 함수의 코드 블록으로 인식하기 때문입니다.

### 실전 예제

### 배열 메소드와 함께 사용

```jsx
const numbers = [1, 2, 3, 4, 5];

// 기존 방식
const doubled = numbers.map(function (num) {
  return num * 2;
});

// Arrow Function
const doubled = numbers.map((num) => num * 2);

console.log(doubled); // [2, 4, 6, 8, 10]

```

Arrow Function을 사용하면 코드가 훨씬 간결해집니다!

### 이벤트 리스너

```jsx
// 기존 방식
button.addEventListener("click", function () {
  console.log("버튼 클릭!");
});

// Arrow Function
button.addEventListener("click", () => {
  console.log("버튼 클릭!");
});
```

### Arrow Function의 제약사항

### 1. arguments 객체가 없음

```jsx
// 일반 함수: arguments 사용 가능
function normalFunc() {
  console.log(arguments);
}

normalFunc(1, 2, 3); // [1, 2, 3]

// Arrow Function: arguments 사용 불가
const arrowFunc = () => {
  console.log(arguments); // ❌ ReferenceError
};

arrowFunc(1, 2, 3);
```

**해결방법:** Rest Parameter 사용

```jsx
const arrowFunc = (...args) => {
  console.log(args);
};

arrowFunc(1, 2, 3); // [1, 2, 3]
```

### 2. this 바인딩 차이

Arrow Function은 일반 함수와 `this` 동작이 다릅니다. (다음 강의에서 자세히 다룸)

### 언제 사용할까?

### Arrow Function을 사용하면 좋을 때

- 짧고 간단한 함수
- 콜백 함수 (배열 메소드, 이벤트 리스너 등)
- 코드를 간결하게 만들고 싶을 때

### 일반 함수를 사용해야 할 때

- `arguments` 객체가 필요할 때
- `this` 바인딩이 중요할 때 (메소드 등)
- 함수 선언 호이스팅이 필요할 때

### 정리

| 특징 | 일반 함수 | Arrow Function |
| --- | --- | --- |
| 문법 | `function() {}` | `() => {}` |
| 간결성 | 보통 | 매우 간결 |
| `arguments` | ✅ 있음 | ❌ 없음 (Rest Parameter 사용) |
| `this` | 동적 바인딩 | 렉시컬 바인딩 (다음 강의) |
| 축약 가능 | ❌ | ✅ |

Arrow Function은 처음에는 어색하지만, 익숙해지면 코드를 훨씬 간결하고 읽기 쉽게 만들어줍니다. 모던 JavaScript에서는 필수적인 문법이므로 꼭 익혀두세요!

### 15. What is this?

JavaScript에서 `this` 키워드는 특별한 동작을 합니다. 함수형 프로그래밍과 객체 지향 프로그래밍의 핵심 개념입니다.

### this의 기본값

```jsx
console.log(this); // Window {...}, global {...}
```

전역에서 `this`는 **전역 객체**(브라우저에서는 `window`, node에서는 `global`)를 가리킵니다.

### 객체 메소드에서의 this

### 문제 상황

```jsx
const user = {
  firstName: "Tess",
  lastName: "Jang",
  getFullName: function () {
    return `${user.firstName} ${user.lastName}`;
  },
};

console.log(user.getFullName()); // Tess Jang
```

메소드가 `user` 객체에 직접 의존하고 있습니다. 다른 객체에서 재사용하면?

```jsx
function getFullName() {
  return `${user.firstName} ${user.lastName}`;
}

const user = {
  firstName: "Tess",
  lastName: "Jang",
  getFullName: getFullName,
};

const admin = {
  firstName: "Alex",
  lastName: "Kim",
  getFullName: getFullName,
};

console.log(user.getFullName()); // Tess Jang
console.log(admin.getFullName()); // Tess Jang  (잘못됨!)
```

`admin`에서 호출해도 `user`의 데이터를 사용합니다!

### this로 해결

```jsx
function getFullName() {
  return `${this.firstName} ${this.lastName}`;
}

const user = {
  firstName: "Tess",
  lastName: "Jang",
  getFullName: getFullName,
};

const admin = {
  firstName: "Alex",
  lastName: "Kim",
  getFullName: getFullName,
};

console.log(user.getFullName()); // Tess Jang
console.log(admin.getFullName()); // Alex Kim  (올바름!)

```

**핵심:** `this`는 **함수를 호출한 객체**를 가리킵니다!

### this의 값이 결정되는 방식

```jsx
console.log(this); // Window

function printThis() {
  console.log(this);
}

printThis(); // Window
```

1. **전역**에서: `window` 객체
2. **일반 함수** 호출: `window` 객체

```jsx
const myObj = {
  content: "myObj",
  printThis: printThis,
};

const otherObj = {
  content: "otherObj",
  printThis: printThis,
};

myObj.printThis(); // myObj 객체
otherObj.printThis(); // otherObj 객체
```

1. **객체 메소드**로 호출: 호출한 객체

**중요:** `this`의 값은 **코드 작성 시**가 아니라 **함수 호출 시**에 결정됩니다!

### Arrow Function과 this

### 일반 함수 vs Arrow Function

```jsx
// 일반 함수
function printThis() {
  console.log(this);
}

const myObj = {
  printThis: printThis,
};

myObj.printThis(); // myObj 객체
```

```jsx
// Arrow Function
const printThis = () => {
  console.log(this);
};

const myObj = {
  printThis: printThis,
};

myObj.printThis(); // Window 객체!
```

### Arrow Function의 this는 고정됨

**일반 함수:**

- `this`는 **호출 시** 결정됨
- 호출한 객체를 가리킴

**Arrow Function:**

- `this`는 **선언 시** 결정됨
- 선언 당시의 `this` 값을 그대로 유지

```jsx
const printThis = () => {
  console.log(this);
};

const myObj = {
  printThis: printThis,
};

const otherObj = {
  printThis: printThis,
};

myObj.printThis(); // Window
otherObj.printThis(); // Window
```

Arrow Function이 선언될 때 `this`는 `window`였으므로, 어디서 호출하든 `window`를 가리킵니다!

### 언제 무엇을 사용할까?

### 객체 메소드: 일반 함수 권장

```jsx
const user = {
  name: "철수",
  greet: function () {
    // ✅ 일반 함수
    console.log(`안녕, 나는 ${this.name}야!`);
  },
};

user.greet(); // 안녕, 나는 철수야!
```

```jsx
const user = {
  name: "철수",
  greet: () => {
    // ❌ Arrow Function
    console.log(`안녕, 나는 ${this.name}야!`);
  },
};

user.greet(); // 안녕, 나는 undefined야!
```

### 콜백 함수: Arrow Function 권장

```jsx
const user = {
  name: "철수",
  friends: ["영희", "민수"],
  printFriends: function () {
    this.friends.forEach((friend) => {
      // ✅ Arrow Function
      console.log(`${this.name}의 친구: ${friend}`);
    });
  },
};

user.printFriends();
// 철수의 친구: 영희
// 철수의 친구: 민수

```

Arrow Function은 외부의 `this`를 그대로 사용하므로 콜백 함수에 적합합니다!

### 정리

**this란?**

- 함수를 호출한 객체를 가리키는 키워드
- 값은 **호출 시점**에 결정됨

**일반 함수의 this:**

- 호출한 객체에 따라 동적으로 변함
- 메소드에 적합

**Arrow Function의 this:**

- 선언 시점의 this를 그대로 유지
- 콜백 함수에 적합

**베스트 프랙티스:**

- 객체 메소드: 일반 함수 사용
- 콜백 함수: Arrow Function 사용
- `this`가 필요한 곳에서는 Arrow Function 주의!

`this`는 JavaScript의 핵심 개념이므로 확실히 이해하고 넘어가세요!

### 02. 삼항 연산자 (조건 연산자) ⭐⭐

기존에 조건을 다룰 때는 `if`문과 `switch`문을 사용했습니다. 하지만 모던 자바스크립트에서는 **삼항 연산자(조건 연산자)**를 사용해 표현식으로도 조건을 다룰 수 있습니다. 

### 기본 구조

```jsx
조건 ? truthy일 때 표현식 : falsy일 때 표현식
```

- **물음표(`?`)**: 조건이 truthy일 때 실행할 표현식과 구분
- **콜론(`:`)**: falsy일 때 실행할 표현식과 구분
- **삼항 연산자**: 연산 기호가 2개, 항이 3개라서 붙은 이름 (조건 연산자라고도 부름)

### 실제 사용 예시

### 합격 여부 판단

```jsx
const PASSING_SCORE = 70;

function checkExam(score) {
  return score >= PASSING_SCORE
    ? "합격입니다!"
    : "불합격입니다.";
}

console.log(checkExam(85)); // 합격입니다!
console.log(checkExam(60)); // 불합격입니다.
```

### if문과 비교

같은 기능을 if문으로 작성하면:

```jsx
function checkExam(score) {
  if (score >= PASSING_SCORE) {
    return "합격입니다!";
  } else {
    return "불합격입니다!";
  }
}
```

조건 연산자가 훨씬 간결하죠?

### 삼항 연산자의 장점

### 1. 간결한 코드

```jsx
const age = 25;
const category = age >= 20 ? "성인" : "미성년자";
console.log(category); // 성인
```

### 2. 변수 할당에 유용

```jsx
const temperature = 15;
const weather =
  temperature >= 25
    ? "더워요"
    : temperature >= 10
    ? "적당해요"
    : "추워요";
console.log(weather); // 적당해요
```

### 조건 연산자의 한계

조건 연산자는 **표현식**이기 때문에 다음과 같은 상황에서는 적합하지 않습니다:

### 1. 조건에 따라 다른 로직을 실행해야 하는 경우

```jsx
const score = 90;

// ✅ 조건 연산자: 값을 결정하는 데 적합
const message = score >= 70 ? "합격" : "불합격";
console.log(message);

// ❌ 조건 연산자 내부에 변수 선언은 불가능 (SyntaxError 발생)
// score >= 70 ? const msg = '합격' : const msg = '불합격';

// ✅ 조건에 따라 다른 변수를 선언하거나 복잡한 로직이 필요하면 if문 사용
if (score >= 70) {
  const bonus = score - 70;
  console.log(`합격! 보너스 점수: ${bonus}점`);
} else {
  const deficit = 70 - score;
  console.log(`불합격. ${deficit}점 부족합니다.`);
}
```

조건 연산자는 **값(표현식)**만 다룰 수 있습니다. 변수 선언(`const`, `let`)은 표현식이 아니라 문(statement)이므로 조건 연산자 내부에서 사용할 수 없습니다.

### 2. 반복문이 필요한 경우

```jsx
// 불가능한 코드
const num = 5;
num > 0 ? for(let i = 0; i < num; i++) { console.log(i); } : console.log('음수');
// if문을 사용해야 함
if (num > 0) {
  for (let i = 0; i < num; i++) {
    console.log(i);
  }
} else {
  console.log('음수입니다');
}
```

### 언제 사용하면 좋을까?

### 조건 연산자를 사용하기 좋은 경우

- 조건에 따라 **간단한 값을 결정**할 때
- 변수에 조건부로 값을 할당할 때
- JSX나 템플릿 문자열 내에서 조건부 렌더링할 때

### if문을 사용해야 하는 경우

- 조건에 따라 **여러 문장**을 실행해야 할 때
- 변수 선언, 반복문 등 **복잡한 로직**이 필요할 때
- 코드의 **가독성**이 더 중요한 경우

### 정리

- **조건 연산자**(삼항 연산자)는 조건을 표현식으로 다룰 수 있게 해줍니다
- `조건 ? truthy일 때 : falsy일 때` 형태로 사용합니다
- if문보다 간결하지만, 복잡한 로직에는 적합하지 않습니다
- 적절한 상황에 맞게 조건 연산자와 if문을 선택해서 사용하세요

### 문제 1

다음 코드의 실행 결과를 예측하세요.

```jsx
const price = 15000;
const discount =
  price >= 20000 ? 3000 : price >= 10000 ? 1000 : 0;
const finalPrice = price - discount;

console.log(discount);
console.log(finalPrice);
```

- 정답 및 해설
    
    정답:
    
    ```
    1000
    14000
    ```
    
    해설:
    
    이 문제는 **중첩된 삼항 연산자**를 다룹니다.
    
    **단계별 평가:**
    
    1. 첫 번째 조건: `price >= 20000`
        - `15000 >= 20000` → `false`
        - falsy이므로 `:` 뒤로 이동
    2. 두 번째 조건: `price >= 10000`
        - `15000 >= 10000` → `true`
        - truthy이므로 `1000` 반환
    3. `discount = 1000`
    4. `finalPrice = 15000 - 1000 = 14000`
    
    **중첩된 삼항 연산자 읽는 법:**
    
    ```jsx
    조건1 ? 값1 : 조건2 ? 값2 : 값3;
    // 조건1이 true → 값1
    // 조건1이 false, 조건2가 true → 값2
    // 둘 다 false → 값3
    ```
    
    **주의사항:**
    
    중첩된 삼항 연산자는 가독성이 떨어집니다. 2개 이상 중첩되면 `if-else if-else`를 사용하는 것이 더 좋습니다.
    
    ```jsx
    // ❌ 가독성이 나쁨
    const grade =
      score >= 90
        ? "A"
        : score >= 80
        ? "B"
        : score >= 70
        ? "C"
        : "D";
    
    // ✅ 가독성이 좋음
    let grade;
    if (score >= 90) grade = "A";
    else if (score >= 80) grade = "B";
    else if (score >= 70) grade = "C";
    else grade = "D";
    ```

### 문제 2

다음 중 **문법 에러**가 발생하거나 **의도대로 동작하지 않는** 코드를 모두 찾으세요.

```jsx
// A
const age = 25;
const status = age >= 20 ? "성인" : "미성년자";

// B
const score = 85;
score >= 80 ? const result = "Good" : const result = "Bad";

// C
const num = 5;
const value = num > 0 ? num * 2 : num * -1;

// D
const isLoggedIn = true;
isLoggedIn ? console.log("환영합니다!") : console.log("로그인하세요");

// E
const count = 10;
const action = count > 0 ? for (let i = 0; i < count; i++) { console.log(i); } : console.log("No items");
```

- 정답 및 해설
    
    정답:
    
    에러 또는 의도대로 동작하지 않는 코드: **B, E**
    
    정상 동작하는 코드: **A, C, D**
    
    해설:
    
    **A (정상):**
    
    ```jsx
    const status = age >= 20 ? "성인" : "미성년자";
    ```
    
    - 조건에 따라 문자열 값을 변수에 할당하는 표현식
    - 삼항 연산자 사용이 올바르므로 정상 동작합니다.
    
    **B (문법 에러):**
    
    ```jsx
    score >= 80 ? const result = "Good" : const result = "Bad";
    ```
    
    - `const result = "Good"` / `const result = "Bad"`는 **문(statement)**입니다.
    - 삼항 연산자에는 **표현식**만 올 수 있습니다.
    - 따라서 **SyntaxError**가 발생합니다.
    
    올바른 수정:
    
    ```jsx
    const result = score >= 80 ? "Good" : "Bad";
    ```
    
    **C (정상):**
    
    ```jsx
    const value = num > 0 ? num * 2 : num * -1;
    ```
    
    - `num * 2`, `num * -1`는 모두 표현식입니다.
    - 조건에 따라 값을 선택하는 전형적인 삼항 연산자 활용 예시로, 정상 동작합니다.
    
    **D (정상):**
    
    ```jsx
    isLoggedIn
      ? console.log("환영합니다!")
      : console.log("로그인하세요");
    ```
    
    - `console.log(...)`는 함수 호출 표현식입니다.
    - 문법적으로 문제는 없고, 조건에 따라 서로 다른 메시지를 출력합니다.
    - 다만 반환값(`undefined`)을 활용하지 않으므로, 가독성 측면에서는 `if`문이 더 적합할 수 있습니다.
    
    **E (문법 에러):**
    
    ```jsx
    const action = count > 0 ? for (let i = 0; i < count; i++) { console.log(i); } : console.log("No items")
    ```
    
    - `for (...) { ... }`는 **반복문(statement)**입니다.
    - 삼항 연산자는 표현식만 사용할 수 있으므로, 이 코드는 **SyntaxError**가 됩니다.
    
    올바른 수정:
    
    ```jsx
    if (count > 0) {
      for (let i = 0; i < count; i++) {
        console.log(i);
      }
    } else {
      console.log("No items");
    }
    ```
    
    **핵심:**
    
    삼항 연산자에는 **표현식(expression)**만 올 수 있습니다.
    
    - ✅ 가능한 것: 값, 연산, 함수 호출, 또 다른 삼항 연산자 등
    - ❌ 불가능한 것: `const`/`let`/`var` 선언, `if`, `for`, `while` 같은 **문(statement)**

다음 두 코드의 차이점을 설명하고, 각각 언제 사용하면 좋을지 설명하세요.

**코드 A: 삼항 연산자**

```jsx

    ? 10000
    : price >= 50000
    ? 5000
    : price >= 30000
    ? 3000
    : 0;
}
```

        - 코드 A는 **값만 반환하는 표현식 중심**
        - 코드 B는 **로그 출력 등 여러 문장을 실행하는 로직 중심**
    - 단순 값 선택에는 코드 A처럼 삼항 연산자를,
        
        여러 문장과 분기가 필요한 경우에는 코드 B처럼 `if-else`를 쓰는 것이 좋습니다.
        
    
    해설:
    
    **주요 차이점 비교**
    
    | 특징 | 삼항 연산자 (코드 A) | if-else문 (코드 B) |
    | --- | --- | --- |
    | 가독성 | 조건이 많을수록 빠르게 나빠짐 | 각 조건과 동작이 명확하게 보임 |

    - 할인 종류에 따라 콘솔 출력, 이벤트 트래킹, 쿠폰 발급 등이 들어가는 경우
    - 조건이 늘어나고, 나중에 다른 사람이 코드를 읽고 수정할 가능성이 클 때

    - **2단계 정도의 간단한 분기** → 삼항 연산자 사용 가능
    
    ```jsx

    ```
    
    - **3단계 이상 복잡해지거나, 여러 문장이 필요** → `if-else` 권장

### 04. Spread 구문 ⭐⭐⭐

ES2015에서 등장한 **Spread 구문**은 배열이나 객체를 펼쳐서 개별 요소로 만드는 문법입니다. 이름처럼 값들을 "펼친다(spread)"는 의미입니다.

### 기본 개념

### 배열 펼치기

```jsx
const numbers = [1, 2, 3];

console.log(numbers); // [1, 2, 3]
console.log(...numbers); // 1 2 3
```

마침표 3개 `...`를 배열 앞에 붙이면, 배열의 대괄호가 벗겨지고 요소들이 개별 값으로 펼쳐집니다!

```jsx
console.log(...numbers); // 1 2 3
console.log(1, 2, 3); // 1 2 3  (똑같은 결과!)
```

### Rest Parameter와의 차이

```jsx
// Rest Parameter: 여러 값을 하나의 배열로 묶기
const sumAll = (...args) => {
  let sum = 0;
  for (const arg of args) {
    sum += arg;
  }
  return sum;
};

console.log(sumAll(1, 2, 3, 4)); // 10
```

- **Rest Parameter** (`...args`): 여러 개의 값 → 하나의 배열로 **묶기**
- **Spread 구문** (`...array`): 하나의 배열 → 여러 개의 값으로 **펼치기**

같은 `...` 기호를 사용하지만 정반대의 역할입니다!

### Spread 구문의 활용

### 1. 배열 복사

```jsx
const original = ["HTML", "CSS"];
const copy = [...original];

copy.push("JavaScript");

console.log(original); // ["HTML", "CSS"]
console.log(copy); // ["HTML", "CSS", "JavaScript"]
```

### 복사하면서 요소 추가

```jsx
const webPublishing = ["HTML", "CSS"];
const interactiveWeb = [...webPublishing, "JavaScript"];

console.log(webPublishing); // ["HTML", "CSS"]
console.log(interactiveWeb); // ["HTML", "CSS", "JavaScript"]
```

### 2. 배열 합치기

```jsx
const arr1 = [1, 2, 3];
const arr2 = [4, 5, 6];

const combined = [...arr1, ...arr2];
console.log(combined); // [1, 2, 3, 4, 5, 6]
```

### 3. 배열 중간에 요소 추가

```jsx
const start = [1, 2];
const end = [5, 6];

const numbers = [...start, 3, 4, ...end];
console.log(numbers); // [1, 2, 3, 4, 5, 6]

```

### 4. 함수 호출 시 argument로 사용

```jsx
function introduce(name, birth, job) {
  console.log(`안녕하세요! 저는 ${name}입니다.`);
  console.log(`${birth}년생이고,`);
  console.log(`직업은 ${job}입니다.`);
}

const myInfo = ["코드잇", 2017, "프로그래밍 강사"];

introduce(...myInfo);
```

### 5. Math 메소드와 함께 사용

```jsx
const numbers = [23, 15, 67, 42, 8];

const max = Math.max(...numbers);
console.log(max); // 67

const min = Math.min(...numbers);
console.log(min); // 8
```

### 객체에 Spread 구문 사용하기

```jsx
const members = ["태호", "종훈", "우재"];
const obj = { ...members };

console.log(obj); // { 0: "태호", 1: "종훈", 2: "우재" }
```

### 중요한 주의사항

### Spread 구문은 값이 아니다!

```jsx
const numbers = [1, 2, 3];

// ❌ 에러 발생
const num = ...numbers;
```

Spread 구문은 **단독으로 사용할 수 없습니다.**

- 배열 리터럴 내부 — 가능
- 함수 호출 시 — 가능
- 객체 리터럴 내부 — 가능

그러나 단독으로 변수에 할당하는 것은 불가능합니다.

### 정리

| 기능 | 코드 예시 |
| --- | --- |
| 배열 복사 | `const copy = [...original];` |
| 배열 합치기 | `const combined = [...arr1, ...arr2];` |
| 요소 추가 | `const newArr = [...arr, "new"];` |
| 함수 argument | `func(...array);` |
| Math 메소드 | `Math.max(...numbers);` |

**핵심:**

- Spread 구문(`...`)은 배열을 개별 요소로 펼친다
- 배열 복사, 합치기, 함수 전달 등에 매우 유용
- Rest Parameter와 정반대의 역할 (펼치기 vs 묶기)

### 07. 모던한 프로퍼티 표기법 ⭐⭐⭐

ES2015에서는 객체의 프로퍼티를 더 간결하게 표현할 수 있는 방법들이 추가되었습니다.

### 1. 프로퍼티 네임 축약 (Property Shorthand)

변수 이름과 프로퍼티 네임이 같을 때, **하나만 작성**할 수 있습니다.

### 기존 방식

```jsx
const brand = "Samsung";
const model = "Galaxy S24";
const price = 1200000;

const phone = {
  brand: brand,
  model: model,
  price: price,
};

console.log(phone);
// { brand: 'Samsung', model: 'Galaxy S24', price: 1200000 }

```

### 모던한 방식 (축약형)

```jsx
const brand = "Samsung";
const model = "Galaxy S24";
const price = 1200000;

const phone = {
  brand,
  model,
  price,
};

console.log(phone);
// { brand: 'Samsung', model: 'Galaxy S24', price: 1200000 }
```

콜론과 중복된 이름을 생략할 수 있어 코드가 훨씬 간결해집니다!

### 2. 메소드 축약 표기

객체 내부에서 메소드를 선언할 때 `function` 키워드와 콜론을 생략할 수 있습니다.

### 기존 방식

```jsx
const calculator = {
  add: function (a, b) {
    return a + b;
  },
  subtract: function (a, b) {
    return a - b;
  },
};

console.log(calculator.add(5, 3)); // 8
console.log(calculator.subtract(5, 3)); // 2

```

### 모던한 방식 (축약형)

```jsx
const calculator = {
  add(a, b) {
    return a + b;
  },
  subtract(a, b) {
    return a - b;
  },
};

console.log(calculator.add(5, 3)); // 8
console.log(calculator.subtract(5, 3)); // 2
```

`function` 키워드와 콜론(`:`)을 생략하여 더 깔끔하게 작성할 수 있습니다.

### 외부 함수를 메소드로 사용할 때도 축약 가능

```jsx
function introduce() {
  return `안녕하세요, 저는 ${this.name}입니다.`;
}

const student = {
  name: "김코딩",
  introduce, // introduce: introduce 와 같음
};

const teacher = {
  name: "박선생",
  introduce,
};

console.log(student.introduce()); // 안녕하세요, 저는 김코딩입니다.
console.log(teacher.introduce()); // 안녕하세요, 저는 박선생입니다.
```

### 3. 계산된 속성명 (Computed Property Name)

표현식을 대괄호(`[]`)로 감싸서 **동적으로** 프로퍼티 네임을 만들 수 있습니다.

### 기본 예시

```jsx
const school = {
  ["student" + "Count"]: 500,
};

console.log(school); // { studentCount: 500 }
```

### 변수를 프로퍼티 네임으로 사용

```jsx
const propertyName = "population";

const city = {
  name: "서울",
  [propertyName]: 9700000,
};

console.log(city); // { name: '서울', population: 9700000 }
```

### 함수 리턴값을 프로퍼티 네임으로 사용

```jsx
const getKey = () => "address";

const company = {
  name: "코드잇",
  ["founded" + "Year"]: 2017,
  [getKey()]: "강남구",
};

console.log(company);
// { name: '코드잇', foundedYear: 2017, address: '강남구' }
```

### 실전 활용 예시

### 사용자 정보 객체 만들기

```jsx
function createUser(name, age, job) {
  return {
    name, // name: name
    age, // age: age
    job, // job: job
```

### 주의사항

### 계산된 속성명은 반드시 값도 함께 지정해야 함

```jsx
const key = "name";

// ❌ 에러! 프로퍼티명만 있고 값이 없음
const obj1 = {
  [key], // SyntaxError: Unexpected token ']'
};

// ✅ 올바른 코드: 값을 반드시 지정
const obj2 = {
  [key]: "홍길동", // { name: '홍길동' }
};

// ✅ 변수를 값으로도 사용하려면 명시적으로 작성
const obj3 = {
  [key]: key, // { name: 'name' }
};
```

프로퍼티 축약(`{ name }`)은 변수명 자체가 프로퍼티명이 될 때만 사용 가능합니다.

계산된 속성명(`[key]`)은 프로퍼티명을 동적으로 계산하는 것이므로 값은 별도로 지정해야 합니다.

### 정리

모던 자바스크립트에서는 객체를 더 간결하게 작성할 수 있는 세 가지 방법을 제공합니다:

1. **프로퍼티 축약**: 변수명과 프로퍼티명이 같으면 하나만 작성
2. **메소드 축약**: `function` 키워드와 콜론 생략 가능
3. **계산된 속성명**: 대괄호로 표현식을 감싸서 동적 프로퍼티 생성

```jsx

  name,
  price,
```

    - 대괄호 안의 표현식은 **평가되어** 프로퍼티명이 됨
    - 계산된 속성명과 일반 프로퍼티를 함께 사용 가능

### 문제 2

다음 중 **문법 에러**가 발생하는 코드를 모두 찾으세요.

```jsx

const obj1 = { x };

// B
const obj2 = {
  method() {
    return "Hello";
  },
};

// C
const key = "value";
const obj3 = { [key] };

  ["prop" + "Name"]: "test",
};

// E
function greet() {
  return "Hi";
}
const obj5 = { greet };
```

    정답:
    
    C
    
    해설:

    - **C**: 문법 에러 ❌


    ```jsx
    const obj2 = {

      },
    };
    ```

    ```jsx
    const key = "value";
    ```

    ```jsx
    const obj3 = { [key]: key }; // { value: 'value' }

    const obj3 = { [key]: "someValue" }; // { value: 'someValue' }
    ```

    ```jsx
    const obj4 = {
      ["prop" + "Name"]: "test", // { propName: 'test' }
    };
    ```
    
    - 계산된 속성명 + 값 제공 ✅
    
    **E (정상):**

    - **계산된 속성명**: 반드시 값 필요 → `{ [key]: value }`
    - **계산된 속성명은 축약 불가**: `{ [key] }` ❌

### 10. Destructuring (구조 분해) ⭐⭐⭐

ES2015에서 등장한 **Destructuring(구조 분해)**은 배열이나 객체의 값을 편리하게 추출하는 문법입니다.

### 배열 Destructuring

### 기존 방식의 불편함

```jsx
const rank = ["효준", "유나", "민환", "재하"];

const first = rank[0];
const second = rank[1];
const third = rank[2];
const fourth = rank[3];

console.log(first); // 효준
console.log(second); // 유나
```

배열의 요소를 하나씩 변수에 할당하려면 코드가 길어집니다.

### Destructuring(구조 분해) 사용

```jsx
const rank = ["효준", "유나", "민환", "재하"];

const [first, second, third, fourth] = rank;

console.log(first); // 효준
console.log(second); // 유나
console.log(third); // 민환
console.log(fourth); // 재하
```

한 줄로 해결되며, 배열의 요소가 순서대로 변수에 할당됩니다.

### 작동 원리

```jsx
const [macbook, ipad, airpods, coupon] = rank;
```

- 왼쪽: 변수들을 배열 형태로 나열
- 오른쪽: 실제 배열
- 결과: 배열 요소가 **순서대로** 변수에 지정됨

### 배열 Destructuring 응용

### 1. 필요한 요소만 가져오기

```jsx
const rank = ["효준", "유나", "민환", "재하", "규식"];

const [first, second] = rank;

console.log(first); // 효준
console.log(second); // 유나

```

### 2. 요소 건너뛰기

```jsx
const rank = ["효준", "유나", "민환", "재하"];

const [first, , third] = rank;

console.log(first); // 효준
console.log(third); // 민환

```

쉼표를 비워두면 해당 위치를 건너뜁니다.

### 3. Rest Parameter와 함께 사용

```jsx
const rank = ["효준", "유나", "민환", "재하", "규식"];

const [first, second, ...others] = rank;

console.log(first); // 효준
console.log(second); // 유나
console.log(others); // ["민환", "재하", "규식"]
```

나머지를 한 번에 배열로 받을 수 있습니다.

### 4. 기본값 설정

```jsx
const rank = ["효준", "유나"];

const [first, second, third = "없음"] = rank;

console.log(first); // 효준
console.log(second); // 유나
console.log(third); // 없음
```

할당할 요소가 없으면 기본값이 사용됩니다.

### 5. 변수 값 교환하기

```jsx
let a = 10;
let b = 20;

[a, b] = [b, a];

console.log(a); // 20
console.log(b); // 10
```

임시 변수 없이 한 줄로 교환할 수 있습니다.

### 주의사항

### 1. 할당할 배열이 없으면 에러

```jsx
// ❌ 에러
const [a, b] = 123;

// ❌ 에러
const [a, b];
```

오른쪽에는 반드시 배열(또는 이터러블)이 필요합니다.

### 2. 길이가 맞지 않아도 됨

```jsx
const [a, b, c] = [1, 2];
console.log(c); // undefined

const [x, y] = [1, 2, 3];
console.log(x, y); // 1 2
```

필요한 만큼만 가져오면 됩니다.

### 실전 예제

### 함수의 리턴값 받기

```jsx
function getWinners() {
  return ["금메달", "은메달", "동메달"];
}

const [gold, silver, bronze] = getWinners();

console.log(gold);
console.log(silver);
console.log(bronze);
```

### 여러 값 동시에 할당

```jsx
const [name, age, job] = ["철수", 25, "개발자"];

console.log(name); // 철수
console.log(age); // 25
console.log(job); // 개발자
```

### 정리

배열 Destructuring의 장점:

- 코드가 간결해짐
- 변수 여러 개를 한 번에 선언 가능
- 값 교환이 쉬움
- 함수 리턴값을 편하게 받을 수 있음

주요 문법:

```jsx
const [a, b] = [1, 2];
const [a, , c] = [1, 2, 3];
const [a, ...rest] = [1, 2, 3];
const [a, b = 0] = [1];
[a, b] = [b, a];
```

다음 강의에서는 **객체 Destructuring**을 배웁니다.

### 문제 1

다음 코드의 실행 결과는 무엇인가요?

```jsx
const numbers = [10, 20, 30, 40, 50];
const [a, , c, ...rest] = numbers;

console.log(a);
console.log(c);
console.log(rest);
```

- 정답 및 해설
    
    정답:
    
    ```
    10
    30
    [40, 50]
    
    ```
    
    해설:
    
    배열 Destructuring에서는 요소가 순서대로 할당됩니다.

    - 두 번째 요소 `20`은 건너뛰기
    - `c` → 세 번째 요소 `30`
    - `rest` → 나머지 요소를 배열로 묶음 → `[40, 50]`
    
    `...rest`는 항상 마지막에 위치해야 하며, 남은 모든 요소를 배열로 받습니다.

### 문제 3

다음 코드를 Destructuring을 사용하여 한 줄로 작성하세요:

```jsx
let first = 5;
let second = 10;

let temp = first;
first = second;
second = temp;

console.log(first); // 10
console.log(second); // 5
```

- 정답 및 해설
    
    정답:
    
    ```jsx
    let first = 5;
    let second = 10;
    
    [first, second] = [second, first];
    
    console.log(first); // 10
    console.log(second); // 5
    ```
    
    해설:
    
    Destructuring을 사용하면 임시 변수가 필요 없습니다.
    
    작동 원리:
    
    1. `[second, first]` → `[10, 5]` 생성
    2. 이를 `[first, second]`에 순서대로 할당
    3. 결과: `first = 10`, `second = 5`
    
    이 문법은 여러 변수의 값 순환에도 활용 가능합니다.
    
    ```jsx
    let a = 1, b = 2, c = 3;
    [a, b, c] = [b, c, a]; // a=2, b=3, c=1
    ```

### 12. Destructuring (구조 분해) II ⭐⭐⭐

ES2015에서 등장한 **Destructuring(구조 분해)**은 배열이나 객체의 값을 편리하게 추출하는 문법입니다.

### 객체 Destructuring 기본

객체는 배열과 달리 **프로퍼티 네임**으로 값을 구분합니다.

### 기존 방식

```jsx
const laptop = {
  brand: "LG",
  model: "Gram",
  price: 1500000,
  weight: "999g",
  display: "15.6인치",
};

const brand = laptop.brand;
const price = laptop.price;

console.log(brand); // LG
console.log(price); // 1500000
```

### Destructuring 방식

```jsx
const laptop = {
  brand: "LG",
  model: "Gram",
  price: 1500000,
  weight: "999g",
  display: "15.6인치",
};

const { brand, price } = laptop;

console.log(brand); // LG
console.log(price); // 1500000
```

- 배열은 대괄호(`[]`), 객체는 중괄호(`{}`) 사용
- **프로퍼티 네임**과 변수명이 일치하면 자동으로 할당됩니다

### 기본값 설정

존재하지 않는 프로퍼티에 접근하면 `undefined`가 할당됩니다. 기본값을 설정할 수 있습니다.

```jsx
const laptop = {
  brand: "LG",
  model: "Gram",
  price: 1500000,
};

const { brand, color } = laptop;
console.log(color); // undefined

const { brand: brand2, color: color2 = "white" } = laptop;
console.log(color2); // white
```

### Rest 문법

나머지 프로퍼티들을 하나의 객체로 모을 수 있습니다.

```jsx
const laptop = {
  brand: "LG",
  model: "Gram",
  price: 1500000,
  weight: "999g",
  display: "15.6인치",
};

const { brand, ...others } = laptop;

console.log(brand); // LG
console.log(others);
// { model: 'Gram', price: 1500000, weight: '999g', display: '15.6인치' }
```

### 다른 변수명으로 할당

프로퍼티 네임과 다른 이름으로 변수를 선언할 수 있습니다.

```jsx
const laptop = {
  brand: "LG",
  model: "Gram",
  price: 1500000,
};

const { brand: manufacturer, model: productName } = laptop;

console.log(manufacturer); // LG
console.log(productName); // Gram
// console.log(brand); // ReferenceError: brand is not defined
```

콜론(`:`) 오른쪽에 있는 것이 **실제 변수명**입니다!

### 왜 필요할까?

### 1. 변수명으로 사용할 수 없는 프로퍼티

```jsx
const product = {
  "product-name": "무선 이어폰",
  "serial-number": "ABC-123-XYZ",
  price: 89000,
};

const { "product-name": productName, "serial-number": serialNumber } = product;

console.log(productName); // 무선 이어폰
console.log(serialNumber); // ABC-123-XYZ

```

### 2. 더 명확한 변수명 사용

```jsx
const response = {
  data: [1, 2, 3],
  status: 200,
};

const { data: numbers, status: httpStatus } = response;

console.log(numbers); // [1, 2, 3]
console.log(httpStatus); // 200
```

### Computed Property Name과 함께 사용

동적으로 프로퍼티를 선택할 수도 있습니다.

```jsx
const laptop = {
  brand: "LG",
  model: "Gram",
  price: 1500000,
};

const key = "brand";
const { [key]: value } = laptop;

console.log(value); // LG
```

계산된 속성명을 사용할 때는 반드시 새 변수명을 지정해야 합니다.

### 중첩 객체 Destructuring

객체 안의 객체도 분해할 수 있습니다.

```jsx
const user = {
  name: "김개발",
  age: 28,
  address: {
    city: "서울",
    district: "강남구",
  },
};

const {
  name,
  address: { city, district },
} = user;

console.log(name); // 김개발
console.log(city); // 서울
console.log(district); // 강남구
// console.log(address); // ReferenceError: address is not defined
```

### 중첩 객체와 변수 모두 필요한 경우

```jsx

  name: "김개발",
  age: 28,

    district: "강남구",
  },

  address,
  address: { city },

console.log(address); // { city: '서울', district: '강남구' }
console.log(city); // 서울
```

### 배열 vs 객체 Destructuring 비교

| 구분 | 배열 | 객체 |
| --- | --- | --- |
| 기호 | `[]` | `{}` |
| 순서 | 인덱스 순서대로 | 프로퍼티 네임으로 |
| 변수명 | 자유롭게 지정 | 프로퍼티명과 동일 (또는 `:` 로 변경) |

```jsx
const [first, second] = [10, 20];
const { name, age } = { name: "김코딩", age: 25 };
```

### 정리

객체 Destructuring은 다음과 같은 특징이 있습니다:

- **중괄호(`{}`)** 사용
- **프로퍼티 네임**으로 자동 매칭
- 기본값 설정 가능 (`=`)
- 다른 변수명으로 할당 가능 (`:`)
- Rest 문법 (`...`)
- 중첩 객체 구조 분해 가능
- Computed Property Name과 함께 사용 가능

### 14. 함수와 Destructuring ⭐⭐⭐

### 함수와 Destructuring

Destructuring 문법은 함수와 함께 사용할 때 더욱 강력해집니다. 특히 **파라미터**에서 Destructuring을 활용하면 코드가 훨씬 간결하고 명확해집니다.

### 1. 배열을 리턴하는 함수

함수가 배열을 리턴할 때 Destructuring으로 쉽게 받을 수 있습니다.

```jsx
function getDevices() {
  return ["노트북", "모니터", "키보드"];
}

const [device1, device2, device3] = getDevices();

console.log(device1); // 노트북
console.log(device2); // 모니터
console.log(device3); // 키보드
```

### 2. Rest Parameter와 Destructuring

Rest Parameter도 배열이므로 Destructuring을 활용할 수 있습니다.

```jsx
function announceWinners(...participants) {
  const [gold, silver, bronze, ...others] = participants;

  console.log("🥇 금메달:", gold);
  console.log("🥈 은메달:", silver);
  console.log("🥉 동메달:", bronze);
  console.log("👏 참가상:", others);
}

announceWinners("철수", "영희", "민수", "지은", "현우");
// 🥇 금메달: 철수
// 🥈 은메달: 영희
// 🥉 동메달: 민수
// 👏 참가상: ['지은', '현우']
```

**파라미터 자체**에서 Destructuring을 할 수 있습니다!

### 16. 에러와 에러 객체 ⭐⭐⭐

프로그래밍을 하다 보면 에러를 완전히 피할 수는 없습니다. 중요한 것은 **에러를 어떻게 다루느냐**입니다.

### 에러가 발생하면 어떻게 될까?

자바스크립트에서 에러가 발생하면 **프로그램이 즉시 멈춥니다**.

```jsx
console.log("프로그램 시작");

const name = "코드잇";
console.log(name);

console.log(codeit); // 에러 발생! (선언되지 않은 변수)

console.log("프로그램 종료"); // 실행되지 않음
```

**실행 결과:**

```
프로그램 시작
코드잇
ReferenceError: codeit is not defined
```

에러가 발생한 순간 프로그램이 멈춰서 `'프로그램 종료'`는 출력되지 않습니다.

### 에러 객체란?

에러가 발생하면 자바스크립트는 자동으로 **에러 객체**를 생성합니다.

### 에러 객체의 구조

에러 객체는 크게 두 가지 프로퍼티를 가집니다:

1. **`name`**: 에러의 종류 (예: `ReferenceError`, `TypeError`)
2. **`message`**: 에러에 대한 구체적인 설명

```jsx
// 에러 발생 시
ReferenceError: codeit is not defined
// ↑ name      ↑ message
```

### 주요 에러 종류

### 1. ReferenceError

선언되지 않은 변수나 함수를 참조할 때 발생합니다.

```jsx
console.log(student); // ReferenceError: student is not defined
```

### 2. TypeError

잘못된 자료형으로 작업을 시도할 때 발생합니다.

```jsx
const message = "안녕하세요";
message(); // TypeError: message is not a function
```

문자열을 함수처럼 호출하려고 해서 에러가 발생했습니다.

### 3. SyntaxError

문법이 잘못되었을 때 발생합니다. **코드 실행 전에** 발생하는 특징이 있습니다.

```jsx
const user-name = '김코딩'; // SyntaxError: Unexpected token '-'
```

변수명에 하이픈(`-`)을 사용할 수 없어서 에러가 발생합니다.

### SyntaxError의 특징

```jsx
console.log('시작');

const my-var = 10; // SyntaxError

console.log('끝');
```

SyntaxError가 있으면 **코드가 아예 실행되지 않습니다**. `'시작'`도 출력되지 않습니다!

### 기타 에러 종류

더 많은 에러 종류가 있습니다:

- `RangeError`: 값이 허용 범위를 벗어남
- `URIError`: URI 처리 함수에 잘못된 매개변수 전달
- `EvalError`: eval() 함수 관련 에러
- `InternalError`: 자바스크립트 엔진 내부 에러

자세한 내용은 MDN 문서를 참고하세요.

### 에러 객체 직접 만들기

우리도 에러 객체를 직접 만들 수 있습니다.

### 에러 객체 생성

```jsx
const error = new TypeError("잘못된 타입입니다");

console.log(error.name); // TypeError
console.log(error.message); // 잘못된 타입입니다
```

### 다양한 에러 객체 생성

```jsx
const refError = new ReferenceError("참조 에러 발생");
const rangeError = new RangeError("범위를 벗어났습니다");
const syntaxError = new SyntaxError("문법이 잘못되었습니다");

console.log(refError.name); // ReferenceError
console.log(rangeError.message); // 범위를 벗어났습니다
```

### 에러 발생시키기 - throw

에러 객체를 만드는 것만으로는 에러가 발생하지 않습니다. **`throw` 키워드**로 에러를 발생시켜야 합니다.

```jsx
console.log("시작");

const error = new TypeError("타입 에러 발생");
throw error;

console.log("끝"); // 실행되지 않음
```

**실행 결과:**

```
시작
TypeError: 타입 에러 발생
```

### 간단한 throw 사용법

```jsx
throw new TypeError("잘못된 타입");
```

변수에 할당하지 않고 바로 `throw`할 수도 있습니다.

### 실전 활용 - 나이 검증 함수

```jsx
function checkAge(age) {
  if (typeof age !== "number") {
    throw new TypeError("나이는 숫자여야 합니다");
  }

  if (age < 0) {
    throw new RangeError("나이는 0보다 커야 합니다");
  }

  if (age > 150) {
    throw new RangeError("나이가 너무 많습니다");
  }

  console.log(`입력하신 나이는 ${age}세입니다`);
}

checkAge(25); // 입력하신 나이는 25세입니다
checkAge("스물다섯"); // TypeError: 나이는 숫자여야 합니다
checkAge(-5); // RangeError: 나이는 0보다 커야 합니다
checkAge(200); // RangeError: 나이가 너무 많습니다
```

### 실전 활용 - 계산기 함수

```jsx
function divide(a, b) {
  if (typeof a !== "number" || typeof b !== "number") {
    throw new TypeError("두 인자 모두 숫자여야 합니다");
  }

  if (b === 0) {
    throw new Error("0으로 나눌 수 없습니다");
  }

  return a / b;
}

console.log(divide(10, 2)); // 5
console.log(divide(10, 0)); // Error: 0으로 나눌 수 없습니다
console.log(divide(10, "2")); // TypeError: 두 인자 모두 숫자여야 합니다
```

### 왜 의도적으로 에러를 발생시킬까?

1. 잘못된 입력을 빨리 발견
2. 명확한 에러 메시지 제공
3. 디버깅이 쉬워짐
4. 예상치 못한 동작을 방지해 안전한 코드 작성 가능

### 정리

- 에러가 발생하면 **프로그램이 멈춤**
- 에러 객체는 `name`과 `message` 프로퍼티를 가짐
- 주요 에러 종류
    - `ReferenceError`
    - `TypeError`
    - `SyntaxError`
- `new ErrorType()`으로 에러 객체 생성
- `throw`로 에러 발생 가능
- 의도적인 에러 발생은 더 안전한 코드 작성에 유리

---

### 주의사항

- SyntaxError는 **코드 실행 전** 발생하며, 하나라도 있으면 전체 코드가 실행되지 않음
- `throw` 이후 코드가 실행되지 않음을 항상 염두에 둘 것

### 용어설명

- **에러 객체(Error Object)**: 에러 발생 시 자동 생성되는 객체
- **name**: 에러 종류
- **message**: 에러 설명
- **throw**: 에러를 강제로 발생시키는 키워드

### 17. try catch문 ⭐⭐⭐

에러가 발생하면 프로그램이 멈춥니다. `try...catch`문을 사용하면 에러가 발생해도 **프로그램을 계속 실행**할 수 있습니다.

### 에러 발생 시 문제점

```jsx
console.log("작업 시작");

const userName = "김코딩";
console.log(userName);

userName = "박코딩"; // 에러 발생! (const 재할당 불가)

const userAge = 25;
console.log(userAge);

console.log("작업 완료");
```

**실행 결과:**

```
작업 시작
김코딩
TypeError: Assignment to constant variable.
```

`userAge`와 `'작업 완료'`는 출력되지 않습니다. 에러 때문에 프로그램이 멈췄기 때문입니다.

### try...catch문이란?

에러가 발생해도 프로그램이 멈추지 않도록 **에러를 처리**하는 구문입니다.

### 기본 구조

```jsx
try {
  // 실행할 코드
} catch (error) {
  // 에러 발생 시 실행할 코드
}
```

- **`try` 블록**: 에러가 발생할 수 있는 코드를 작성
- **`catch` 블록**: 에러가 발생했을 때 실행할 코드를 작성
- **`error`**: 에러 객체를 받는 파라미터 (이름은 자유롭게 변경 가능)

### try...catch 동작 방식

### 1. 에러가 발생하지 않을 때

```jsx
try {
  console.log("계산 시작");
  const result = 10 + 20;
  console.log("결과:", result);
} catch (error) {
  console.log("에러가 발생했습니다");
}

console.log("프로그램 종료");
```

**실행 결과:**

```
계산 시작
결과: 30
프로그램 종료
```

`try` 블록의 코드가 모두 실행되고, `catch` 블록은 실행되지 않습니다.

### 2. 에러가 발생할 때

```jsx
try {
  console.log("작업 시작");

  const userName = "김코딩";
  console.log(userName);

  userName = "박코딩"; // 에러 발생!
```

- `try` 블록에서 에러가 발생하면 **그 시점부터 try 블록의 나머지 코드는 실행되지 않습니다**
- 대신 `catch` 블록이 실행됩니다
- 프로그램은 멈추지 않고 계속 진행됩니다

### 주의사항

### 1. try 블록에서 에러 발생 시점 이후는 실행되지 않음

```jsx
try {
  console.log("1번 실행");
  throw new Error("에러 발생!");
  console.log("2번 실행"); // 실행되지 않음
  console.log("3번 실행"); // 실행되지 않음

  console.log("에러 처리");
}
```

**실행 결과:**

```
1번 실행
에러 처리
```

### 2. SyntaxError는 try...catch로 잡을 수 없음

```jsx
try {
  const my-var = 10; // SyntaxError: 코드가 아예 실행되지 않음
} catch (error) {
  console.log('에러 처리'); // 실행되지 않음
}
```

SyntaxError는 코드 실행 전에 발생하므로 try...catch로 처리할 수 없습니다.

### 3. 블록 스코프 주의

`try`와 `catch`는 각각 별도의 **블록 스코프**를 가집니다.

```jsx
try {
  const message = "안녕하세요";
} catch (error) {
  console.log(message); // ReferenceError: message is not defined
}

console.log(message); // ReferenceError: message is not defined
```

`try` 블록에서 선언한 변수는 `catch` 블록이나 외부에서 사용할 수 없습니다.

### 정리

- **`try...catch`문**은 에러가 발생해도 프로그램이 멈추지 않게 합니다
- **`try` 블록**: 실행할 코드를 작성
- **`catch` 블록**: 에러 발생 시 실행할 코드를 작성
- `catch`의 파라미터로 **에러 객체**를 받을 수 있습니다
- `try` 블록에서 에러가 발생하면 **그 이후 코드는 실행되지 않고** `catch` 블록으로 넘어갑니다
- `console.error()`로 에러를 더 명확하게 표시할 수 있습니다
- `try`와 `catch`는 각각 별도의 **블록 스코프**를 가집니다

다음 시간에는 `try...catch`를 실제로 어떻게 활용하는지 더 자세히 알아보겠습니다!

```jsx
  x();
} catch (error) {
  console.log('예외 처리됨'); // 실행됨
}

// 예외 X - try...catch로 처리 불가
try {
  const my-var = 10; // SyntaxError: 코드가 실행되지 않음
} catch (error) {
  console.log('예외 처리됨'); // 실행되지 않음
}

```

### 예외 처리 용어

- **예외 처리 (Exception Handling)**: 실행 가능한 코드의 에러를 처리하는 과정
- 일반적으로 "에러 처리"와 "예외 처리"를 구분 없이 사용하지만, 엄밀히는 다릅니다

### 실전 패턴

### 1. 사용자 친화적인 에러 메시지

```jsx
function processPayment(amount) {
  try {
    if (amount <= 0) {
      throw new Error("결제 금액은 0보다 커야 합니다");

    if (amount > 1_000_000) {
      throw new Error("한 번에 100만원까지만 결제 가능합니다");

  }
}

processPayment(1_000) // 1000원 결제 완료
processPayment(100_000) // 100000원 결제 완료
```

### 정리

- `try...catch`는 에러가 발생해도 **프로그램을 계속 실행**하게 합니다
- 에러를 적절히 처리하여 **사용자에게 안내**할 수 있습니다
- 여러 작업을 수행할 때 **하나의 에러가 전체를 막지 않도록** 할 수 있습니다
- `try`와 `catch`는 **각각 별도의 블록 스코프**를 가집니다
- **예외(Exception)**: 실행 가능한 코드에서 발생하는 에러
- **예외 처리(Exception Handling)**: 예외를 다루는 과정

`try...catch`를 활용하면 훨씬 더 안정적이고 사용자 친화적인 프로그램을 만들 수 있습니다!

- try와 catch는 각각 별도의 블록 스코프를 가짐

### 문제 3

다음 코드의 문제점을 찾고 수정해보세요.

```jsx
try {
  const userInput = prompt("숫자를 입력하세요:");
  const result = parseInt(userInput) * 2;
} catch (error) {
  console.error("에러 발생:", error.message);
}

console.log(`결과: ${result}`);
```

이 코드를 실행하면 어떤 에러가 발생할까요?

- 정답 및 해설
    
    정답:
    
    - `result`가 `try` 블록 안에서 `const`로 선언되어 블록 밖에서 접근할 수 없기 때문에
        
        `ReferenceError: result is not defined` 에러가 발생합니다.
        
    
    해결 방법 1: 변수를 외부에 선언
    
    ```jsx
    let result;
    
    try {
      const userInput = prompt("숫자를 입력하세요:");
      result = parseInt(userInput) * 2;
    } catch (error) {
      console.error("에러 발생:", error.message);
      result = 0; // 기본값 설정
    }
    
    console.log(`결과: ${result}`);
    ```
    
    해결 방법 2: 함수로 감싸기
    
    ```jsx
    function calculateResult() {
      try {
        const userInput = prompt("숫자를 입력하세요:");
        const result = parseInt(userInput) * 2;
        return result;
      } catch (error) {
        console.error("에러 발생:", error.message);
        return 0; // 기본값 반환
      }
    }
    
    const result = calculateResult();
    console.log(`결과: ${result}`);
    ```
    
    해설:
    
    `try`와 `catch`는 각각 하나의 블록이기 때문에, 그 안에서 `const`나 `let`으로 선언한 변수는 블록 바깥에서 사용할 수 없습니다. 원래 코드에서는 `result`를 `try` 블록 내부에서 선언했기 때문에 `console.log`에서 `result`를 읽으려 하면 `ReferenceError`가 발생합니다. 외부에서 사용할 값이라면 `try` 바깥에서 변수를 선언해 두고 안에서 할당하거나, 함수를 만들어 `return`으로 값을 돌려주는 방식으로 스코프 문제를 해결해야 합니다.

### 01. 배열 메소드 I: forEach와 map ⭐⭐⭐

### 배열 메소드 I: forEach와 map

배열의 요소를 하나씩 순회하면서 작업할 때 `for...of`문을 사용할 수 있습니다. 하지만 배열에는 더 강력한 메소드들이 있습니다!

### for...of 문 복습

```jsx
const members = ["영훈", "윤수", "동욱", "태호"];

for (const member of members) {
  console.log(`${member}님이 입장하셨습니다.`);
}
```

배열의 각 요소를 순회하면서 작업을 수행합니다.

### forEach 메소드

### 기본 사용법

```jsx
const members = ["영훈", "윤수", "동욱", "태호"];

members.forEach((member) => {
  console.log(`${member}님이 입장하셨습니다.`);
});
```

`forEach` 메소드는 **콜백 함수**를 받아서, 배열의 각 요소마다 그 함수를 실행합니다.

### forEach의 파라미터

forEach의 콜백 함수는 **3개의 파라미터**를 받을 수 있습니다:

```jsx
array.forEach((요소, 인덱스, 배열) => {
  // 작업
});
```

1. **요소** (필수): 현재 처리 중인 배열의 요소
2. **인덱스** (선택): 현재 요소의 인덱스
3. **배열** (선택): forEach를 호출한 배열 자체

### 인덱스 활용하기

```jsx
const members = ["영훈", "윤수", "동욱", "태호"];

members.forEach((member, index) => {
  console.log(`${index + 1}. ${member}님이 입장하셨습니다.`);
});
// 1. 영훈님이 입장하셨습니다.
// 2. 윤수님이 입장하셨습니다.
// 3. 동욱님이 입장하셨습니다.
// 4. 태호님이 입장하셨습니다.

```

### 인덱스로 다른 배열 매칭하기

```jsx
const firstNames = ["영훈", "윤수", "동욱", "태호"];
const lastNames = ["강", "이", "손", "성"];

firstNames.forEach((firstName, i) => {
  console.log(`${lastNames[i]}${firstName}님이 입장하셨습니다.`);
});
// 강영훈님이 입장하셨습니다.
// 이윤수님이 입장하셨습니다.
// 손동욱님이 입장하셨습니다.
// 성태호님이 입장하셨습니다.
```

### 세 번째 파라미터 (배열)

```jsx
const members = ["영훈", "윤수", "동욱"];

members.forEach((member, index, arr) => {
  console.log(`${member} (전체: ${arr.length}명)`);
});
// 영훈 (전체: 3명)
// 윤수 (전체: 3명)
// 동욱 (전체: 3명)
```

보통은 배열 자체를 변수로 접근할 수 있어서 잘 사용하지 않지 않습니다.

### map 메소드

### forEach와의 차이점

```jsx
const numbers = [1, 2, 3, 4, 5];

// forEach: 리턴값 없음
const result1 = numbers.forEach((num) => {
  console.log(num * 2);
});
console.log(result1); // undefined

// map: 새로운 배열 리턴
const result2 = numbers.map((num) => {
  return num * 2;
});
console.log(result2); // [2, 4, 6, 8, 10]
```

- **forEach**: 단순 반복 작업, 리턴값 없음
- **map**: 각 요소를 변환하여 **새로운 배열 생성**

### map의 핵심: 변환

```jsx
const numbers = [1, 2, 3, 4, 5];

const doubled = numbers.map((num) => num * 2);

console.log(numbers); // [1, 2, 3, 4, 5] (원본 유지)
console.log(doubled); // [2, 4, 6, 8, 10] (새 배열)
```

원본 배열은 변경하지 않고, 변환된 새로운 배열을 만듭니다!

### 실전 예제

**이름 배열 → 인사말 배열**

```jsx
const names = ["철수", "영희", "민수"];

const greetings = names.map((name) => `안녕하세요, ${name}님!`);

console.log(greetings);
// ["안녕하세요, 철수님!", "안녕하세요, 영희님!", "안녕하세요, 민수님!"]
```

**점수 배열 → 등급 배열**

```jsx
const scores = [85, 92, 78, 95, 88];

const grades = scores.map((score) => {
  if (score >= 90) return "A";
  if (score >= 80) return "B";
  return "C";
});

console.log(grades); // ["B", "A", "C", "A", "B"]
```

### map도 3개의 파라미터 사용 가능

```jsx
const numbers = [10, 20, 30];

const result = numbers.map((num, index) => {
  return `${index}번: ${num}`;
});

console.log(result); // ["0번: 10", "1번: 20", "2번: 30"]
```

### forEach vs map 비교

| 특징 | forEach | map |
| --- | --- | --- |
| 리턴값 | 없음 (undefined) | 새로운 배열 |
| 용도 | 단순 반복 작업 | 배열 변환 |
| 원본 배열 | 변경 안함* | 변경 안함 |
| 체이닝 | 불가능 | 가능 |
- 직접 수정하지 않는 한

### 언제 무엇을 사용할까?

**forEach 사용:**

- 단순히 배열을 순회하며 작업만 할 때
- 콘솔 출력, DOM 조작 등
- 새로운 배열이 필요 없을 때

```jsx
// forEach가 적합
members.forEach((member) => {
  console.log(member);
});
```

**map 사용:**

- 배열의 각 요소를 변환할 때
- 새로운 배열이 필요할 때
- 데이터 가공 및 변환

```jsx
// map이 적합
const doubled = numbers.map((num) => num * 2);
```

### 주의사항

### 최대 반복 횟수는 처음 결정됨

```jsx
const members = ["영훈", "윤수", "동욱", "태호"];

members.forEach((member) => {
  console.log(`${member}님이 입장하셨습니다.`);
  members.push("코드잇"); // 매번 요소 추가
});

console.log(members);
// ["영훈", "윤수", "동욱", "태호", "코드잇", "코드잇", "코드잇", "코드잇"]
```

반복 중에 요소를 추가해도, **처음 시작할 때의 길이만큼만 반복**합니다! (무한 루프 방지)

### 배열 길이가 줄어들면?

```jsx
const members = ["영훈", "윤수", "동욱", "태호"];

members.forEach((member) => {
  console.log(`${member}님이 입장하셨습니다.`);
  members.pop(); // 매번 마지막 요소 제거
});
// 영훈님이 입장하셨습니다.
// 윤수님이 입장하셨습니다.

console.log(members); // []

```

길이가 줄어들면 반복 횟수도 줄어듭니다. 주의하세요!

### 정리

**forEach:**

- 배열의 각 요소마다 콜백 함수 실행
- 리턴값 없음
- 단순 반복 작업에 적합

**map:**

- 배열의 각 요소를 변환
- 새로운 배열 리턴
- 데이터 변환에 적합

**공통점:**

- 3개의 파라미터: (요소, 인덱스, 배열)
- 원본 배열을 직접 수정하지 않음
- `for...of`보다 간결하고 함수형 프로그래밍 스타일

### 주의사항

- forEach와 map은 원본 배열을 직접 수정하지 않지만, 콜백 내부에서 원본을 조작하면 결과가 예측과 달라질 수 있음
- 반복 도중 배열 길이를 변화시키면 반복 횟수는 처음 시점의 길이 기준으로 결정됨
- map은 반드시 새로운 배열을 반환해야 하며, forEach는 반환값이 없음

### 용어설명

- **콜백 함수**: 다른 함수의 인자로 전달되는 함수
- **변환(transform)**: 기존 값을 새로운 형태로 바꾸는 작업
- **iterable**: `for...of`로 순회할 수 있는 객체

```jsx

  return price * 1.1;
});

const result2 = prices.map((price) => {
  return price * 1.1;
});

console.log(result1);
console.log(result2);
```

- 정답 및 해설

    ```
    [1100, 2200, 3300]
    ```

    핵심은 forEach는 리턴값이 없어서 단순 반복 작업에 사용하고, map은 각 요소를 변환한 새로운 배열을 만들 때 사용한다는 점입니다. forEach에서 return을 작성해도 메소드의 리턴값은 항상 undefined입니다.

### 문제 3

다음 코드의 실행 결과를 예상하고, 왜 그런 결과가 나오는지 설명해보세요.

```jsx
const numbers = [1, 2, 3];

numbers.forEach((num, index) => {
  console.log(`${index}: ${num}`);
  numbers.push(num + 10);
});

console.log(numbers);
```

- 정답 및 해설
    
    정답:
    
    ```
    0: 1
    1: 2
    2: 3
    [1, 2, 3, 11, 12, 13]
    ```
    
    해설:
    
    forEach가 시작될 때 배열 numbers의 길이는 3이므로, 최대 반복 횟수는 처음 길이인 3번으로 정해집니다. 그래서 인덱스 0, 1, 2에 해당하는 요소만 순회합니다.
    
    첫 번째 반복에서는 0: 1이 출력되고 numbers.push(11)이 실행되어 배열이 [1, 2, 3, 11]이 됩니다. 두 번째 반복에서는 1: 2가 출력되고 12가 추가되어 [1, 2, 3, 11, 12]가 됩니다. 세 번째 반복에서는 2: 3이 출력되고 13이 추가되어 [1, 2, 3, 11, 12, 13]이 됩니다. 새로 추가된 11, 12, 13은 forEach가 반복 대상으로 삼지 않습니다.
    
    이후 console.log(numbers)에서 최종 배열 [1, 2, 3, 11, 12, 13]이 출력됩니다. forEach와 map은 무한 루프에 빠지지 않도록 처음 시작 시점의 배열 길이를 기준으로 반복 횟수를 결정합니다. 그래서 반복 도중에 요소를 추가해도 반복 횟수가 늘어나지 않습니다.
    
    주의해야 할 점은 배열 길이를 줄이는 경우 반복 횟수도 줄어든다는 것입니다. 예를 들어 다음 코드에서는 1과 2만 출력되고 배열은 빈 배열이 됩니다.
    
    ```jsx
    const items = [1, 2, 3, 4];
    
    items.forEach((item) => {
      console.log(item);
      items.pop(); // 마지막 요소 제거
    });
    // 출력: 1, 2
    // items: []
    ```
    
    핵심은 forEach와 map의 최대 반복 횟수는 시작 시점에 결정된다는 것, 요소를 추가해도 반복 횟수는 늘어나지 않고, 요소를 제거하면 반복 횟수가 줄어들 수 있다는 점입니다. 반복 중에는 배열을 수정하지 않는 것이 가장 안전합니다.

### 04. 배열 메소드 II: filter와 find ⭐⭐⭐

배열에서 **특정 조건을 만족하는 요소**를 찾을 때 사용하는 메소드를 알아봅시다.

### filter 메소드

**조건을 만족하는 모든 요소**를 모아서 **새로운 배열**로 반환합니다.

### 기본 사용법

```jsx
const numbers = [1,2,3,4,5]

const odds = numbers.filter((number) => number % 2 === 1);

console.log(odds) // [1,3,5]
```

```jsx
const products = [
  { name: "노트북", brand: "Apple", price: 2500000 },
  { name: "마우스", brand: "Logitech", price: 50000 },
  { name: "키보드", brand: "Apple", price: 150000 },
  { name: "모니터", brand: "LG", price: 400000 },
  { name: "이어폰", brand: "Apple", price: 300000 },
  { name: "스피커", brand: "Sony", price: 200000 },
];

const appleProducts = products.filter((product) => product.brand === "Apple");

console.log(appleProducts);
```

```jsx
[
  { name: "노트북", brand: "Apple", price: 2500000 },
  { name: "키보드", brand: "Apple", price: 150000 },
  { name: "이어폰", brand: "Apple", price: 300000 }
]
```

### filter의 특징

1. **콜백 함수는 조건식을 반환**해야 합니다
2. 조건식이 true인 요소만 모아 **배열로 반환**합니다
3. **원본 배열은 변경되지 않습니다**

### 다양한 필터링 예시

```jsx
const scores = [45, 72, 88, 93, 55, 67, 81, 90];

const highScores = scores.filter((score) => score >= 80);
const lowScores = scores.filter((score) => score < 60);
const evenScores = scores.filter((score) => score % 2 === 0);

console.log(highScores); // [88, 93, 81, 90]
console.log(lowScores);  // [45, 55]
console.log(evenScores); // [72, 88, 90]

```

### find 메소드

**조건을 만족하는 첫 번째 요소**를 반환합니다.

### 기본 사용법

```jsx
const products = [
  { name: "노트북", brand: "Apple", price: 2500000 },
  { name: "마우스", brand: "Logitech", price: 50000 },
  { name: "키보드", brand: "Apple", price: 150000 },
  { name: "모니터", brand: "LG", price: 400000 },
  { name: "이어폰", brand: "Apple", price: 300000 },
];

const lgProduct = products.find((product) => product.brand === "LG");

console.log(lgProduct);
```

```jsx
{ name: '모니터', brand: 'LG', price: 400000 }
```

### find의 특징

1. **조건을 만족하는 첫 번째 요소만 반환**합니다
2. 반환값은 **요소 자체**입니다 (배열 아님)
3. 조건을 만족하는 요소가 없으면 **undefined**

### 다양한 찾기 예시

```jsx
const numbers = [5, 12, 8, 21, 15, 3, 18];

console.log(numbers.find((num) => num > 10)); // 12
console.log(numbers.find((num) => num > 20)); // 21
console.log(numbers.find((num) => num % 2 === 0)); // 12
```

### filter vs find 비교

### 1. 반환 타입

```jsx
const numbers = [1, 2, 3, 4, 5];

console.log(numbers.filter((num) => num > 3)); // [4, 5]
console.log(numbers.find((num) => num > 3));   // 4
```

### 2. 반복 횟수

```jsx
const numbers = [1, 2, 3, 4, 5];

// filter는 전체 순회
numbers.filter((num, i) => {
  console.log("filter:", i);
  return num > 3;
});

// find는 조건을 만족하면 즉시 종료
numbers.find((num, i) => {
  console.log("find:", i);
  return num > 3;
});
```

### 3. 사용 시나리오

| 메소드 | 언제 사용? | 반환 |
| --- | --- | --- |
| filter | 조건을 만족하는 **모든 요소** 필요할 때 | 배열 |
| find | 조건을 만족하는 **첫 요소** 필요할 때 | 값 |

### 정리

### filter 메소드

- 조건을 만족하는 **모든 요소**를 **배열로 반환**
- 배열 전체 순회
- 없으면 **빈 배열**
- 여러 요소를 추출할 때 사용

### find 메소드

- 조건을 만족하는 **첫 번째 요소**를 **값으로 반환**
- 조건 만족하면 즉시 종료
- 없으면 **undefined**
- 특정 대상을 빠르게 찾고 싶을 때 사용

두 메소드는 상황에 맞게 선택해 사용하면 됩니다!

### 주의사항

- filter와 find의 콜백은 반드시 **true/false를 반환**
- 원본 배열은 변경되지 않음
- 조건에 따라 찾을지, 추출할지를 구분하여 선택

### 문제 3

다음 코드의 실행 결과를 예상하고, find와 filter의 반복 횟수 차이를 설명해보세요.

```jsx
const users = [
  { id: 1, name: "김철수", age: 25 },
  { id: 2, name: "이영희", age: 30 },
  { id: 3, name: "박민수", age: 28 },
  { id: 4, name: "정지은", age: 30 },
];

// Case 1
const filterResult = users.filter((user, index) => {
  console.log("filter index:", index);
  return user.age === 30;
});

// Case 2
const findResult = users.find((user, index) => {
  console.log("find index:", index);
  return user.age === 30;
});

console.log("Filter result:", filterResult);
console.log("Find result:", findResult);

```

- 정답 및 해설
    
    정답:
    
    ```
    filter index: 0
    filter index: 1
    filter index: 2
    filter index: 3
    Filter result: [
      { id: 2, name: "이영희", age: 30 },
      { id: 4, name: "정지은", age: 30 }
    ]
    
    find index: 0
    find index: 1
    Find result: { id: 2, name: "이영희", age: 30 }
    ```
    
    해설:
    
    filter는 조건을 만족하는 모든 요소를 찾기 위해 배열의 **모든 요소를 끝까지 순회합니다.** 위 코드에서는 인덱스 0, 1, 2, 3이 모두 한 번씩 출력되고, 나이가 30인 사용자 두 명(이영희, 정지은)이 배열에 담겨 filterResult에 저장됩니다.
    
    find는 조건을 만족하는 **첫 번째 요소를 찾으면 즉시 순회를 종료합니다.** 인덱스 0에서 age가 25라 조건을 만족하지 못하고, 인덱스 1에서 age가 30이므로 조건을 만족하는 순간 순회가 멈춥니다. 따라서 find index는 0과 1만 출력되고, findResult에는 { id: 2, name: "이영희", age: 30 }만 담깁니다.
    
    성능 측면에서 filter는 항상 전체를 순회하지만, find는 평균적으로 더 적은 횟수만 순회할 수 있어 조건을 만족하는 요소를 하나만 찾으면 되는 상황에서는 find가 더 효율적입니다.

### 07. 배열 메소드 III: some과 every ⭐⭐⭐

### some 메소드

배열에서 **조건을 만족하는 요소가 1개 이상 있는지** 확인합니다.

### 기본 사용법

```jsx
const ages = [15, 22, 18, 30, 17];

const hasAdult = ages.some((age) => age >= 20);
console.log(hasAdult); // true
```

### some의 특징

1. **true 또는 false**만 반환합니다
2. **조건을 만족하는 요소를 찾으면 즉시 종료**합니다
3. 요소가 **1개라도 조건을 만족하면 true**

### 다양한 예시

```jsx
const numbers = [1, 3, 5, 7, 9];

console.log(numbers.some((num) => num > 5)); // true
console.log(numbers.some((num) => num % 2 === 0)); // false
console.log(numbers.some((num) => num > 10)); // false
```

### 객체 배열에서 사용

```jsx
const students = [
  { name: "김민수", score: 85 },
  { name: "이지은", score: 72 },
  { name: "박서준", score: 90 },
  { name: "최유리", score: 68 },
];

const hasTopStudent = students.some((s) => s.score >= 90);
const hasFailingStudent = students.some((s) => s.score < 60);

console.log(hasTopStudent); // true
console.log(hasFailingStudent); // false
```

### every 메소드

배열의 **모든 요소가 조건을 만족하는지** 확인합니다.

### 기본 사용법

```jsx
const ages = [22, 25, 30, 28, 24];

const allAdults = ages.every((age) => age >= 20);
console.log(allAdults); // true
```

### every의 특징

1. **true 또는 false** 반환
2. 조건을 만족하지 않는 요소 발견 시 **즉시 종료**
3. **모든 요소**가 조건을 만족해야 true

### 다양한 예시

```jsx
const numbers = [2, 4, 6, 8, 10];

console.log(numbers.every((num) => num % 2 === 0)); // true
console.log(numbers.every((num) => num > 5)); // false
console.log(numbers.every((num) => num > 0)); // true
```

### 객체 배열에서 사용

```jsx
const products = [
  { name: "노트북", price: 1500000, inStock: true },
  { name: "마우스", price: 30000, inStock: true },
  { name: "키보드", price: 80000, inStock: true },
];

console.log(products.every((p) => p.inStock)); // true
console.log(products.every((p) => p.price <= 1000000)); // false
```

---

### some vs every 비교

### 의미 차이

```jsx
const scores = [85, 92, 78, 88, 95];

console.log(scores.some((score) => score >= 90)); // true
console.log(scores.every((score) => score >= 90)); // false
```

### 반복 종료 시점

```jsx
const numbers = [1, 3, 5, 7, 9];

const someResult = numbers.some((num, i) => {
  console.log("some:", i);
  return num > 5;
});

const everyResult = numbers.every((num, i) => {
  console.log("every:", i);
  return num > 5;
});
```

### 표로 정리

| 메소드 | 의미 | true 반환 조건 | 즉시 종료 기준 |
| --- | --- | --- | --- |
| some | 1개라도 조건 만족? | 조건 만족 요소 1개 이상 | 조건 만족 요소 발견 |
| every | 모두 조건 만족? | 모든 요소 조건 만족 | 조건 불만족 요소 발견 |

### filter/find vs some/every

### filter & find → **요소를 반환**

```jsx
numbers.filter((n) => n > 5); // [7, 9]
numbers.find((n) => n > 5);   // 7
```

### some & every → **조건 확인(boolean)**

```jsx
numbers.some((n) => n > 5);  // true
numbers.every((n) => n > 5); // false
```

---

### 정리

### some

- 1개라도 조건 만족하면 true
- 조건 만족 시 즉시 종료
- 빈 배열 → false

### every

- 모든 요소가 조건 만족해야 true
- 조건 불만족 시 즉시 종료
- 빈 배열 → true

### 사용 기준

- 일부라도 조건 만족? → some
- 모두 조건 만족? → every

---

### 용어설명

- **조건식**: true/false를 반환하는 표현식
- **콜백 함수**: 배열 메소드 내부에서 실행되는 함수
- **즉시 종료(short-circuit)**: 불필요한 반복을 중단하는 동작

```jsx
const ages = [18, 22, 16, 30, 25];
```

### 문제 2

다음 코드를 실행하면 콜백 함수는 총 몇 번 실행될까요?

```jsx
const numbers = [2, 4, 6, 7, 8];

const result1 = numbers.some((num) => num % 2 === 1);
const result2 = numbers.every((num) => num % 2 === 0);

// result1을 구할 때: ? 번
// result2를 구할 때: ? 번

```

- 정답 및 해설
    
    정답:
    
    ```
    result1을 구할 때: 4번
    result2를 구할 때: 4번
    
    ```
    
    해설:
    
    `some((num) => num % 2 === 1)`은 홀수가 하나라도 있는지 확인합니다.
    
    인덱스 0: 2(짝수) → 계속
    
    인덱스 1: 4(짝수) → 계속
    
    인덱스 2: 6(짝수) → 계속
    
    인덱스 3: 7(홀수, 조건 만족) → true를 반환하고 바로 종료합니다.
    
    따라서 총 4번 실행됩니다.
    
    `every((num) => num % 2 === 0)`은 모든 수가 짝수인지 확인합니다.
    
    인덱스 0: 2(짝수, 조건 만족) → 계속
    
    인덱스 1: 4(짝수, 조건 만족) → 계속
    
    인덱스 2: 6(짝수, 조건 만족) → 계속
    
    인덱스 3: 7(홀수, 조건 불만족) → false를 반환하고 바로 종료합니다.
    
    마찬가지로 총 4번 실행됩니다.
    
    두 메소드 모두 조건에 따라 더 이상 볼 필요가 없으면 **즉시 종료**된다는 점이 핵심입니다.

```jsx
const allSubmitted = /* 코드 작성 */;
```

    ```jsx
    const students = [
      { name: "김철수", score: 85, submitted: true },
      { name: "이영희", score: 92, submitted: true },
      { name: "박민수", score: 78, submitted: false },
      { name: "최지은", score: 88, submitted: true },

    const hasTopScore = students.some((student) => student.score >= 90);

    const allSubmitted = students.every((student) => student.submitted);

    const hasFailure = students.some((student) => student.score < 60);
    ```

    메소드 선택 기준은 간단합니다.
    
    “하나라도 있는가?” → some, “전부 그런가?” → every.

### 09. 배열 메소드 IV: reduce ⭐⭐⭐⭐⭐

`reduce`는 배열의 모든 요소를 **순회하면서 하나의 결과값**으로 만들어내는 강력한 메소드입니다.

### reduce의 기본 구조

```jsx
배열.reduce((accumulator, element, index, array) => {
  // 다음 accumulator로 전달할 값을 return
  return nextAccumulatorValue;
}, initialValue);
```

### 파라미터 설명

- **`accumulator`** (누산기): 이전 콜백의 반환값 (축적되는 값)
- **`element`**: 현재 처리 중인 배열 요소
- **`index`**: 현재 요소의 인덱스 (선택)
- **`array`**: reduce를 호출한 배열 (선택)
- **`initialValue`**: accumulator의 초기값

### 기본 사용 예시 - 합계 구하기

```jsx
const numbers = [1, 2, 3, 4, 5];

const sum = numbers.reduce((acc, num) => {
  return acc + num;
}, 0);

console.log(sum); // 15
```

### 동작 과정 상세 분석

```jsx
const numbers = [1, 2, 3, 4, 5];

const sum = numbers.reduce((acc, num, index) => {
  console.log(`${index}번째: acc = ${acc}, num = ${num}`);
  return acc + num;
}, 0);

console.log("최종 결과:", sum);
```

**실행 결과:**

```
0번째: acc = 0, num = 1
1번째: acc = 1, num = 2
2번째: acc = 3, num = 3
3번째: acc = 6, num = 4
4번째: acc = 10, num = 5
최종 결과: 15
```

### 단계별 설명

1. **1회차**: `acc = 0` (초기값), `num = 1` → 반환: `0 + 1 = 1`
2. **2회차**: `acc = 1` (이전 반환값), `num = 2` → 반환: `1 + 2 = 3`
3. **3회차**: `acc = 3`, `num = 3` → 반환: `3 + 3 = 6`
4. **4회차**: `acc = 6`, `num = 4` → 반환: `6 + 4 = 10`
5. **5회차**: `acc = 10`, `num = 5` → 반환: `10 + 5 = 15`
6. **최종**: `15` 반환

### 초기값의 중요성

### 초기값을 생략하면?

```jsx
const numbers = [1, 2, 3, 4, 5];

const sum = numbers.reduce((acc, num, index) => {
  console.log(`${index}번째: acc = ${acc}, num = ${num}`);
  return acc + num;
});

console.log("최종 결과:", sum);
```

**실행 결과:**

```
1번째: acc = 1, num = 2
2번째: acc = 3, num = 3
3번째: acc = 6, num = 4
4번째: acc = 10, num = 5
최종 결과: 15
```

초기값을 생략하면:

- **첫 번째 요소**가 accumulator의 초기값이 됩니다
- 반복이 **1번 인덱스**부터 시작됩니다 (0번 인덱스 생략)

### 초기값을 명시하는 것이 안전합니다!

```jsx
// ✅ 안전 - 초기값 명시
const sum1 = [10, 20, 30].reduce((acc, num) => acc + num, 0);
console.log(sum1); // 60

// ⚠️ 위험 - 초기값 생략
const sum2 = [10, 20, 30].reduce((acc, num) => acc + num);
console.log(sum2); // 60 (이 경우는 동일)

// ❌ 빈 배열에서는 에러 발생!
const sum3 = [].reduce((acc, num) => acc + num);
// TypeError: Reduce of empty array with no initial value

```

**권장**: 항상 초기값을 명시하세요!

### 실전 활용 - 평균 구하기

```jsx
const scores = [85, 92, 78, 95, 88];

const average = scores.reduce((sum, score, index, array) => {
  sum += score;

  // 마지막 요소일 때 평균 계산
  if (index === array.length - 1) {
    return sum / array.length;
  }

  return sum;
}, 0);

console.log(`평균 점수: ${average}`); // 평균 점수: 87.6
```

### reduce vs 다른 메소드

```jsx
const numbers = [1, 2, 3, 4, 5];

// forEach 사용
let sum1 = 0;
numbers.forEach((num) => {
  sum1 += num;
});
console.log(sum1); // 15

// reduce 사용
const sum2 = numbers.reduce((acc, num) => acc + num, 0);
console.log(sum2); // 15

```

`reduce`가 더 함수형 프로그래밍 스타일이며, **외부 변수를 사용하지 않아** 더 안전합니다.

### 주의사항

### 1. 초기값을 항상 제공하세요

```jsx
// ✅ 좋은 예
const sum = numbers.reduce((acc, num) => acc + num, 0);

// ⚠️ 나쁜 예 (빈 배열에서 에러)
const sum = numbers.reduce((acc, num) => acc + num);
```

### 2. accumulator를 반드시 반환하세요

```jsx
// ❌ 잘못된 예 - 반환 없음
const wrong = numbers.reduce((acc, num) => {
  acc + num; // return이 없음!
}, 0);
console.log(wrong); // undefined

// ✅ 올바른 예
const correct = numbers.reduce((acc, num) => {
  return acc + num;
}, 0);
console.log(correct); // 15
```

### 3. 객체를 accumulator로 사용할 때

```jsx
// ✅ 올바른 예 - 객체를 수정하고 반환
const result = array.reduce((obj, item) => {
  obj[item.key] = item.value;
  return obj; // 반드시 객체를 반환
}, {});
```

### 정리

- *`reduce`*는 배열을 **하나의 값**으로 축약합니다
- **accumulator**: 이전 콜백의 반환값 (누적값)
- **초기값**을 항상 제공하는 것이 안전합니다
- 초기값이 없으면 **첫 번째 요소**가 초기값이 되고, **1번 인덱스**부터 시작합니다
- 합계, 평균, 최댓값, 그룹화, 변환 등 다양한 작업에 활용 가능합니다
- 콜백에서 **accumulator를 반드시 반환**해야 합니다

`reduce`는 처음에는 어렵게 느껴질 수 있지만, 익숙해지면 매우 강력하고 유연한 도구입니다. 다양한 예시를 통해 연습해보세요!

### 용어설명

- **reduce**: 배열의 모든 요소를 순회하면서 하나의 값으로 축약하는 배열 메소드
- **누산기(accumulator)**: 각 단계에서 계산 결과를 계속 누적해서 담는 변수
- **초기값(initialValue)**: reduce가 처음 시작할 때 누산기에 설정되는 시작 값
- **콜백 함수**: reduce가 각 요소마다 호출하는 함수로, 다음 단계에서 사용할 누산기 값을 반환해야 합니다
