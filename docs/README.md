# codeit-fs-15 docs

이 디렉터리는 이 레포의 운영 문서와 제작 기준을 모아두는 문서 루트입니다.

실제 15기 강의자료 본문은 `courses/` 아래에 둡니다. `docs/`에는 강의자료 제작 기준, 커리큘럼 진행 상태, 참고자료 정리 원칙만 둡니다.

## 구성

- [00. 문서 운영](./00.%20문서%20운영/목차.md): 15기 강의자료 개선 기준
- [강의자료 검토 및 업로드 절차](./00.%20문서%20운영/강의자료%20검토%20및%20업로드%20절차.md): 현재 기준이 되는 검토, 수정, Notion 부분 수정 및 명시적 전체 교체 절차
- [중요도 별표 기준](./00.%20문서%20운영/중요도%20별표%20기준.md): 강의 제목에 붙는 `⭐` 판정 기준
- [제목 계층 기준](./00.%20문서%20운영/제목%20계층%20기준.md): `##`, `###`, `####` 사용 기준
- [01 기준 샘플](./00.%20문서%20운영/01%20기준%20샘플.md): `01. 프로그래밍 시작하기` 동기화, formatter, 품질 점검 기준
- [전체 구성](./00.%20문서%20운영/전체%20구성.md): 기준 과정 목록

## 현재 작성 기준

- 15기 강의 순서와 과정 제목은 `references/raw/notion`의 `NN.제목` 폴더명을 기준으로 유지합니다.
- 과정별 강의자료는 `courses/NN.제목/강의자료.md`처럼 과정 폴더 바로 아래에 둡니다.
- 내부 내용은 원문을 보존한 상태에서 내용, 말투, 억양이 교과서적으로 적절한지 검토하고, 교과서적으로 적절하지 않은 학생용 본문은 수정합니다.
- 명확하게 잘못된 내용이 있으면 `courses/*/강의자료.md`에서 수정합니다.
- 최신 라이브러리, 패키지, 외부 서비스 정책처럼 현재 문서 확인이 필요한 내용은 직접 수정하지 않고 사용자에게 보고합니다.
- `#스크립트`는 강사용 수업 진행 대본으로 보고 구어체를 허용합니다.
- 같은 강의 항목 안에서 자료 유형이 섞여 있으면 `본문 → #스크립트 → #실습 → #퀴즈` 순서로 정리합니다.
- `####` 제목4는 하나의 `###` 강의 항목 안에서 핵심 개념, 설명 흐름 전환, 코드 예시, 정리/주의를 구분할 때만 사용합니다.
- 중요도 별표는 [중요도 별표 기준](./00.%20문서%20운영/중요도%20별표%20기준.md)에 따라 학생용 개념 본문 제목에만 제한적으로 붙입니다.
- Notion 업로드용 `_notion.md`는 원본 Markdown이 확정된 뒤 이 저장소 전용 로컬 `$notion-formatter`로 생성합니다.
- Notion 반영은 사용자의 명시 요청이 있을 때만 진행합니다. 기본은 필요한 블록만 부분 수정하고, 전체 교체 `replace` 방식은 사용자가 전체 교체를 명시적으로 요청한 경우에만 사용합니다.
- 세부 작업 기준은 [강의자료 검토 및 업로드 절차](./00.%20문서%20운영/강의자료%20검토%20및%20업로드%20절차.md)와 [15기 강의자료 개선 기준](./00.%20문서%20운영/15기%20강의자료%20개선%20기준.md)을 따릅니다.

## 정리 원칙

- 공통 템플릿을 채우기 위한 문서는 만들지 않습니다.
- 실제 과정 문서는 `courses/`에만 추가합니다.
- `docs/`에는 원본을 그대로 옮기지 않고 운영 기준과 진행 기록만 남깁니다.
- Notion 변환본은 업로드용 파생 파일이므로 원본 문서와 섞어서 관리하지 않습니다.
