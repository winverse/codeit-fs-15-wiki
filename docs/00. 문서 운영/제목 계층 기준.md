# 제목 계층 기준

이 문서는 강의자료에서 제목 단계를 사용할 때의 기준입니다. 목적은 문서 구조를 크게 바꾸는 것이 아니라, 학생이 한 강의 항목 안의 핵심 개념을 빠르게 파악할 수 있도록 작은 목차 역할을 만드는 것입니다.

## 기본 계층

- `##`: 과정 안의 큰 단원입니다. 예: `02. HTML 시작하기`, `03. CSS 시작하기`
- `###`: 하나의 강의 항목입니다. Notion에서는 기본적으로 토글 제목으로 사용합니다. 예: `03. 한글이 깨져요! (<meta> 태그, UTF-8) ⭐⭐`
- `####`: 하나의 `###` 강의 항목 안에서 개념을 나누는 소제목입니다.

## 제목4를 사용하는 경우

`####`는 하나의 `###` 강의 항목 안에서 아래 기준 중 하나에 해당할 때 사용합니다.

- 핵심 개념 이름을 구분할 때
  예: `문자 인코딩`, `UTF-8`, `charset`, `블록 요소`, `인라인 요소`
- 설명 흐름이 명확하게 바뀔 때
  예: `한글이 깨지는 이유`, `브라우저가 해석하는 방식`, `태그 작성 위치`
- 코드 예시의 목적을 구분할 때
  예: `HTML 문서 예시`, `CSS 적용 예시`, `잘못된 예시와 올바른 예시`
- 비교, 주의, 정리처럼 학습 포인트를 따로 묶을 때
  예: `정리`, `주의할 점`, `비교`

## 제목4를 사용하지 않는 경우

아래 항목은 제목4로 올리지 않습니다.

- 한 문장짜리 보충 설명
- 단순 목록의 항목명
- `참고`, `예시 설명`, `좀 더 쉽게 설명하면`처럼 의미가 느슨한 라벨
- 강사용 `#스크립트` 안의 말하기 흐름
- 실습 링크만 들어 있는 `#실습` 항목
- 퀴즈의 개별 문제 제목

## 문체 기준

- 제목4는 명사형 개념 이름을 우선합니다. 예: `문자 인코딩`, `HTML 문서의 기본 구조`
- `~해보기`, `~해볼게요`, `좀 더 쉽게 설명하면`처럼 말하기 표현은 학생용 본문의 제목4로 사용하지 않습니다.
- 제목4 아래 설명은 `합니다/입니다`체의 교과서형 문체로 작성합니다.
- 제목4를 너무 많이 만들지 않습니다. 하나의 `###` 안에서 3~5개 정도를 기준으로 보고, 내용이 짧으면 굵은 문단이나 목록으로 둡니다.

## Notion 반영 기준

- Markdown 원본에서는 `####`를 사용합니다.
- Notion에서는 `heading_4` 블록으로 반영합니다.
- `###` 토글 내부의 작은 목차로만 사용하고, `####`를 새로운 강의 항목처럼 사용하지 않습니다.
- Notion 변환 과정에서 `####`가 일반 문단으로 내려가면 formatter를 수정해야 합니다.

## 예시

```markdown
### 03. 한글이 깨져요! (<meta> 태그, UTF-8) ⭐⭐

#### 문자 인코딩

문자 인코딩은 문자를 컴퓨터가 저장하고 전송할 수 있는 바이트 값으로 표현하는 규칙입니다.

#### UTF-8

UTF-8은 유니코드 문자를 표현하는 대표적인 문자 인코딩 방식입니다.

#### <meta charset="UTF-8">

`<meta charset="UTF-8">`는 현재 HTML 문서가 UTF-8 문자 인코딩으로 작성되었다는 사실을 브라우저에 알려 주는 태그입니다.

#### 정리

- HTML 문서는 기본적으로 `<head>` 안에 `<meta charset="UTF-8">`를 작성합니다.
- `charset`은 문서를 해석할 때 사용할 문자 인코딩을 지정합니다.
```
