# 강의자료 검토 및 업로드 절차

이 문서는 `courses/*/강의자료.md`를 검토하고 Notion에 반영할 때의 기준 절차입니다. 현재 기준은 원문을 크게 재구성하는 것이 아니라, 원문을 보존한 상태에서 교과서형 문체와 명확한 오류만 점검하는 것입니다.

## 핵심 원칙

1. 원문에서 내용, 말투, 억양이 교과서적으로 적절한지 검토하고, 교과서적으로 적절하지 않은 학생용 본문은 수정합니다.
2. 명확하게 잘못된 내용이 있으면 `courses/*/강의자료.md`에서 수정합니다.
3. 최신 라이브러리, 패키지, 외부 서비스 정책처럼 현재 문서 확인이 필요한 내용은 직접 수정하지 않고 사용자에게 확인 필요 항목으로 보고합니다.

## 원문 보존 원칙

- 강의 순서, 제목, 코드 예시, 이미지, zip 링크, 스크립트 흐름은 원칙적으로 유지합니다.
- 원문에 있던 코드나 예시를 삭제하지 않습니다.
- 문서 구조를 크게 바꾸거나 섹션을 새로 나누지 않습니다.
- 같은 강의 항목 안에서 자료 유형이 섞여 있으면 `본문 → #스크립트 → #실습 → #퀴즈` 순서로 정리합니다.
- `#스크립트`는 강사용 대본이므로 구어체를 허용합니다.
- `#스크립트`를 교과서형 문장으로 전면 변환하지 않습니다.
- 퀴즈, 실습, 문제은행, 숙제, 추가 설명은 사용자가 명시적으로 요청하지 않으면 새로 만들지 않습니다.

## 수정할 수 있는 범위

아래 항목은 직접 수정할 수 있습니다.

- 교과서형 문체에 맞지 않는 학생용 본문 표현
- 명확한 오탈자와 비문
- HTML, CSS, JavaScript 문법상 잘못된 예시
- 코드블록이 깨져 제목이나 본문이 코드블록 안으로 들어가는 문제
- 링크, 이미지, zip 표기가 명백히 깨진 경우
- 원문 내용과 충돌하는 잘못된 설명

아래 항목은 직접 수정하지 않고 사용자에게 보고합니다.

- 최신 라이브러리나 패키지 버전 변경
- 설치 방법, CLI 명령, 외부 서비스 UI처럼 현재 상태가 바뀔 수 있는 내용
- 원문에 없는 실습 목표, 완료 기준, 문제은행, 숙제 추가
- 코드잇 실습 페이지 내부 문제를 확인해야 판단할 수 있는 내용
- 강의 구성 자체를 바꾸는 큰 구조 변경

## 검토 절차

1. `references/raw/notion/NN.제목`의 원문과 `courses/NN.제목/강의자료.md`를 대조합니다.
2. 원문에 있던 코드, 이미지, zip, 스크립트가 빠지지 않았는지 확인합니다.
3. 같은 강의 항목 안의 순서가 `본문 → #스크립트 → #실습 → #퀴즈`인지 확인합니다.
4. 원문에서 내용, 말투, 억양이 교과서적으로 적절한지 확인합니다.
5. 잘못된 내용, 교과서형 문체에 맞지 않는 학생용 본문, 명확한 비문은 직접 수정합니다.
6. 코드 예시가 문법적으로 맞고 코드블록이 정상적으로 닫혔는지 확인합니다.
7. 별표가 [중요도 별표 기준](./중요도%20별표%20기준.md)에 맞는지 확인합니다.
8. 현재 문서 확인이 필요한 최신성 이슈는 수정하지 않고 목록으로 보고합니다.
9. 직접 수정한 뒤에는 수정 범위와 이유를 짧게 남깁니다.

## Notion 반영 절차

Notion에 반영할 때는 항상 로컬 원본을 기준으로 전체 교체 업로드합니다.

1. `courses/NN.제목/강의자료.md`를 수정합니다.
2. `$notion-formatter`로 `강의자료_notion.md`를 다시 생성합니다.
3. publisher dry-run으로 이미지, 파일, 퀴즈 토글, 코드블록 개수를 확인합니다.
4. 사용자가 업로드를 명시적으로 요청한 경우에만 Notion에 업로드합니다.
5. 업로드는 기존 Notion 페이지 내용을 archive 처리하고, 생성된 `강의자료_notion.md`를 통째로 다시 올리는 `replace` 방식으로 진행합니다.

`$notion-formatter`는 이 저장소 전용 로컬 스킬입니다. Codex가 레포 로컬 스킬로 읽는 `.agents/skills/notion-formatter`에 두고, 명령은 저장소 루트에서 해당 경로로 실행합니다. 전역 스킬 위치인 `~/.codex/skills`나 `~/.agents/skills`에는 같은 스킬을 두지 않습니다.

기본 생성 명령:

```bash
node .agents/skills/notion-formatter/scripts/format_notion_markdown.mjs \
  --input "courses/NN.제목/강의자료.md"
```

업로드 전 확인 명령:

```bash
node .agents/skills/notion-formatter/scripts/format_and_publish_notion_course.mjs \
  --input "courses/NN.제목/강의자료.md" \
  --page-id "<notion-page-id-or-url>" \
  --dry-run
```

실제 업로드 명령:

```bash
node .agents/skills/notion-formatter/scripts/format_and_publish_notion_course.mjs \
  --input "courses/NN.제목/강의자료.md" \
  --page-id "<notion-page-id-or-url>"
```

## 업로드 후 확인

- 기존 top-level block이 archive 처리되었는지 확인합니다.
- 새 top-level block, toggle heading, image, file, quiz toggle 개수를 확인합니다.
- 업로드 결과에 `verifiedTopLevel`, `verifiedToggleHeading3` 값이 있는지 확인합니다.
- Notion 화면에서 대표 섹션 1-2개를 직접 확인합니다.

## 보고 형식

검토만 한 경우:

```markdown
- 수정 필요:
- 이유:
- 직접 수정 가능 여부:
- 사용자 확인 필요:
```

수정 및 업로드까지 한 경우:

```markdown
- 수정 파일:
- 수정 범위:
- formatter 결과:
- dry-run 결과:
- Notion 업로드 결과:
- 남은 확인 필요 항목:
```
