# 강의자료 검토 및 업로드 절차

이 문서는 과정별 강의자료를 최초 정리하고 Notion에 반영한 뒤, 업로드된 Notion 페이지를 섹션 단위로 교정할 때의 기준 절차입니다. 현재 기준은 원문을 크게 재구성하는 것이 아니라, 원문을 보존한 상태에서 교과서형 문체와 명확한 오류만 점검하는 것입니다.

## 핵심 원칙

1. 원문에서 내용, 말투, 억양이 교과서적으로 적절한지 검토하고, 교과서적으로 적절하지 않은 학생용 본문은 수정합니다.
2. 명확하게 잘못된 내용이 있으면 최초 Notion 업로드 전에는 `courses/*/강의자료.md`에서 수정합니다.
3. 최초 Notion 업로드 이후에는 `courses/*/강의자료.md`와 `_notion.md`를 갱신하지 않고, Notion MCP/API로 업로드된 페이지의 해당 섹션 또는 블록만 수정합니다.
4. 최신 라이브러리, 패키지, 외부 서비스 정책처럼 현재 문서 확인이 필요한 내용은 직접 수정하지 않고 사용자에게 확인 필요 항목으로 보고합니다.
5. 최초 Notion 업로드 이후 페이지 전체나 `heading_2` 단위 검토가 필요하면 [Notion 페이지 검토 방법](./Notion%20페이지%20검토%20방법.md)에 따라 수정 없이 문제 후보를 먼저 보고합니다.

## 기준 문서 우선순위

강의자료 작업에서 기준이 충돌하거나 애매하면 아래 순서로 판단합니다.

1. 학생용 본문의 개념 설명, 문체, 원문 보존, 수정 가능 범위는 이 문서와 [15기 강의자료 개선 기준](./15기%20강의자료%20개선%20기준.md)을 따릅니다.
2. 제목 단계는 [제목 계층 기준](./제목%20계층%20기준.md), 중요도 별표는 [중요도 별표 기준](./중요도%20별표%20기준.md), 퀴즈 개선은 [퀴즈 작성 기준](./퀴즈%20작성%20기준.md)을 따릅니다.
3. `$notion-formatter`는 최초 Notion 업로드 또는 사용자가 명시한 전체 교체를 위한 변환/업로드 도구입니다. 학생용 본문의 개념 설명이나 문체를 판단하는 기준으로 사용하지 않습니다.
4. 최초 Notion 반영 이후 섹션 단위 교정에서는 이 문서와 관련 기준 문서를 먼저 보고, Notion MCP/API로 해당 섹션 또는 블록만 수정합니다. 이때 `$notion-formatter`는 코드블록, 인라인 코드, quote 블록 같은 Notion 표시 방식 확인이 필요할 때만 보조로 참고합니다.
5. `AGENTS.md`는 작업자가 빠르게 따르기 위한 요약입니다. 세부 판단은 `docs/`의 기준 문서를 우선합니다.

## 작업 단계 구분

과정별 문서는 `최초 전체 정리 → 최초 Notion 반영 → 섹션 단위 교정` 순서로 다룹니다.

### 1. 최초 전체 정리

사용자가 `03을 진행해줘`, `03을 새롭게 만들어줘`처럼 아직 정리되지 않은 과정을 요청하면 최초 전체 정리 단계로 봅니다.

이 단계에서는 해당 과정 전체를 읽고 아래 기준을 한 번에 점검할 수 있습니다.

- 학생용 본문의 교과서형 문체
- `###`, `####` 제목 계층
- 본문, 스크립트, 실습, 퀴즈의 배치
- 강사용 스크립트와 학생용 본문의 분리
- 중요도 별표 기준
- 코드블록 언어와 코드 예시의 문법
- 백틱 인라인 코드의 Notion 변환 규칙
- 제목에 백틱이나 인라인 코드 스타일이 들어가지 않았는지 여부
- 이미지, zip, 링크 누락 여부

단, 이 단계에서도 원문에 있던 코드, 이미지, zip, 링크, 스크립트 흐름을 임의로 삭제하지 않습니다.

### 2. 최초 Notion 반영

최초 전체 정리가 끝난 뒤 사용자가 Notion 반영을 요청하면, 해당 과정의 정리본을 기준으로 전체 업로드 또는 전체 교체를 진행할 수 있습니다.

- 업로드 전까지는 `courses/NN.제목/강의자료.md`를 기준 초안으로 둡니다.
- `$notion-formatter`로 `강의자료_notion.md`를 생성합니다.
- publisher dry-run으로 이미지, 파일, 퀴즈 토글, 코드블록, 인라인 코드 규칙을 확인합니다.
- 최초 반영 단계에서는 전체 교체 업로드를 사용할 수 있습니다.
- 최초 업로드가 끝난 뒤부터는 Notion 페이지가 교정 대상이 되며, 로컬 `강의자료.md`와 `_notion.md`는 최초 업로드용 스냅샷으로 취급합니다.

### 3. 섹션 단위 교정

최초 Notion 반영이 끝난 뒤에는 사용자가 Notion 화면을 보며 한 줄 또는 한 섹션씩 교정하는 단계로 봅니다.

이 단계에서는 사용자가 명시적으로 요청하지 않는 한 아래 작업을 하지 않습니다.

- 과정 전체를 다시 교과서 톤으로 재작성
- 전체 제목 계층 재구성
- 전체 `_notion.md` 기준 재업로드
- 기존 Notion 페이지 전체 삭제 후 재업로드
- 요청받지 않은 섹션의 구조 변경

기본 작업 단위는 사용자가 지적한 섹션 또는 블록입니다. 이 단계에서는 로컬 `강의자료.md`와 `강의자료_notion.md`를 수정하지 않습니다. Notion MCP/API로 업로드된 페이지에서 해당 섹션 또는 블록을 찾아 필요한 범위만 `update`, `append`, `delete`로 수정합니다.

사용자가 특정 `heading_2` 단원 전체나 페이지 전체 검토를 요청하면 이 단계에서도 바로 수정하지 않습니다. 먼저 [Notion 페이지 검토 방법](./Notion%20페이지%20검토%20방법.md)에 따라 해당 범위의 스냅샷을 만들고, 제목 스타일, 번호, 개념 본문, `#스크립트`, `#실습`, `#요약`, `#퀴즈`의 연결성을 검토한 뒤 문제 후보만 보고합니다.

## 원문 보존 원칙

- 강의 순서, 제목, 코드 예시, 이미지, zip 링크, 스크립트 흐름은 원칙적으로 유지합니다.
- 원문에 있던 코드나 예시를 삭제하지 않습니다.
- 문서 구조를 크게 바꾸거나 섹션을 새로 나누지 않습니다.
- 같은 주제를 다루는 개념 본문, `#스크립트`, `#실습`, `#퀴즈`는 같은 섹션 번호를 유지합니다. 번호는 새 개념이 시작될 때만 증가합니다.
- 같은 강의 항목 안에서 자료 유형이 섞여 있으면 `본문 → #스크립트 → #실습 → #퀴즈` 순서로 정리합니다.
- `#스크립트`는 강사용 대본이므로 구어체를 허용합니다.
- `#스크립트`를 교과서형 문장으로 전면 변환하지 않습니다.
- Notion의 왼쪽 세로 바(`|`, quote 블록)는 `#스크립트`에만 사용합니다. 스크립트가 아닌 개념 설명은 quote 블록으로 만들지 않고 일반 문단, 목록, 소제목으로 정리합니다.
- 퀴즈, 실습, 문제은행, 숙제, 추가 설명은 사용자가 명시적으로 요청하지 않으면 새로 만들지 않습니다.

## 수정할 수 있는 범위

아래 항목은 직접 수정할 수 있습니다.

- 교과서형 문체에 맞지 않는 학생용 본문 표현
- 명확한 오탈자와 비문
- HTML, CSS, JavaScript 문법상 잘못된 예시
- 코드블록이 깨져 제목이나 본문이 코드블록 안으로 들어가는 문제
- 링크, 이미지, zip 표기가 명백히 깨진 경우
- 원문 내용과 충돌하는 잘못된 설명

아래 항목은 직접 수정하지 않고 사용자에게 보고합니다.

- 최신 라이브러리나 패키지 버전 변경
- 설치 방법, CLI 명령, 외부 서비스 UI처럼 현재 상태가 바뀔 수 있는 내용
- 원문에 없는 실습 목표, 완료 기준, 문제은행, 숙제 추가
- 코드잇 실습 페이지 내부 문제를 확인해야 판단할 수 있는 내용
- 강의 구성 자체를 바꾸는 큰 구조 변경

## 검토 절차

1. 최초 업로드 전에는 `references/raw/notion/NN.제목`의 원문과 `courses/NN.제목/강의자료.md`를 대조합니다.
2. 최초 업로드 이후에는 Notion MCP/API로 업로드된 페이지의 해당 섹션을 읽고, 그 섹션 안에서만 검토합니다.
3. 원문에 있던 코드, 이미지, zip, 스크립트가 빠지지 않았는지 확인합니다.
4. 같은 강의 항목 안의 순서가 `본문 → #스크립트 → #실습 → #퀴즈`인지 확인합니다.
5. 원문에서 내용, 말투, 억양이 교과서적으로 적절한지 확인합니다.
6. 잘못된 내용, 교과서형 문체에 맞지 않는 학생용 본문, 명확한 비문은 작업 단계에 따라 직접 수정합니다.
7. 퀴즈를 검토하거나 사용자의 요청으로 개선할 때는 [퀴즈 작성 기준](./퀴즈%20작성%20기준.md)에 따라 단순 암기형 비율, 코드 해석형/상황 판단형/오류 수정형 비율, 정답과 해설의 명확성을 확인합니다.
8. 코드 예시가 문법적으로 맞고 코드블록이 정상적으로 닫혔는지 확인합니다.
9. 별표가 [중요도 별표 기준](./중요도%20별표%20기준.md)에 맞는지 확인합니다.
10. 현재 문서 확인이 필요한 최신성 이슈는 수정하지 않고 목록으로 보고합니다.
11. 직접 수정한 뒤에는 수정 범위와 이유를 짧게 남깁니다.

## Notion 반영 절차

Notion 반영 방식은 작업 단계에 따라 다릅니다.

- 최초 Notion 반영 단계: 전체 업로드 또는 전체 교체를 사용할 수 있습니다.
- 최초 반영 이후 교정 단계: 로컬 Markdown을 갱신하지 않고 Notion MCP/API로 필요한 섹션 또는 블록만 부분 수정합니다.
- 최초 반영 이후 전체 교체 업로드는 사용자가 "전체 교체", "`replace`로 업로드", "통째로 다시 올려"처럼 명시적으로 요청한 경우에만 사용합니다.

최초 Notion 반영 전:

1. `courses/NN.제목/강의자료.md`를 수정합니다.
2. `$notion-formatter`로 `강의자료_notion.md`를 다시 생성합니다.
3. publisher dry-run으로 이미지, 파일, 퀴즈 토글, 코드블록 개수를 확인합니다.
4. 최초 반영이거나 사용자가 전체 교체를 명시한 경우에만 `--replace`를 붙여 업로드합니다.

최초 Notion 반영 후:

1. Notion MCP/API로 이미 올라간 페이지에서 수정 대상 섹션 또는 블록을 찾습니다.
2. 수정 전 해당 섹션의 현재 내용을 읽고, 사용자가 요청한 범위와 맞는지 확인합니다.
3. 한 섹션, 코드블록 언어, 인라인 스타일처럼 좁은 변경은 해당 블록만 `update` 또는 필요한 최소 API 호출로 수정합니다.
4. 필요한 경우 섹션 안의 일부 자식 블록만 삭제하고 다시 추가합니다.
5. 로컬 `courses/NN.제목/강의자료.md`와 `강의자료_notion.md`는 수정하지 않습니다.
6. 최초 반영 이후 전체 교체가 필요하다고 판단되더라도 먼저 사용자에게 전체 교체 여부를 확인합니다.

`$notion-formatter`는 이 저장소 전용 로컬 스킬입니다. Codex가 레포 로컬 스킬로 읽는 `.agents/skills/notion-formatter`에 두고, 명령은 저장소 루트에서 해당 경로로 실행합니다. 전역 스킬 위치인 `~/.codex/skills`나 `~/.agents/skills`에는 같은 스킬을 두지 않습니다.

최초 업로드 전 기본 생성 명령:

```bash
node .agents/skills/notion-formatter/scripts/format_notion_markdown.mjs \
  --input "courses/NN.제목/강의자료.md"
```

최초 업로드 전 확인 명령:

```bash
node .agents/skills/notion-formatter/scripts/format_and_publish_notion_course.mjs \
  --input "courses/NN.제목/강의자료.md" \
  --page-id "<notion-page-id-or-url>" \
  --dry-run
```

최초 Notion 반영이거나 사용자가 전체 교체를 명시적으로 요청한 경우에만 실행하는 명령:

```bash
node .agents/skills/notion-formatter/scripts/format_and_publish_notion_course.mjs \
  --input "courses/NN.제목/강의자료.md" \
  --page-id "<notion-page-id-or-url>" \
  --replace
```

## 업로드 후 확인

- 부분 수정인 경우 수정한 블록만 Notion 화면에서 직접 확인합니다.
- 최초 업로드 이후 부분 수정인 경우 로컬 Markdown을 갱신하지 않았음을 보고합니다.
- 전체 교체인 경우에만 기존 top-level block archive 여부와 새 top-level block, toggle heading, image, file, quiz toggle 개수를 확인합니다.
- 전체 교체인 경우 업로드 결과에 `verifiedTopLevel`, `verifiedToggleHeading3` 값이 있는지 확인합니다.
- Notion 화면에서 대표 섹션 1-2개를 직접 확인합니다.

## 보고 형식

검토만 한 경우:

```markdown
- 수정 필요:
- 이유:
- 직접 수정 가능 여부:
- 사용자 확인 필요:
```

수정 및 업로드까지 한 경우:

```markdown
- 수정 파일:
- 수정 범위:
- formatter 결과:
- dry-run 결과:
- Notion 업로드 결과:
- 로컬 Markdown 갱신 여부:
- 남은 확인 필요 항목:
```
