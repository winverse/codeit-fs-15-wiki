# 강의자료 최종 검수 절차

이 문서는 강의자료를 최종 검수할 때 사용하는 절차입니다. 최초 Notion 업로드 전에는 `courses/*/강의자료.md`를 검수하고, 최초 업로드 이후에는 Notion MCP/API로 업로드된 페이지의 해당 섹션을 검수합니다. 현재 최종 검수의 목적은 원문을 크게 바꾸는 것이 아니라, 학생용 본문이 교과서적으로 읽히는지와 명확한 오류가 있는지를 확인하는 것입니다.

상세 작업 범위와 Notion 반영 방식은 [강의자료 검토 및 업로드 절차](./강의자료%20검토%20및%20업로드%20절차.md)를 따릅니다.
최초 Notion 업로드 이후 페이지 전체나 `heading_2` 단위로 검토할 때는 [Notion 페이지 검토 방법](./Notion%20페이지%20검토%20방법.md)을 따릅니다.

## 기본 원칙

- 자동 검사는 최종 판단이 아니라 보조 도구입니다.
- 최종 통과 여부는 문서를 직접 읽고 판단합니다.
- 원문에 있던 코드, 이미지, zip, 스크립트 흐름은 보존합니다.
- `#스크립트`는 강사용 수업 진행 대본이므로 구어체를 허용합니다.
- 학생용 본문은 `합니다/입니다`체의 교과서형 문체로 읽혀야 합니다.
- 학생용 개념 설명에는 Notion 왼쪽 세로 바(`|`, quote 블록)를 사용하지 않습니다. quote 블록은 `#스크립트` 말하기 흐름에만 사용합니다.
- 최신 라이브러리, 패키지, 외부 서비스 정책처럼 현재 문서 확인이 필요한 내용은 직접 수정하지 않고 사용자에게 보고합니다.

## 검수 대상

- 학생용 본문: 개념 설명, 정의, 이유, 예시, 정리
- 강사용 대본: `#스크립트` 섹션
- 실습 안내: `#실습` 섹션과 코드잇 링크
- 퀴즈: `#퀴즈` 섹션, 문제, 선택지, 정답 및 해설
- 코드와 코드블록: HTML, CSS, JavaScript, 명령어, 예시 코드
- 자료 링크: 이미지, zip, 외부 링크

## 자동 검사

자동 검사는 아래 문제를 빠르게 찾기 위한 1차 필터입니다.

- `다룬다`, `일반적이다`, `확인한다` 같은 학생용 본문의 `이다/한다`체
- `해볼게요`, `보시면`, `하겠습니다` 같은 학생용 본문의 말하기 표현
- `하습니다`, `것입니다..`, `것을로` 같은 비문
- 코드블록 개수 변화
- 코드블록 안에 `###` 제목이 들어간 경우
- `// 추가`처럼 언어에 맞지 않는 주석 문법
- Notion 변환 후 원본과 `_notion.md`의 구조 차이

자동 검사는 문제 후보를 찾는 용도입니다. 자동 치환으로 문체를 고치지 않습니다.

## 수동 검수

1. 원문 보존 확인
   - 원문에 있던 코드, 이미지, zip, 링크가 빠지지 않았는지 확인합니다.
   - 강사용 스크립트나 코드 예시를 임의로 삭제하지 않았는지 확인합니다.
   - 원문 구조를 크게 바꾸지 않았는지 확인합니다.
   - 같은 주제를 다루는 개념 본문, `#스크립트`, `#실습`, `#퀴즈`의 섹션 번호가 같은지 확인합니다.
   - 같은 강의 항목 안에서 자료 유형이 섞여 있으면 `본문 → #스크립트 → #실습 → #퀴즈` 순서인지 확인합니다.

2. 교과서형 문체 확인
   - 학생용 본문이 `합니다/입니다`체로 자연스럽게 읽히는지 확인합니다.
   - 원문의 내용, 말투, 억양이 교과서적으로 적절한지 확인합니다.
   - 문장이 교과서처럼 보이지만 설명 흐름이 어색하면 수정 필요로 봅니다.
   - `#스크립트`는 말하기 대본이므로 학생용 본문과 같은 문체 기준을 적용하지 않습니다.

3. 내용 정확성 확인
   - HTML, CSS, JavaScript 예시가 문법적으로 맞는지 확인합니다.
   - 설명이 코드 예시와 충돌하지 않는지 확인합니다.
   - 원문과 다른 내용으로 바뀐 경우 의도된 수정인지 확인합니다.
   - 같은 단원에서 반복되는 예시 값, 파일명, URL, 코드 결과가 개념 본문, 스크립트, 실습, 퀴즈 사이에서 일관되는지 확인합니다.
   - 누적 프로젝트 코드가 이전 섹션의 결과와 다음 섹션의 출발점으로 자연스럽게 이어지는지 확인합니다.

4. 중요도 별표 확인
   - 별표가 학생용 개념 본문 제목에만 붙어 있는지 확인합니다.
   - `#스크립트`, `#실습`, `#퀴즈`, `#요약`, `#Video`, 설치, 링크, 참고 자료에는 별표가 붙어 있지 않은지 확인합니다.
   - `⭐⭐⭐`가 너무 많아 학생의 우선순위를 흐리지 않는지 확인합니다.
   - 세부 기준은 [중요도 별표 기준](./중요도%20별표%20기준.md)을 따릅니다.

5. 퀴즈 품질 확인
   - 사용자의 요청으로 퀴즈를 개선한 경우 [퀴즈 작성 기준](./퀴즈%20작성%20기준.md)을 따랐는지 확인합니다.
   - 단순 암기형 문제가 전체의 20%를 넘지 않는지 확인합니다.
   - 15분 퀴즈라면 최소 6문항 이상이 코드 해석, 상황 판단, 오류 수정형인지 확인합니다.
   - 정답이 하나로 명확하고, `정답 및 해설`에 정답 이유와 주요 오답 이유가 설명되어 있는지 확인합니다.

6. 최신성 확인
   - 최신 라이브러리, 패키지, 외부 서비스 UI, 설치 명령은 직접 수정하지 않습니다.
   - 변경 가능성이 있는 내용은 확인 필요 항목으로 보고합니다.

7. Notion 반영 전 확인
   - 최초 업로드 전에는 `_notion.md`가 원본 수정 후 다시 생성되어야 합니다.
   - 최초 업로드 이후에는 로컬 Markdown을 갱신하지 않고 Notion MCP/API로 필요한 블록만 부분 수정합니다.
   - 전체 교체를 사용자가 명시적으로 요청한 경우에만 dry-run으로 이미지, 파일, 퀴즈 토글, 코드블록 개수를 확인한 뒤 `replace` 방식으로 업로드합니다.

8. Notion 페이지 단위 검토
   - 사용자가 Notion의 특정 `heading_2` 단원이나 페이지 전체 검토를 요청하면 [Notion 페이지 검토 방법](./Notion%20페이지%20검토%20방법.md)에 따라 스냅샷을 먼저 만듭니다.
   - 제목 스타일뿐 아니라 개념 본문, `#스크립트`, `#실습`, `#요약`, `#퀴즈`가 같은 흐름으로 이어지는지 확인합니다.
   - 검토 단계에서는 Notion과 로컬 Markdown을 수정하지 않고, 문제 후보만 보고합니다.

## 통과 기준

다음 조건을 만족해야 최종 통과로 봅니다.

- 학생용 본문이 교과서형 문체로 자연스럽게 읽힙니다.
- 명확한 내용 오류와 비문이 없습니다.
- 코드 예시와 코드블록 구조가 깨지지 않았습니다.
- 원문에 있던 주요 코드, 이미지, zip, 스크립트 흐름이 보존되어 있습니다.
- 같은 주제의 개념 본문, `#스크립트`, `#실습`, `#퀴즈`가 같은 섹션 번호로 묶여 있습니다.
- 같은 강의 항목 안의 자료 유형 순서가 `본문 → #스크립트 → #실습 → #퀴즈`로 정리되어 있습니다.
- 같은 단원 안의 예시 값, 파일명, URL, 코드 결과가 서로 충돌하지 않습니다.
- 별표가 [중요도 별표 기준](./중요도%20별표%20기준.md)에 따라 제한적으로 붙어 있습니다.
- 사용자의 요청으로 퀴즈를 개선한 경우 [퀴즈 작성 기준](./퀴즈%20작성%20기준.md)에 따라 문제 유형, 난이도, 시간, 해설이 검수되었습니다.
- 최신성 확인이 필요한 항목은 직접 수정하지 않고 보고되었습니다.
- 최초 업로드 또는 전체 교체 업로드가 필요한 경우 dry-run을 통과했습니다.
- 최초 업로드 이후 섹션 교정은 로컬 Markdown을 갱신하지 않고 Notion MCP/API에서 필요한 블록만 수정했습니다.
- 최초 업로드 이후 페이지 전체나 `heading_2` 단위 검토를 한 경우, 수정 전에 문제 후보와 확인 필요 항목을 먼저 보고했습니다.

## 보고 형식

```markdown
## 최종 검수 결과

- 대상:
- 판정: 통과 / 수정 필요
- 직접 수정한 내용:
- 수정하지 않고 보고할 내용:
- Notion 업로드 여부:
- 로컬 Markdown 갱신 여부:
- 남은 확인 필요 항목:
```

## 현재 확인된 엣지 케이스

- `합니다/입니다`로 끝나도 설명 흐름이 어색하면 통과가 아닙니다.
- 대본을 본문으로 바꾸거나 본문을 대본으로 옮기는 구조 변경은 사용자가 요청한 경우에만 진행합니다. 단, 같은 강의 항목 안에서 `본문 → #스크립트 → #실습 → #퀴즈` 순서를 맞추기 위한 최소 재배치는 허용합니다.
- 자동 치환은 `사용하습니다`, `것을로` 같은 비문을 만들 수 있으므로 사용하지 않습니다.
- 예시 코드 안의 뉴스레터 문장, HTML 본문 샘플, 외부 원문은 실제 교재 설명문과 분리해서 판단합니다.
- Notion 업로드용 문서에서는 `` **`<title>`** ``처럼 굵게와 인라인 코드를 중첩하지 않습니다. 태그명은 `` `<title>` ``처럼 인라인 코드만 사용하고, 업로드 과정에서 파란색 인라인 코드로 변환합니다.
- Notion 업로드 결과에서 백틱으로 감쌌던 인라인 코드는 파란색 텍스트로 표시되어야 하며, 굵게는 적용하지 않습니다. 이 기준은 인라인 코드에만 적용하며, 코드블록에는 적용하지 않습니다.
- `##`, `###`, `####` 제목에는 백틱을 사용하지 않습니다. 제목에 HTML 태그나 코드 조각이 포함되어도 인라인 코드 스타일을 적용하지 않고 일반 제목 텍스트로 둡니다.
- 코드 일부를 생략해야 할 때는 해당 언어에서 유효한 주석이나 설명 방식으로 표시합니다. 코드블록 안에 해당 언어의 문법이 아닌 임의의 설명 조각을 넣지 않습니다.
