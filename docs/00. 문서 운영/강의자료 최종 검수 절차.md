# 강의자료 최종 검수 절차

이 문서는 `courses/*/강의자료.md`를 최종 검수할 때 사용하는 절차입니다. 목적은 자동 검사로 잡을 수 있는 오류와 사람이 읽어야만 판단할 수 있는 품질 문제를 분리하고, 최종 통과 여부를 명확히 판단하는 것입니다.

## 기본 원칙

- 자동 검사는 최종 판단이 아니라 보조 도구입니다.
- 최종 통과 여부는 제가 문서를 직접 읽고 판단합니다.
- `#스크립트`는 강사용 수업 진행 대본이므로 구어체를 허용합니다.
- `#스크립트`가 아닌 본문, 퀴즈, 실습 안내, 꿀팁은 `합니다/입니다`체의 교과서형 문체로 작성합니다.
- 문장이 `합니다/입니다`로 끝나더라도 설명 흐름이 어색하거나 강의 대본처럼 보이면 최종 통과로 보지 않습니다.
- 검수 중 새 엣지 케이스가 발견되면 이 문서를 업데이트합니다.

## 검수 대상 구분

강의자료를 먼저 아래 유형으로 나눕니다.

- 학생용 본문: 개념 설명, 정의, 이유, 예시, 정리
- 강사용 대본: `#스크립트` 섹션
- 실습 안내: `#실습` 섹션, 링크, 준비물, 완료 기준
- 퀴즈: `#퀴즈` 섹션, 문제, 선택지, 정답 및 해설
- 코드와 코드블록: HTML, CSS, JavaScript, 명령어, 예시 코드
- 원본 보존 대상: 코드잇 링크, 이미지, zip, 원본 예시 문장

## 자동 검사

자동 검사는 아래 문제를 빠르게 찾기 위한 1차 필터입니다.

- `다룬다`, `일반적이다`, `확인한다` 같은 `이다/한다`체
- `해볼게요`, `보시면`, `하겠습니다` 같은 학생용 영역의 말하기 표현
- `하습니다`, `것입니다..`, `것을로` 같은 자동 치환 비문
- `#스크립트`가 아닌 영역에 남은 긴 인용문
- 코드블록 개수 변화
- Notion 변환 후 원본과 `_notion.md`의 구조 차이

자동 검사는 문제 후보를 찾는 용도이며, 자동 치환으로 문체를 고치지 않습니다. 문체 수정은 섹션 단위로 읽고 직접 고칩니다.

## 수동 최종 검수

자동 검사 후에는 제가 아래 기준으로 직접 읽습니다.

1. 구조 검수
   - 학생용 본문과 강사용 대본이 섞여 있지 않은지 확인합니다.
   - 원본 강의 항목 내부에 대본이 들어 있으면 같은 번호의 `#스크립트` 섹션으로 분리합니다.
   - 같은 번호 안에서는 `본문 > #스크립트 > #실습 > #퀴즈` 순서를 우선합니다.

2. 문체 검수
   - 학생용 본문은 `합니다/입니다`체로 통일합니다.
   - `이다/한다`체와 말하기 표현은 학생용 영역에서 제거합니다.
   - 퀴즈 선택지와 해설도 학생이 보는 문장이므로 같은 문체 기준을 적용합니다.
   - 코드 주석, 예시 콘텐츠, 외부 원문은 문체 변환 대상인지 먼저 판단합니다.

3. 설명 품질 검수
   - 정의, 이유, 예시, 정리 순서가 자연스러운지 확인합니다.
   - 문장이 교과서처럼 보이지만 실제 설명이 어색한 경우 다시 씁니다.
   - 용어가 처음 나올 때 필요한 만큼만 정의되어 있는지 확인합니다.
   - 같은 개념을 불필요하게 반복하지 않는지 확인합니다.

4. 실습 검수
   - 코드잇 실습 페이지의 내부 문제를 확인하지 못한 경우, 목표나 성공 기준을 추정해 쓰지 않습니다.
   - 실습 링크만 있는 경우 원문을 유지하거나 최소 안내만 둡니다.
   - 실제 코드가 포함된 경우 파일명, 실행 흐름, 결과 확인 방법이 맞는지 확인합니다.

5. 퀴즈 검수
   - 문제 난이도와 풀이 시간이 적절한지 확인합니다.
   - 정답만 있지 않고 해설이 충분한지 확인합니다.
   - 오답이 왜 오답인지 설명할 필요가 있는 문제는 해설을 보강합니다.
   - 단순 링크형 퀴즈는 링크 내용을 확인하기 전까지 문제를 임의로 추가하지 않습니다.

## 통과 기준

다음 조건을 만족해야 최종 통과로 봅니다.

- 학생용 영역에 강사용 대본이 남아 있지 않습니다.
- 학생용 본문, 실습 안내, 퀴즈가 `합니다/입니다`체로 자연스럽게 읽힙니다.
- 자동 검사에서 발견된 비문과 명확한 금지 표현이 없습니다.
- 사람이 읽었을 때 문장 흐름이 교과서형 설명으로 자연스럽습니다.
- 코드, 링크, 이미지, zip 같은 자료 연결이 깨지지 않습니다.
- `_notion.md`는 원본 수정 후 다시 생성되어 있습니다.

## 판정 방식

검수 후 결과는 아래처럼 남깁니다.

```markdown
## 최종 검수 결과

- 대상:
- 판정: 통과 / 수정 필요
- 자동 검사 결과:
- 수동 검수 결과:
- 수정한 내용:
- 남은 보류 사항:
```

`통과`는 바로 Notion 업로드 또는 커밋 가능한 상태를 의미합니다. `수정 필요`는 자동 검사를 통과했더라도 사람이 읽었을 때 구조나 문체가 아직 맞지 않는 상태를 의미합니다.

## 현재 확인된 엣지 케이스

- `합니다/입니다`로 끝나도 강사용 대본처럼 읽히는 문장은 최종 통과가 아닙니다.
- `#스크립트`가 아닌 영역의 긴 인용문은 학생용 본문인지 강사용 대본인지 먼저 판정해야 합니다.
- 자동 치환은 `사용하습니다`, `것을로` 같은 비문을 만들 수 있으므로 사용하지 않습니다.
- 퀴즈 선택지는 짧은 문장이지만 학생이 보는 문장이므로 문체 기준을 적용합니다.
- 예시 코드 안의 뉴스레터 문장, HTML 본문 샘플, 외부 원문은 실제 교재 설명문과 분리해서 판단합니다.

## 운영 방식

- 실제 강의자료를 검수할 때마다 새로 발견한 예외를 이 문서에 추가합니다.
- 기준이 바뀌면 [15기 강의자료 개선 기준](./15기%20강의자료%20개선%20기준.md)에도 필요한 만큼 반영합니다.
- 특정 과정에서 반복되는 문제는 과정별 개선 메모에 남기고, 다음 과정 작업 전에 먼저 확인합니다.

## 시범 적용 기록

### 2026-06-04: 01, 02 문체 검수

- `01. 프로그래밍 시작하기`
  - 자동 검사: 학생용 일반 본문 기준 `이다/한다`체와 비문 후보가 남지 않았습니다.
  - 수동 검수: 구조상 `#스크립트`, `#퀴즈`, 실습 링크가 비교적 분리되어 있어 문체 기준을 대체로 통과할 수 있습니다.
  - 보류: 세부 퀴즈 품질과 풀이 시간은 별도 검수 대상입니다.

- `02. 웹 퍼블리싱 시작하기`
  - 자동 검사: `#스크립트` 밖의 인용문과 말하기식 표현 후보가 남지 않았습니다.
  - 수동 검수: 일반 본문 안에 섞여 있던 강사용 대본 성격의 인용문을 같은 번호의 `#스크립트` 섹션으로 분리했습니다.
  - Notion 변환: `courses/02.웹 퍼블리싱 시작하기/강의자료.md`를 기준으로 `강의자료_notion.md`를 다시 생성했습니다. 과정형 `## 01. ...` 제목 경고는 현재 레포 구조에서 허용합니다.
  - 보류: 세부 퀴즈 품질과 실습 링크 내부 문제 확인은 별도 검수 대상입니다.

### 다음 반복에서 확인할 점

- 새 과정 문서를 정리할 때 `#스크립트`가 아닌 영역의 `>` 인용문을 먼저 검토합니다.
- 학생용 본문으로 남길 인용문은 교과서형 문장으로 다시 씁니다.
- 강사용 대본으로 판단되는 인용문은 같은 번호의 `#스크립트` 섹션으로 분리합니다.
- 문체 자동 치환을 사용하지 않고, 수정은 섹션 단위로 직접 수행합니다.
