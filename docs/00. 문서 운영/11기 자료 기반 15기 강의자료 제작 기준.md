# 11기 자료 기반 15기 강의자료 제작 기준

이 문서는 `references/raw`에 모아둔 11기 자료와 11기 코드 저장소를 바탕으로 15기 강의 자료를 만들거나 수정할 때 따를 기준입니다.

## 목적

- 11기에서 사용한 강의 흐름을 기준으로 15기 자료를 만든다.
- 강의 순서와 과정 제목은 유지한다.
- 내부 설명, 예제, 실습, 코드, 버전, 안내 문구는 15기 기준으로 개선한다.
- 원본 자료와 작성 결과물을 분리해서, 무엇을 참고했고 무엇을 새로 작성했는지 추적 가능하게 만든다.

## 기준 자료

| 구분 | 위치 | 역할 |
| :--- | :--- | :--- |
| 강의 순서와 제목 | `references/raw/notion/NN.제목` | 15기 과정 순서와 과정명 기준 |
| 11기 강의 원문 | `references/raw/notion/NN.제목/**/*.md` | 기존 설명, 목차, 실습 흐름 참고 |
| 이미지와 첨부 자료 | `references/raw/notion/NN.제목/**` | 기존 화면 자료와 첨부 파일 확인 |
| 11기 코드 | `https://github.com/winverse/codeit-fs.git`의 `codeit-fs-11/` | 예제 코드, 실습 코드, 완성 코드 확인 |
| 15기 최종 산출물 | `docs/` | 학생에게 제공할 문서와 강의 제작 자료 |

11기 코드 저장소는 이 문서를 만들 때 `4fb9847bc1574660ea5b3ccf3a2e0b3781982c15` 커밋 기준으로 확인했습니다. 실제 강의 자료를 작성할 때는 다시 최신 상태를 확인합니다.

## 고정 규칙

- 과정 폴더의 번호와 제목은 `references/raw/notion`과 동일하게 유지한다.
- 사용자 지시 없이 과정을 합치거나 나누거나 제목을 바꾸지 않는다.
- `references/raw` 아래 원본 자료는 수정하지 않는다.
- 11기 문서를 그대로 복사하지 않고, 15기 기준으로 설명과 실습을 다시 정리한다.
- 오래된 라이브러리 버전, 명령어, 정책, UI는 최종 작성 전에 현재 공식 문서나 실제 코드 실행 결과로 확인한다.
- 코드 예제가 있는 과정은 11기 코드 저장소의 경로를 문서에 남긴다.
- 원문과 코드가 충돌하면 원문은 강의 의도, 코드는 실습 상태로 보고 둘 다 확인한 뒤 15기 기준으로 정리한다.

## 과정별 작업 순서

1. `references/raw/notion/NN.제목` 폴더를 연다.
2. 루트 Markdown과 하위 Markdown을 읽고 기존 목차, 강의 단위, 실습 단위를 확인한다.
3. 과정 번호와 제목은 그대로 둔다.
4. 11기 코드 저장소의 `codeit-fs-11/`에서 관련 예제 또는 프로젝트를 찾는다.
5. 15기에서 유지할 내용, 수정할 내용, 삭제할 내용을 구분한다.
6. `docs/` 아래에 15기용 문서를 작성한다.
7. 작성한 문서에 참고한 원본 Markdown 경로와 코드 경로를 남긴다.
8. 코드가 포함된 문서는 설치, 실행, 테스트 명령을 실제 프로젝트 기준으로 확인한다.

## 과정 문서에 남길 추적 정보

각 과정 문서에는 가능한 한 아래 정보를 남깁니다.

```markdown
## 참고 자료

- 원본 Notion: references/raw/notion/NN.제목/...
- 참고 코드: codeit-fs-11/프로젝트명/...
- 확인한 실행 명령: npm run dev
- 업데이트 메모: 15기 기준으로 바꾼 내용
```

## 코드 확인 기준

- 패키지 매니저는 프로젝트별 lock 파일을 기준으로 확인한다.
- `package.json`의 `scripts`를 먼저 보고 실행 명령을 정한다.
- 스타터, 실습, 완성본이 나뉜 과정은 학생용 자료에는 스타터를 기준으로 쓰고, 강사용 검토에는 완성본 경로를 함께 남긴다.
- 11기 코드가 현재 버전에서 바로 실행되지 않으면, 오류 원인과 15기 수정 방향을 문서에 기록한다.
- 보안, OAuth, AWS, 배포, 데이터베이스처럼 외부 정책이 바뀌는 주제는 11기 코드만 믿지 않고 현재 공식 문서를 다시 확인한다.

## 산출물 작성 기준

과정별 15기 문서는 기본적으로 아래 구성을 따른다.

```text
NN.제목/
  목차.md
  교재/
  실습/
  퀴즈/
  대본/
  검토-메모.md
```

과정 특성상 필요 없는 유형은 만들지 않습니다. 다만 `목차.md`와 참고 자료 경로는 남겨서 다음 작업자가 같은 기준으로 이어서 작성할 수 있게 합니다.
