# 작성 규칙

이 문서는 `docs` 아래에 추가되는 모든 수업 자료의 기본 작성 규칙입니다.

## 디렉터리 구조

과정 단위 자료는 아래 구조를 기본으로 합니다.

```text
docs/
  NN. 과정명/
    목차.md
    교재/
    대본/
    실습/
    강의/
      S-LL. 강의명/
        학생용.md
        quiz.md
    자료/
    완성본/
```

## 번호 규칙

- 과정 번호는 두 자리 숫자를 사용합니다. 예: `01. 과정 개요`
- 섹션과 강의 번호는 `S-LL` 형식을 사용합니다. 예: `2-03. 상태 관리`
- 실습 파일은 검색하기 쉽게 언더스코어를 사용할 수 있습니다. 예: `2-03_상태_관리_실습.md`
- 같은 과정 안에서는 같은 번호를 재사용하지 않습니다.

## 자료 유형

| 유형 | 용도 |
| :--- | :--- |
| `목차.md` | 과정의 전체 흐름, 섹션, 강의 목록을 정리합니다. |
| `교재/` | 개념 설명 중심의 본문 자료를 작성합니다. |
| `대본/` | 영상 또는 라이브 강의 진행용 말하기 문서를 작성합니다. |
| `실습/` | 학생이 따라 할 수 있는 단계별 작업 지시를 작성합니다. |
| `강의/` | 짧은 강의 단위 자료를 관리합니다. |
| `학생용.md` | 학생에게 직접 제공할 강의 자료입니다. |
| `quiz.md` | 강의와 연결된 퀴즈, 정답, 해설입니다. |
| `자료/` | 이미지, 참고 링크, API 문서 등 보조 자료를 둡니다. |
| `완성본/` | 실습 완료 코드 또는 검증 자료를 둡니다. |

## Markdown 스타일

- 문서 제목은 `# 번호. 제목` 형식을 우선 사용합니다.
- 큰 흐름은 `##`, 세부 설명은 `###`로 나눕니다.
- 코드 블록에는 언어를 명시합니다.

```javascript
console.log("Hello, Codeit");
```

- 실습 문서는 체크리스트 형태를 우선 사용합니다.

```markdown
## 체크리스트

### □ 1단계: 프로젝트 생성

작업 설명을 작성합니다.
```

- 정답이나 해설이 필요한 문서는 문제와 해설을 명확히 분리합니다.
- 아직 확정되지 않은 내용은 `TODO:`로 표시합니다.

## 작성 원칙

- 강의 자료는 학생이 바로 읽고 따라 할 수 있게 씁니다.
- 대본은 말하기 흐름을 유지하되, 명령어와 파일명은 코드 형식으로 표시합니다.
- 실습은 한 단계에 하나의 주요 작업만 담습니다.
- 오래된 버전 정보, 명령어, 패키지명은 최종 배포 전에 다시 확인합니다.
