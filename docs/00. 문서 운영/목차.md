# 00. 문서 운영

15기 강의 자료를 만들 때의 최소 운영 기준입니다.

## 문서

### 01. 15기 강의자료 개선 기준

- 11기 원문을 보존하면서 15기 자료로 검토하는 기준
- 교과서형 문체와 명확한 내용 오류를 판단하는 기준
- 직접 수정할 항목과 사용자에게 보고할 항목을 구분하는 기준
- 최초 업로드 전 `courses/NN.제목/강의자료.md` 형태의 과정별 단일 초안 관리 기준
- Notion 업로드 직전에 이 저장소 전용 로컬 `$notion-formatter`로 `_notion.md`를 생성하는 기준
- 최초 업로드 이후 Notion MCP/API로 해당 섹션만 교정하는 기준

### 02. 강의자료 검토 및 업로드 절차

- 현재 작업 기준이 되는 검토, 수정, 업로드 절차
- 원문 보존 원칙
- 같은 강의 항목 안의 `본문 → #스크립트 → #실습 → #퀴즈` 순서 기준
- 중요도 별표를 검토 절차에 포함하는 기준
- 최신 라이브러리/패키지 이슈를 직접 수정하지 않고 보고하는 기준
- 최초 업로드 이후 로컬 Markdown을 갱신하지 않고 Notion MCP/API로 섹션만 수정하는 원칙
- 명시적 전체 교체 `replace` 절차

### 03. 중요도 별표 기준

- `⭐`, `⭐⭐`, `⭐⭐⭐`를 붙이는 판정 기준
- 별표를 붙이지 않는 제목 유형
- 단원별 별표 개수 제한 기준

### 04. 퀴즈 작성 기준

- HTML, CSS, JavaScript, React, Express, Jest, AWS 등 전체 과정에 적용하는 퀴즈 작성 기준
- 단순 암기형, 코드 해석형, 상황 판단형, 오류 수정형 문제의 비율 기준
- 5분, 10분, 15분 퀴즈의 문항 수와 난이도 기준
- AWS SAA형 상황 판단 문제 작성 기준

### 05. 제목 계층 기준

- `##`, `###`, `####`를 구분해서 쓰는 기준
- `###` 강의 항목 안에서 제목4를 사용하는 기준
- 제목4를 쓰지 않아야 하는 경우와 Notion 반영 기준

### 06. Notion 페이지 검토 방법

- 최초 업로드 이후 Notion 페이지나 `heading_2` 단원을 검토하는 방법
- 제목 스타일, 번호, 개념 본문, `#스크립트`, `#실습`, `#요약`, `#퀴즈` 연결성 검토 기준
- 스냅샷 기반 멀티 에이전트 검토와 문제 후보 보고 기준

### 07. 전체 구성

- 01부터 38까지의 기준 과정 목록
- 01~10 과정의 현재 문서 위치
- 아직 15기 문서를 만들지 않은 과정의 원본 위치

### 08. 01 기준 샘플

- `01. 프로그래밍 시작하기`의 최초 업로드 전 로컬 초안과 `_notion.md` 생성 기준
- 최초 업로드 이후 Notion MCP/API 섹션 교정 원칙
- formatter와 publisher dry-run 검증 절차
- 퀴즈 시간, 태그, 이미지, 파일 링크, 강사용/학생용 영역 점검 결과

불필요한 공통 템플릿은 두지 않습니다. 과정별 자료는 원본 흐름을 유지하면서 필요한 부분만 수정합니다.
