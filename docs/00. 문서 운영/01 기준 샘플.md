# 01 기준 샘플

`01. 프로그래밍 시작하기`는 이후 과정 문서를 정리할 때 기준으로 삼는 첫 샘플입니다.

## 기준 파일

- 원본 기준: `courses/01.프로그래밍 시작하기/강의자료.md`
- Notion 업로드용 파생본: `courses/01.프로그래밍 시작하기/강의자료_notion.md`
- 로컬 자산: `courses/01.프로그래밍 시작하기/01 프로그래밍 시작하기/`

## 동기화 절차

1. Notion에 사용자가 직접 수정한 내용이 있으면 Notion MCP/API로 읽기만 합니다.
2. 읽어 온 내용을 로컬 `강의자료.md`에 반영합니다.
3. Notion에는 `update`, `append`, `delete`, `patch`, `publish` 계열 작업을 하지 않습니다.
4. 로컬 원본이 확정되면 `$notion-formatter`로 `_notion.md`를 생성합니다.
5. Notion 반영이 필요할 때만 사용자의 명시 요청을 받은 뒤 `_notion.md`를 업로드합니다.

기본 변환 명령:

```bash
node ~/.codex/skills/notion-formatter/scripts/format_notion_markdown.mjs \
  --input "courses/01.프로그래밍 시작하기/강의자료.md"
```

업로드 전 구조 확인 명령:

```bash
node ~/.codex/skills/notion-formatter/scripts/publish_notion_course.mjs \
  --input "courses/01.프로그래밍 시작하기/강의자료_notion.md" \
  --page-id "<notion-page-id-or-url>" \
  --dry-run \
  --image-mode placeholder
```

실제 업로드는 사용자가 명시적으로 요청한 경우에만 수행합니다.

## 주의 사항

- `_notion.md`는 파생본입니다. 내용 오류는 `강의자료.md`에서 고치고 다시 생성합니다.
- formatter 경고 중 `section heading is not N-XX format`은 현재 과정형 제목인 `## 01. Web 소개` 구조 때문에 발생합니다. 이 repo의 `courses/NN.제목/강의자료.md` 구조에서는 허용합니다.
- API 업로드에서 로컬 이미지를 `github-raw` 모드로 쓰려면 이미지 파일이 git에 포함되어 있어야 합니다. 새로 내려받은 이미지는 커밋 대상에 포함해야 합니다.
- 수동 Notion import를 해야 하는 경우에는 이미지 번들 방식이 별도로 필요할 수 있습니다. 현재 기준은 Notion API publisher를 사용하는 흐름입니다.
