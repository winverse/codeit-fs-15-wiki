# 15기 강의자료 개선 기준

이 문서는 11기 자료를 15기 강의 자료로 수정할 때의 기준입니다. 목표는 문서 포맷을 크게 바꾸는 것이 아니라, 기존 흐름을 유지하면서 내용의 품질을 높이는 것입니다.

## 기준

- 과정 순서와 과정 제목은 `references/raw/notion`의 `NN.제목` 폴더명을 유지합니다.
- 11기 원문은 `references/raw/notion/NN.제목/**/*.md`에서 확인합니다.
- 실습 코드는 `references/code/*` 아래 자료를 기준으로 확인합니다.
- 원본 자료는 수정하지 않고, 15기용 결과는 `courses/`에 작성합니다.
- `docs/`에는 이 레포의 운영 기준과 진행 기록만 작성합니다.
- 불필요한 새 형식은 만들지 않습니다. 필요한 문서만 과정별로 추가합니다.
- 과정별 결과물은 `courses/NN.제목/v1/강의자료.md`처럼 버전 폴더 아래에 둡니다.
- Notion 업로드용 문서는 원본 Markdown 작성이 끝난 뒤에만 별도로 생성합니다.

## 개선 범위

15기 작업은 아래 세 가지에 집중합니다.

### 1. 퀴즈 개선

- 기존 퀴즈는 유지하되, 난이도를 `기본`, `중급`, `심화`로 조정합니다.
- 정답만 쓰지 않고, 왜 정답인지와 오답이 왜 틀렸는지 설명합니다.
- 반복 복습이 필요한 주제는 문제은행형 퀴즈를 추가합니다.
- 문제은행은 강의 흐름을 방해하지 않도록 별도 문서로 둘 수 있습니다.
- 퀴즈 섹션에 단순 링크만 들어 있는 경우에는 원문을 그대로 둡니다.
- 단순 링크형 퀴즈에 문제은행을 추가해야 한다면, 먼저 링크 내용을 직접 확인한 뒤 해당 내용에 근거해서만 문항을 만듭니다.
- 링크 내용을 확인하지 못한 상태에서는 퀴즈 문항이나 해설을 임의로 추가하지 않습니다.

예시:

```text
퀴즈/
  1-01_강의명_퀴즈.md
  1-01_강의명_문제은행.md
```

### 2. 문장 개선

- 구어체 설명은 교과서형 문장으로 다듬습니다.
- 정의, 이유, 예시, 정리 순서가 자연스럽게 이어지도록 고칩니다.
- 지나치게 긴 문장은 나누고, 한 문장에는 하나의 핵심만 담습니다.
- 대본이 아닌 교재 문서에서는 `해보죠`, `보시면` 같은 말하기 표현을 줄입니다.
- 어려운 용어는 처음 등장할 때 짧게 정의합니다.

### 3. 코드와 실습 개선

- `references/code/*`에 있는 실습 코드를 먼저 확인합니다.
- 11기 코드가 오래된 라이브러리나 명령어를 사용하면 15기 기준으로 업데이트합니다.
- 패키지 버전은 `package.json`과 lock 파일을 함께 보고 판단합니다.
- 실습이 교재를 따라가기 어렵다면 단계, 파일명, 실행 명령, 확인 방법을 다시 정리합니다.
- 새로운 실습 코드나 개선된 실습 코드가 필요하면 `references/code/*`의 기존 코드를 바탕으로 15기용 실습 흐름에 맞게 작성합니다.
- 코드가 포함된 문서는 실제 실행 명령과 확인 결과를 남깁니다.

## 코드 자료 사용 방식

`references/code/*`는 실습 코드 개선을 위한 기준 자료입니다.

- `references/code/codeit-fs-11/`: 11기에서 사용한 참고 코드
- 그 밖의 하위 폴더: 사용자가 추가한 실습 코드 또는 개선 후보 코드

작업할 때는 아래 순서로 봅니다.

1. 원본 Notion 문서에서 강의 의도와 실습 흐름을 확인합니다.
2. `references/code/*`에서 연결되는 코드를 찾습니다.
3. 현재 라이브러리 기준으로 유지할 코드와 바꿀 코드를 나눕니다.
4. 15기 문서에는 새 실습 흐름, 실행 명령, 확인 방법을 씁니다.
5. 새 코드가 필요하면 원본 코드를 그대로 복사하지 않고, 15기 실습 목적에 맞게 개선합니다.

## 과정별 기록

과정별 문서를 작성하거나 수정할 때는 필요한 경우 아래 정도만 남깁니다. 모든 강의에 같은 포맷을 강제하지 않습니다.

```markdown
## 개선 메모

- 원본 Notion: references/raw/notion/NN.제목/...
- 참고 코드: references/code/...
- 개선한 내용: 퀴즈 / 문장 / 코드 / 실습
- 확인한 명령: npm run dev
- 확인 결과:
- 보류 사항:
```

## 폴더 기반 버전 관리

과정별 문서는 폴더 단위로 버전을 나눕니다.

```text
courses/
  01.프로그래밍 시작하기/
    v1/
      강의자료.md
    v2/
      강의자료.md
```

- `v1`은 11기 자료를 바탕으로 처음 정리한 15기 개선 초안입니다.
- 같은 과정에서 문장, 퀴즈, 실습 코드가 크게 바뀌면 기존 버전을 직접 덮어쓰기보다 새 버전 폴더를 만듭니다.
- 작은 오탈자나 링크 수정은 현재 버전 안에서 수정할 수 있습니다.
- Notion 업로드용 `_notion.md`는 버전 폴더 안의 원본 Markdown이 확정된 뒤에만 생성합니다.

## Notion 변환 기준

`courses/`의 원본 Markdown이 15기 강의자료의 기준본입니다. Notion import용 파일은 업로드 직전에만 만드는 파생본입니다.

- 원본 문서 작성과 Notion 포맷 변환을 같은 작업으로 섞지 않습니다.
- 내용 수정은 항상 `courses/` 원본 Markdown에서 먼저 합니다.
- Notion 업로드가 필요할 때만 로컬 스킬 `$notion-formatter`를 사용해 `_notion.md`를 생성합니다.
- `_notion.md`는 제목, 토글, 이미지 마커, 퀴즈 정답 토글 같은 업로드 형식만 다루는 파일입니다.
- `_notion.md`에서 내용 오류를 발견하면 원본 Markdown을 수정한 뒤 다시 변환합니다.
- 기본 변환은 `##` 절 전체를 토글로 만들지 않습니다. 퀴즈 정답과 해설, 보충 설명처럼 접어둘 필요가 있는 부분만 토글로 둡니다.
- 이미지가 있는 문서는 Notion 업로드 직전에 이미지 번들과 업로드 가이드를 함께 생성합니다.

## 유지할 것과 삭제할 것

- 강의 순서와 제목은 유지합니다.
- 좋은 설명, 좋은 예시, 검증된 실습 흐름은 유지합니다.
- 오래된 버전 정보, 실행되지 않는 명령, 흐름을 방해하는 중복 설명은 수정하거나 삭제합니다.
- 단순 템플릿을 채우기 위한 문서는 만들지 않습니다.
- 문서 구조보다 실제 학습 흐름과 실습 성공 가능성을 우선합니다.
