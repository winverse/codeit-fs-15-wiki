# 15기 강의자료 개선 기준

이 문서는 11기 자료를 15기 강의 자료로 검토하고 필요한 부분만 수정할 때의 기준입니다. 현재 목표는 문서 포맷이나 강의 흐름을 크게 바꾸는 것이 아니라, 원문을 보존하면서 교과서형 문체와 명확한 오류를 점검하는 것입니다.

상세 절차는 [강의자료 검토 및 업로드 절차](./강의자료%20검토%20및%20업로드%20절차.md)를 기준으로 합니다.
제목 단계는 [제목 계층 기준](./제목%20계층%20기준.md)을 따릅니다.
퀴즈 검토와 개선은 [퀴즈 작성 기준](./퀴즈%20작성%20기준.md)을 따릅니다.

## 기본 기준

- 과정 순서와 과정 제목은 `references/raw/notion`의 `NN.제목` 폴더명을 유지합니다.
- 11기 원문은 `references/raw/notion/NN.제목/**/*.md`에서 확인합니다.
- 원본 자료는 수정하지 않고, 15기용 결과는 `courses/`에 작성합니다.
- `docs/`에는 이 레포의 운영 기준과 진행 기록만 작성합니다.
- 과정별 최초 정리 결과물은 `courses/NN.제목/강의자료.md`처럼 과정 폴더 바로 아래에 둡니다.
- 최초 Notion 업로드 전에는 `강의자료.md`가 기준 초안이고, `강의자료_notion.md`는 Notion 업로드용 파생본입니다.
- 최초 Notion 업로드 이후에는 업로드된 Notion 페이지가 섹션 교정 대상입니다. 이 단계에서는 `강의자료.md`와 `강의자료_notion.md`를 갱신하지 않습니다.
- 별도 `v1`, `v2` 폴더는 만들지 않습니다. 변경 이력은 Git 커밋과 운영 문서로 관리합니다.

## 현재 작업 원칙

1. 원문에서 내용, 말투, 억양이 교과서적으로 적절한지 검토하고, 교과서적으로 적절하지 않은 학생용 본문은 수정합니다.
2. 명확하게 잘못된 내용이 있으면 최초 Notion 업로드 전에는 `courses/*/강의자료.md`에서 수정합니다.
3. 최초 Notion 업로드 이후에는 Notion MCP/API로 해당 페이지의 요청받은 섹션 또는 블록만 수정합니다.
4. 최신 라이브러리, 패키지, 외부 서비스 정책처럼 현재 문서 확인이 필요한 내용은 직접 수정하지 않고 사용자에게 확인 필요 항목으로 보고합니다.

## 작업 단계 기준

- 아직 정리되지 않은 과정을 처음 다룰 때는 최초 전체 정리 단계로 봅니다. 이때는 과정 전체를 대상으로 교과서형 문체, 제목 계층, 본문/스크립트 분리, 중요도 별표, 코드블록, 이미지와 링크를 점검합니다.
- 최초 전체 정리가 끝난 뒤 Notion에 처음 반영할 때는 전체 업로드 또는 전체 교체를 사용할 수 있습니다.
- 최초 Notion 반영 이후에는 섹션 단위 교정 단계로 봅니다. 이 단계에서는 `courses/*/강의자료.md`를 수정하지 않고, Notion MCP/API로 사용자가 요청한 섹션이나 블록만 수정합니다. 사용자가 명시적으로 요청하지 않는 한 과정 전체를 다시 수정하거나 Notion 페이지를 전체 교체하지 않습니다.

## 원문 보존

- 원문에 있던 코드, 이미지, zip, 링크, 스크립트 흐름은 삭제하지 않습니다.
- 원문 구조를 크게 바꾸거나 섹션을 새로 나누지 않습니다.
- 같은 주제를 다루는 개념 본문, `#스크립트`, `#실습`, `#퀴즈`는 같은 섹션 번호를 유지합니다. 번호가 다르면 학생과 강사가 같은 항목을 서로 다른 수업 흐름으로 오해할 수 있습니다.
- 번호는 새 개념이 시작될 때만 증가합니다. 같은 주제의 실습이나 퀴즈는 별도 번호를 새로 받지 않습니다.
- 같은 강의 항목 안에서 자료 유형이 섞여 있으면 `본문 → #스크립트 → #실습 → #퀴즈` 순서로 정리합니다.
- 같은 강의 항목 안에서 핵심 개념이나 설명 흐름을 나눌 필요가 있으면 `####` 제목4를 사용합니다.
- 학생용 개념 본문의 제목4는 핵심 개념, 적용 방식, 주의점처럼 개념 흐름을 나누는 용도로만 사용합니다. `확인하기`, `사용해보기`, `넣기1`, `Live Server로 확인하기`, `마무리`처럼 영상 진행형 제목은 학생용 개념 본문에 사용하지 않습니다.
- 단원에 `#요약` 섹션이 별도로 있으면 개별 개념 섹션마다 `정리` 제목4를 반복하지 않습니다. 짧은 마무리는 일반 문단이나 목록으로 처리합니다.
- `#스크립트` 안에서는 강사용 진행을 위해 제목4를 사용할 수 있지만, 학생용 개념 본문과 같은 교과서형 제목 기준을 그대로 적용하지 않습니다. 다만 같은 주제의 개념 본문과 `#스크립트`는 설명과 시연 순서가 비교 가능해야 합니다.
- `##`, `###`, `####` 제목에는 백틱을 추가하지 않습니다. HTML 태그나 코드 조각이 제목에 들어가더라도 제목 텍스트에는 그대로 적고, 파란색 인라인 코드 스타일은 본문 설명에만 적용합니다.
- `#스크립트`는 강사용 수업 진행 대본이므로 구어체를 허용합니다.
- `#스크립트`를 교과서형 문장으로 전면 변환하지 않습니다.
- Notion 화면에서 보이는 왼쪽 세로 바(`|`, quote 블록)는 `#스크립트` 말하기 흐름에만 사용합니다. 학생용 개념 설명, 정의, 예시 해설, 정리에는 quote 블록을 쓰지 않고 일반 문단, 목록, 소제목을 사용합니다.
- 퀴즈, 실습, 문제은행, 숙제, 추가 설명은 사용자가 명시적으로 요청하지 않으면 새로 만들지 않습니다.
- 사용자의 요청으로 퀴즈를 개선할 때는 단순 암기형 문제를 줄이고, 코드 해석, 상황 판단, 오류 수정형 문제를 중심으로 구성합니다. 15분 퀴즈는 8-10문항을 기준으로 하며, 최소 6문항 이상은 적용형 문제로 구성합니다.
- 코드잇 실습 페이지의 내부 문제를 직접 확인하지 못한 경우, 파일명이나 주변 코드만 보고 목표, 진행 단계, 성공 확인을 추정해 쓰지 않습니다.
- 같은 단원에서 반복되는 예시 값, 파일명, URL, 코드 결과는 개념 본문, 스크립트, 실습, 퀴즈 사이에서 일관되게 유지합니다. 같은 대상을 설명하면서 이름이나 주소가 섞이면 수정 후보로 봅니다.

## 직접 수정할 수 있는 항목

- 교과서형 문체에 맞지 않는 학생용 본문 표현
- 원문에서 내용, 말투, 억양이 교과서적으로 적절하지 않은 학생용 본문
- 명확한 오탈자와 비문
- HTML, CSS, JavaScript 문법상 잘못된 예시
- 코드블록이 깨져 제목이나 본문이 코드블록 안으로 들어가는 문제
- 링크, 이미지, zip 표기가 명백히 깨진 경우
- 원문 내용과 충돌하는 잘못된 설명

## 보고만 할 항목

- 최신 라이브러리나 패키지 버전 변경
- 설치 방법, CLI 명령, 외부 서비스 UI처럼 현재 상태가 바뀔 수 있는 내용
- 원문에 없는 실습 목표, 완료 기준, 문제은행, 숙제 추가
- 코드잇 실습 페이지 내부 문제를 확인해야 판단할 수 있는 내용
- 강의 구성 자체를 바꾸는 큰 구조 변경

## 문체 기준

- 학생용 본문은 `합니다/입니다`체의 교과서형 문체로 작성합니다.
- `다룬다`, `일반적이다`, `확인한다` 같은 `이다/한다`체는 학생용 본문에서 사용하지 않습니다.
- `해볼게요`, `보시면`, `하겠습니다` 같은 말하기 표현은 학생용 본문에서 줄입니다.
- `#스크립트`는 강사용 대본이므로 말하기 표현을 허용합니다.
- 자동 검사는 금지 표현과 비문을 찾는 보조 수단입니다. 문체 수정은 자동 치환으로 처리하지 않고, 문단 단위로 직접 읽고 수정합니다.

## 중요도 별표 기준

- 별표는 학생의 복습 우선순위를 표시하기 위한 것입니다.
- 별표는 학생용 개념 본문 제목에만 붙입니다.
- `#스크립트`, `#실습`, `#퀴즈`, `#요약`, `#Video`, 설치, 링크, 참고 자료에는 기본적으로 붙이지 않습니다.
- 별표 개수와 판정 기준은 [중요도 별표 기준](./중요도%20별표%20기준.md)을 따릅니다.
- 기존 원문에 별표가 많으면 새 기준에 따라 줄이는 쪽으로 검토합니다.

## 코드와 실습 기준

- 문서 안의 코드 예시는 문법적으로 맞아야 합니다.
- 코드블록은 반드시 정상적으로 열리고 닫혀야 합니다.
- HTML 예시에는 HTML 주석 문법, CSS 예시에는 CSS 주석 문법처럼 해당 언어에 맞는 문법을 사용합니다.
- 코드 일부를 생략해야 한다면 해당 언어에서 유효한 주석이나 설명 방식으로 표시합니다. 학생이 코드블록을 복사했을 때 문법적으로 깨지는 형태는 사용하지 않습니다.
- 같은 누적 프로젝트 안의 코드 예시는 이전 섹션의 결과와 다음 섹션의 출발점이 자연스럽게 이어져야 합니다.
- `references/code/*`는 참고 자료로 확인할 수 있습니다.
- 최신 라이브러리나 패키지 업데이트가 필요해 보이면 직접 수정하지 않고 사용자에게 보고합니다.

## Notion 변환 기준

최초 Notion 업로드 전에는 `courses/`의 원본 Markdown이 15기 강의자료의 기준 초안입니다. Notion 업로드용 파일은 업로드 직전에만 만드는 파생본입니다. 최초 업로드가 끝난 뒤에는 업로드된 Notion 페이지를 교정 대상으로 삼고, 로컬 Markdown은 최초 업로드용 스냅샷으로 둡니다.

- 최초 업로드 전 내용 수정은 `courses/` 원본 Markdown에서 먼저 합니다.
- 최초 업로드 이후 내용 수정은 `courses/` 원본 Markdown이 아니라 Notion MCP/API로 해당 섹션 또는 블록에 직접 반영합니다.
- Notion 업로드가 필요할 때만 로컬 스킬 `$notion-formatter`를 사용해 `_notion.md`를 생성합니다.
- `$notion-formatter`는 Notion 변환과 업로드를 위한 도구이며, 학생용 본문의 개념 설명, 문체, 교육적 적절성을 판단하는 기준이 아닙니다. 이런 판단은 이 문서와 [강의자료 검토 및 업로드 절차](./강의자료%20검토%20및%20업로드%20절차.md)를 우선합니다.
- `$notion-formatter`는 이 저장소 전용 스킬입니다. Codex가 레포 로컬 스킬로 읽는 `.agents/skills/notion-formatter`에만 두고, 전역 스킬 위치인 `~/.codex/skills`나 `~/.agents/skills`에는 두지 않습니다.
- 최초 업로드 전 `_notion.md`에서 내용 오류를 발견하면 원본 Markdown을 수정한 뒤 다시 변환합니다.
- Notion 반영은 사용자가 명시적으로 요청한 경우에만 진행합니다.
- 최초 Notion 반영 단계에서는 전체 업로드 또는 전체 교체를 사용할 수 있습니다.
- 최초 Notion 반영 이후의 교정 단계에서는 로컬 Markdown을 갱신하지 않고 Notion MCP/API로 필요한 블록만 부분 수정합니다.
- 기존 Notion 페이지 내용을 archive 처리하고 `강의자료_notion.md`를 통째로 다시 올리는 `replace` 방식은 최초 반영이거나 사용자가 전체 교체를 명시적으로 요청한 경우에만 사용합니다.
- Markdown에서 백틱으로 감싼 인라인 코드는 Notion 업로드 시 인라인 코드로 유지하고 텍스트 색상을 `blue`로 지정합니다. 굵게는 적용하지 않으며, 코드블록의 언어 설정이나 코드블록 내부 텍스트 색상은 이 규칙의 대상이 아닙니다.

기본 생성 명령:

```bash
node .agents/skills/notion-formatter/scripts/format_notion_markdown.mjs \
  --input "courses/NN.제목/강의자료.md"
```

업로드 전 확인 명령:

```bash
node .agents/skills/notion-formatter/scripts/format_and_publish_notion_course.mjs \
  --input "courses/NN.제목/강의자료.md" \
  --page-id "<notion-page-id-or-url>" \
  --dry-run
```

최초 Notion 반영이거나 사용자가 전체 교체를 명시적으로 요청한 경우에만 실행하는 명령:

```bash
node .agents/skills/notion-formatter/scripts/format_and_publish_notion_course.mjs \
  --input "courses/NN.제목/강의자료.md" \
  --page-id "<notion-page-id-or-url>" \
  --replace
```

## 확인 결과 기록

검토나 수정 결과는 필요할 때만 짧게 남깁니다.

```markdown
## 검토 결과

- 대상:
- 수정한 내용:
- 수정하지 않고 보고한 내용:
- Notion 업로드 여부:
- 남은 확인 필요 항목:
```
