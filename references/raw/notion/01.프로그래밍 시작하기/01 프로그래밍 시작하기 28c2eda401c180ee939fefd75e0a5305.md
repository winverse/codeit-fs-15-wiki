### 01. #스크립트 웹의 탄생 - 원시 웹

```jsx
웹의 탄생에 대해서 이야기 하기 전에

먼저, 자주 혼용되는 인터넷과 웹의 관계부터 명확히 해보죠.
비유하자면, **인터넷은 '도시'**와 같습니다. 전 세계를 연결하는 거대한 인프라이죠.
그렇다면 **웹은 그 도시 위에 있는 '건물 하나'** 또는 **'도로 위를 달리는 자동차 한 대'**와 같습니다.
인터넷이라는 전체 안에 존재하는, 수많은 서비스 중 하나일 뿐입니다.
```

```jsx
이 둘은 역사도 다릅니다.
1960년대는 핵전쟁의 공포가 크던 시절이었습니다. 
당시 미국의 통신 시스템은 중앙집중적이어서, 만약 지휘부가 핵 공격을 당하면 전체 통신망이 
마비될 수 있는 심각한 약점이 있었죠. 
이 문제를 해결하기 위해, 핵 공격에도 견딜 수 있는 강인한 통신 시스템의 필요성이 대두되었고, 
그 결과 인터넷이 탄생했습니다.

인터넷의 핵심은 **'중앙이 없는 분산형 시스템'**입니다. 
수많은 통신 장치들이 각각 작은 전화국처럼 동작하여, 
한 지점이 파괴되어도 나머지 지점들이 그 역할을 대신해 전체 통신망이 유지되는 구조입니다. 

바로 이 강인한 분산 구조가, 오늘날 구글이나 페이스북 같은 거대 기업들을 묵묵히 
지탱하는 거대한 인프라가 된 것입니다.

인터넷은 1960년부터 30년 동안 천천히 전 세계적으로 확산되었습니다.
이 기간 동안 인터넷은 기업, 연구소, 대학, 군대 등 거대한 기관에서 사용되던 통신 시스템이었기 
때문에 대중에게는 잘 알려지지 않았습니다.

그리고 웹은 그로부터 30년 후인 1990년에 등장하여, 이 거대한 인프라의 잠재력을 폭발시켰습니다.
```

```jsx
웹은 의외의 장소, 스위스 제네바에 위치한 **유럽 입자 물리 연구소**, CERN에서 태어났습니다.
그리고 이곳엔 인류 최대의 장치인 '강입자 가속기'를 이용하여 우주의 근원을 탐구하는 곳입니다. 
전 세계에서 모여, 만 여명이 넘는 천재 과학자와 엔지니어들이 엄청난 양의 데이터를 쏟아내는 곳이었죠.
1980년대 말, **유럽 입자 물리 연구소**는 정보의 바다에서 허우적대는 '작은 지구'와 같았습니다. 
바로 이런 정보 과잉 환경이, 웹과 같은 혁신적인 시스템을 필요로 했습니다.
```

```jsx
1980년, **팀 버너스리**라는 비정규직 프로그래머가 **유럽 입자 물리 연구소**에 합류합니다.

그리고 1990년 말, 단 몇 달 만에 웹의 핵심 요소들이 탄생합니다.

- 10월: 세계 최초의 '웹페이지 편집기' 개발
- 11월: 세계 최초의 '웹 브라우저' (이름도 'World Wide Web'이었습니다) 개발
- 12월 24일: 세계 최초의 '웹서버'가 가동되고, `info.cern.ch` 라는 인류 최초의 웹 주소가 부여됩니다.

info.cern.ch에 접속해보시겠어요?

이게 바로 가장 필수적인 정보만을 담고 있는 원시 웹이라고 할 수 있습니다.

그리고 아래 Learn about the birth of web을 클릭하시면 이렇게 팀 버너스리 경(sir)의 얼굴이 나오는 
것을 확인 할 수 있습니다.
```

### 02. #Quiz 웹의 탄생 - 원시 웹 (5분)

**문제 1.** 인터넷과 웹의 관계를 흔히 도시에 비유합니다. '인터넷'이 도시 전체라면, '웹'은 무엇에 해당한다고 볼 수 있을까요?

1. 또 다른 도시
2. 도시를 연결하는 도로
3. 도시 안의 건물 하나
4. 도시의 시장

- 정답 확인하기
    
    **정답: 3. 도시 안의 건물 하나**
    
    > 해설: 인터넷이라는 거대한 인프라(도시) 안에 웹은 수많은 서비스(건물) 중 하나라고 설명하고 있습니다.
    > 

---

**문제 2.** 1960년대에 인터넷이 처음 설계될 때, 핵 공격에도 견딜 수 있도록 구현된 가장 핵심적인 특징은 무엇이었나요?

1. 중앙집중형 시스템
2. 분산형 시스템
3. 단일 서버 시스템
4. 암호화 통신 시스템

- 정답 확인하기
    
    **정답: 2. 분산형 시스템**
    
    > 해설: 초기 인터넷은 특정 지휘부가 공격받아도 전체 통신망이 마비되는 것을 막기 위해 설계되었습니다. 이를 위해 중앙 서버 없이 여러 장치가 동등하게 통신을 중계하는 '분산형 시스템' 방식을 채택하여, 일부가 파괴되어도 네트워크가 유지되도록 했습니다.
    > 

---

**문제 3.** 웹의 아버지, 팀 버너스리가 1990년 말에 웹을 탄생시킬 당시 소속되어 있던 연구소의 이름은 무엇일까요?

1. NASA (미국 항공우주국)

    > 해설: 웹은 스위스 제네바에 위치한 CERN에서, 수많은 연구자들의 정보 공유 문제를 해결하기 위한 목적으로 탄생했습니다.
    > 

---

### 01. #Quiz 웹에서 정보 주고 받기 (5분)

**문제 1.** 정보를 요청하는 컴퓨터를 무엇이라고 부르나요?

1. 서버 (Server)
2. 클라이언트 (Client)
3. 인터넷 (Internet)
4. 웹 (Web)

- 정답 확인하기
    
    **정답: 2. 클라이언트 (Client)**
    
    > 해설: 클라이언트는 서버에게 정보를 **요청**하는 역할을 합니다.
    > 

---

**문제 2.** 다음 중 서버(Server)에 대한 설명으로 가장 올바른 것은 무엇인가요?

1. 유튜브 영상을 시청하는 나의 스마트폰
2. 정보를 요청해서 제공받는 컴퓨터
3. 서비스를 사용하는 일반 사용자의 컴퓨터
4. 요청을 받아 정보를 제공해주는 컴퓨터

- 정답 확인하기
    
    **정답: 4. 요청을 받아 정보를 제공해주는 컴퓨터**
    
    > 해설: 서버는 클라이언트의 요청에 따라 정보를 '제공(Serve)'해주는**(응답)** 역할을 합니다. 1, 2, 3번은 모두 클라이언트에 대한 설명입니다.
    > 

---

**문제 3.** (단답형) 우리가 앞으로 배울 내용에서, 웹 브라우저에 보이는 화면을 만드는 '프론트엔드'는 서버와 클라이언트 중 어떤 역할을 하게 될까요?

- 정답 확인하기
    
    **정답: 클라이언트**
    
    > 해설: '프론트엔드'는 사용자가 브라우저에서 직접 보고 상호작용하는 화면을 의미합니다. 이 화면은 사용자에게 필요한 데이터(예: 게시글 목록)를 보여주기 위해 서버에게 정보를 **요청**하는 역할을 합니다. 이처럼 정보를 요청하는 쪽이 '클라이언트'이므로, 프론트엔드는 클라이언트의 역할을 담당하게 됩니다.
    > 

---

**문제 4.** (응용) 온라인 쇼핑몰에서 옷을 주문하려고 합니다. 이 상황에서 '서버'의 역할을 하는 것은 무엇일까요?

1. 내 컴퓨터의 웹 브라우저 (크롬, 사파리 등)
2. 주문 요청을 처리하는 쇼핑몰의 컴퓨터 시스템
3. 내 컴퓨터의 운영체제 (윈도우, macOS 등)
4. 주문한 옷을 배송하는 택배 회사

- 정답 확인하기
    
    **정답: 2. 주문 요청을 처리하는 쇼핑몰의 컴퓨터 시스템**
    
    > 해설: 내 웹 브라우저(클라이언트)가 '주문'이라는 **요청**을 보내면, 쇼핑몰의 컴퓨터 시스템(서버)이 이 요청을 받아 처리하고**(응답)** 주문 완료 페이지를 보여줍니다.
    > 

---

**문제 5.** (응용) Gmail을 사용해서 친구에게 이메일을 보냈습니다. 이때 내 컴퓨터는 ( A ) 역할을 한 것이고, Google의 컴퓨터는 ( B ) 역할을 한 것입니다. A와 B에 들어갈 말로 올바른 것은?

1. A: 서버, B: 클라이언트

- 정답 확인하기
    
    **정답: 2. A: 클라이언트, B: 서버**
    
    > 해설: 내 컴퓨터(클라이언트)가 '이메일을 보내달라'는 **요청**을 한 것이고, Google의 컴퓨터(서버)가 그 요청을 받아**(응답)** 이메일을 친구에게 전송해준 것입니다.

### 07. #실습 크롬 개발자 도구 이용해보기1 (10분) ⭐⭐⭐

웹 개발자에게 크롬 개발자 도구는 코드를 진단하고 수정하는 의사의 청진기와 같습니다. 이 가이드는 초보자가 가장 먼저 알아야 할 핵심 기능 4가지를 중심으로 개발자 도구와 친해지는 법을 알려줍니다.

---

### **개발자 도구, 어떻게 여나요?**

어떤 웹사이트(www.naver.com, www.google.com)에서든 다음 방법 중 하나를 사용해 보세요.

- **마우스 우클릭:** 페이지의 아무 곳이나 우클릭 후 **[검사(Inspect)]** 메뉴 선택
- **단축키 (Mac):** `Cmd` + `Opt` + `I`
- **단축키 (Windows):** `Ctrl` + `Shift` + `I` 또는 `F12`

---

### 초보자를 위한 4가지 핵심 패널

개발자 도구를 열면 여러 탭(패널)이 보입니다. 처음에는 복잡해 보이지만, 딱 4가지만 알면 충분합니다.

### 1. Elements 패널: HTML 구조와 CSS 스타일 확인하기

![스크린샷 2025-10-14 15.27.05.png](01%20%ED%94%84%EB%A1%9C%EA%B7%B8%EB%9E%98%EB%B0%8D%20%EC%8B%9C%EC%9E%91%ED%95%98%EA%B8%B0/%E1%84%89%E1%85%B3%E1%84%8F%E1%85%B3%E1%84%85%E1%85%B5%E1%86%AB%E1%84%89%E1%85%A3%E1%86%BA_2025-10-14_15.27.05.png)

Elements 패널은 웹페이지의 실시간 뼈대(HTML)와 디자인(CSS)을 보여주는 곳입니다.

**주요 기능:**

- **HTML 구조 확인:** 삼각형(▶)을 클릭해 태그를 열어보며 페이지가 어떻게 구성되었는지 파악할 수 있습니다.
- **CSS 스타일 확인 및 수정:** 특정 HTML 요소를 클릭하면, 오른쪽 `Styles` 탭에 해당 요소에 적용된 모든 CSS 속성이 나타납니다. 여기서 체크박스를 끄거나 값을 바꿔보세요. 페이지가 실시간으로 변하는 것을 볼 수 있습니다.
- **내용 임시 수정:** HTML 부분을 더블클릭하면 페이지의 텍스트를 내 마음대로 임시로 바꿀 수 있습니다.

**실습:** 아무 웹사이트나 열고, 버튼 하나의 색깔을 `Styles` 탭에서 `background-color: red;`로 바꿔보세요. (새로고침하면 원래대로 돌아옵니다!)

### 2. Console 패널: JavaScript와 대화하기

![스크린샷 2025-10-14 15.28.01.png](01%20%ED%94%84%EB%A1%9C%EA%B7%B8%EB%9E%98%EB%B0%8D%20%EC%8B%9C%EC%9E%91%ED%95%98%EA%B8%B0/%E1%84%89%E1%85%B3%E1%84%8F%E1%85%B3%E1%84%85%E1%85%B5%E1%86%AB%E1%84%89%E1%85%A3%E1%86%BA_2025-10-14_15.28.01.png)

Console 패널은 JavaScript의 놀이터이자, 코드의 상태를 알려주는 로그 게시판입니다.

**주요 기능:**

- **JavaScript 코드 실행:** 간단한 계산(`2 + 2`)부터 복잡한 명령어까지 직접 입력하고 실행 결과를 바로 확인할 수 있습니다.
- **에러 메시지 확인:** 웹페이지가 오작동할 때, 대부분의 경우 Console에 빨간색으로 에러 메시지가 표시됩니다. 이 메시지는 버그를 잡는 가장 중요한 단서입니다.
- **로그 확인:** 개발자들이 코드 중간에 `console.log()`로 출력한 값들을 확인하며 코드가 어떻게 실행되고 있는지 흐름을 파악할 수 있습니다.

**실습:** Console에 `console.log("Hello, Developer!");` 라고 입력하고 엔터를 쳐보세요.

### 3. Sources 패널: 웹사이트의 소스 코드 열람하기

![스크린샷 2025-10-14 15.29.36.png](01%20%ED%94%84%EB%A1%9C%EA%B7%B8%EB%9E%98%EB%B0%8D%20%EC%8B%9C%EC%9E%91%ED%95%98%EA%B8%B0/%E1%84%89%E1%85%B3%E1%84%8F%E1%85%B3%E1%84%85%E1%85%B5%E1%86%AB%E1%84%89%E1%85%A3%E1%86%BA_2025-10-14_15.29.36.png)

Sources 패널은 이 웹사이트를 구성하는 모든 원본 파일(JavaScript, CSS, 이미지 등)을 볼 수 있는 파일 탐색기입니다.

**주요 기능:**

- **원본 코드 확인:** 다른 웹사이트는 어떤 코드로 만들어졌는지 궁금할 때, 여기서 JavaScript 파일을 열어볼 수 있습니다.
- **코드 디버깅 (다음 단계):** '브레이크포인트(Breakpoint)'라는 기능을 사용해 코드 실행을 특정 지점에서 멈추고, 변수 값을 확인하며 오류를 잡을 수 있습니다. (매우 중요하지만, 처음에는 이런 게 있다는 것만 알아두세요!)

**실습:** 왼쪽 파일 탐색기에서 `.js`로 끝나는 파일을 아무거나 열어보세요.

### 4. Network 패널: 웹사이트의 통신 과정 엿보기

![스크린샷 2025-10-14 15.30.11.png](01%20%ED%94%84%EB%A1%9C%EA%B7%B8%EB%9E%98%EB%B0%8D%20%EC%8B%9C%EC%9E%91%ED%95%98%EA%B8%B0/%E1%84%89%E1%85%B3%E1%84%8F%E1%85%B3%E1%84%85%E1%85%B5%E1%86%AB%E1%84%89%E1%85%A3%E1%86%BA_2025-10-14_15.30.11.png)

Network 패널은 웹 브라우저가 서버와 주고받는 모든 통신 내역을 기록하는 곳입니다.

**주요 기능:**

- **리소스 로딩 확인:** 이 페이지를 그리기 위해 어떤 파일들(이미지, 폰트, 데이터 등)을 다운로드했는지 목록으로 보여줍니다.
- **로딩 속도 확인:** 각 파일이 로딩되는 데 얼마나 걸리는지 확인할 수 있어, 웹사이트 성능 개선의 기초 자료가 됩니다.
- **통신 상태 확인:** 각 요청이 성공했는지(상태 코드 `200`), 실패했는지(`404` 등) 알 수 있습니다.

**실습:** Network 탭을 열어둔 채로 페이지를 새로고침(`Cmd`+`R` 또는 `Ctrl`+`R`) 해보세요. 파일 목록이 주르륵 뜨는 것을 볼 수 있습니다.

### 02. #Quiz JSON

**문제 1.** API를 통해 클라이언트와 서버가 데이터를 주고받을 때, 현재 사실상의 표준으로 사용되는 데이터 형식은 무엇일까요?

1. HTML
2. CSS
3. JSON
4. SQL

- 정답 확인하기
    
    **정답: 3. JSON**
    
    > 해설: JSON은 자바스크립트 객체 표기법을 따르는 문자열 기반의 데이터 형식으로, 웹 API 통신에서 데이터를 교환하기 위한 표준처럼 사용됩니다.
    > 

---

**문제 2.** JSON이 웹 개발, 특히 자바스크립트 환경에서 널리 사용되는 가장 큰 이유는 무엇일까요?

1. 보안이 매우 뛰어나기 때문에
2. 사람이 읽기에는 어렵기 때문에
3. 자바스크립트의 객체 문법과 거의 동일하여 처리하기가 매우 쉽기 때문에
4. 오직 자바스크립트에서만 사용할 수 있기 때문에

- 정답 확인하기
    
    **정답: 3. 자바스크립트의 객체 문법과 거의 동일하여 처리하기가 매우 쉽기 때문에**
    
    > 해설: JSON은 'JavaScript Object Notation'의 약자로, 이름처럼 자바스크립트 객체 표기법에 기반하고 있습니다. 이 덕분에 웹의 핵심 언어인 자바스크립트에서 별도의 복잡한 변환 과정 없이 데이터를 쉽게 사용할 수 있습니다.
    > 

---

**문제 3.** (실습) 아래 정보를 바탕으로, 사용자에 대한 정보를 담은 JSON 객체를 직접 만들어 보세요.

- `name`은 `"홍길동"` (문자열)
- `age`는 `30` (숫자)
- `isStudent`는 `false` (불리언)

- 정답 확인하기
    
    **정답:**
    
    ```json
    {
      "name": "홍길동",
      "age": 30,
      "isStudent": false
    }
    ```
    
    > 해설: JSON은 중괄호({})로 객체를 감싸고, "key": value 형식의 쌍으로 데이터를 표현합니다. 문자열은 큰따옴표로 감싸야 합니다.
    > 

---

**문제 4.** JSON 데이터 형식에 대한 설명으로 올바르지 **않은** 것은 무엇일까요?

1. Key-Value 쌍으로 이루어져 있다.
2. 사람이 읽고 이해하기 쉬운 구조이다.
3. 자바스크립트에서만 사용할 수 있고 다른 언어(Java, Python 등)에서는 처리할 수 없다.
4. 객체 안에 또 다른 객체를 넣어 중첩된 구조를 만들 수 있다.

- 정답 확인하기
    
    **정답: 3. 자바스크립트에서만 사용할 수 있고 다른 언어(Java, Python 등)에서는 처리할 수 없다.**
    
    > 해설: JSON은 이제 사실상의 표준 데이터 교환 형식이 되었기 때문에, Java, Python 등 대부분의 주요 프로그래밍 언어에서 JSON 데이터를 쉽게 처리할 수 있는 라이브러리나 도구를 잘 지원합니다.
    >
