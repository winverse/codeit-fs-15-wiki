### 기본 position: static

- **`Computed`** → **`Show all`** 에서 확인해보면 기본 position은 static으로 되어 있음
- 별도로 position을 지정하지 않은 모든 HTML 요소는 **`position: static`**이 적용됨
- "문서의 흐름(normal flow)"에 따라 원래 있어야 할 위치에 그대로 배치되는 것을 의미함

![스크린샷 2025-11-04 12.28.38.png](05%20CSS%20%EB%A0%88%EC%9D%B4%EC%95%84%EC%9B%83/%E1%84%89%E1%85%B3%E1%84%8F%E1%85%B3%E1%84%85%E1%85%B5%E1%86%AB%E1%84%89%E1%85%A3%E1%86%BA_2025-11-04_12.28.38.png)

### position: relative

- 요소를 "자기 자신의 원래 위치"를 기준으로 이동시키겠다는 뜻입니다.
- 원래 static일 때의 위치를 기준 삼아, top/right/bottom/left 값을 활용해 **원래 위치에서 상대적으로 이동**만 시킵니다.

```css
/* style.css */
.green {
  background-color: #32b9c1;
  position: relative;
  top: 30px;
  left: 50px;
}
```

### margin과의 차이

- margin 요소가 이동하면 주변 요소들도 함께 밀려나며, 문서의 레이아웃 전체에 영향을 줌

```css
/* style.css */
.green {
  background-color: #32b9c1;
```

### position: absolute;

- absolute는 절대값이라는 뜻
- 가장 가까운 포지셔닝(position이 relative, absolute, fixed, sticky)된 부모 요소를 기준으로 정확히 좌표(top, right, bottom, left)로 배치할 때 사용하는 속성입니다.
- 따라서 부모의 요소의 position이 중요함

### 부모 position 설정하기

```css
/* style.css */
.green {
  position: relative;
  top: 40px;
  left: 90px;
  background-color: #32b9c1;
  width: 300px;
  height: 300px;
}
```

### position: absolute 사용해보기

- orange box와 겹치는 것을 알 수 있음

```css
.green {
  position: relative;
  top: 0px;
  left: 0px;
  background-color: #32b9c1;
  width: 300px;
  height: 300px;
}
```

### 겹치지 않게 수정

```css
.blue {
  position: absolute;
  top: 40px;
  right: 40px;
  background-color: #5195ee;
  width: 100px;
  height: 100px;
}
```

### 부모를 변경해보기

- 가장 가까운 포지셔닝 된 부모를 찾아서 위치를 설정하는 것을 알 수 있음

```css
.red {
  position: relative; /* green에서 변경 */
  background-color: #e46e80;
  width: 500px;
  height: 500px;
}

.green {
  background-color: #32b9c1;
  width: 300px;
  height: 300px;
}
```

### position: absolute; 의 영역

- orange 영역은 <div> 태그가 `display: block;` 라서 가로 공간이 100%가 됨
- 그러나 blue 영역은 position이 absolute;이기 때문에, 원래 자리에서 사라지게 됨
    - 텍스트를 추가하거나
    - width: 100%
    - top, left, right, bottom을 0으로 설정

```css
.blue { /* 가로 사이즈 제거 */
  position: absolute; 
  top: 40px;
  right: 40px;
  background-color: #5195ee;
  height: 100px;
}

.orange { /* 가로 사이즈 제거 */
  background-color: #f18575;
  height: 100px;
}
```

```css
  left: 0;
  right: 0;
  bottom: 0;
```

### 07. fixed 포지션 ⭐⭐

### position: fixed; 사용해보기

- 화면(브라우저 창)을 기준으로 요소를 고정시키는 위치 속성
- 부모와 상관 없이 브라우저 화면(뷰포트)를 기준으로 고정
- 스크롤을 해도 항상 같은 위치에 유지

```css
.red {
  background-color: #e46e80;
  width: 100px;
  height: 100px;
  position: fixed;
  top: 30px;
  left: 50px;
}
```

### **fixed vs 다른 position 차이**

- `relative` : 요소의 원리 위치를 기준으로 배치
- `absolute` : 가장 가까운 포지셔닝 된 조상 요소를 기준으로 배치
- `fixed` : 브라우저 화면 기준으로 배치

### 다른 곳에 배치해보기

```css
.red {
  ...
  position: fixed;
  bottom: 100px;
  right: 70px;
}
```

### 실전 활용

- 내비게이션, 팝업, 퀵메뉴, Floating 버튼 등에 많이 사용

```css
.red {
  background-color: #e46e80;
  width: 100%;
  height: 100px;
  position: fixed;
  top: 0;
  left: 0;
}
```

- 겹치는 부분이 있다면 margin-top으로 해결 하면 됨

```html
<main style="margin-top: 120px">
  <img class="skeleton" src="skeleton.png" alt="긴 글" />
  <img class="skeleton" src="skeleton.png" alt="긴 글" />
  <img class="skeleton" src="skeleton.png" alt="긴 글" />
</main>
```

```css

```

### position: sticky

- 기본적으로 static처럼 제자리에 있다가
- 스크롤해서 top: 0 위치에 닿는 순간 fixed처럼 화면에 고정됨
- 다시 원래 위치로 돌아가면 또 제자리 배치됨

### **sticky vs 다른 position 차이**

- `relative`**:** 원래 위치 기준 이동
- `absolute`**:** 가장 가까운 position된 조상 기준
- `fixed`**:** 브라우저 화면 기준으로 고정 (부모 무시)
- `sticky`**:** 원래 위치+화면 기준, 지정 위치에 닿으면 고정, 부모 요소 안에서만 동작

### **부모 요소의 영향**

- sticky 포지션은 부모 요소가 화면 밖으로 사라지면 같이 사라짐
- 즉, 부모의 영역을 벗어나면 더 이상 고정되지 않고 다시 제자리로 돌아옴

```html
  <body>
    <!-- <div class="header"></div>
    <div class="red"></div> -->
    <div class="main">
      <div style="border: 1px solid red">
        <div class="blue"></div>
        <img class="skeleton" src="skeleton.png" alt="긴 글" />
      </div>
      <img class="skeleton" src="skeleton.png" alt="긴 글" />
      <img class="skeleton" src="skeleton.png" alt="긴 글" />
    </div>
  </body>
```

### 실제 사용해보기

- 목차 별 header

```html
<div class="main">
  <div style="border: 1px solid red">
    <div class="blue">제목1</div>
    <img class="skeleton" src="skeleton.png" alt="긴 글" />
  </div>
  <div style="border: 1px solid green">
    <div class="blue">제목2</div>
    <img class="skeleton" src="skeleton.png" alt="긴 글" />
  </div>
  <div style="border: 1px solid yellow">
    <div class="blue">제목3</div>
    <img class="skeleton" src="skeleton.png" alt="긴 글" />
  </div>
</div>
<footer style="height: 2000px"></footer>
```

### 코드 분석 및 현재 문제점

### z-index를 사용해보자

- 요소의 **앞/뒤(겹침) 순서**를 정하는 속성
- top/bottom/left/right가 위치(방향)라면, z-index는 앞/뒤(레이어) 방향
- 값이 높을수록 **더 앞쪽**(위쪽 레이어)으로 나타남
- 숫자 단위 없이 정수(+,- 모두 가능)

### 겹친 다는 것이 무엇이지?

- paints 옵션 체크
- header 영역 보다 blue 영역이 앞에 있는 것을 알 수 있음
- z-index를 지정하지 않으면 코드상 더 아랫줄(나중에 작성된 태그)일수록 화면에서는 앞에 보임

### display: flex 사용해보기

- **기본 방향:**
    - flex-direction: row (기본값)
    - 왼쪽에서 오른쪽, 즉 `"가로"` 방향으로 박스들이 나란히 배열됨

```css
.container {
  ...
  display: flex;
}
```

### 세로 방향으로 배열

```css
.container {
  ...
  display: flex;
  flex-direction: column;
}
```

### **방향 뒤집기 (순서도 바뀜)**

- flex-direction: row-reverse
- flex-direction: column-reverse

```css
.container {
	...
  flex-direction: row-reverse;
}
```

### 정리

- Flexbox의 핵심은 "배치 방향을 자유롭게 설정할 수 있다"는 점
- row = 가로(왼→오) / row-reverse = 가로(오→왼)
- column = 세로(위→아래) / column-reverse = 세로(아래→위)
- flex-direction을 활용하면 훨씬 다양한 정렬 효과를 쉽게 만들 수 있다

### 기본 축**(주 축)** 정렬하기

- **`justify-content`** 속성 사용
    - 기본값: **`flex-start`** (앞쪽 정렬)
    - 주요 값:
        - **`center`**: 가운데 정렬
        - **`flex-end`**: 끝쪽 정렬
        - **`space-between`**: 양 끝에 붙이고 나머지 공간에 고르게 배치
        - **`space-around`**: 양쪽에 동일한 여백(균등)으로 띄엄띄엄 배치
        - **`space-evenly`**: 모든 공간(요소 사이, 양끝)이 동일하게 배치

### **교차 축 정렬**

- **`align-items`** 속성 사용
    - 기본값: **`stretch`** (교차축 방향으로 요소를 꽉 채움)
    - 주요 값:
        - **`center`**: 교차축 중앙 정렬
        - **`flex-end`**: 교차축 끝(아래 혹은 오른쪽) 정렬
        - **`flex-start`**: 교차축 앞(위 혹은 왼쪽) 정렬
        - **`stretch`**: 교차축 영역 꽉 채우기(기본)

### 기본 축과 교차축 바꿔보기

```css
.container {
  border: 5px dashed #cacfd9;
  width: 800px;
  height: 500px;
  display: flex;
  flex-direction: column; /* 축 변경하기 */
}
```

### 넘칠때는 flex-wrap

- 기본 Flexbox는 display: flex 상태에서 요소들을 한 줄(row)로 배치함
- 요소 개수가 많아 컨테이너 너비(또는 높이)를 초과하면, 기본값(nowrap)에서는 요소가 Flexbox를 벗어나서(흘러넘쳐서) 배치됨.
    
    ![스크린샷 2025-11-05 12.39.03.png](05%20CSS%20%EB%A0%88%EC%9D%B4%EC%95%84%EC%9B%83/%E1%84%89%E1%85%B3%E1%84%8F%E1%85%B3%E1%84%85%E1%85%B5%E1%86%AB%E1%84%89%E1%85%A3%E1%86%BA_2025-11-05_12.39.03.png)
    
- 줄 바꿈하려면 `.container { flex-wrap: wrap; }`로 지정

```css
.container {
  ...
  display: flex;
  flex-wrap: wrap;
}
```

### **flex-direction에 따른 동작**

- 방향을 row로 두면(기본값), 넘치는 요소가 '아래'로 줄 바꿈
- flex-direction: column;을 주면, 넘치는 요소가 '오른쪽'으로 넘어가며 배치됨(주 축과 교차 축이 바뀌기 때문)

```css
.container {
	...
  flex-direction: column;
}
```

### 마진을 이용한 간격

- 각 요소(margin)나 특정 선택자를 이용해 직접 여백 설정 가능
- 단점: **요소 추가/순서 변경** 시 예외 처리가 불편하고 코드가 길어짐

```css
.green {
  background-color: #32b9c1;
  margin: 30px;
}
```

### gap을 이용한 간격

- Flex 컨테이너에 gap: 30px;을 쓰면, 모든 요소 사이에 30px 간격이 자동 적용됨.
- gap: 30px 60px;처럼 두 개의 값을 쓰면, **첫 번째는 세로(row-gap), 두 번째는 가로**(column-gap)를 의미.
- **flex-direction과 무관하게** gap의 첫 번째 값은 항상 세로(위아래), 두 번째 값은 가로(좌우) 간격.

```css
.container {
  ...
  gap: 30px; 
}
```

### **flex-grow: 빈 공간을 채우는 방법**

- flex-grow는 남은 공간을 어떻게 분배할지 결정하는 속성
- 기본값은 0. 값이 1이면 남은 공간을 같은 비율로 가져가며, 값이 다르면 비율에 따라 나눔
    - 예) green에 1, blue에 2 → 남은 공간의 1:2로 분배.
        - 단, padding 혹은 컨텐츠에 따라서 정확한 수치는 다를 수 있음

```css
.green { flex-grow: 1; }
.blue { flex-grow: 2; }
.red  { flex-grow: 0; } /* 늘어나지 않음 */
```

### box 요소 사이즈 키우기

- 사이즈를 변경하였지만 여전히 container 안에 유지하고 있는 것을 알 수 있음
- 개발자 도구에서 확인해보면 화살표를 통해서 원래 사이즈에서 줄어든 정도를 볼 수 있음

```css
.box {
  ...
  width: 300px; /* 여전히 container 안에 다 들어 있음 */
}
```

### **flex-shrink: 요소 크기 줄이기**

- flex-shrink는 컨테이너 공간이 부족할 때 얼마나 줄일지 결정.
- 기본값 1. 값이 크면 더 빨리 줄어듦. 0이면 지정된 크기(width 등)를 유지.
- 사이즈를 유지하고 싶을때 사용

```css
.red  { flex-shrink: 0; }
.green{ flex-shrink: 1; }
.blue { flex-shrink: 2; }
```

### display: grid;

- 그리드 적용: `display: grid;`
- 열(column) 나누기: `grid-template-columns: 100px 300px 100px;` → 각각의 column에 너비 100px, 300px, 100px 적용.
- 행(row) 나누기: `grid-template-rows: 200px 200px 100px;` → 각각의 row에 높이 200px, 200px, 100px 적용.
- 개발자 도구에서 `grid` 버튼을 눌러서 확인 가능
    
    ![스크린샷 2025-11-05 13.50.06.png](05%20CSS%20%EB%A0%88%EC%9D%B4%EC%95%84%EC%9B%83/%E1%84%89%E1%85%B3%E1%84%8F%E1%85%B3%E1%84%85%E1%85%B5%E1%86%AB%E1%84%89%E1%85%A3%E1%86%BA_2025-11-05_13.50.06.png)
    

```css
.container {
  border: 5px dashed #cacfd9;
  width: 500px;
  height: 500px;
  display: grid;
  grid-template-columns: 100px 300px 100px;
  /* colums 확인 이후 */
  grid-template-rows: 200px 200px 100px;
}
```

### 단축 속성 grid-template

- 복수 속성을 한 줄로 작성할 때:
    
    `grid-template: 200px 200px 100px / 100px 300px 100px;`
    
    - **`/`** 앞이 row, 뒤가 column.

```css
.container {
  ...
  display: grid
  grid-template: 200px 200px 100px / 100px 300px 100px;
}
```
