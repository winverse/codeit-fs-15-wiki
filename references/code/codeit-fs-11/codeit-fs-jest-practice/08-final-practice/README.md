# 08-final

이 폴더는 8장 실습의 상세 기준입니다. 교재 본문에서는 시작 경로만 안내하고, 실제 과제 내용은 이 README를 기준으로 진행합니다.

## 역할

- 최종 종합 실습 시작점입니다.
- 런타임 구조는 이미 `src/`에 준비되어 있고, 학생은 `test/` 아래에 테스트 파일을 직접 만듭니다.

## 현재 상태

- `src/`에는 Express 런타임 코드가 들어 있습니다.
- 이 런타임은 `Controller -> App -> errorHandler` 형식으로 이미 조립되어 있습니다.
- `test/`는 비어 있거나 최소 상태이며, 실습 문제에 맞춰 테스트 파일을 직접 추가하는 구조입니다.
- Jest 실행과 coverage 확인에 필요한 스크립트와 설정은 이미 들어 있습니다.
- ESLint와 Prettier 설정도 이미 들어 있습니다.
- 따라서 시작 직후에는 테스트 파일이 아직 없어서 `pnpm test`가 실패하는 것이 정상입니다.

## 시작 코드

이 폴더는 최종 종합 실습 시작점입니다. 런타임 코드는 이미 들어 있고, `test/`는 최소 상태로 두었습니다. 지금까지 배운 기준을 이용해 테스트 파일을 직접 만들고, 구조와 품질 기준을 함께 점검하면 됩니다.

작성 순서는 아래처럼 고정해 두는 편이 좋습니다.

1. 먼저 5장 기준으로 `test/unit` 아래 단위 테스트를 만듭니다.
2. 그다음 6장 기준으로 `test/integration/app.spec.ts`를 만들고, 필요하면 같은 장에서 본 방식대로 `test/helper/app-environment.ts` 같은 환경 helper를 추가합니다.
3. 최소 테스트 파일이 생긴 뒤 `pnpm test`, `pnpm run test:cov`, 필요하면 `pnpm run test:runInBand`를 실행합니다.

## 해야 할 일

- `test/` 아래에 필요한 단위 테스트와 통합 테스트 파일을 직접 추가합니다.
- 테스트 이름은 조건, 행동, 결과가 드러나게 적습니다.
- `pnpm test`, `pnpm run test:cov`, 필요하면 `pnpm run test:runInBand`까지 실행합니다.

## 완료 기준

- 단위 테스트와 통합 테스트가 각자의 책임 경계를 지키고 있습니다.
- 커버리지 수치와 테스트 품질을 구분해서 설명할 수 있습니다.
- 반복 실행 가능한 테스트 환경을 직접 점검할 수 있습니다.
