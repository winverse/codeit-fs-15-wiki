# 05. GitHub 퀴즈 I

### 문제 1
로컬 저장소의 `main` 브랜치에 새로운 커밋이 있습니다. 이 내용을 `origin`이라는 이름의 리모트 저장소로 처음 전송하면서, 앞으로의 `push`와 `pull` 작업을 더 간단하게 할 수 있도록 로컬 브랜치와 리모트 브랜치를 연결하려고 합니다. 다음 중 가장 적절한 명령어는 무엇일까요?

<details>
<summary>정답 및 해설 보기</summary>
<div markdown="1">

- **정답:** `git push --set-upstream origin main`

- **해설:** `git push`는 로컬의 변경 내용을 리모트로 보냅니다. `--set-upstream` 옵션은 로컬의 현재 브랜치(이 경우 `main`)가 리모트 저장소(origin)의 특정 브랜치(`main`)를 추적하도록 설정합니다. 이 설정을 한번 해두면, 다음부터는 `git push` 와 `git pull`만으로 간단하게 동기화할 수 있습니다.

</div>
</details>

### 문제 2
팀 동료가 리모트 저장소에 새로운 기능을 추가하고 `push`를 완료했습니다. 동료의 최신 작업을 내 로컬 저장소로 가져와서 내용을 통합(병합)하고 싶을 때 사용하는 가장 일반적인 명령어는 무엇인가요?

<details>
<summary>정답 및 해설 보기</summary>
<div markdown="1">

- **정답:** `git pull`

- **해설:** `git pull`은 리모트 저장소의 최신 변경 내용을 내 로컬 작업 공간으로 가져와(fetch) 자동으로 병합(merge)하는 두 가지 주요 작업을 한 번에 수행합니다. 이를 통해 로컬 환경을 항상 최신 상태로 유지할 수 있습니다.

</div>
</details>

### 문제 3
`git push origin main`이라는 명령어에서, `origin`은 일반적으로 무엇을 의미하나요?

<details>
<summary>정답 및 해설 보기</summary>
<div markdown="1">

- **정답:** 리모트 저장소의 주소를 가리키는 **별명(alias)**입니다.

- **해설:** Git은 긴 URL 주소 대신 `origin`과 같은 간단한 이름으로 리모트 저장소를 관리할 수 있게 해줍니다. 이 별명은 `git remote add <별명> <URL>` 명령어를 통해 설정하며, `origin`은 관례적으로 가장 많이 사용되는 기본 별명입니다.

</div>
</details>

### 문제 4
로컬 저장소에서 `git push`를 실행했을 때와 `git pull`을 실행했을 때, 데이터는 각각 어느 방향으로 흐르나요?

<details>
<summary>정답 및 해설 보기</summary>
<div markdown="1">

- **정답:**
  - `git push`: **로컬 저장소 → 리모트 저장소**
  - `git pull`: **리모트 저장소 → 로컬 저장소**

- **해설:** `push`는 내 컴퓨터의 변경 내용을 '밀어내는(push)' 것이고, `pull`은 리모트의 변경 내용을 '당겨오는(pull)' 것이라고 생각하면 데이터의 흐름을 쉽게 이해할 수 있습니다.

</div>
</details>
