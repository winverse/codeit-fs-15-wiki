### 06. 두 커밋 간의 차이 보기

- 특정 두 커밋 사이에 어떤 코드 변화가 있었는지 비교하고 싶을 때 `git diff` 명령어를 사용함
- `diff`는 'difference'의 줄임말로, 두 지점 간의 차이를 보여줌

---

### 사용 방법

1. `git log`를 통해 비교하고 싶은 두 커밋의 아이디(해시)를 확인
2. `git diff [오래된 커밋 아이디] [최신 커밋 아이디]` 순서로 명령어를 입력

---

### 결과 해석

- `git show`와 유사한 형태로 결과가 출력됨
- `-` 기호가 붙은 줄: 이전 커밋(a)에는 있었지만, 이후 커밋(b)에서 삭제된 내용
- `+` 기호가 붙은 줄: 이전 커밋(a)에는 없었지만, 이후 커밋(b)에서 추가된 내용

---

### 예시

```bash
# 1. 먼저 git log로 커밋 히스토리를 확인
git log --pretty=oneline

# 2. 확인한 두 커밋 아이디를 사용해 diff 실행
# (오래된 커밋 아이디를 먼저 입력)
git diff 172a5da f160bf7
```

```diff
diff --git a/README.md b/README.md
index e69de29..b546591 100644
--- a/README.md
+++ b/README.md
@@ -0,0 +1 @@
+This is calculator for study.

```

---

### 주의사항

- 커밋 아이디는 전체를 다 쓸 필요 없이, 다른 커밋과 구별될 정도의 앞 몇 글자만 입력해도 됨
