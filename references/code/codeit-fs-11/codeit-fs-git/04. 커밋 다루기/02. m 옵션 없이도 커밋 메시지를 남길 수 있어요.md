### 02. m 옵션 없이도 커밋 메시지를 남길 수 있어요

- `git commit` 명령어에서 `-m` 옵션을 생략하면, Git이 기본 텍스트 편집기(예: Vim)를 실행하여 커밋 메시지를 작성할 수 있도록 함
- 이 방법을 사용하면 제목과 본문을 구분하여 더 상세한 커밋 메시지를 작성하기 용이함

---

### 사용 방법

1. 파일을 수정한 후 `git add` 명령어로 Staging Area에 추가
2. 터미널에 `git commit`을 입력하고 엔터를 누름
3. 텍스트 편집기가 열리면 커밋 메시지를 작성
    - 첫 번째 줄에는 커밋 제목(요약)을 작성
    - 한 줄을 비우고 다음 줄부터 상세한 커밋 내용(본문)을 작성
4. 메시지 작성이 끝나면 편집기를 저장하고 종료함 (예: Vim에서 `Esc` 누른 후 `:wq` 입력)
5. 편집기를 종료하면 커밋이 완료됨

---

### 예시 (Vim 편집기)

```
Add multiply function

calculator.js supports multiply now

# Please enter the commit message for your changes. Lines starting
# with '#' will be ignored, and an empty message aborts the commit.
#
# On branch master
# Changes to be committed:
#       modified:   calculator.js
#
```

---

### 주의사항

- 커밋 메시지를 작성하지 않고 편집기를 그냥 종료하면 해당 커밋은 취소됨
- `#`으로 시작하는 줄은 주석으로 처리되어 커밋 메시지에 포함되지 않음

---

### 용어설명

- **Vim**: 리눅스 환경에서 많이 사용되는 기본 텍스트 편집기 중 하나
    - `i`: 입력 모드로 전환
    - `Esc`: 입력 모드에서 일반 모드로 전환
    - `:wq`: 저장(write)하고 종료(quit)
