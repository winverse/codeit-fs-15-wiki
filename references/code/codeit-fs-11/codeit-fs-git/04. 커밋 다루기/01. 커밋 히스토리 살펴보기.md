### 01. 커밋 히스토리 살펴보기

- `git log` 명령어로 커밋 히스토리를 확인할 수 있음
- 커밋 히스토리는 최신순으로 위에서부터 아래로 나열됨
- 각 커밋은 고유한 아이디, 작성자, 날짜, 그리고 커밋 메시지를 포함함

---

### 예시

- 전체 커밋 히스토리 보기

```bash
git log
```

- 커밋 히스토리를 한 줄씩 간략하게 보기

```bash
git log --pretty=oneline
```

- 특정 커밋의 변경 내용 상세보기

```bash
git show [커밋 아이디]
```

---

### 옵션

- `--pretty=oneline`: 커밋 히스토리를 한 줄씩 간결하게 표시

---

### 주의사항

- `git show` 사용 시, 전체 커밋 아이디 대신 앞 일부만 입력하고 `Tab` 키를 누르면 자동 완성 기능을 사용할 수 있음
- `git log` 또는 `git show`의 결과 화면에서 나가려면 `q`를 입력
- 터미널 명령어가 익숙하지 않다면, VS Code의 `Git File History` 같은 확장 프로그램을 통해 GUI 환경에서 더 편리하게 커밋 히스토리를 조회할 수 있음

---

### 용어설명

- **커밋 해시(Commit Hash)**: 각 커밋을 식별하기 위한 고유한 아이디
- **a, b**: `git show` 결과에서 `a`는 변경 전 파일, `b`는 변경 후 파일을 의미
