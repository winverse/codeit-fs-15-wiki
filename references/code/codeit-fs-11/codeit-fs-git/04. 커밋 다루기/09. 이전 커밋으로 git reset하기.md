### 09. 이전 커밋으로 git reset하기

- 특정 시점 이후의 작업 내용이 전부 마음에 들지 않아 과거의 특정 커밋으로 완전히 되돌아가고 싶을 때 `git reset` 명령어를 사용함
- `reset`은 `HEAD`의 위치를 바꾸고, 그에 맞게 워킹 디렉토리의 상태도 변경할 수 있음

---

### 사용 방법

- `git reset --hard [돌아갈 커밋 아이디]`

1. `git log`로 돌아가고 싶은 과거 커밋의 아이디를 확인
2. `git reset --hard [커밋 아이디]` 명령어를 실행

---

### 결과

- `HEAD`가 `[돌아갈 커밋 아이디]`가 가리키는 커밋으로 이동함
- 워킹 디렉토리의 모든 파일 내용이 해당 과거 커밋의 상태와 동일하게 변경됨
- `git log`를 확인하면, 돌아간 시점 이후에 있던 커밋들이 모두 사라진 것처럼 보임

---

### 예시

```bash
# 1. 로그를 확인해 돌아갈 커밋의 아이디(dd91)를 확인
git log --pretty=oneline

# 2. --hard 옵션과 함께 reset 실행
git reset --hard dd91
```

---

### 옵션

- `--hard`: `reset`의 여러 옵션 중 하나로, 워킹 디렉토리의 내용까지 지정한 커밋의 상태로 완전히 되돌리는 강력한 옵션

---

### 주의사항

- `git reset --hard` 명령어는 지정한 커밋 이후의 모든 변경 이력을 삭제하는 것처럼 보이기 때문에 매우 신중하게 사용해야 함
- 실제로 커밋이 영구 삭제되는 것은 아니지만, 복구하기 번거로우므로 중요한 작업 내용이 있다면 사용에 주의가 필요함
