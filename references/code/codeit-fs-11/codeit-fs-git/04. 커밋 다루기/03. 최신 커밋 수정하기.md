### 03. 최신 커밋 수정하기

- 커밋을 한 직후에 코드 수정이 필요하거나 메시지를 잘못 작성한 경우, 새로운 커밋을 만드는 대신 가장 최근의 커밋을 수정할 수 있음
- `git commit --amend` 명령어를 사용하면 최신 커밋에 변경사항을 추가하거나 커밋 메시지를 변경할 수 있음

---

### 사용 방법

1. 코드 수정 등 원하는 작업을 완료
2. `git add` 명령어로 변경된 파일들을 Staging Area에 추가
3. 터미널에 `git commit --amend`를 입력
4. 텍스트 편집기가 열리면, 기존 커밋 메시지를 수정하거나 그대로 둔 채로 저장하고 종료

---

### 결과

- 기존의 최신 커밋은 사라지고, 새로운 변경사항과 메시지를 담은 **새로운 커밋**이 그 자리를 대체함
- 커밋이 새로 만들어지는 것이므로, 커밋 아이디(해시)도 변경됨

---

### 예시

```bash
# 1. 마지막 커밋에 추가할 내용을 마저 수정

# 2. 수정한 파일을 스테이징
git add .

# 3. --amend 옵션으로 커밋 실행
git commit --amend
```

---

### 주의사항

- **매우 중요**: `git push`를 통해 이미 원격 저장소(GitHub 등)에 공유된 커밋은 절대 `amend`하면 안 됨
- 공유된 커밋을 수정하면 다른 팀원들의 작업 히스토리와 충돌하여 큰 문제를 일으킬 수 있음
- `--amend`는 아직 push하지 않은, 자신의 로컬 저장소에만 있는 커밋을 수정할 때만 사용하는 것이 안전함
