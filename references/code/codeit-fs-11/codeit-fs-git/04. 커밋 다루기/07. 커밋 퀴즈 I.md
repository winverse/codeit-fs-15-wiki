### 07. 커밋 퀴즈 I

지금까지 배운 내용을 바탕으로 퀴즈를 풀어보세요

---

### 질문 1

커밋 히스토리를 조회하려고 합니다. 각 커밋이 한 줄로, 간결하게 표시되도록 보고 싶을 때 `git log` 명령어에 추가해야 할 옵션은 무엇일까요?

<details>
<summary>정답 확인하기</summary>

`--pretty=oneline`

</details>

---

### 질문 2

가장 최신의 커밋 1개에 대한 상세한 변경 내용(코드 변경점 포함)을 확인하고 싶을 때 사용하는 명령어는 무엇인가요?

<details>
<summary>정답 확인하기</summary>

`git show`

</details>

---

### 질문 3

`git commit` 명령어를 `-m` 옵션 없이 실행하여 텍스트 편집기가 열렸습니다. 하지만 커밋 메시지를 작성하지 않고 그냥 편집기를 종료하면 어떻게 될까요?

<details>
<summary>정답 확인하기</summary>

커밋이 취소(abort)됩니다

</details>

---

### 질문 4

방금 커밋을 완료했는데, 커밋 메시지에 오타가 있는 것을 발견했습니다. 새로운 커밋을 만들지 않고, 방금 전의 최신 커밋 메시지를 수정하고 싶을 때 사용하는 명령어는 무엇일까요?

<details>
<summary>정답 확인하기</summary>

`git commit --amend`

</details>

---

### 질문 5

`git commit --amend`를 사용하여 최신 커밋을 수정했습니다. 이 경우, 수정된 커밋의 아이디(해시)는 어떻게 될까요?

1. 이전 커밋 아이디와 동일하게 유지된다
2. 이전 커밋 아이디와 다른 새로운 아이디로 변경된다
3. 커밋 아이디가 없어진다

<details>
<summary>정답 확인하기</summary>

2번 (이전 커밋 아이디와 다른 새로운 아이디로 변경된다)

</details>

---

### 질문 6

**매우 중요!** `git commit --amend` 명령어 사용을 반드시 피해야 하는, 위험한 상황은 언제인가요?

<details>
<summary>정답 확인하기</summary>

원격 저장소(GitHub 등)에 이미 `push`한 커밋을 수정할 때입니다. 다른 사람의 작업 내역과 충돌을 일으킬 수 있습니다

</details>

---

### 질문 7

두 개의 특정 커밋 사이의 코드 변경점을 확인하고 싶습니다. 이때 사용하는 명령어는 무엇일까요?

<details>
<summary>정답 확인하기</summary>

`git diff [오래된 커밋 아이디] [최신 커밋 아이디]`

</details>

---

### 질문 8

`git diff`를 사용해 두 커밋을 비교할 때, `git diff [최신 커밋] [오래된 커밋]`처럼 순서를 반대로 입력하면 어떻게 될까요?

<details>
<summary>정답 확인하기</summary>

변경 사항이 반대로 표시됩니다. 즉, 추가된 내용이 삭제된 것처럼(`-`), 삭제된 내용이 추가된 것처럼(`+`) 보입니다

</details>
