### 11. 잘못된 브랜치에서 작업하고 있었다면?

개발을 하다 보면, 당연히 `feature` 브랜치인 줄 알았는데 나중에 보니 `main` 브랜치에서 작업을 하고 있었던 아찔한 실수를 할 때가 있습니다.

다행히 Git은 이런 실수를 되돌릴 수 있는 여러 방법을 제공합니다. 상황에 따라 대처법이 다르므로, 두 가지 시나리오로 나누어 알아보겠습니다.

---

#### **시나리오 1: 아직 커밋하지 않고, 파일만 수정한 경우**

`main` 브랜치에서 `calculator.js`에 `power` 함수를 추가하던 중, 브랜치를 잘못 선택했다는 것을 깨달았습니다.

**해결책: `git stash` 활용하기**

1.  **현재 변경사항 임시 저장**: `main` 브랜치에서 작업한 내용을 `stash`에 임시 저장합니다.
    ```bash
    git stash
    ```
2.  **올바른 브랜치로 이동**: 원래 작업했어야 할 `feature/calculator` 브랜치로 이동합니다.
    ```bash
    git switch feature/calculator
    ```
3.  **임시 저장 내용 불러오기**: `stash`에 저장했던 내 작업 내용을 현재 브랜치로 가져옵니다.
    ```bash
    git stash pop
    ```
-   이제 당신의 변경사항은 `main` 브랜치가 아닌, `feature/calculator` 브랜치 위에 안전하게 옮겨졌습니다.

---

#### **시나리오 2: 이미 커밋까지 해버린 경우**

`main` 브랜치에서 `power` 함수를 추가하고, 무심코 커밋까지 완료해버렸습니다. 이 커밋은 사실 `feature/calculator` 브랜치에 있었어야 합니다.

**해결책: `git cherry-pick`과 `git reset` 활용하기**

1.  **올바른 브랜치로 커밋 복사**: `main`에 잘못 올린 커밋을 `feature/calculator` 브랜치로 '복사'합니다.
    ```bash
    # 1. 올바른 브랜치로 이동
    git switch feature/calculator

    # 2. main 브랜치에 있던 잘못된 커밋을 현재 브랜치로 가져온다
    git cherry-pick [main에_잘못_올린_커밋_아이디]
    ```
2.  **잘못된 브랜치의 커밋 삭제**: `main` 브랜치로 돌아가, 잘못 올렸던 커밋을 삭제합니다.
    ```bash
    # 3. 다시 main 브랜치로 이동
    git switch main

    # 4. main 브랜치의 최신 커밋을 삭제하고, 이전 상태로 되돌린다
    git reset HEAD~1 --hard
    ```
-   **결과**: `main` 브랜치는 깨끗해지고, 잘못 만들었던 커밋은 `feature/calculator` 브랜치에 안전하게 복원되었습니다.
