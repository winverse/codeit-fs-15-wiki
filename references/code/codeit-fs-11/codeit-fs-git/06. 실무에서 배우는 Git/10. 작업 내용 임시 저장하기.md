### 10. 작업 내용 임시 저장하기 (git stash)

작업을 하다 보면, 아직 커밋하기에는 애매한데 급하게 다른 브랜치로 전환해야 할 때가 있습니다. 예를 들어, 한창 새로운 기능을 개발하는 중에 긴급한 버그 수정 요청이 들어온 경우입니다.

이럴 때 아직 완료되지 않은 작업을 어설프게 커밋하고 싶지는 않을 겁니다. `git stash`는 이럴 때 사용하는 아주 유용한 명령어입니다.

---

#### **`git stash`란?**

`stash`는 '숨겨두다', '넣어두다'라는 뜻입니다. `git stash`는 아직 커밋하지 않은 변경사항(Staged, Unstaged 상태의 모든 변경사항)을 잠시 '임시 저장' 공간으로 치워두고, 워킹 디렉토리를 마지막 커밋 상태로 깨끗하게 만들어주는 명령어입니다.

이를 통해 현재 작업 내용을 안전하게 보관한 채로, 다른 브랜치로 자유롭게 이동하여 다른 작업을 수행할 수 있습니다.

---

#### **`git stash` 기본 사용법**

1.  **변경 내용 임시 저장하기**
    -   작업하던 내용이 있는 상태에서 아래 명령어를 실행합니다.

    ```bash
    git stash
    ```
    -   실행 즉시, 모든 변경사항이 임시 저장되고 워킹 디렉토리는 깨끗해집니다.

2.  **임시 저장한 내용 다시 불러오기**
    -   원래 작업하던 브랜치로 돌아와, 아래 명령어를 실행하여 가장 최근에 `stash` 했던 내용을 다시 적용합니다.

    ```bash
    git stash pop
    ```
    -   `pop`은 임시 저장소에서 내용을 가져와 적용함과 동시에, 해당 `stash`를 목록에서 삭제합니다.

---

#### **`git stash` 관련 명령어**

-   `git stash list`: 저장된 `stash` 목록을 확인합니다.
-   `git stash apply`: `pop`과 비슷하게 `stash`를 적용하지만, 목록에서 삭제하지는 않습니다.
-   `git stash drop`: 가장 최근의 `stash`를 목록에서 삭제합니다.

---

### **시나리오: 긴급 버그 수정하기**

1.  `feature/calculator` 브랜치에서 `calculator.js` 파일에 새로운 `power` (거듭제곱) 함수를 추가하는 작업을 하고 있습니다.
    ```javascript
    // ... 기존 코드 ...

    // 제곱
    export const power = (a, b) => a ** b; // 작업 중인 코드
    ```
2.  이때 `main` 브랜치에서 `divide_free` 함수에 0으로 나눌 때 에러가 발생하는 긴급 버그를 수정해달라는 요청을 받았습니다.
3.  아직 커밋하기는 이른 내 작업을 `stash`를 이용해 임시 저장합니다.
    ```bash
    git stash
    ```
4.  깨끗해진 상태에서 `main` 브랜치로 이동하여 버그를 수정하고 커밋합니다.
    ```bash
    git switch main
    # ... calculator.js에서 버그 수정 및 커밋 ...
    ```
5.  버그 수정이 끝나면, 다시 원래 작업하던 브랜치로 돌아옵니다.
    ```bash
    git switch feature/calculator
    ```
6.  `stash` 해두었던 내 작업을 다시 불러와 `power` 함수 개발을 이어나갑니다.
    ```bash
    git stash pop
    ```
