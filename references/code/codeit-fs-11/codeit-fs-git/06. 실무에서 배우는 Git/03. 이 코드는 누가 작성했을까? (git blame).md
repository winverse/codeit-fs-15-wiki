### 03. 이 코드는 누가 작성했을까? (git blame)

프로젝트를 진행하다 보면, 특정 코드 한 줄이 언제, 왜, 그리고 누구에 의해 추가되었는지 궁금할 때가 많습니다. 예를 들어, 버그를 유발하는 것으로 의심되는 코드를 발견했거나, 코드의 작성 의도를 파악하고 싶을 때 유용합니다.

Git은 파일의 각 줄이 어떤 커밋에 의해 마지막으로 수정되었는지 추적하는 강력한 기능을 제공합니다.

---

#### **`git blame` 사용하기**

`blame`은 '비난하다', '책임을 묻다'라는 뜻을 가지고 있습니다. 이름처럼, `git blame` 명령어는 파일의 각 줄에 대해 누가 책임을 가지고 있는지, 즉 누가 마지막으로 수정했는지 알려줍니다.

**사용법**

```bash
git blame [파일 이름]
```

**명령 결과 (터미널 출력 예)**

```
^a1b2c3d (John Doe 2025-11-10 10:30:00 +0900 1) function add(a, b) {
^a1b2c3d (John Doe 2025-11-10 10:30:00 +0900 2)   return a + b;
^a1b2c3d (John Doe 2-25-11-10 10:30:00 +0900 3) }
b4c5d6e (Jane Smith 2025-11-11 15:00:00 +0900 4) function multiply(a, b) {
b4c5d6e (Jane Smith 2025-11-11 15:00:00 +0900 5)   return a * b;
b4c5d6e (Jane Smith 2025-11-11 15:00:00 +0900 6) }
```

-   각 줄의 맨 앞에 **커밋 아이디(해시)**, **작성자**, **수정한 날짜와 시간**이 표시됩니다. 이를 통해 `multiply` 함수는 `b4c5d6e` 커밋에서 `Jane Smith`가 작성했음을 알 수 있습니다.

---

#### **`git show`로 상세 내역 추적하기**

`blame`으로 특정 변경이 이루어진 커밋을 찾았다면, `git show`를 통해 해당 커밋의 전체 변경 내용을 상세히 확인할 수 있습니다.

```bash
# 1. blame 결과에서 궁금한 부분의 커밋 아이디를 복사합니다.

# 2. git show로 해당 커밋의 상세 내역을 확인합니다.
git show b4c5d6e
```

-   이 과정을 통해, 단순히 코드 한 줄의 작성자를 넘어 그 코드가 어떤 맥락에서, 다른 어떤 파일들과 함께 변경되었는지 파악할 수 있습니다.

---

#### **GUI 환경에서의 활용 (feat. GitLens)**

터미널에서 `blame`과 `show`를 번갈아 사용하는 것은 다소 번거로울 수 있습니다. VS Code에서는 **GitLens**라는 매우 강력한 확장 프로그램을 통해 이 모든 과정을 훨씬 편리하게 수행할 수 있습니다.

-   **GitLens 기능**:
    -   코드를 한 줄 클릭하기만 하면, 해당 줄 바로 옆에 누가, 언제, 어떤 커밋 메시지로 수정했는지 희미하게 표시됩니다.
    -   이 표시된 부분을 클릭하면 해당 커밋의 상세 변경 내용, 이전 버전과 비교 등 훨씬 더 많은 정보를 시각적으로 탐색할 수 있습니다.

-   **결론**: `git blame`은 강력한 기능이지만, 실무에서는 GitLens와 같은 GUI 도구를 통해 훨씬 더 빠르고 직관적으로 코드의 히스토리를 추적하는 경우가 많습니다.

---

### **요약**

-   `git blame [파일 이름]`: 파일의 각 줄을 누가, 언제 마지막으로 수정했는지 보여줍니다.
-   `blame` 결과로 얻은 커밋 아이디를 `git show`에 넘겨주면, 해당 변경점의 구체적인 맥락을 파악할 수 있습니다.
-   VS Code의 **GitLens** 확장 프로그램을 사용하면 이 과정을 훨씬 더 편리하게 수행할 수 있습니다.
