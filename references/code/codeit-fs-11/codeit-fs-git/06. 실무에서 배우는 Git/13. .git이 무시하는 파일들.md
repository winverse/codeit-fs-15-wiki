### 13. Git이 무시하는 파일들 (.gitignore)

프로젝트를 진행하다 보면, Git으로 굳이 버전 관리를 할 필요가 없거나, 해서는 안 되는 파일들이 생깁니다.

-   `node_modules`, `venv` 등 라이브러리/패키지 폴더 (설치하면 자동으로 생기므로)
-   `.log` 등 시스템이 자동으로 생성하는 로그 파일
-   `build/`, `dist/` 등 빌드 결과물
-   `.env` 등 API 키나 비밀번호가 담긴 환경 설정 파일 (보안상 절대 올리면 안 됨)

이런 파일들이 `git status`에 계속 `Untracked files`로 나타나면 지저분하고, 실수로 `git add .` 명령어를 실행했을 때 원치 않는 파일들이 커밋될 위험이 있습니다.

---

#### **`.gitignore` 파일의 역할**

`.gitignore`는 이름 그대로, Git이 특정 파일이나 폴더를 **처음부터 추적하지 않도록(ignore)** 설정하는 파일입니다. 이 파일에 무시하고 싶은 파일의 이름이나 패턴을 적어두면, Git은 해당 파일들을 없는 셈 치고 관리 대상에서 제외합니다.

---

#### **`.gitignore` 작성법**

1.  프로젝트의 최상위 폴더(root)에 `.gitignore` 라는 이름의 파일을 만듭니다.
2.  파일 안에 무시하고 싶은 파일/폴더의 이름이나 패턴을 한 줄에 하나씩 적습니다.

**패턴 예시**

```
# 주석을 달 수 있습니다.

# 특정 파일 무시
secret.key
.env

# 특정 폴더와 그 안의 모든 내용 무시 (슬래시/는 필수는 아님)
node_modules/
build/

# 특정 확장자를 가진 모든 파일 무시
*.log
*.tmp

# 예외 처리: 특정 패턴은 무시하지만, 그중 일부는 무시하지 않음
# 모든 .log 파일을 무시하지만, important.log는 무시하지 않음
*.log
!important.log
```

-   `.gitignore` 파일 자체는 Git으로 관리하여 커밋하고 `push`해야, 팀원 모두에게 동일한 무시 규칙이 적용됩니다.

---

#### **⚠️ 이미 추적 중인 파일 무시하기**

`.gitignore`는 **아직 한 번도 Git이 추적한 적 없는(Untracked) 파일에 대해서만 동작**합니다.

만약 실수로 `secret.key` 파일을 이미 커밋해버렸다면, 나중에 `.gitignore`에 `secret.key`를 추가해도 Git은 계속해서 그 파일의 변경사항을 추적합니다.

이 경우, 아래 명령어를 통해 Git의 추적 대상에서만 파일을 제거해야 합니다.

```bash
# 저장소의 추적 목록에서만 파일을 제거하고, 실제 파일은 남겨둠
git rm --cached secret.key

# 이제 .gitignore 규칙이 적용되어, 다음부터는 추적되지 않음
```
