### 14. Git 실전 퀴즈 IV

`stash`, `cherry-pick`, `.gitignore` 등 실무에서 유용한 기능들에 대해 잘 이해했는지 퀴즈를 통해 확인해보세요.

---

### 질문 1

현재 브랜치에서 한창 작업하던 중, 아직 커밋하기는 애매한데 급하게 다른 브랜치로 이동하여 다른 일을 처리해야 합니다. 이때 내 작업 내용을 안전하게 임시 저장하기 위해 사용하는 명령어는 무엇인가요?

<details>
<summary>정답 및 해설 보기</summary>

`git stash`

</details>

---

### 질문 2

`git stash`로 임시 저장했던 가장 최근의 작업을 다시 가져와 적용하고, `stash` 목록에서도 삭제하고 싶습니다. 어떤 명령어를 사용해야 할까요?

<details>
<summary>정답 및 해설 보기</summary>

`git stash pop`

</details>

---

### 질문 3

다른 브랜치에 있는 여러 커밋들 중, 특정 커밋 하나만 골라서 현재 내 브랜치에 똑같이 적용하고 싶을 때 사용하는 명령어는 무엇인가요?

<details>
<summary>정답 및 해설 보기</summary>

`git cherry-pick [가져올_커밋_아이디]`

</details>

---

### 질문 4

프로젝트의 로그 파일(`*.log`)이나, API 키가 들어있는 `.env` 파일 등을 Git이 버전 관리하지 않도록 무시하게 만들려면 어떤 파일을 생성하고 내용을 작성해야 하나요?

<details>
<summary>정답 및 해설 보기</summary>

`.gitignore` 파일

</details>

---

### 질문 5

실수로 `credentials.json` 파일을 커밋하고 원격 저장소에 `push`까지 해버렸습니다. 나중에 이 파일이 민감한 정보를 담고 있다는 것을 깨닫고 `.gitignore`에 `credentials.json`을 추가했지만, 여전히 Git이 이 파일의 변경사항을 추적합니다.

이미 추적되고 있는 이 파일을 Git의 관리 대상에서 제외하기 위해 어떤 명령어를 먼저 실행해야 할까요? (단, 내 컴퓨터에서 파일이 삭제되면 안 됩니다.)

<details>
<summary>정답 및 해설 보기</summary>

`git rm --cached credentials.json`

이 명령어는 실제 파일은 남겨둔 채, Git의 추적 대상(Staging Area)에서만 파일을 제거합니다. 이 명령어를 실행하고 커밋해야, 비로소 `.gitignore` 규칙이 적용되어 다음부터는 해당 파일이 추적되지 않습니다. (물론, 이미 원격 저장소에 올라간 과거 기록까지 지워주지는 않으므로 주의해야 합니다.)

</details>
