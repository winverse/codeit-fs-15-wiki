### 12. 필요한 커밋만 가져오는 git cherry-pick

`cherry-pick`은 이름 그대로, 다른 브랜치에 있는 수많은 커밋 중에서 내가 원하는 커밋 몇 개만 '체리처럼 콕 집어서' 현재 내 브랜치로 가져오는 기능입니다.

`merge`가 브랜치의 모든 변경사항을 통째로 합치는 것이라면, `cherry-pick`은 특정 커밋 하나의 변경사항만 복사하여 새로운 커밋으로 만들어줍니다.

---

#### **`git cherry-pick`은 언제 사용할까?**

1.  **잘못된 브랜치에 커밋한 내용을 옮길 때**
    -   이전 강의에서 본 것처럼, `main`에 실수로 올린 커밋을 `feature` 브랜치로 가져올 때 유용합니다.

2.  **급한 버그 수정사항만 먼저 반영하고 싶을 때**
    -   `hotfix` 브랜치에서 여러 버그를 수정한 커밋들이 있는데, 그중 가장 치명적인 버그를 수정한 커밋 하나만 `main` 브랜치에 긴급하게 먼저 적용해야 할 때 사용합니다.

3.  **다른 동료의 유용한 기능만 가져오고 싶을 때**
    -   동료의 `feature/new-payment` 브랜치에 아직 개발 중인 기능들이 많지만, 그중 '결제 유효성 검사' 기능 커밋은 내 작업에도 바로 필요할 때, 해당 커밋만 가져올 수 있습니다.

---

#### **`git cherry-pick` 사용법**

1.  **가져올 커밋의 아이디(해시) 확인**: `git log` 등을 사용하여, 다른 브랜치에 있는 커밋 중 가져오고 싶은 커밋의 아이디를 복사합니다.

2.  **내 브랜치에서 명령어 실행**: 커밋을 적용하고 싶은 내 브랜치로 이동한 후, 아래 명령어를 실행합니다.

    ```bash
    git cherry-pick [가져올_커밋_아이디]
    ```

-   실행 즉시, 해당 커밋의 변경사항이 그대로 적용된 **새로운 커밋**이 현재 브랜치에 생성됩니다. (커밋 아이디는 달라집니다)

---

### **시나리오: 다른 브랜치의 유용한 기능만 가져오기**

1.  `feature/advanced-math` 브랜치에 "Feat: Add power function" 커밋과 "Feat: Add factorial function" 커밋이 순서대로 있습니다.
2.  `main` 브랜치에서 작업하던 중, "Feat: Add power function" 커밋의 내용이 지금 바로 필요해졌습니다.
3.  `main` 브랜치에서 아래 명령어를 실행합니다.
    ```bash
    # 1. 'Add power function' 커밋의 아이디를 복사 (예: a1b2c3d)

    # 2. main 브랜치에서 cherry-pick 실행
    git cherry-pick a1b2c3d
    ```
4.  이제 `main` 브랜치의 `calculator.js` 파일을 확인하면, `feature/advanced-math` 브랜치 전체를 `merge`하지 않았음에도 `power` 함수가 추가된 것을 볼 수 있습니다.
