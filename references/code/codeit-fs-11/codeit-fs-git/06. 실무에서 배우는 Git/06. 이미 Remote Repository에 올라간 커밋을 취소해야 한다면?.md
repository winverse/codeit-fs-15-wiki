### 06. 이미 Remote Repository에 올라간 커밋을 취소해야 한다면?

협업을 하다 보면, 분명히 작업을 완료해서 원격 저장소에 `push`까지 했는데, "그 기능 다시 빼주세요" 라는 요청을 받을 때가 있습니다. 이미 팀원들과 공유된 커밋을 어떻게 안전하게 되돌릴 수 있을까요?

---

#### **1. 문제 상황: 공유된 커밋을 되돌려야 할 때**

`main` 브랜치에 `square` (제곱) 함수를 추가하고, 원격 저장소에 `push`까지 완료한 상황이라고 가정해봅시다.

```bash
# 1. calculator.js에 square 함수 추가
# ... 코드 수정 ...

# 2. 커밋 및 푸시 완료
git add calculator.js
git commit -m "Add square function"
git push
```

그런데 갑자기 팀 회의에서 "제곱 기능은 이번 버전에서 제외"하기로 결정되었습니다. 이미 원격 저장소에 올라가 버린 `Add square function` 커밋을 없었던 일로 만들어야 합니다.

---

#### **2. `git revert`: 안전하게 변경사항 되돌리기**

이럴 때 사용하는 명령어가 바로 `git revert` 입니다. `revert`는 '되돌리다', '복귀하다'라는 뜻으로, 특정 커밋에서 이루어진 변경사항을 **거꾸로 되돌리는 새로운 커밋**을 만들어줍니다.

**`git reset`과의 차이점**

-   `git reset`: 기존 커밋을 **삭제**하고 히스토리를 과거로 되돌립니다. (과거를 바꾸는 것)
-   `git revert`: 기존 커밋은 그대로 둔 채, 그 커밋의 변경 내용을 취소하는 **새로운 커밋**을 추가합니다. (실수를 만회하는 새로운 행동을 추가하는 것)

---

#### **3. `git revert` 사용법**

1.  **되돌릴 커밋 확인**: `git log`로 취소하고 싶은 커밋(`Add square function`)의 아이디를 확인합니다.

2.  **`revert` 명령어 실행**:

    ```bash
    git revert [취소할_커밋_아이디]
    ```

3.  **커밋 메시지 작성**: 명령을 실행하면, 어떤 커밋을 되돌렸는지 알려주는 기본 커밋 메시지가 작성된 텍스트 편집기가 열립니다.
    -   `Revert "Add square function"`
    -   메시지를 그대로 저장하고 편집기를 종료하면, `revert` 커밋이 생성됩니다.

4.  **결과 확인**: `calculator.js` 파일을 다시 열어보면 `square` 함수가 사라진 것을 확인할 수 있습니다. `git log`를 보면, `Add square function` 커밋 위에 `Revert "Add square function"` 이라는 새로운 커밋이 쌓여있습니다.

5.  **원격 저장소에 반영**: 이 `revert` 커밋은 새로운 커밋이므로, 원격 저장소에 안전하게 `push` 할 수 있습니다.

    ```bash
    git push
    ```

---

#### **4. 왜 `reset`을 사용하면 안 될까?**

만약 이 상황에서 `git reset --hard [이전_커밋_아이디]`를 사용했다면 어떻게 될까요?

1.  로컬 저장소의 `main` 브랜치는 `Add square function` 커밋이 없던 과거로 돌아갑니다.
2.  하지만 원격 저장소의 `main` 브랜치에는 여전히 `Add square function` 커밋이 남아있습니다. 즉, **원격 저장소가 로컬보다 더 최신인 상태**가 됩니다.
3.  이 상태에서 `git push`를 시도하면, `non-fast-forward` 에러가 발생하며 거부됩니다.
4.  이를 강제로 밀어 넣으려면 `git push --force`를 사용해야 하는데, 이는 원격 저장소의 히스토리를 강제로 덮어쓰는 매우 위험한 행동입니다. 다른 팀원이 `Add square function` 커밋을 기반으로 이미 다른 작업을 시작했다면, 팀 전체의 히스토리가 꼬여버리는 대참사가 발생할 수 있습니다.

---

### **요약: `reset` vs. `revert`**

-   **`git reset`**: **아직 `push`하지 않은, 나만 보는 로컬 저장소의 커밋**을 되돌릴 때 사용합니다.
-   **`git revert`**: **이미 `push`하여 팀원들과 공유된 커밋**의 변경사항을 안전하게 취소하고 싶을 때 사용합니다. 새로운 커밋을 생성하여 작업 내역을 명확히 남기므로, 협업 시 반드시 `revert`를 사용해야 합니다.
