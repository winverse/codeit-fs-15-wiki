### 15. Git 토픽 내용 총정리

이 문서는 튜토리얼에서 다룬 모든 Git 명령어와 핵심 개념을 빠르게 찾아볼 수 있도록 정리한 종합 요약본입니다.

---

### **1. 로컬 저장소 설정 및 기본 작업**

-   **`git init`**
    -   현재 폴더에 Git 저장소(.git 폴더)를 생성하여 버전 관리를 시작합니다.
-   **`git add [파일 이름]`**
    -   작업 디렉토리의 변경사항을 스테이징 영역(Staging Area)으로 추가합니다.
-   **`git add .`**
    -   현재 폴더와 하위 폴더의 모든 변경사항을 스테이징 영역에 추가합니다.
-   **`git commit -m "커밋 메시지"`**
    -   스테이징 영역의 변경사항을 메시지와 함께 저장소에 영구적으로 기록(커밋)합니다.
-   **`git commit`**
    -   `-m` 옵션 없이 실행하면, 텍스트 편집기가 열려 더 상세한 커밋 메시지를 작성할 수 있습니다.
-   **`git status`**
    -   파일의 현재 상태(Untracked, Modified, Staged 등)를 확인합니다.
-   **`git log`**
    -   저장소의 커밋 히스토리를 최신순으로 조회합니다.
    -   `--oneline`, `--graph` 등 다양한 옵션과 함께 사용됩니다.
-   **`git diff`**
    -   작업 디렉토리와 스테이징 영역, 또는 두 커밋 간의 코드 차이를 비교합니다.
-   **`git restore --staged [파일]`**
    -   스테이징 영역에 올린 파일(`git add`한 파일)을 다시 내립니다. (Unstage)

---

### **2. 원격 저장소(GitHub) 및 협업**

-   **`git clone [원격 저장소 주소]`**
    -   원격 저장소를 내 컴퓨터에 그대로 복제하여 로컬 저장소를 만듭니다.
-   **`git remote add origin [주소]`**
    -   로컬 저장소에 원격 저장소 주소를 `origin`이라는 이름으로 연결합니다.
-   **`git push`**
    -   로컬 브랜치의 커밋을 원격 저장소로 업로드합니다.
-   **`git push -u origin [브랜치명]`**
    -   로컬 브랜치를 원격 저장소의 특정 브랜치와 연결하며 처음으로 `push`할 때 사용합니다.
-   **`git pull`**
    -   원격 저장소의 최신 내용을 가져와(fetch) 현재 로컬 브랜치와 병합(merge)합니다.
-   **`git pull --no-rebase`**
    -   `pull` 시, `rebase`가 아닌 `merge` 방식으로 합치도록 명시합니다.
-   **`git blame [파일 이름]`**
    -   파일의 각 줄을 누가, 언제, 어떤 커밋에서 마지막으로 수정했는지 추적합니다.

---

### **3. 커밋 되돌리기 및 수정**

-   **`git commit --amend`**
    -   가장 최신의 커밋을 수정합니다. (메시지 변경 또는 파일 추가)
    -   **주의**: 원격 저장소에 `push`하지 않은 커밋에만 사용해야 합니다.
-   **`git reset [옵션] [커밋]`**
    -   브랜치 포인터를 지정한 커밋으로 이동시켜 과거로 되돌립니다. (히스토리 변경)
    -   `--hard`: 모든 변경사항(커밋, 스테이징, 작업 디렉토리)을 완전히 삭제하며 되돌립니다.
    -   **주의**: `reset` 역시 `push`하지 않은 로컬 커밋에만 사용하는 것이 안전합니다.
-   **`git revert [커밋]`**
    -   지정한 커밋의 변경 내용을 **거꾸로 되돌리는 새로운 커밋**을 생성합니다.
    -   기존 히스토리를 삭제하지 않으므로, **이미 원격 저장소에 `push`한 커밋을 안전하게 취소**할 때 사용합니다.
-   **`git reflog`**
    -   `HEAD`가 이동했던 모든 기록(커밋, 리셋, 체크아웃 등)을 보여줍니다. `reset`으로 사라진 것처럼 보이는 커밋을 복구할 때 유용합니다.

---

### **4. 브랜치 및 고급 기능**

-   **`git branch`**
    -   브랜치 목록을 보여줍니다.
-   **`git switch [브랜치명]`**
    -   지정한 브랜치로 작업 공간을 전환합니다.
-   **`git switch -c [브랜치명]`**
    -   새로운 브랜치를 생성하고, 즉시 그 브랜치로 전환합니다.
-   **`git branch -d [브랜치명]`**
    -   지정한 브랜치를 삭제합니다.
-   **`git merge [브랜치명]`**
    -   지정한 브랜치의 내용을 현재 브랜치로 병합합니다.
-   **`git merge --no-ff`**
    -   `Fast-forward`가 가능한 상황에서도, 병합 내역을 남기기 위해 강제로 병합 커밋(Merge Commit)을 생성합니다.
-   **`git stash`**
    -   커밋하지 않은 변경사항을 임시 저장 공간에 숨겨둡니다.
-   **`git stash pop`**
    -   가장 최근에 `stash`한 내용을 다시 적용하고, 목록에서 삭제합니다.
-   **`git cherry-pick [커밋 아이디]`**
    -   다른 브랜치에 있는 특정 커밋 하나만 현재 브랜치로 가져와 적용합니다.

---

### **5. 기타 설정**

-   **`.gitignore` 파일**
    -   Git이 추적하지 않아야 할 파일이나 폴더의 목록을 지정합니다. (예: `node_modules/`, `*.log`, `.env`)
-   **`git rm --cached [파일]`**
    -   이미 Git이 추적하고 있는 파일을, 실제 파일은 삭제하지 않고 Git의 관리 대상에서만 제외시킵니다. `.gitignore`에 뒤늦게 추가한 파일을 적용할 때 사용합니다.
