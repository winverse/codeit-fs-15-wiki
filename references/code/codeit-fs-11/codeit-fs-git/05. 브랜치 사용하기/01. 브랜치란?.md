### 01. 브랜치란?

브랜치(Branch)는 Git에서 가장 중요하고 강력한 기능 중 하나로, 독립적인 코드 관리 및 개발 흐름을 의미합니다.

---

### 브랜치의 개념

- **나뭇가지 모델**: Git의 커밋 히스토리는 하나의 뿌리(root commit)에서 시작해 나뭇가지(branch)처럼 여러 갈래로 뻗어 나갈 수 있습니다. 이 각각의 갈래가 바로 '브랜치'입니다.
- **독립적인 작업 공간**: 브랜치를 사용하면 기존의 메인 코드(예: `main` 브랜치)에 영향을 주지 않고, 새로운 기능을 추가하거나 버그를 수정하는 등 독립적인 작업을 진행할 수 있습니다.
- **활용 예시**: 유료 버전과 무료 버전을 동시에 개발할 때, 공통 기능은 `main` 브랜치에서 작업하고 각 버전에만 해당하는 기능은 `premium`, `free` 등 별도의 브랜치를 만들어 관리하면 효율적입니다.

---

### 새로운 브랜치 만들고 작업하기

1.  **브랜치 생성 및 이동**: `git switch -c [새 브랜치 이름]` 명령어로 새로운 브랜치를 만들고, 즉시 해당 브랜치로 작업 위치를 옮길 수 있습니다.

2.  **독립적인 커밋**: 새로 만든 브랜치에서 파일을 수정하고 커밋하면, 해당 변경 내용은 오직 현재 브랜치에만 기록됩니다. 다른 브랜치에는 아무런 영향이 없습니다.

---

### 예시: `premium` 브랜치 생성 및 커밋

```bash
# 'premium'이라는 새 브랜치를 만들고 그 브랜치로 이동
git switch -c premium

# # (LICENSE 파일 생성 및 내용 추가 등 작업 수행)

# 변경사항을 스테이징
git add .

# 'premium' 브랜치에만 새로운 커밋을 생성
git commit -m "Add License for premium"
```

- 위와 같이 실행하면, `git graph`에서 `premium` 브랜치는 새로운 커밋을 가리키며 앞으로 나아가고, `main` 브랜치는 원래 위치에 그대로 머물러 있는 것을 확인할 수 있습니다. 이를 통해 두 작업 흐름이 분리되었음을 알 수 있습니다.

---

### 용어설명

- **브랜치(Branch)**: 독립적인 코드 관리 및 개발 흐름. 나뭇가지처럼 하나의 커밋에서 분기하여 별도의 작업 내역을 기록할 수 있음
- **`git pull`**: 원격 저장소의 최신 내용을 가져와 현재 로컬 브랜치와 동기화하는 명령어
- **`git switch -c`**: 새로운 브랜치를 생성(create)하고, 그 브랜치로 즉시 전환(switch)하는 명령어
- **Fast-forward**: 히스토리가 분기되지 않고 직선으로 이어질 때, 병합 커밋 없이 브랜치 포인터만 최신 커밋으로 이동시키는 병합 방식
