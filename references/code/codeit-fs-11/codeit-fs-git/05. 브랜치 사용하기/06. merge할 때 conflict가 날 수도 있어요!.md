### 06. merge할 때 conflict가 날 수도 있어요! ⭐⭐

`merge`가 항상 성공하는 것은 아닙니다. 만약 서로 다른 두 브랜치에서 **같은 파일의 같은 부분을 다르게 수정**했다면, Git은 어떤 변경사항을 선택해야 할지 스스로 판단할 수 없어 **Merge 충돌(Conflict)**을 발생시킵니다.

---

### Merge 충돌이란?

- **원인**: Git이 자동으로 병합할 수 없는 상황. 예를 들어 `main`에서는 함수의 이름을 `A`로 바꾸고, `premium`에서는 `B`로 바꾼 뒤 `merge`를 시도하는 경우입니다.
- **Git의 동작**: Git은 병합을 잠시 멈추고, 개발자에게 어떤 코드가 최종 결과물에 남아야 하는지 직접 해결해달라고 요청합니다.

---

### 충돌 해결하기 (VS Code 기준)

1.  **충돌 확인**: `git merge` 실행 시 터미널에 `CONFLICT` 메시지가 나타나고, 해당 파일을 열면 아래와 같은 충돌 표시자가 보입니다.

    ```javascript
    <<<<<<< HEAD
    // 현재 브랜치(premium)의 변경사항
    const divide_premium = (a, b) => a / b;
    =======
    // 병합하려는 브랜치(main)의 변경사항
    const divide_free = (a, b) => a / b;
    >>>>>>> main
    ```

2.  **변경사항 선택 및 수정**:
    - **소통**: **(가장 중요)** 실무에서는 이 코드를 수정한 다른 개발자와 함께 상의하여 어떤 내용을 남길지, 혹은 어떻게 합칠지 결정해야 합니다.
    - **코드 수정**: VS Code에서는 충돌 부분 위에 나타나는 버튼(`Accept Current Change`, `Accept Incoming Change` 등)을 클릭하거나, '병합 편집기(Open in Merge Editor)'를 열어 시각적으로 편리하게 코드를 수정할 수 있습니다. 원하는 최종 결과가 되도록 코드를 직접 편집합니다.

3.  **병합 완료(Commit)**:
    - 충돌을 모두 해결하고 파일을 저장합니다.
    - `git add [충돌이 해결된 파일]` 명령어로 해결된 내용을 스테이징합니다.
    - `git commit`을 실행하여 병합을 최종적으로 완료합니다. (이때 커밋 메시지는 보통 자동으로 작성됩니다)

---

### 충돌에 대한 마음가짐

- **충돌은 에러가 아닙니다**: 충돌은 Git을 사용한 협업 과정에서 발생하는 매우 자연스러운 일입니다. 절대 당황하거나 두려워할 필요가 없습니다.
- **핵심은 소통**: 충돌이 발생했다는 것은 "어떤 코드가 최선인지 개발자들끼리 상의해서 결정해주세요"라는 Git의 신호입니다. 침착하게 코드를 분석하고 동료와 대화하는 것이 가장 중요합니다.
