### 18. 개발에서 흔히 발생하는 시나리오로 Git 이해하기

지금까지 Git의 여러 기능들을 나누어 배웠습니다. 이제 실제 개발 과정과 유사한 시나리오를 통해, 배운 명령어들이 어떻게 유기적으로 사용되는지 체험해보겠습니다.

---

### **시나리오: 새로운 '로그인 기능' 개발하기**

당신은 팀 프로젝트에 참여한 개발자입니다. 당신에게 '로그인 기능 개발'이라는 임무가 주어졌습니다. 지금부터 아래의 과정을 차근차근 따라 해보세요.

#### **0. 사전 준비: 프로젝트 시작하기**

먼저 바탕화면에 이 시나리오를 진행할 새로운 프로젝트 폴더를 만들고, Git 저장소로 초기화한 후 GitHub 원격 저장소까지 연결하겠습니다.

```bash
# 1. 바탕화면으로 이동합니다.
cd ~/Desktop

# 2. 'git-scenario' 라는 이름의 새 폴더를 만들고, 그 안으로 이동합니다.
mkdir git-scenario
cd git-scenario

# 3. 이 폴더를 Git 저장소로 초기화합니다.
git init
```

이제 GitHub에 새로운 원격 저장소를 만들 차례입니다.
**`03. Github 시작하기` 챕터의 `01. GitHub 계정과 Remote Repository 만들기` 내용을 참고하여, `git-scenario` 라는 이름의 새로운 원격 저장소를 GitHub에 만들어주세요.**

저장소를 만들었다면, 아래 명령어를 터미널에 입력하여 로컬 저장소와 원격 저장소를 연결합니다. `<repository_url>` 부분에는 방금 만든 GitHub 저장소의 주소를 복사하여 붙여넣으세요.

```bash
# 4. 로컬 저장소에 원격 저장소 주소를 'origin'이라는 이름으로 추가합니다.
git remote add origin <repository_url>

# 5. 프로젝트의 기본이 될 README.md 파일을 만들고 첫 커밋을 남깁니다.
touch README.md
git add README.md
git commit -m "Initial commit"

# 6. 현재 브랜치 이름을 'main'으로 설정하고, 원격 저장소에 첫 커밋을 push합니다.
git branch -M main
git push -u origin main
```
- **학습 포인트**: 로컬에서 Git 프로젝트를 시작하고(`init`), 원격 저장소를 만들어 연결(`remote add`)한 후, 첫 커밋을 `push`하여 프로젝트의 기반을 마련했습니다. 이제 협업할 준비가 되었습니다.

---

#### **1. 기능 개발 시작: 새 브랜치 만들기**

`main` 브랜치에 직접 기능을 개발하면, 아직 완성되지 않은 코드가 다른 팀원에게 영향을 줄 수 있습니다. 항상 독립적인 브랜치를 만들어 작업을 시작하는 것이 좋습니다.

```bash
# 1. 현재 브랜치가 main인지 확인합니다. (이미 main 브랜치에 있습니다)
git switch main

# 2. 'feature/login'이라는 이름의 새 브랜치를 만들고, 그 브랜치로 이동합니다.
git switch -c feature/login
```
- **학습 포인트**: `switch -c`를 사용하여 독립적인 작업 공간(브랜치)을 확보했습니다.

---

#### **2. 첫 번째 기능 구현 및 커밋**

로그인 페이지의 기본 UI를 구성하는 코드를 작성하고 첫 커밋을 남깁니다.

```bash
# 1. 'login.js' 파일을 새로 만듭니다. (실제 코드를 작성했다고 가정)
touch login.js

# 2. 변경된 내용을 스테이징합니다.
git add login.js

# 3. "로그인 UI 추가"라는 메시지와 함께 커밋합니다.
git commit -m "Feat: Add login UI"
```
- **학습 포인트**: `add`, `commit`을 통해 작업한 내용을 의미 있는 단위로 저장했습니다.

---

#### **3. 두 번째 기능 구현 및 커밋**

이제 실제 로그인을 처리하는 로직을 `login.js` 파일에 추가하고 또 한 번 커밋을 합니다.

VS Code로 `login.js` 파일을 열고, 아래 내용을 추가한 뒤 저장해주세요.

```javascript
console.log('Login logic is here');
```

> **💡 자바스크립트를 모르셔도 괜찮아요!**
> `console.log()`는 자바스크립트에서 괄호 안의 내용을 개발자용 콘솔(터미널과 비슷한 창)에 출력해주는 기능입니다. 코드가 잘 동작하는지 중간중간 확인하기 위해 사용하는, 가장 기본적인 명령어 중 하나입니다.

이제 파일이 실제로 변경되었으니, 커밋을 진행해봅시다.

```bash
# 1. 변경된 'login.js' 파일을 스테이징합니다.
git add login.js

# 2. "로그인 로직 구현"이라는 메시지와 함께 커밋합니다.
git commit -m "Feat: Implement login logic"
```
- **학습 포인트**: 기능 개발을 진행하며 변경사항을 단계별로 커밋하여 히스토리를 쌓았습니다.

---

#### **4. 실수 발견 및 수정: 최신 커밋 보완하기 (`--amend`)**

방금 커밋을 하고 코드를 다시 보니, `console.log`에 적은 메시지에 오타가 있는 것을 발견했습니다. 이 작은 수정을 위해 새로운 커밋을 만들기보다, 방금 한 커밋을 보완(amend)하여 히스토리를 깔끔하게 유지해봅시다.

VS Code로 `login.js` 파일을 다시 열고, 아래와 같이 `console.log`의 내용을 수정한 뒤 저장해주세요.

*   **변경 전**: `console.log('Login logic is here');`
*   **변경 후**: `console.log('Login logic is complete');`

이제 수정된 내용을 커밋에 반영해봅시다.

```bash
# 1. 수정된 'login.js' 파일을 스테이징합니다.
git add login.js

# 2. --amend 옵션을 사용하여 최신 커밋에 덮어씌웁니다.
#    (커밋 메시지 편집기 창이 뜨면, 수정 없이 그대로 저장 및 종료)
git commit --amend
```
- **학습 포인트**: `commit --amend`를 사용해, 아직 원격에 올리지 않은 커밋을 수정하여 불필요한 "버그 수정" 커밋을 만들지 않고 히스토리를 관리했습니다.

---

#### **5. 원격 저장소에 공유하기 (`push`)**

이제 기능 개발이 완료되었습니다. 팀원들이 내 코드를 리뷰하고 사용할 수 있도록 원격 저장소(GitHub)에 `feature/login` 브랜치를 올립니다.

```bash
# 'feature/login' 브랜치는 로컬에만 있으므로, 처음 올릴 때는 --set-upstream 옵션이 필요합니다.
git push --set-upstream origin feature/login
```
- **학습 포인트**: `--set-upstream` 옵션을 사용하여 로컬 브랜치를 원격 브랜치와 연결하고 공유하는 방법을 배웠습니다.

---

#### **6. 작업 내용 통합하기 (`merge`)**

코드 리뷰가 끝나고, 이제 내가 만든 로그인 기능을 프로젝트의 메인 코드인 `main` 브랜치에 합칠 차례입니다.

```bash
# 1. 통합의 주체가 될 'main' 브랜치로 이동합니다.
git switch main

# 2. --no-ff 옵션으로 'feature/login' 브랜치의 작업 내역을 명확히 남기며 병합합니다.
git merge --no-ff feature/login
```
- **학습 포인트**: `switch`로 브랜치를 이동하고, `merge --no-ff`를 통해 기능 브랜치의 작업 히스토리를 명확하게 남기면서 `main` 브랜치에 통합했습니다.

---

#### **7. 최종 결과 반영하기**

`main` 브랜치에 성공적으로 병합된 최신 결과를 원격 저장소에도 반영하여 모든 팀원이 볼 수 있도록 합니다.

```bash
# main 브랜치는 이미 원격과 연결되어 있으므로, 간단히 push만 실행합니다.
git push
```
- **학습 포인트**: `main` 브랜치에 반영된 모든 작업 내용을 `push`하여 프로젝트 전체에 동기화했습니다.
