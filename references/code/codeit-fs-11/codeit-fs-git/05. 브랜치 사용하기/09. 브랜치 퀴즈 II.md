### 09. 브랜치 퀴즈 II

브랜치 병합(merge)과 충돌(conflict) 해결에 대해 잘 이해했는지 퀴즈를 통해 확인해보세요.

---

### 질문 1

`feature` 브랜치에서 작업한 새로운 기능들을 `main` 브랜치로 합치고(merge) 싶습니다. 가장 먼저 실행해야 할 명령어는 무엇일까요?

1.  `git merge feature`
2.  `git switch main`
3.  `git switch feature`

<details>
<summary>정답 확인하기</summary>

2번. 내용을 받아야 하는 브랜치(`main`)로 먼저 이동(switch)해야 합니다.

</details>

---

### 질문 2

`main` 브랜치에서 `git merge feature`를 실행했더니, 별도의 merge commit 없이 `main` 브랜치가 `feature` 브랜치의 위치로 그냥 이동해버렸습니다(Fast-forward).

`feature` 브랜치의 작업 내역을 히스토리에 명확히 남기기 위해, merge commit을 강제로 생성하려면 어떤 명령어를 사용해야 할까요?

<details>
<summary>정답 확인하기</summary>

`git merge --no-ff feature`

</details>

---

### 질문 3

`git merge`를 시도했을 때 Merge Conflict(충돌)가 발생하는 가장 일반적인 원인은 무엇인가요?

<details>
<summary>정답 확인하기</summary>

합치려는 두 브랜치에서 **같은 파일의 같은 라인**을 서로 다르게 수정했기 때문입니다.

</details>

---

### 질문 4

Merge Conflict가 발생하여 VS Code에서 충돌 부분을 원하는 대로 수정한 후, 이 병합을 최종적으로 완료하기 위해 순서대로 실행해야 할 명령어는 무엇인가요?

<details>
<summary>정답 확인하기</summary>

`git add [수정한 파일]` 과 `git commit` 입니다. (충돌을 해결한 파일을 스테이징하고, 커밋을 통해 병합을 마무리합니다.)

</details>

---

### 질문 5

`merge`를 시도하다가 예상치 못한 큰 충돌이 발생했습니다. 일단 이 병합 작업을 깨끗하게 **취소**하고, `merge`를 시도하기 전의 상태로 완전히 되돌아가고 싶을 때 사용하는 명령어는 무엇인가요?

<details>
<summary>정답 확인하기</summary>

`git merge --abort`

</details>
