### 04. 브랜치 퀴즈 I

브랜치의 개념과 기본 명령어에 대해 잘 이해했는지 퀴즈를 통해 확인해보세요.

---

### 질문 1

Git에서 브랜치(Branch)를 사용하는 가장 주된 이유는 무엇일까요?

1.  커밋 메시지를 더 길게 쓰기 위해서
2.  기존 코드에 영향을 주지 않고 새로운 기능을 개발하는 등 독립적인 작업 흐름을 갖기 위해서
3.  커밋 히스토리를 압축해서 저장 공간을 아끼기 위해서

<details>
<summary>정답 확인하기</summary>

2번. 브랜치는 다른 작업에 영향을 주지 않는 독립적인 작업 공간을 제공하는 것이 핵심입니다.

</details>

---

### 질문 2

`feature/login`이라는 이름의 새 브랜치를 '생성'하고, 곧바로 그 브랜치로 '이동'까지 하고 싶습니다. 어떤 명령어를 사용해야 할까요?

<details>
<summary>정답 확인하기</summary>

`git switch -c feature/login`

</details>

---

### 질문 3

`git branch` 명령어를 실행했을 때, 여러 브랜치 이름 중 현재 내가 작업하고 있는 브랜치임을 알려주는 기호는 무엇인가요?

<details>
<summary>정답 확인하기</summary>

`*` (별표)

</details>

---

### 질문 4

`test` 브랜치를 삭제하기 위해 `git branch -d test` 명령어를 실행했지만 에러가 발생했습니다. 가장 가능성이 높은 원인은 무엇일까요?

<details>
<summary>정답 확인하기</summary>

현재 내가 `test` 브랜치 위에서 작업하고 있기 때문입니다. (브랜치를 삭제하려면 먼저 다른 브랜치로 이동해야 합니다.)

</details>

---

### 질문 5

`bugfix` 브랜치를 만들고 그곳에서 작업을 한 뒤, 다시 `main` 브랜치로 돌아가려고 합니다. 아래 빈 칸에 들어갈 올바른 명령어들을 순서대로 골라주세요.

```bash
# 1. bugfix 브랜치 생성 및 이동
[           ]

# (버그 수정 작업 및 커밋)

# 2. main 브랜치로 돌아가기
[           ]
```

<details>
<summary>정답 확인하기</summary>

1. `git switch -c bugfix`
2. `git switch main`

</details>
