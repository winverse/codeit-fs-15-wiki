### 11. main 브랜치와 premium 브랜치 둘다 push하기

지금까지의 모든 작업은 내 컴퓨터(로컬 저장소)에서만 이루어졌습니다. 이제 `main`과 `premium` 브랜치에서 작업한 내용을 다른 사람과 공유할 수 있도록 원격 저장소(GitHub)에 업로드(`push`) 해보겠습니다.

---

### 1. 이미 연결된 브랜치 `push`하기

`main` 브랜치는 처음부터 원격 저장소의 `origin/main`과 연결되어 있었으므로, 간단히 `git push` 명령어로 변경사항을 올릴 수 있습니다.

```bash
# 1. main 브랜치로 이동
git switch main

# 2. 원격 저장소에 변경사항 업로드
git push
```

- `push`가 완료되면, `git graph`에서 `main`과 `origin/main`이 같은 커밋을 가리키는 것을 확인할 수 있습니다.

---

### 2. 새로운 브랜치 처음으로 `push`하기

로컬에서 새로 만든 `premium` 브랜치는 아직 원격 저장소에 존재하지 않으며, 어느 리모트 브랜치와 연결되어야 하는지 정보가 없습니다.

#### 문제 상황

- 이 상태에서 `git push`를 시도하면, Git은 어느 브랜치로 내용을 보내야 할지 모르기 때문에 아래와 같은 `no upstream branch` 에러를 발생시킵니다.

#### 해결 방법

- **최초 1회**에 한해, `--set-upstream` 옵션을 사용하여 로컬 브랜치가 원격 저장소의 어떤 브랜치를 추적(연결)할지 명시적으로 알려주어야 합니다.

```bash
# 1. premium 브랜치로 이동
git switch premium

# 2. --set-upstream 옵션으로 원격 브랜치를 지정하여 push
# "origin이라는 원격 저장소에 premium이라는 이름으로 브랜치를 올리고, 앞으로도 계속 추적하겠습니다" 라는 의미
git push --set-upstream origin premium
```

- 이 명령을 실행하면 원격 저장소에 `premium` 브랜치가 새로 생성되고, 로컬 `premium` 브랜치와 연결됩니다.
- **이후부터는** `premium` 브랜치에서도 `git push` 명령어만 사용하면 됩니다.

---

### 용어설명

- **Upstream Branch**: 로컬 브랜치가 추적하도록 설정된 원격 브랜치. `push`할 때 변경사항이 올라가고, `pull`할 때 변경사항을 가져오는 기준점이 됩니다. `--set-upstream` 옵션은 이 기준점을 설정하는 역할을 합니다.
