# 09. git add 취소하기

실수로 커밋에 포함되면 안 되는 파일을 `git add` 했거나, 스테이징한 파일의 일부 내용을 수정해서 다시 `add` 해야 할 때가 있습니다. 이럴 때는 스테이징 영역에 올린 파일을 다시 내릴 수 있습니다.

---
### Staged 상태의 파일을 내리는 명령어

`git add`로 스테이징 영역에 추가된 파일은 'Staged' 상태가 됩니다. 이 파일을 다시 'Modified' 또는 'Untracked' 상태로 되돌리는 것을 'Unstage'라고 합니다.

- **명령어**
  ```bash
  git restore --staged [파일 이름]
  ```

---
### 예시: `git add` 취소하기

1.  `calculator.js`와 `style.css` 두 파일을 수정했다고 가정합니다.
    -   두 파일 모두 `Modified` 상태입니다.

2.  `git add .` 명령어로 모든 변경 내용을 스테이징합니다.
    ```bash
    git add .
    ```
    -   이제 두 파일 모두 `Staged` 상태가 됩니다.

3.  `style.css`는 아직 커밋할 준비가 되지 않아 스테이징에서 제외하고 싶습니다.
    ```bash
    git restore --staged style.css
    ```
    -   `style.css` 파일은 다시 `Modified` 상태로 돌아옵니다.
    -   `calculator.js` 파일만 `Staged` 상태로 남아있게 됩니다.

---
### 용어 설명

- **`git restore --staged`**: 스테이징 영역에 있는 파일을 작업 디렉토리의 상태로 되돌리는 명령어입니다. 즉, `git add`를 하기 전 상태로 돌려줍니다. 파일의 변경 내용이 사라지는 것은 아닙니다.

- **참고: 오래된 명령어**
  - 예전에는 `git reset HEAD [파일 이름]`이라는 명령어를 사용했습니다. 지금도 작동하지만, `git restore`가 역할을 더 명확하게 표현하여 최근 Git 버전에서는 이 명령어를 권장합니다.
