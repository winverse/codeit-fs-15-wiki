# 05. Git의 3가지 작업 영역

Git은 파일의 변경사항을 관리하기 위해 세 가지 다른 공간을 사용합니다. 바로 **작업 디렉토리**, **스테이징 영역**, **저장소**입니다.

---
### 1. 작업 디렉토리 (Working Directory)
-   우리가 실제로 파일을 만들고, 코드를 수정하는 프로젝트 폴더입니다.
-   눈에 보이는 폴더 그 자체를 의미합니다.

---
### 2. 스테이징 영역 (Staging Area)
-   커밋할 변경사항을 모아두는 준비 공간입니다.
-   `git add` 명령어를 사용하면 작업 디렉토리의 변경사항이 이곳으로 옮겨집니다.
-   장바구니에 살 물건을 담는 것에 비유할 수 있습니다.

---
### 3. 저장소 (Repository)
-   스테이징 영역에 있던 변경사항들이 `git commit` 명령어를 통해 영구적으로 저장되는 곳입니다.
-   `.git` 폴더 안에 모든 커밋 기록이 차곡차곡 쌓입니다.
-   장바구니의 물건을 계산하고 영수증과 함께 보관하는 것에 비유할 수 있습니다.

---
### 작업 흐름 요약
1.  **작업 디렉토리**에서 파일을 수정합니다.
2.  `git add`로 변경된 파일을 **스테이징 영역**에 올립니다.
3.  `git commit`으로 스테이징 영역의 파일들을 **저장소**에 기록합니다.
