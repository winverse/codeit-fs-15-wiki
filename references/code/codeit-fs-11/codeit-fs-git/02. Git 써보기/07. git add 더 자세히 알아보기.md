# 07. git add 더 자세히 알아보기

`git add`는 단순히 파일을 추가하는 것 이상의 유용한 기능들을 가지고 있습니다. 이번에는 `git add`를 더 효과적으로 사용하는 몇 가지 방법을 알아보겠습니다.

---
### 1. 특정 파일만 추가하기

가장 기본적인 사용법으로, 특정 파일의 변경 사항만 스테이징 영역에 추가합니다.

- **예시**
  ```bash
  # calculator.js 파일의 변경 내용만 스테이징
  git add calculator.js
  ```

---
### 2. 여러 파일을 한 번에 추가하기

여러 파일의 변경 사항을 한 번에 스테이징 영역에 추가할 수 있습니다.

- **예시**
  ```bash
  # 두 파일의 변경 내용을 한 번에 스테이징
  git add calculator.js style.css
  ```

---
### 3. 현재 폴더의 모든 변경 내용 추가하기

폴더 내에서 변경된 모든 파일(새 파일 포함)을 한 번에 스테이징 영역에 추가하고 싶을 때 사용합니다. `.`은 '현재 폴더'를 의미합니다.

- **예시**
  ```bash
  # 현재 폴더의 모든 변경 내용을 스테이징
  git add .
  ```

- **주의사항**
  - `git add .` 명령어는 현재 내가 위치한 폴더를 기준으로 동작합니다.
  - 원하지 않는 파일까지 추가될 수 있으므로, `git status` 명령어로 변경 내용을 미리 확인하는 습관을 들이는 것이 좋습니다.

---
### `git add`의 핵심 역할

`git add`는 작업 디렉토리의 수많은 변경 내용 중에서, 의미 있는 단위로 묶어 커밋할 내용들을 선별하는(Staging) 역할을 합니다. 즉, 이번 커밋에 포함될 변경 사항을 직접 선택하는 '문지기'와 같습니다.
