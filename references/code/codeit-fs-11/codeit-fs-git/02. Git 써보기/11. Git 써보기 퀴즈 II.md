# 11. Git 써보기 퀴즈 II

### 문제 1
현재 폴더와 그 하위 폴더에 있는 모든 변경 사항(새 파일, 수정된 파일)을 한 번에 스테이징 영역으로 추가하는 명령어는 무엇인가요?
<details>
<summary>정답 및 해설 보기</summary>
<div markdown="1">

- **정답:** `git add .`

- **해설:** `.`은 현재 디렉토리를 의미합니다. `git add .` 명령어는 현재 디렉토리 이하의 모든 변경 내용을 스테이징 영역에 추가하는 가장 일반적인 방법 중 하나입니다.

</div>
</details>

---
### 문제 2
Git 저장소에서 새로 생성한 파일(`notes.txt`)은 Git이 관리하지 않는 상태입니다. 이 상태를 무엇이라고 부르나요?
<details>
<summary>정답 및 해설 보기</summary>
<div markdown="1">

- **정답:** **Untracked (추적되지 않음)**

- **해설:** `Untracked` 상태는 Git이 해당 파일의 변경 내역을 전혀 추적하고 있지 않음을 의미합니다. 이 파일을 관리하려면 `git add`를 통해 `Staged` 상태로 만들어야 합니다.

</div>
</details>

---
### 문제 3
마지막 커밋 이후 아무것도 변경하지 않은 `Unmodified` 상태의 파일이 있습니다. 이 파일을 수정한 후, `git add`를 실행했습니다. 현재 이 파일의 상태는 무엇일까요?
<details>
<summary>정답 및 해설 보기</summary>
<div markdown="1">

- **정답:** **Staged (스테이징됨)**

- **해설:** `Unmodified` 상태의 파일은 수정되는 순간 `Modified` 상태가 되고, 이 파일을 `git add`하면 커밋 준비가 완료된 `Staged` 상태가 됩니다.

</div>
</details>

---
### 문제 4
`Staged` 상태의 파일들을 성공적으로 커밋(`git commit`)했습니다. 이제 이 파일들의 상태는 무엇으로 변경될까요?
<details>
<summary>정답 및 해설 보기</summary>
<div markdown="1">

- **정답:** **Unmodified (수정되지 않음)**

- **해설:** 커밋이 완료되면 스테이징되었던 변경 사항이 저장소에 안전하게 기록되고, 파일은 다시 변경 내역이 없는 깨끗한 `Unmodified` 상태로 돌아갑니다.

</div>
</details>

---
### 문제 5
`git add` 명령어로 스테이징 영역에 올린 파일(`Staged` 상태)을 다시 작업 디렉토리(`Modified` 상태)로 되돌리고 싶을 때 사용하는 명령어는 무엇인가요?
<details>
<summary>정답 및 해설 보기</summary>
<div markdown="1">

- **정답:** `git restore --staged [파일 이름]`

- **해설:** 이 명령어는 스테이징 영역에만 영향을 주며, 파일의 실제 변경 내용은 그대로 유지됩니다. 즉, `git add`를 실행하기 바로 전 상태로 되돌립니다.

</div>
</details>

---
### 문제 6
`git commit` 명령어의 자세한 사용법과 모든 옵션이 궁금할 때, 터미널에서 공식 설명서를 확인하는 명령어는 무엇인가요? (2가지 모두 작성)
<details>
<summary>정답 및 해설 보기</summary>
<div markdown="1">

- **정답:** `git commit --help` 또는 `git help commit`

- **해설:** `git help <명령어>` 또는 `git <명령어> --help` 패턴을 사용하면 Git의 모든 명령어에 대한 공식 문서를 빠르고 정확하게 찾아볼 수 있습니다.

</div>
</details>
