# 06. Git 써보기 퀴즈 I

### 문제 1
새로운 프로젝트 폴더(`mathTool`)에서 Git 버전 관리를 시작하기 위해 가장 먼저 실행해야 할 명령어는 무엇인가요?
<details>
<summary>정답 및 해설 보기</summary>
<div markdown="1">

- **정답:** `git init`

- **해설:** `git init` 명령어는 현재 폴더에 Git 저장소(.git 디렉토리)를 생성하여 버전 관리를 시작할 수 있도록 초기화하는 역할을 합니다.

</div>
</details>

---
### 문제 2
Git에서 '저장소(Repository)'란 무엇을 의미하며, 어떤 역할을 하나요?
<details>
<summary>정답 및 해설 보기</summary>
<div markdown="1">

- **정답:** 저장소(Repository)는 프로젝트의 파일 및 모든 변경 이력을 관리하는 데이터베이스입니다.

- **해설:** Git 저장소는 모든 커밋, 브랜치, 태그 등 프로젝트의 스냅샷을 시간순으로 저장합니다. 이를 통해 사용자가 언제든지 특정 버전으로 돌아가거나 변경 내역을 추적할 수 있게 해주는 핵심적인 공간입니다.

</div>
</details>

---
### 문제 3
`git init` 실행 후 생성되는, 프로젝트의 모든 버전 관리 기록을 담는 핵심 폴더의 이름은 무엇이며, 만약 이 폴더를 삭제하면 어떻게 되나요?
<details>
<summary>정답 및 해설 보기</summary>
<div markdown="1">

- **정답:** 폴더 이름은 **`.git`**이며, 이 폴더를 삭제하면 프로젝트의 모든 커밋 이력과 버전 관리 정보가 영구적으로 사라집니다.

- **해설:** `.git` 디렉토리는 Git의 내부 데이터베이스와 같습니다. 따라서 이 폴더가 없으면 Git은 더 이상 해당 프로젝트를 버전 관리 대상으로 인식하지 못합니다.

</div>
</details>

### 문제 4
작업 디렉토리에서 수정한 파일의 변경 내용을 커밋에 포함시키려면, `git commit` 이전에 반드시 실행해야 하는 명령어는 무엇이며, 이 명령의 역할은 무엇인가요?
<details>
<summary>정답 및 해설 보기</summary>
<div markdown="1">

- **정답:** `git add [파일 이름]`

- **해설:** `git add` 명령어는 작업 디렉토리의 변경 사항을 커밋을 위해 준비하는 **스테이징 영역(Staging Area)**으로 옮기는 역할을 합니다. 커밋은 스테이징 영역에 있는 변경 사항들만을 대상으로 이루어집니다.

</div>
</details>

---
### 문제 5
`git commit` 명령어 사용 시 `-m` 옵션은 어떤 역할을 하며, 왜 중요한가요?
<details>
<summary>정답 및 해설 보기</summary>
<div markdown="1">

- **정답:** `-m` 옵션은 커밋에 대한 설명을 담은 **커밋 메시지(Commit Message)**를 함께 기록하는 역할을 합니다.

- **해설:** 커밋 메시지는 해당 커밋에서 어떤 변경 작업이 이루어졌는지 요약하는 정보입니다. 좋은 커밋 메시지는 미래의 자신이나 다른 동료가 변경 이력을 쉽게 이해하는 데 큰 도움을 주기 때문에 매우 중요합니다.

</div>
</details>

### 문제 6
`file1.js`와 `file2.js` 두 파일을 수정했지만, `file1.js`의 변경사항만 먼저 커밋하고 싶을 때 사용해야 하는 명령어들을 순서대로 나열하세요.
<details>
<summary>정답 및 해설 보기</summary>
<div markdown="1">

- **정답:**
  1. `git add file1.js`
  2. `git commit -m "file1.js 수정"`

- **해설:** `git add` 명령어를 통해 원하는 파일(`file1.js`)만 스테이징 영역에 추가할 수 있습니다. `git commit`은 스테이징 영역에 있는 내용만 커밋하므로, `file2.js`의 변경 내용은 커밋에 포함되지 않습니다.

</div>
</details>

### 문제 7
Git의 세 가지 작업 영역(저장소, 작업 디렉토리, 스테이징 영역)의 올바른 작업 흐름 순서는 무엇인가요?
<details>
<summary>정답 및 해설 보기</summary>
<div markdown="1">

- **정답:** 작업 디렉토리 → 스테이징 영역 → 저장소

- **해설:**
    1. **작업 디렉토리**에서 파일을 수정합니다.
    2. `git add` 명령어로 **스테이징 영역**에 변경 사항을 추가합니다.
    3. `git commit` 명령어로 **저장소**에 최종적으로 기록합니다.

</div>
</details>
