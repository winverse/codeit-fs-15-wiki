# 08. Git이 보는 파일의 4가지 상태

Git은 프로젝트 폴더 안의 모든 파일을 추적하고 관리합니다. 이때 Git은 각 파일을 4가지 상태 중 하나로 인식합니다. 이 상태를 이해하면 `git add`와 `git commit`의 동작을 더 깊이 파악할 수 있습니다.

---
### 파일의 4가지 상태

1.  **Untracked (추적되지 않음)**
    -   새로 생성되어 Git이 한 번도 관리한 적 없는 파일의 상태입니다.
    -   이 상태의 파일은 `git add`를 통해 처음으로 Git의 관리하에 들어갑니다.

2.  **Unmodified (수정되지 않음)**
    -   파일이 마지막 커밋 상태 그대로이며, 작업 디렉토리에서 아무런 변경이 없는 깨끗한 상태입니다.

3.  **Modified (수정됨)**
    -   마지막 커밋 이후 파일의 내용이 변경된 상태입니다.
    -   Git은 이 파일이 변경되었음을 감지하고, 커밋할 준비를 하라고 알려줍니다.

4.  **Staged (스테이징됨)**
    -   수정된 파일이 `git add` 명령어를 통해 '커밋될 준비가 된' 상태로, 스테이징 영역에 올라간 것을 의미합니다.
    -   이 상태의 파일들만 `git commit` 시 저장소에 기록됩니다.

---
### 파일 상태의 흐름

파일의 상태는 아래와 같은 흐름으로 계속해서 변합니다.

1.  **파일 생성** → `Untracked`
2.  `git add` 실행 → `Staged`
3.  `git commit` 실행 → `Unmodified`
4.  파일 내용 수정 → `Modified`
5.  다시 `git add` 실행 → `Staged`
6.  다시 `git commit` 실행 → `Unmodified`

이 흐름을 이해하면, 내가 작업한 내용이 현재 어떤 상태에 있는지, 그리고 다음에는 어떤 명령어를 사용해야 하는지 명확하게 알 수 있습니다. `git status` 명령어를 사용하면 언제든지 파일들의 현재 상태를 확인할 수 있습니다.
