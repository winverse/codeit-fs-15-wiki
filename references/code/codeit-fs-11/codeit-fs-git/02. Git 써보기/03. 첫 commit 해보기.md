# 03. 첫 commit 해보기

### **Commit의 개념**

- 작업한 내용을 저장하는 행위
- 변경 사항에 대한 스냅샷을 생성

---

- **첫 Commit 과정**
    1. 파일을 생성하거나 수정
    2. `git add`로 Staging Area에 추가
    3. `git commit`으로 변경 사항을 저장소에 기록

---

- **예시**
    
    `calculator.js` 파일 생성 또는 수정
    
    ```jsx
    function add(a, b) {
      return a + b;
    }
    
    function subtract(a, b) {
      return a - b;
    }
    ```
    
    Staging Area 추가 및 커밋
    
    ```bash
    git add calculator.js
    git commit -m "Add calculator.js for basic calculations"
    ```
    

---

## 주요 커맨드

| 명령어 | 설명 |
| --- | --- |
| `git add [파일 이름]` | 파일을 Staging Area로 이동 |
| `git commit -m "[메시지]"` | Staging Area의 내용을 커밋 |

---

## 용어설명

- **Staging Area**
    
    커밋할 변경 사항들이 임시로 대기하는 공간
    
- **Commit Message**
    
    커밋 시 기록되는 변경 사항 요약 설명
